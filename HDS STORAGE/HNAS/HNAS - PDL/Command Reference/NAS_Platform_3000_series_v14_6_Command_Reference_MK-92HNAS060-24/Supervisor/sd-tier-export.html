sd-tier-export

Export SD tiers to a text file

Syntax

sd-tier-export <filename> [<SDs-see-'man sd-spec'>]

Description

Exports SD tiers from the server's registry into a text file. If you specify a set of SDs, those SDs' tiers are exported; by default, the command exports tiers from all SDs that have been placed into tiers.

Inside an SSC session, the command will create a file on your local client. In other environments, the file will be created on the server's local file system.

This command is rarely needed. There is no need to make regular backups of a server's SD tiers: tiers are stored in the registry, and a registry backup will back them up. Exporting and importing tiers is part of the process of migrating a span between clusters, but the SMU can carry out that task more simply and efficiently.

FILE FORMAT

Here is a simple tier file:

Schema 0
0 0 [01:02:00]HITACHI 37:35:30:31:32:30:31:34:30:30:30:30
1 0 [01:02:00]HITACHI 37:35:30:31:32:30:30:32:30:30:30:30
2 1 [01:02:00]HITACHI 37:35:30:31:32:30:31:34:30:30:30:31
3 1 [01:02:00]HITACHI 37:35:30:31:32:30:30:32:30:30:30:31
4 250 [01:02:00]HITACHI 37:35:30:31:32:30:31:34:30:30:30:32

The file must start with a line saying 'Schema 0'; the importer will reject it if it does not. After that, each line contains three pieces of information:

  1. The device ID of the SD, as the exporting server sees it
  2. The SD's tier number; the special value 250 means that the SD is not assigned to a tier
  3. The SD's Luid, which is used because SD device IDs may not be the same in the exporting server and the importing server. (In fact, apart from checking for duplicates, the importer ignores device IDs completely, and identifies SDs by their Luids.)

Being a text file, an exported file can be edited by hand. You can add blank lines and comment lines wherever you wish -- even before the 'Schema 0' line. A comment line starts with optional white space and a hash (#) mark. One simple use of a comment is to disable one or more lines without deleting them.

Examples

sd-tier-export some-sds.txt 20-39


sd-tier-export all-tiered-sds.txt

Applies To

Cluster wide
