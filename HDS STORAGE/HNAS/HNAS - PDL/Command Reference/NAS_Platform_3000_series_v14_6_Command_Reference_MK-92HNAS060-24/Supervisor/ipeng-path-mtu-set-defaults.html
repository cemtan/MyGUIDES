ipeng-path-mtu-set-defaults

Reset Path MTU configuration to defaults.

Syntax

ipeng-path-mtu-set-defaults

Description

The ipeng-path-mtu commands control the Path MTU Discovery feature that supports the dynamic learning and application of per-path MTU limitations. These are global settings which act on all ports.

Only use under instruction from your support provider.

The Path MTU Discovery only applies to IPv6 traffic and operates in accordance with RFC-1981.

The commands available are as follows.

ipeng-path-mtu-show

Displays the Path MTU Discovery configuration options.

ipeng-path-mtu-link-flush-enable

Enable Path MTU flushing on a link-down event.

ipeng-path-mtu-link-flush-disable

Disable Path MTU flushing on a link-down event.

ipeng-path-mtu-timeout [<value>]

If a parameter is given, sets the timeout value, in seconds, for discovered Path MTU information. The valid range for this parameter is 300 to 3600 seconds.

With no parameter, displays the current and default settings.

ipeng-path-mtu-set-defaults

Reset the Path MTU Discovery configuration options to their default values. This enables Path MTU flushing on a link-down event and sets the Path MTU Timeout to 600 seconds.

Examples

To change the timeout to 900 seconds:

    ipeng-path-mtu-timeout 900

To reset the timeout to the default:

    ipeng-path-mtu-set-defaults

Applies To

Cluster wide
