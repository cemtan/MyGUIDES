migration-cloud-schedule-delete

Delete an external cloud migration schedule

Syntax

migration-cloud-schedule-delete <policy name> <schedule id> [ --stop-active | -f ]

Description

Deletes a schedule entry for cloud migration. The delete attempt will normally fail if a currently-running migration was launched by the entry to be deleted. This behavior may be changed by using the --stop-active flag.

<policy name>

Name of the policy applied by the schedule. Alternatively may be a policy ID (UUID), as shown in "migration-cloud-schedule-list -v" output.

<schedule id>

ID of the schedule. To get a list of IDs, use the migration-cloud-schedule-list command.

--stop-active

If present, this flag indicates that the delete request should proceed even if a migration initiated by the schedule entry is currently running. If such a migration is currently running, specifying this flag will cause the migration to be terminated as well as allowing deletion of the selected schedule entry.

Examples

migration-cloud-schedule-delete DefaultPolicy 3

Deletes the cloud migration schedule with the specified ID (3) for the policy named 'DefaultPolicy'. Deletion is not permitted if a migration is presently running for this policy / schedule ID combination.

migration-cloud-schedule-delete SomePolicy 7 --stop-active

Deletes the cloud migration schedule with the specified ID (7) for the policy named 'SomePolicy'. If a migration is presently running for this schedule ID / policy combination, the migration will be halted. The schedule entry for the specified policy ID / schedule ID will be deleted regardless of whether a migration for it was running.
