smb3-reject-unencrypted-access-disable

Disable 'reject unencrypted access' on the current EVS

Syntax

smb3-reject-unencrypted-access-disable

Description

The smb3-reject-unencrypted-access-* commands determine whether SMB2 clients can access an EVS. These commands are only available on an EVS if it has the maximum supported SMB version set to SMB3. Note that access by SMB1 clients can only be prevented by turning off the SMB1 server. The SMB1 server can be turned off by using the smb-min-supported-version command to set the minimum supported version to SMB2.

smb3-reject-unencrypted-access-status

Display the smb3-reject-unencrypted-access status of the current EVS.

smb3-reject-unencrypted-access-enable

Enable encryption on the current EVS.

smb3-reject-unencrypted-access-disable

Disable encryption on the current EVS.

smb3-reject-unencrypted-access-default

Set the SMB3 'reject-unencrypted-access' status of the current EVS to the default setting of enabled.

Applies To

EVS
