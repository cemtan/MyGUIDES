replication-recovery-default-transfer-setting-set

Set the per file system default access point transfer disaster recovery setting.

Syntax

replication-recovery-default-transfer-setting-set <file-system> (on|off)

Description

This command applies to WFS-2 file systems only.

Sets a file system's default disaster recovery behaviour for shares and exports. This setting is used to decide whether the shares/exports that have their '--disaster-recovery-setting' field set to 'use-fs-default' are recovered or not when an Object Replication target of the file system is promoted to read-write.

Note that when a share/export is created, its '--disaster-recovery-setting' field is set to 'use-fs-default' by default, i.e., its recovery is controlled using the file system level switch. Also note that the file system level switch can be overridden on an individual basis by setting a share/export's '--disaster-recovery-setting' field to 'transfer' or 'do-not-transfer' explicitly (see cifs-share/nfs-export add/mod commands).

<file system>

The file system to which this setting needs to be applied.

<configuration-transfer-setting>

Set this option to one of the following:

on

Enable the recovery of all shares and exports that have their '--disaster-recovery-setting' field set to 'use-fs-default'.

off

Disable the recovery of all shares and exports that have their '--disaster-recovery-setting' field set to 'use-fs-default'.

Examples

    replication-recovery-default-transfer-setting-set fs0 on
    Disaster recovery access point transfer setting for file system 'fs0' is now enabled

Enables the recovery of all shares and exports that target the file system 'fs0' and that have their '--disaster-recovery-setting' field set to 'use-fs-default'.
