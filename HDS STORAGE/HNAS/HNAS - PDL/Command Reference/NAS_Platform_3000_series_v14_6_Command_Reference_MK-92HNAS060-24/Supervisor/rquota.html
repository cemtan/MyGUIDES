rquota

Queries/modifies the rquota protocol settings

Syntax

rquota [--on | --off] [--criteria restrictive | matching]
	restrictive - report the most restrictive limits from
	              all applicable quotas
	matching    - report the first applicable quota

Description

This command allows the settings for the rquota RPC service to be modified. The rquota service allows network clients to query the quota details for a particular user and file system on the server exported over NFS.

Usage:

rquota [--on | --off] [--criteria restrictive | matching]

By default the rquota RPC service is enabled. This can be changed using the --off switch. This setting requires a reboot to take effect.

For any given user on any give file system there are a number of different quotas that can apply. The criteria option specifies how the server should select the quota as follows:

restrictive

Report the most restrictive limits from all applicable quotas

matching

Report the first applicable quota

Examples

rquota --off

Will disable the rquota service on the next reboot.

Applies To

Cluster wide

Privilege Level

Supervisor
