si-superflush-statistics

Display superflush statistics

Syntax

si-superflush-statistics [-r] [ -a | [-g] [ { -d | -s } [<identifier>] | [-d|-s] -f ] ]

Description

Display or reset superflush statistics. Statistics are maintained globally, for each system drive, and for performance-info-report's current file system of interest (if any).

-f, --filesystem

Display or reset superflush statistics for performance-info-report's currently selected file system.

-s, --span

With -f, equivalent to -d. Otherwise the identifier, if present, should be a span label, and all the system drives in that span are selected; if no identifier is given, select all SDs from all spans.

-d, --system-drive

With -f, select all the SDs in the span hosting that file system, in addition to the file system. Otherwise the identifier, if present, should be a system drive ID; if no identifier is given, select all SDs with superflush configured.

-g, --global

Select the global statistics. This is the default if none of -f -s -d is used.

-a, --all

Display or reset all available superflush statistics.

-r, --reset

Reset the selected statistics instead of displaying them.

NB: Per-SD statistics are normally not gathered and will therefore be reported as 0. You must enable them using enhanced-perf-stats; counts will accumulate from that point.

Only a lower bound is available for the global and FS individual buffer flushes counts. This also affects calculation of the proportion of stripe set flushed buffers. (When enhanced statistics gathering is enabled, per-SD counts will be added together to give an accurate global count.)

Examples

Show the global statistics:

si-superflush-statistics

Show statistics for SD 3:

si-superflush-statistics -d 3

Show statistics for every SD with superflush configured and the global statistics:

si-superflush-statistics -g -d

Reset statistics for all SDs in span "Archive":

si-superflush-statistics -r -s Archive

Reset all superflush statistics:

si-superflush-statistics -r -a

Applies To

Cluster node
