ndmp-device-assigned-name-set

Assign a name to an NDMP device

Syntax

ndmp-device-assigned-name-set { -s <serial_num> | -n <node_wwn> [-l <lun>]} [--override | -o] <name>

Description

The command assigns a name to a tape drive or media changer device that can be used by NDMP clients. If no name is assigned to a device the system assigns a name. (See man page ndmp-devices for details.)

The device to be named can be identified either by its serial number or by the fibre channel node name of the device with an optional logical unit number.

The serial number is specified by the -s option.

The fibre channel device node name is specified by the -n option. Use the prefix 0x to indicate that the number supplied is hexadecimal. If more than one subdevice is hosted on the main FC device then the logical unit number of the subdevice must be specified using the -l option.

If the name is already assigned to a device or if the device addressed is assigned another name then the command will have no effect unless the --override (-o for short) option is specified.

Examples

Change the name of the tape drive with serial number PMB51S0594 to be td2.

ndmp-device-assigned-name-set -s PMB51S0594 td2

Change the name of the tape drive with logical unit number 3 within tape library with fibre channel node name 5005084000278000 to tape-drive3.

ndmp-device-assigned-name-set -n 0x5005084000278000 -l 3 tape-drive3

Change the name of the native fibre channel tape drive with node name a005084000abcdef to tape1.

ndmp-device-assigned-name-set -n 0xa005084000abcdef tape1

Applies To

Cluster wide
