sd-identify-encroachment

Try to determine which operating system encroached on one or more system drives

Syntax

sd-identify-encroachment <System-drives-see-'man sd-spec'>

Description

If a foreign server is connected to this server's storage and switches are not zoned correctly, it is possible that the foreign server will write to this server's storage. If this happens, the first thing that the foreign server writes is usually a signature. You can use 'sd-identify-encroachment' to scan one or more system drives (SDs) for foreign signatures and try to identify the operating system running on the foreign server.

If the server detects no encroachment by a foreign server, the command will produce the following output:

x86 MBR - Partitions: (1) Ours

Any other output strongly suggests that a foreign server has written to the system drive at some time in the past. However, if the SD was connected to a different vendor's storage before being used with the server and the signature is not shown as 'x86 MBR', it's possible that no problem has occurred and the signature was written before the system drive was first used with the server.

'sd-identify-encroachment' reads directly from the storage, ignoring the server's cached state. It's therefore possible for this command to report encroachments that don't yet appear in the output of other commands.

To see how to specify more than one SD, consult the 'sd-spec' man page.

Examples

sd-identify-encroachment 7

Scans for signatures on system drive 7

sd-identify-encroachment Accounts:has-bad-encroachment

Scans for signatures on all SDs in span Accounts on which an encroachment has been detected.

Applies To

Cluster wide
