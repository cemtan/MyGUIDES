sd-fp-pending

Dump failed or timed-out fast-path management I/Os that haven't yet completed

Syntax

sd-fp-pending

Description

If an I/O error or timeout occurs while the server is reading or writing encroachment data or Cod signatures, it does not wait the (potentially very long) time required for the I/O to time out or complete: instead, it aborts the operation and lets all pending I/Os time out asynchronously. 'sd-fp-pending' lists all I/Os that are still waiting to time out.

The I/Os in question arise mainly from automatic actions that occur (for instance) when you create or expand a span or mount a file system, or which run regularly in the background. Commands such as 'sd-dump-encroachment-area' use a different I/O path and can't cause this command to display any output.

If 'sd-fp-pending' reports that a span has outstanding failed I/Os, there are several consequences:

  • The span can't be deleted ('span-delete')
  • The span can't be unlicensed ('span-deny-access' with no switches)
  • The server can't load an integrated Cod backup that would replace the span in memory
  • The server can't read or write encroachment data to the span or check any of its SDs' Cod signatures

Other operations, including file service on the affected span, can continue normally.

The condition will clear, without user intervention, after a few seconds or minutes.

Examples

server:$ sd-fp-pending
HDP pre-allocation write to SD 11 (used in span Accounts)  at offset 81622204416 (0x1301100000)
server:$

The pieces of information listed on each line include

  • The I/O type
  • The system drive number
  • The span instance name and ID
  • The offset of the I/O and the structure that resides there

The I/Os are listed in chronological order: the first I/Os in the list are most likely to complete first.

Applies To

Cluster node
