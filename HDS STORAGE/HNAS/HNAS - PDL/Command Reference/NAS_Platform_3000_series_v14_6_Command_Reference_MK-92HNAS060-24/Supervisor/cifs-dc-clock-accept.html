cifs-dc-clock-accept

Adjust tolerance of server / DC clock skew.

Syntax

cifs-dc-clock-accept [ --on | --off ]

Description

The cifs-dc-clock-accept command allows you to control the system's clock tolerance when connecting to a CIFS domain controller (DC). Ordinarily one setting applies cluster wide, however, when multi-tenancy is enabled configuration is per EVS.

In a normal ADS environment, CIFS servers will refuse to connect to a DC if their clocks differ by more than five minutes. Note that this is a more stringent requirement than for regular ADS client / server connections.

To ease the administrative burden at sites which have not deployed NTP, the server may optionally pretend that its clock matches that of a DC when negotiating a connection with that DC. This does not change the server's actual clock value, merely how it is presented to the DC.

By default this option is enabled. It may be disabled using the --off switch, and enabled again using --on.

Changes made by this command will take effect with the next DC connection attempt.

Note that the setting controlled by this command only influences routine DC connections. The DC connections formed by the cifs-name command's add and remove options still require that the server and DC clocks be in sync. The intent of this command is to accommodate clock drift, not to render the server's clock irrelevant.

If possible, it is recommended that you use NTP instead and leave this option off.

Examples

To display the current cifs-dc-clock-accept setting:

cifs-dc-clock-accept

To enable DC clock acceptance (the default):

cifs-dc-clock-accept --on

To disable DC clock acceptance (normal MS ADS server behavior):

cifs-dc-clock-accept --off

Applies To

Cluster wide (per EVS when multi-tenancy is enabled).
