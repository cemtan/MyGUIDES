span-release

Allow auto-expansion of all unconfined filesystems on a span

Syntax

span-release <span-instance-name>
Alias:       relspan

Description

In normal use, the server automatically expands filesystems when they become full. It is possible to 'confine' a span: this prevents automatic expansion of all filesystems on the span while still allowing manual filesystem expansion. Releasing the span undoes the effect of confining it, and allows the server to auto-expand filesystems again.

File systems that are very full may auto-expand, perhaps several times, after the command is run. The command returns control quickly, without waiting for any auto-expansion to take place.

You can also confine a filesystem to a specified capacity. Releasing a span does not remove its filesystems' confining capacities or permit them to auto-expand beyond those confining capacities.

There is no need to release a span or filesystem before expanding a filesystem manually with (for example) the 'filesystem-expand' command: only automatic expansion is affected by confinement.

On a tiered span, all tiers are affected by 'span-confine' and 'span-release'. There is no mechanism for confining or releasing a single tier of a span. For a similar facility, see 'filesystem-confine' and 'filesystem-release', each of which has a --tier switch.

If 'span-hdp-preallocation no' is in force, no filesystem auto-expansion takes place on spans that reside on HDP storage, even if you release those spans. Auto-expansion is also impossible on spans that reside on UVM storage.

Examples

span-release Accounts

Allows all filesystems on span Accounts to auto-expand, subject to any limits imposed on them by 'filesystem-confine'.

Applies To

Cluster wide
