cvl-convert

Schedule conversion from CVL1 to a CVL2

Syntax

cvl-convert 
schedule [-p <file-path>] [--fix-flags-only] <file-system>
stop <file-system>
status <file-system>

Description

The cvl-convert command is used to manage the conversion of files of type Cross Volume Link 1 (CVL1) to files of type Cross Volume Link 2 (CVL2) for a given file system.

Usage:

cvl-convert schedule [OPTION] <file-system>

Schedule CVL1 to CVL2 conversion for a file system. Events will be raised to indicate the status of the conversion process. When the conversion actually begins, the file system will be scanned for CVL1 files, which will be converted into CVL2. If the file system goes offline while this conversion is in progress or is scheduled for it, the process will be paused automatically, and will be resumed when the file system comes back up online again.

Additionally, this tool will fix the OFFLINE DOS flag and clear the ReadOnlyLocked flag for any remote CVL2 links that it encounters. The OFFLINE DOS flag will be set or cleared as specified by the No-DOS-Offline-for-XVL flag for the file system. (Use the '--fix-flags-only' to perform this step alone).

-p, --path <file-path>

Process the specified file only.

--fix-flags-only

Fix the OFFLINE DOS flag and clear the ReadOnlyLocked flag of remote CVL2 links only, do not perform any CVL1 to CVL2 conversions.

cvl-convert stop <file-system>

This command will stop the conversion of CVL1 to CVL2 for a file system.

cvl-convert status <file-system>

Output the conversion status for a filesystem.

Examples

To schedule CVL1 to CVL2 conversion for a file system:

  cvl-convert schedule fs0
  CVL1 to CVL2 conversion scheduled. An event will be raised when the conversion process has completed

To stop conversion for a file system:

  cvl-convert stop fs0
  CVL1 to CVL2 conversion has been stopped.

To view the status of conversion for a particular file system:

  cvl-convert status fs0
  CVL1 to CVL2 conversion state: InProgress

Applies To

Cluster node
