span-allow-access

Allow access to all the system drives in a span

Syntax

span-allow-access <span-instance-name>

Description

This is one of a sequence of commands used when you are moving a span from one server or cluster to another. It licenses (allows access to) all unlicensed system drives (SDs) in the specified span.

If the 'span-deny-access' command has previously configured the span to permit mirrored partners to be unlicensed, 'span-allow-access' undoes that configuration and tells the server to issue warnings if any SD in the span is unlicensed.

The 'span-assign-to-cluster' man page shows the full procedure for moving a span between clusters.

If you merely wish to undo the effect of 'span-deny-access' without moving the span between clusters, here is the procedure:

  • Use 'sd-allow-access' to license any one primary SD in the span. (A primary SD is usually a stand-alone SD or a mirrored P-Vol; the 'storage-based-mirror' man page describes the exceptions.)
  • Pause for a few seconds (or, in a script, run 'span-wait-for-loading') to wait for the server to load Cod from the SD you have just licensed. You can confirm it using 'span-list'.
  • Use 'span-allow-access' to license all the other SDs in the span.
  • In a script, run 'span-wait-for-loading --mount' to wait until the resulting background processes have completed.

    • If working interactively, you can use 'sd-list-queues' and 'filesystem-list-queues' to watch the progress of these background processes without waiting for them to complete, but they normally run fast enough to make monitoring unnecessary.

  • Use 'evsfs' to bind filesystems to EVSs.

Examples

span-allow-access ProjectX

Applies To

Cluster wide
