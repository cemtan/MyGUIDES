timestamp-encode

Convert human-readable date and time to a 32-bit or 64-bit Unix time integer

Syntax

timestamp-encode [--nanosecond-resolution] [--hex] {--date yyyy-mm-dd --time hh:mm:ss[.fraction][+|-hh:mm] } | { --iso-format yyyy-mm-ddThh:mm:ss[.fraction][+|-hh:mm] }

Description

This command converts a file system timestamp in human-readable form to a 32-bit or 64-bit Unix time integer value.

It should be pointed out that conversion of date time strings to integer values and vice versa is also supported in the 'strtoken' filter and built-in functions that can be specified in the 'calculate' command.

For the purpose of this command, a 32-bit Unix time value is the number of seconds since midnight of 1970/01/01 and a 64-bit Unix time value is defined as follows:

   - the high-order 32-bit value is the number of seconds since midnight of 1970/01/01.
   - the low-order 32-bit value contains the number of nanoseconds in the second fraction.

Note that the human-readable timestamp has 100-nanosecond resolution. Thus, when it is converted to Unix time value, the result also has 100-nanosecond resolution.

Usage:

--nanosecond-resolution, -n

If this option is specified, display the Unix time value as a 64-bit integer with 100-nanosecond resolution. Otherwise, display the Unix time value as a 32-bit integer with one-second resolution.

--hex, -h

Display the Unix time value in hex. If this option is not specified, the Unix time value is displayed in decimal.

--date, -d <yyyy-mm-dd>

Specify the date in the form of yyyy-mm-dd such as -d 2015-08-20. If --date is not specified, the date string is assumed to be "1970-01-01".

--time, -t <hh:mm:ss[.fraction][+|-hh:mm]>

Specify the time string in the 24-hour format where hh is the hour number from 0 to 23, mm is the minute number from 0 to 59, ss is the integer number of seconds from 0 to 60 (60 to support leap second). The fractional part of a second appears after a decimal point with at most nanosecond resolution, i.e., at most 9 decimal digits. The GMT time offset is specified at the end of the string as +hh:mm or -hh:mm. If the GMT time offset is not specified, it will be assumed to be "+00:00".

If --time is not specified, the time string is assumed to be "00:00:00+00:00"

--iso-format, -i <yyyy-mm-dd>T<hh:mm:ss[.fraction][+|-hh:mm]>

Specify the date and time in ISO 8601 format. In this format, the string starts with a date string of the form yyyy-mm-dd, then 'T', then a time string as described in the --time option.

Note that if --iso-format is specified, options --date and --time cannot be specified and vice versa.

Examples

1. Convert a human-readable file system timestamp to 32-bit Unix time in decimal:

   $ timestamp-encode --date 2015-08-16 --time 21:01:31-07:00
   1439784091


   $ timestamp-encode --iso-format 2015-08-16T21:01:31-07:00
   1439784091


   $ timestamp-encode --iso-format 2015-08-16T21:01:31.326712334-07:00
   1439784091

2. Convert a human-readable timestamp to 32-bit Unix time in hex:

   $ timestamp-encode --hex --date 2015-08-16 --time 21:01:31-07:00
   0x55d15c9b

3. Specify --nano to convert a human-readable timestamp to 64-bit time in decimal:

   $ timestamp-encode --nano --date 2015-08-16 --time 21:01:31-07:00
   6183825584146087936

4. Same as (3) but display in hex:

   $ timestamp-encode --hex --nano --date 2015-08-16 --time 21:01:31-07:00
   0x55d15c9b00000000
