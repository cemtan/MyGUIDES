replication-policy-modify

Modify an object replication policy

Syntax

replication-policy-modify [--new-name <name>] [--no-target-check] [--target <target>] [--target-fs-label <target fs label>] [--source-snapshot-option <source-snapshot-option>] [--source-snapshot-rule <source-snapshot-rule>] [--target-snapshot-option <target-snapshot-option>] [--target-snapshot-rule <target-snapshot-rule>] [-s <source ip address>] [-p <target port>] [--enable|--disable] <policy name> <source fs label>

Description

Modifies a policy for object replication.

--no-target-check

By default a message is sent to the target server to check that the replication target with the given name is available. If it is not available, the policy modification will fail. Providing this option will enable a policy to be modified even if the replication target is not available.

<policy name>

Name of the policy.

<source fs label>

Source file system label.

--new-name, -n <policy name>

New name of the policy.

--target, -t <target>

Target EVS host name or IP address.

An IP address can be specified as either an IPv4 address in dotted decimal notation (e.g. 203.0.113.0) or as an IPv6 address (e.g. 2001:db8:fe58:3efa:deca:212::c0a9)

--target-fs-label <target fs label>

Target file system label.

--source-ip-address, -s <source ip address>

Source EVS host name or IP address. This is useful when a customer has an EVS with IP addresses on different physical networks, where some IP addresses are routable to the target server and others are not. In this case, the admin will need to choose a source IP address manually.

An IP address can be specified as either an IPv4 address in dotted decimal notation (e.g. 203.0.113.0) or as an IPv6 address (e.g. 2001:db8:fe58:3efa:deca:212::c0a9)

--source-snapshot-option <source snapshot option>

Source snapshot option, allowed values are:

auto

Take an automatic snapshot and replicate that snapshot. After the object replication completes, the previous snapshot is deleted.

use-named

Replicate the latest snapshot taken with the named snapshot rule.

--source-snapshot-rule <source-snapshot-rule>

Name of the snapshot rule when the source snapshot option is set to use-named.

--target-ip-port, -p <target port>

Specify object replication listening port of the target host. The default is 59550.

--target-snapshot-option <target snapshot option>

Target snapshot option, allowed values are:

auto

When the object replication completes, take an automatic snapshot.

use-named

When the object replication completes, take a snapshot using the named snapshot rule.

--target-snapshot-rule <target snapshot rule>

Name of the snapshot rule when the target snapshot option is set to use-named.

--enable

Enable the object replication policy thus allowing it to run.

--disable

Disable the object replication policy thus disallowing it from running.

--force

Forcibly modify an object replication policy that is currently running.

Examples

replication-policy-modify --target FileServer2 --target-fs-label fs0r DefaultPolicy fs0

Modifies an object replication policy named 'DefaultPolicy' on file system fs0 to replicate to target file system fs0r on EVS hostname FileServer2.

Applies To

EVS
