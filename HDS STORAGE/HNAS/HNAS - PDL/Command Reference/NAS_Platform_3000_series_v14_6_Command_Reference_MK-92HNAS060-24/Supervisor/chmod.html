chmod

Change Unix file access permissions

Syntax

chmod <mode> <file1> [<file2>...]

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The chmod command is used to change the Unix access permissions (i.e. mode) for a set of files and/or directories. The format of a mode, as used by the chmod command, is one of:

Symbolic
chmod <[augo] { [+-=] [rwxst] } [,...]> <filename>

The letters 'augo' control which users' access permissions will be modified. Options are all users (a), the file's owner user (u), other users in the file's group (g) or all other users that are neither the file's owner nor in the file's group (o).

The + and - operators add and remove the given permissions from the mode respectively. The = operator sets the mode to that given.

The letters 'rwxst' identify which permissions shall be added, removed or set. Options are read (r), write (w), execute (x), set user ID/set group ID (s) or sticky bit (t).

In order to set the extended mode bits (Set User ID/Set Group ID/Sticky) you must use the letter 's' for user and group but the letter 't' for other. User and group will ignore the 't' permission modifier and other will ignore the 's'.

Numeric
chmod <nnnn> <filename>

Where nnnn is an octal number specifying the mode.

A numeric mode is from one to four octal digits (0 -7), derived by adding up the bits with values 4, 2, and 1. Omitted digits are assumed to be leading zeros. The first digit selects the set user ID (4) and set group ID (2) and restricted deletion or sticky (1) attributes. The second digit selects permissions for the user who owns the file: read (4), write (2), and execute (1); the third selects permissions other users in the file’s group, with the same values; and the fourth for other users not in the file’s group, with the same values.

NOTE: Set Group ID is ignored and cannot be set or cleared using numeric format. In order to set/clear Set Group ID you must use symbolic format (g+s | g-s).

Examples

Symbolic

To allow all users to read the file 'myfile':

chmod a+r myfile

To deny write and execute rights to the files 'file1', 'file2' and 'file3' to everyone apart from their owners:

chmod go-wx file1 file2 file3

To allow write and execute but deny read to group of file 'foo' and deny any access to others:

chmod g+wx-r,o= foo

NOTE: There can be no spaces in the mode-specifier. If you use a comma to extend the mode-set you must follow it with the next specifier without a space.

Numeric

To allow user read, write and execute permissions, allow group read and execute and deny others access:

chmod 0750 foo

Or you may omit the leading 0:

chmod 750 foo

To set the same permissions but with Set User ID

chmod 4750 foo
