migration-cloud-service-mode-set

Updates the data transfer mode and restarts Cloud Gateway process

Syntax

migration-cloud-service-mode-set --mgmnt-ports | --agg-ports | --agg-ports-zero-copy

Description

Sets the data transfer mode of Data Migrator to Cloud process for all physical nodes.

--agg-ports, -a

In this mode Cloud Gateway will use HNAS aggregate ports for its connections to cloud servers.

--agg-ports-zero-copy, -z

In this mode Cloud Gateway will use HNAS aggregate ports for its connections to cloud servers. This mode should give the best performance since it doesn't copy the data from kernel to user space. However, the only cloud provider that can benefit from this mode is HCP when configured to transfer data with HTTP protocol.

All other vendors, including HCPS3, when in this mode, will fall back to use aggregate ports and will behave the same way as '-a' option.

An additional mode, --mgmnt-ports (or -m), is unsupported for production use because these ports are architecturally unsuited to high bandwidth data movement. It's of no use on the Unified platform, where the management ports cannot communicate outside the chassis.

Examples

server$:migration-cloud-service-mode-set --agg-ports
Cloud Gateway is not running.

Cloud gateway transfers data via aggregate ports

Sets the Data Migrator to Cloud process to transfer data via HNAS aggregate ports.

Applies To

Cluster wide
