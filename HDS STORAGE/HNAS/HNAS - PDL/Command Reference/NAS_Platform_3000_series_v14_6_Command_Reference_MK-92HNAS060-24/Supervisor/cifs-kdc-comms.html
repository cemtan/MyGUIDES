cifs-kdc-comms

List / change CIFS KDC communications type.

Syntax

cifs-kdc-comms [  -t  tcp-always | udp-default  ]

Description

The cifs-kdc-comms command allows you to configure, on a per-EVS basis, whether the server uses TCP or UDP when communicating with a kerberos KDC on behalf of CIFS protocol operations (e.g., DC connections, cifs-name DDNS registration).

By default, kerberos requests initiated by the server on behalf of CIFS operations will use TCP communications only. This may be changed so that the server first uses UDP, switching to TCP only when kerberos credential size exceeds the capacity of UDP (typically when the authenticated CIFS account has many group memberships).

If multiple EVSs are configured to use common security settings (see evs-security) then those EVSs will share a single cifs-kdc-comms setting.

Changes made by this command take effect immediately.

Examples

To display the server's present CIFS kerberos transport selection:

cifs-kdc-comms

To select TCP exclusively for the server's CIFS kerberos transport:

cifs-kdc-comms -t tcp-always

To select UDP, with TCP used only for large kerberos credentials:

cifs-kdc-comms -t udp-default

Applies To

EVS security context

See Also

cifs-dc cifs-ddns cifs-name evs-security

Privilege Level

Supervisor
