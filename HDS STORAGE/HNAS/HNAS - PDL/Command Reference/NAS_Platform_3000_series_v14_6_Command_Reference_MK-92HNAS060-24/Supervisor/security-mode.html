security-mode

Get/Set security mode

Syntax

security-mode 
	get [-f <file-system> [-i <vivol-name>]]
	set [-f <file-system> [-i <vivol-name>]] mixed|unix
	clear <file-system> [<vivol-name>]

Description

The security-mode command is used to set the file system security mode for the current EVS security context, a file system or a virtual volume.

Unless explicitly configured, a file system will inherit the configuration of the EVS it's assigned to, and a virtual volume will inherit the configuration of its file system.

Two security modes are available - Mixed and Unix. In Mixed security mode, files and directories can have security descriptors to describe their access rights, as used by Microsoft Windows. This security mode usually provides the best 'experience' for CIFS clients. In Unix security mode, files and directories cannot have security descriptors, so instead rely on the Unix UID, GID and mode to store access rights. This security mode usually provides the best 'experience' for Unix clients.

Note that CIFS clients can still access file systems in Unix security mode. However, if they apply complicated security descriptors to files, they may not be stored or enforced as expected. This is because the Unix UID, GID and mode can't store the same amount of complex security information as a security descriptor.

Similarly, Unix clients can still access file systems in Mixed security mode. However, under certain circumstances access may be denied when the user expects it to be allowed. This is again because the Unix UID, GID and mode returned to the Unix client can't report the same amount of complex security information as may be applied to the file by its security descriptor.

Usage:

security-mode get [OPTION]...

Display the current security mode. If no options are provided, the security mode for the current EVS security context (cluster-wide or per-EVS) is displayed.

-f, --file-system <file-system>

Display the security mode for the given file system.

-i, --vivol-name <vivol>

Display the security mode for the given virtual volume.

security-mode set [OPTION]... mixed|unix

Set the security mode. If no file system or virtual volume is provided, the change is applied to the current EVS security context (cluster-wide or per-EVS).

-f, --file-system <file-system>

Set the security mode for the given file system.

-i, --vivol-name <vivol>

Set the security mode for the given virtual volume.

mixed

Set the security mode to Mixed.

unix

Set the security mode to Unix.

security-mode clear <file-system> [<vivol-name>]

Clear the security mode for the given file system. The security mode for the current EVS will then apply to this file system.

<vivol-name>

Clear the security mode for the given virtual volume. The security mode for the virtual volume's file system or, if it doesn't have a security mode, the current EVS, will then apply to this virtual volume.

Applies To

EVS security context

Privilege Level

Supervisor
