span-translate-offset

Translate span and offset into the same offset in other formats

Syntax

span-translate-offset [--dump <nr-bytes> [--verbose] [--timeout <nr-seconds>]] <span> <span-offset-in-bytes>

Description

This command is intended for use by your support provider. Given a span instance name and an offset into the span, it prints several other representations of the same offset: as an offset into a system drive, as an offset into a chunk within the span, and so on.

See 'filesystem-translate-offset' for an explanation of the output.

Specifying '--dump' and a byte-count will read the corresponding data directly from the storage (bypassing the server's caches) and hexdump it on the console. The command always dumps a whole number of disk blocks, and so '--dump 0' will dump the whole of the disk block that contains the offset you specify. If you add '--verbose', the command prints the system drive device ID and offset before each block and dumps the data in full; otherwise, repeated lines are suppressed. '--verbose' implies '--dump'. This command can't dump more than a mebibyte (that is, 2**20 bytes) or beyond the end of a span; to dump further, issue another '*-translate-offset' command.

The '--timeout' switch limits the amount of time that the command spends dumping data. If the specified time is exceeded, the command will abort. Without '--dump', '--timeout' is ignored. The default timeout is 120 seconds.

Space occupied by a filesystem in the recycle bin will be shown as unused. If necessary, undelete a filesystem in order to see offsets into it.

Examples

See the examples under 'sd-translate-offset'.
