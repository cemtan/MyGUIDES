security-decode-file

Displays a file's security information.

Syntax

security-decode-file <file name or handle>

Description

The security-decode-file command is used to display information about a file's security attributes.

When a security descriptor is reported to a network client the server attempts to map all principals in the SD to equivalent principals in the client's security model. When returning results to an SMB client, for example, "NFSv4 user\bob@example.com" or "Unix user\518" would be translated to something like "PROTOCOLS\bob", if such a mapping is available.

This command reports a file's security descriptor in two forms, firstly how it appears on disk (or for files without an SD, how it would be derived from the file's Unix security) and secondly how it would appear to an SMB client.
