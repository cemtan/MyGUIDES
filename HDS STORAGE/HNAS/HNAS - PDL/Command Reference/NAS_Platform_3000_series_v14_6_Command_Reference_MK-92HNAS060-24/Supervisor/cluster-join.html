cluster-join

Join an existing cluster

Syntax

cluster-join [-c | --confirm] [-a | --admin_ip] [-i | --ip <ipaddr>] <cluster name>

Description

The cluster-join command will join a server to an existing cluster. The server will automatically reboot as a result of this command. In order to join a cluster, the server must have a cluster node IP address; this can be previously assigned with the "ipaddr" command, or the server's administrative IP address can be re-used via the "-a" switch. The server's configuration (including all EVSs and file systems) will be removed when the node joins the cluster and replaced with the cluster's existing configuration. In case of metro cluster where the cluster nodes reside on different subnets, "-i" option needs to be used to specify the target cluster node IP address.

See the 'cluster-interfaces' man page for guidelines on configuring the cluster network.

Usage:

-c, --confirm

Confirms joining the cluster.

-a, --admin_ip

Uses the current Admin IP address as the new Cluster Node address. If a Cluster Node address already exists, this option will be ignored and the Cluster Node address will be used to join the Cluster.

-i, --ip <ipaddr>

Specifies the cluster node IP address of one of the nodes in the target cluster.

<cluster name>

Specifies the name of the cluster.

Examples

server:$ cluster-join MY-CLUSTER
This command will remove the configuration and reboot the system.
Do you want to continue?(Y/N)[N]:
y


server:$ cluster-join -a -i 10.20.1.10 M-CLUSTER
This command will remove the configuration and reboot the system.
Do you want to continue?(Y/N)[N]:
y

Applies To

Cluster node
