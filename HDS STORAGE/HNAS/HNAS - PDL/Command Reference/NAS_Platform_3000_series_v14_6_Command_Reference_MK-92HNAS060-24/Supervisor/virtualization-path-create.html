virtualization-path-create

Creates a virtualization path for the specified target.

Syntax

virtualization-path-create [--characterset {latin1|utf8}] [--port <port>] [--mountport <mountport>] -t <fs label> [-v <vivol>] [-d <root of association>] [--with-metadata-ext-copy] -n <association name> -s <source>

Description

Creates a virtualization path association between a remote NFS export source and a local file system, vivol or directory target.

To ensure data can be managed correctly between the Legacy NAS and the HNAS, it is recommended that the mount on the Legacy NAS is exported with rw, sync, no_subtree_check, no_root_squash. It is also advised that due to no_root_squash, the mount is available only to the HNAS.

Options:

--characterset <latin1|utf8>

Character set of the legacy NAS source. The default is utf8.

--port <port>

NFS port of the legacy NAS source.

--mountport <mountport>

Mount port of the legacy NAS source.

-t, --target <fs label>

Target file system. Selected filesystem has to be empty if the directory or Virtual Volume is not selected for the association being created.

-v, --vivol <vivol>

Name of target Virtual Volume (optional). The vivol has to be empty, otherwise creation of the association will fail.

-d, --dir <directory>

Directory relative to the vivol or the filesystem in which to create the association (optional). Selected directory must exist and has to be empty, otherwise creation of the association will fail.

-n, --name <association name>

Logical name of the association.

-s, --source <legacy NAS path URI>

Legacy NAS path URI. The URI can contain a hostname, an IPv4 literal address, or an IPv6 literal address.

--with-metadata-ext-copy

Enable metadata external copy mode. Refer to your Support Provider for further details.

The target entity (file-system, vivol or directory) has to be empty prior to creation of a new association. It isn't possible to re-create the virtualization path association if any artifacts from previous virtualization attempts remain there.

Examples

virtualization-path-create -t fs1 -n assoc1 -s nfs://legacy.example.com/example_fs/example_dir -d example_dir

Creates a virtualization path association to the directory example_dir on filesystem fs1 from the legacy NAS path nfs://legacy.example.com/example_fs/example_dir.

virtualization-path-create --characterset latin1 -t fs1 -n assoc1 -s nfs://legacy.example.com/example_fs

Creates a virtualization path to file system fs1 from the legacy NAS path nfs://legacy.example.com/example_fs using character set Latin-1.
