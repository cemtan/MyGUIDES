file-clone-declone

Declone a file's unnamed stream.

Syntax

file-clone-declone [-f <file-system>] <clone>

Description

Takes a cloned file and removes the dependency on its Snapshot File object, turning it into a normal file. This involves copying data in shared blocks into new unshared blocks.

A de-cloning task completes asynchronously, so users should consult the event log to discover its result. A de-cloning task will be aborted if the file it's operating on is re-cloned or its file system is unmounted.

Currently only one file can be de-cloned at a time.

De-cloning is not compatible with object-based snapshots.

-f, --file-system

The file system to work on. The currently selected file system is used if this option is not specified.

<clone>

The path of the file to declone.

Examples

To declone /tmp/file-clone

$ file-clone-declone /tmp/file-clone

Applies To

File system
