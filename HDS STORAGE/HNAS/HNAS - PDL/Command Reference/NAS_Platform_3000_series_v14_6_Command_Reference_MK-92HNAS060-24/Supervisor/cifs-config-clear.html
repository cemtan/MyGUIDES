cifs-config-clear

Clear all CIFS serving names for the EVS in context.

Syntax

cifs-config-clear [ --confirm ]

Description

This command directly clears all CIFS name records for the EVS from the server's registry. If the per EVS security status of the EVS is 'individual' as opposed to 'global' it also clears its domain configuration data. It is not intended for routine operation; the cifs-name command should be used instead.

cifs-config-clear makes no attempt to communicate with domain controllers for any affected domains in order to remove machine account records corresponding to the removed CIFS names.

Because this command only affects the server configuration stored in the registry, existing CIFS names cached for an online EVS will remain present until their host EVS is restarted. However, since domain trust account credentials are drawn directly from the (cleared) configuration, DC connectivity will be lost and further CIFS operation will be problematic.

If the --confirm option is not supplied, the user will be prompted to confirm the cifs-config-clear request.

Examples

To remove CIFS-name and (conditionally) domain configuration data from EVS 3 in the cluster:

vn 3 cifs-config-clear

To remove CIFS-name and (conditionally) domain configuration data from all EVSs in the cluster:

for-each-evs cifs-config-clear --confirm

Applies To

EVS security context

See Also
