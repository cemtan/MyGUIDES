migration-expand-external-path

Adds new targets to an external migration path.

Syntax

migration-expand-external-path -s <fs label> [-v <vivol>] [-A] [-H] [-R] [-W] [--type <type>] [-u <user> [-p <password>]] -n <logical name> -t <target> [--force] [--use-ignore-file]

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol>

Name of source Virtual Volume (optional).

-W, --WORM

Flag this target as being a system with WORM behavior (optional).

-R, --RAIN

Flag this target as being a system with RAIN behavior (optional).

-A, --HCAP-archive

Flags this target as being a Hitachi Content Archive Platform, HCAP (optional).

-T, --type { HCAP2 | HCP-default-namespace | HCP-namespace }

Flags this target as being of the type specified. Note that --type HCAP2 is equivalent to --HCAP-archive.

-H, --hash

Turn on hash generation for files as they are migrated (optional).

-n, --name <logical name>

Logical name of the migration target.

-t, --target <external path URI>

Protocol specific external path.

-F, --force

Force the addition of the external target even when it is not accessible.

-I, --use-ignore-file

Attempt to find and use an ignorefile on the external target. See migration-add-external-path for more information on ignore files.

-u, --user <username>

Specify the data access account username for an HCP namespace target.

-p, --password <password>

Specify the data access account password for an HCP namespace target. If this option is not supplied on the command line, the command will prompt for the password interactively, in which case the password will not be echoed to the terminal.

Adds an additional external migration target to an external data migration path.

The options are the same as for the migration-add-external-path command.

Examples

migration-expand-external-path -s fs1 -W -H -n Archive2 -t nfs://second-archive.example.com/fcfs_data/example_archive

Adds the /fcfs_data/example_archive mount point on second-archive.example.com as an external migration target with the logical name "Archive2" to the existing migration path from fs1.

Applies To

EVS
