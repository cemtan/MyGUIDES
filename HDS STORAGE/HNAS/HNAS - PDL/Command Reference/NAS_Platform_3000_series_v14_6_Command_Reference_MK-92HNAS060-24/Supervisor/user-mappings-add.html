user-mappings-add

Creates a user mapping

Syntax

user-mappings-add     [--unix-name <name>] [--unix-id <id>] [--nt-name <name>]
    [--nt-id <id>] [--nfsv4-name <name>] [--kerberos-name <name>]
    [--no-unix-mapping] [--no-nt-mapping] [--no-nfsv4-mapping]
    [--no-kerberos-mapping]

Description

Adds a new entry to the user mappings table.

Options:

--unix-name

Sets the Unix name to be used for the new entry

--unix-id

Sets the Unix user id for the new entry

--nt-name

Sets the NT name (of the form DOMAIN\user) for the new entry

--nt-id

Sets the NT SID for the new entry

--nfsv4-name

Sets the NFSv4 name (of the form user@domain) for the new entry

--kerberos-name

Sets the Kerberos principal (of the form user@realm) for the new entry

--no-unix-mapping

The entry will never have a valid Unix mapping

--no-nt-mapping

The entry will never have a valid NT mapping

--no-nfsv4-mapping

The entry will never have a valid NFSv4 mapping

--no-kerberos-mapping

The entry will never have a valid Kerberos mapping

Any fields set from the command line will be marked as fixed and won't be changed by the auto mapper.

Any fields marked as having no mapping will be ignored by the auto mapper.

The auto mapper will attempt to fill in any fields that haven't been either explicitly set or marked as having no mapping.

If the server finds an existing entry that matches some of the fields set from the command line, the command will not create a new entry or modify the existing entry (use user-mappings-modify to modify an existing entry).

Examples

$: # Add the Unix user joe with a user id of 100 who maps to the NT user
$: # EXAMPLE\Joe Blogs, NFSv4 user joeblogs@example.com and Kerberos
$: # principal joeblogs@example.com
$: user-mappings-add --unix-name joe --unix-id 100 --nt-name "EXAMPLE\Joe Blogs" --nfsv4-name joeblogs@example.com --kerberos-name joeblogs@EXAMPLE.COM
Created entry
Unix name     : joe*
Unix user id  : 100*
NT name       : EXAMPLE\Joe Blogs*
NT id         : *unknown*
NFSv4 name    : joeblogs@example.com*
Kerberos id   : joeblogs@example.COM*

Applies To

EVS security context
