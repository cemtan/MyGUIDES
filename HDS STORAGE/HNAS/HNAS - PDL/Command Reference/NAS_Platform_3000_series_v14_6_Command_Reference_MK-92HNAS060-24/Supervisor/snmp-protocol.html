snmp-protocol

Report/configure SNMP Agent Protocol Usage

Syntax

snmp-protocol [--version snmpver] | [disable]
Alias:       snmpprot

Description

The snmp-protocol command manages the versions of the SNMP protocol the server will respond to. The server supports SNMPv1, SNMPv2c, SNMPv3, and bilingual operation, where both SNMPv1 and SNMPv2c are accepted.

snmp-protocol is used to display the protocols supported, enable or disable the agent, and change the protocols supported. The --version option followed by v1, v2, v3, or bi enables SNMPv1, SNMPv2c, SNMPv3, or bilingual operation respectively. The disable operation disables the agent entirely. If the agent is disabled and the --version option is used, the agent is automatically enabled with support for the protocol version specified as an argument to the --version option.

The shortcut -v can be used in place of --version.

Examples

Display protocol support:

snmp-protocol
Protocol:    Disabled

Enable agent access using SNMPv1:

snmp-protocol --version v1
snmp-protocol
Protocol:    SNMPv1

Enable agent access using SNMPv2c:

snmp-protocol --version v2
snmp-protocol
Protocol:    SNMPv2c

Enable agent access using SNMPv3:

snmp-protocol --version v3
snmp-protocol
Protocol:    SNMPv3

Enable agent access using either SNMPv1 or SNMPv2c:

snmp-protocol --version bi
snmp-protocol
Protocol:    Bilingual

Disable the agent (disallow any SNMP access):

snmp-protocol disable
snmp-protocol
Protocol:    Disabled

Applies To

Cluster wide
