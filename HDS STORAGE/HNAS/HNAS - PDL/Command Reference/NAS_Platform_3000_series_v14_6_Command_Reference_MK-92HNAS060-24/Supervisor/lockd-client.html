lockd-client

Indicates whether the specified client may have NLM state and allows state to be removed

Syntax

lockd-client [-u] <client IP addr>

Description

This command provides the ability to query the Network Lock Manager on the server to find out if a specified client may have byte-range locks, and additionally allows these locks to be removed. This is only applicable to NFS v2/v3, not NFSv4 as per protocol definition.

This command should only be used to repair the case where network client crashes and fails to clear its Network Lock Manager state on reboot. Clearing Network Lock Manager state held by an active client may cause applications to fail in an unexpected manner.

This command is only available at the support level.

Usage:

[-u]

The -u switch specifies that the Network Lock Manager state for the specified client should be removed.

<client IP addr>

The client is specified by an IP address in dot notation.

Examples

To find out whether the server may have Network Lock Manager state for client 10.1.1.17:

lockd-client 10.1.1.17
Specified client is known by NLM, and therefore may have lock state.

To remove the state:

lockd-client -u 10.1.1.17

Applies To

Cluster node

Privilege Level

Supervisor
