user-mappings-list

Lists users mappings

Syntax

user-mappings-list [--unix-name <name> | --unix-id <id> | --nt-name <name> | --nt-id <id> |
 --nfsv4-name <name> | --kerberos-name <name>]
[--fixed] [--invalid] [--fixed-unix] [--fixed-nt] [--fixed-nfsv4] [--fixed-kerberos] [--invalid-unix] [--invalid-nt] [--invalid-nfsv4] [--invalid-kerberos]

Description

List entries in the user mapping table.

Options:

--fixed

List entries where any field is fixed (a fixed field will not be changed by the auto mapper)

--fixed-unix

List entries where the Unix name/Unix id fields are fixed

--fixed-nt

List entries where the NT name/NT id fields are fixed

--fixed-nfsv4

List entries where the NFSv4 name is fixed

--fixed-kerberos

List entries where the Kerberos principal is fixed

--invalid

List entries with invalid fields (an invalid field can not be resolved by the auto mapper)

--invalid-unix

List entries where the Unix name/Unix id fields are invalid

--invalid-nt

List entries where the NT name/NT id fields are invalid

--invalid-nfsv4

List entries where the NFSv4 name is invalid

--invalid-kerberos

List entries where the Kerberos principal is invalid

--unix-name

Lists an entry with the given Unix name

--unix-id

Lists an entry with the given Unix id

--nt-name

Lists an entry with the given NT name

--nt-id

Lists an entry with the given NT id

--nfsv4-name

Lists an entry with the given NFSv4 name

--kerberos-name

Lists an entry with the given Kerberos principal

The results are output in a table. The Unix column shows the Unix name with the associated user id in brackets. The NT column shows the NT name in the form DOMAIN\User along with its NT SID in brackets. The NFSv4 column shows NFSv4 names in the form user@domain. Finally, the Kerberos field shows the Kerberos id in the form user@realm.

Any field with an asterisk (*) printed after it indicates the field has been marked fixed and will not be changed by the auto mapper. Any field that contains *unknown* indicates the auto mapper has been unable to resolve a value for that field. A field with a blank entry indicates the field has been flagged to say it will never contain a valid entry, and will not be changed by the auto mapper.

Unless you specify you want to list a single entry (by supplying the Unix/NT/NFSv4/Kerberos name/id) the output may be thousands of lines long. You may wish to slice it with grep, page it with less or check the size first with wc.

Examples

$: # List all mappings with fixed entries
$: user-mappings-list --fixed | less


                  Unix                                                                   NT           NFSv4      Kerberos
----------------------     ----------------------------------------------------------------     -----------     ---------
      nobody* (65534*)                              NT AUTHORITY\Anonymous Logon (S-1-5-7*)     ANONYMOUS@*
            root* (0*)
      joe* (*unknown*)                                                *unknown* (*unknown*)       *unknown*     *unknown*

Applies To

EVS security context
