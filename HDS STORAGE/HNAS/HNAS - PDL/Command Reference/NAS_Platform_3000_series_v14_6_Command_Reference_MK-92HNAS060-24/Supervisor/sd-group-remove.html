sd-group-remove

Remove system drives from groups

Syntax

sd-group-remove <SD-device-IDs-see-'man sd-spec'>...
Alias:       trimsdg

Description

Removes one or more system drives (SDs) from one or more SD groups. See the 'sd-group' man page for background.

Specify only the SDs you wish to remove from their groups. There is no need to specify the groups themselves; the server knows which group contains each SD.

The SDs you remove from their groups must not be used in open spans. (A span is open if any of its file systems is mounted or is being checked, fixed, etc.) SDs that remain in groups after this command is run are not subject to the same restriction.

Using 'sd-forget' to remove an unhealthy or deleted SD from the server's registry will automatically remove the SD from its SD group.

Removing all the SDs from a group (with either 'sd-group-remove' or 'sd-forget') will delete the group. There is no need for further manual cleanup.

Examples

server:$ sd-group-list -m
Serial: 0 1 2 4


Serial: 3 5 6


Serial: 7 8


Serial: 9


server:$ sd-group-remove 1 2 5 9
Success
server:$ sd-group-list
Serial: 0 4


Serial: 3 6


Serial: 7 8


server:$

Applies To

Cluster wide
