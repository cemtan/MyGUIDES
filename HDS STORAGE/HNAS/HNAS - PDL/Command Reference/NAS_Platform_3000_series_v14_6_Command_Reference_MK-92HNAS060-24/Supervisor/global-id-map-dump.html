global-id-map-dump

Dump current state of Global Id lists

Syntax

global-id-map-dump

Description

There are a number of objects on the NAS server which are tracked and cached based on a globally unique identifier. There are a series of maps of these globally unique identifiers which are used to retrieve objects based on their globally unique identifier. This command is used to print out all of these maps and hence the output can be quite large.

SCOPING

The scope that an object is cached at is dependent on that object's natural scoping. We list the scoping of each class of object in the TRACKED CLASSES section below. The command shows the scoping that applies to each map as it prints them. In most cases this is simply the evs identifier but in some cases it can be something else, e.g. the virtual volume identifier in the case of quotas.

Note that on a cluster what you see when you run this command will depend on the current evs, e.g. if evs 1 is in context you will see the maps for the cluster node that hosts evs 1 regardless of which cluster node you actually run the command on.

TRACKED CLASSES

The classes of objects which are currently tracked are as follows.

exports

Exports are scoped at the evs level.

shares

Shares are scoped at the evs level.

user mappings

User mappings are scoped within the current EVS security context.

group mappings

Group mappings share the same scoping as user mappings.

quotas

Quotas are scoped at the file system level.

Examples

Display all objects referenced by a globally unique identifier.

global-id-map-dump

Applies To

Cluster node
