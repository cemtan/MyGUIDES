fs-quarantine-add

Add an object to quarantine

Syntax

fs-quarantine-add [--is-directory] -f <file-system> -o <object-number> -r <reuse-count>

Description

When supplied with a file system, and object-number, and a reuse-count, this utility will add the object to quarantine. If the file system is currently mounted, then the quarantine will take effect immediately, else it will take effect the next time the file system is mounted.

Quarantine itself has a number of limitations. It is not persistent over reboot. Automatic quarantining of objects is only possible if the preceding volume failure was provoked by the OBJ chip (not, for example, if the failure was provoked by WDIR). Quarantining an object does not prevent auto-inquiry access. If a quarantined object has a corrupt root onode, then a subsequent access to that target will likely still cause a file system failure.

The fs-quarantine-add command is most useful when a problem object has been identified, but the object has not been automatically added to quarantine by software (e.g. in the case of a previous file system failure caused by a corrupt WDIR payload).

The is-directory switch must be specified if the target object is a directory, as the software cannot reliably identify the object type.

Examples

To add a directory with object-number 0x40000, reuse-count 0, to file system hippo:

fs-quarantine-add --is-directory -f hippo -o 0x40000 -r 0

To add a file with object-number 0x60000, reuse-count 1, to file system possum:

fs-quarantine-add -f possum -o 0x60000 -r 1

Applies To

EVS
