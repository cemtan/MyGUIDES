autoinquiry-config

Enables/disables autoinquiry persistently.

Syntax

autoinquiry-config [--type (all|nfs|smb1|smb2)] [--operation (all|nfs-all|getattr|lookup|read|readdir|readdirplus|readlink|access|commit|gns|smb1-all|smb1-read|smb1-flush|smb1-trans2-query-file] [--confirm] [on|off|default]

Description

The autoinquiry-config command is used to manage which protocols are subject to autoinquiry by the server.

Usage:

autoinquiry-config [--type <protocol_type>] [--operation <protocol_operation>] [--confirm] [<state>]

Options:

-t, --type <protocol_type>

Selects a protocol to configure. Valid choices are (all|nfs|smb1|smb2). The 'smb2' option specifies SMB2.x/3.x. Defaults to all.

-o, --operation <protocol_operation>

This allows DEV users to configure autoinquiry for individual operations; it is not available to any other user levels at the present time. Using the --type parameter will enable and disable supported operations as required. Selects a specific protocol operation to configure. Defaults to all.

for --type all

Valid choices are: all

for --type nfs

Valid choices are: (nfs-all|getattr|lookup|read|readdir|readdirplus|readlink|access|commit|gns)

for --type smb1

Valid choices are: (smb1-all|smb1-read|smb1-flush|smb1-trans2-query-file)

for --type smb2

Operation level control is not supported.

<state>

Turn autoinquiry on or off, or reset to the system default for the given protocol. If no state is given then the current state is shown. State must be the last parameter in a command.

--confirm

Bypass any warning messages which ask the user to confirm their actions on the terminal.

Examples

autoinquiry-config --type all

Displays the configuration details for all protocols

autoinquiry-config --type smb1 on

Enables autoinquiry for SMB1

autoinquiry-config --type all default

Restores the system default state for all protocols

Applies To

Cluster node
