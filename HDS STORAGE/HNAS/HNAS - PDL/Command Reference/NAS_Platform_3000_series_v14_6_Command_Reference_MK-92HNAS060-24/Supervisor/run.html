run

Run command(s) and send the output to email recipient(s)

Syntax

run [-m <mail-list>] <command-list>

Description

The run command allows you to schedule a command to be run immediately by cron and send the output via email.

run [-m <mail-list>] <command-list>
mail-list

The full email address of the user(s) to mail output to. Multiple users can be specified, in quotes, in the form "user1;user2".

Note: If no email addresses are specified, the output will be displayed in the console.

command-list

The command(s) that should be run.

e.g. "command -option argument"

Multiple commands can be specified, in quotes, in the form

"command1; command2"

Another way to send command output via email is to use the mail filter.

Examples

run -m support@example.com "ps -r;sleep 10;ps"

Applies To

Cluster wide
