local-group-disable-saving

Disables automatic saving of the local group table

Syntax

local-group-disable-saving

Description

The local-group- commands manage the local group table.

local-group-disable-saving

Disables automatic saving of the local group table

local-group-save

Saves the local group table immediately and synchronously

Saving of local group updates is automatic by default but delayed by a few seconds in case another update arrives. local-group-save does not return until persistence is ensured. Before release 5.0, extant credentials were synchronously reassessed when saving took place. This now happens in the background (and even if local-group-disable-saving is in effect). Where it is important to ensure that changes have been activated, the credentials-reassess command is now required. connection -v all and credentials-list can be used to inspect the extant credentials. Newly created credentials always use the latest copy of the local group table.

Disabling saving during batch updates used to be a significant performance win, especially in clusters. The automatic deferral of saving on each update obsoletes local-group-disable-saving, which may be removed in a future release. local-group-save retains some small utility for scripted testing scenarios where a reboot or migration of the target EVS is issued immediately after a local group update. local-group-save has the side effect of re-enabling automatic saving.

Examples

Most of the local group table manipulation functionality currently still resides in the localgroup command. See its page for examples.
