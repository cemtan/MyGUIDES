cifs-user-lookup-access

List / change anonymous client LSA access via the server.

Syntax

cifs-user-lookup-access [-e anonymous|authenticated]

Description

By default, the server refuses user lookup queries via the LSA service from anonymous client sessions (or "NULL session connections"). This command may be used to override that behavior. Doing so is not recommended, however, as it poses a potential security risk.

The cifs-user-lookup-access command provides control equivalent to that made available by the RestrictAnonymous LSA registry entry in Microsoft Windows Server products. See Microsoft Knowledge Base articles 246261 and 823659 for further background information.

The cifs-user-lookup-access setting is cluster-wide: there are not separate settings for each EVS.

Usage:

--enable, -e anonymous|authenticated

Set LSA access setting.

anonymous

Permits unknown clients to look up CIFS domain users and SIDs via the server's LSA RPC interface. This is a security risk and is disabled by default.

authenticated

Only allows authenticated clients to use the server's LSA RPC interface.

Examples

To display the current setting:

cifs-user-lookup-access

To allow only authenticated clients to lookup CIFS data via the server (the default):

cifs-user-lookup-access -e authenticated

To permit anonymous clients to lookup CIFS data via the server:

cifs-user-lookup-access -e anonymous

Applies To

Cluster wide

Privilege Level

Supervisor
