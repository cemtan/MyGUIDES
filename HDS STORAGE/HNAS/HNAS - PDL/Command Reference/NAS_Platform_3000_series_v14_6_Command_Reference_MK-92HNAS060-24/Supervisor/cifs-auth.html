cifs-auth

Enable/disable CIFS authentication for current EVS

Syntax

cifs-auth [on|off|list]

Description

The cifs-auth command allows you to configure, on a per-EVS basis, whether incoming CIFS connections are authenticated using a domain controller (DC). Except for the "cifs-auth list" form, the evs-select command must be used first to select which EVS the cifs-auth command will apply to.

By default, normal domain-based authentication is employed for CIFS connections to all EVSs. The cifs-auth command is provided for those sites that wish to use CIFS clients with the server in the absence of a domain controller.

When cifs-auth is set to "off" for a given EVS, incoming CIFS connection client credentials are ignored and all CIFS connections to that EVS appear to be from the "anonymous" user.

If multiple EVSs are configured to use common security settings (see evs-security) then those EVSs will share a single cifs-auth setting. In this case, the restrictions discussed above apply to an EVS using common security settings if the stated conditions are met by any EVS also using common security settings.

Changes made by this command take effect immediately.

Usage:

cifs-auth on

Enables authentication of incoming CIFS connections.

cifs-auth off

Disables authentication of incoming CIFS connections.

cifs-auth list

Lists authentication settings for all EVSs.

cifs-auth

Shows authentication settings for current EVS.

Applies To

EVS security context

See Also

cifs-name evs evs-security evs-select

Privilege Level

Supervisor
