migration-cloud-schedule-modify

Modify an external cloud migration schedule

Syntax

migration-cloud-schedule-modify <policy name> <schedule id> [ ( --run-type | -t ) <run-type> ] [ ( --next-run | -n ) <run-time>] [--max-duration | -d <max-duration>] [--enable | --disable]
where
  <run-type> = simulateOnly | onceOnly | daily | weekly
  <run-time> = YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]
  <max-duration> = [nD]hh:mm

Description

Modifies a schedule for cloud migration.

<policy name>

Name of the policy applied by this schedule.

<schedule id>

Schedule ID.

--next-run <run-time>

The next run time in the format YYYY-MM-DD{T| }HH:MM[:SS][+hh:mm].

--max-duration <limit>

The maximum amount of time a given migration run started by this schedule will be allowed to run. If this limit is hit then the migration will be halted until the next scheduled run-time, if any.

The limit is in the format [nD]HH:MM. For example, 1D12:30 is 1 day, 12 and one half hours (or 36.5 hours).

If the --max-duration option is not given, the default is no limit.

--run-type <run type>

Schedule run type, allowed values are:

onceOnly

Run a cloud migration once at the next run time.

daily

Run a cloud migration at the next run time, and at the same time every day after that.

weekly

Run a cloud migration at the next run time, and then at the same time once every week after that.

simulateOnly

Run a cloud migration detailed test once at the next run time. This exercises the selected policy, but no files are actually migrated.

The scheduler will run the cloud migration at or slightly after the next run time.

--enable

Enable the cloud migration schedule thus allowing it to be scheduled to run.

--disable

Disable the cloud migration schedule thus disallowing it from being scheduled to run.

Examples

migration-cloud-schedule-modify hcp-pol1 1 --next-run 2011-02-03T03:30:00-08:00 --run-type daily

Modifies the cloud migration schedule with the specified ID (1) for the policy named 'hcp-pol1' so that it is run once every day, beginning 3 March 2011 at 3:30am PST.
