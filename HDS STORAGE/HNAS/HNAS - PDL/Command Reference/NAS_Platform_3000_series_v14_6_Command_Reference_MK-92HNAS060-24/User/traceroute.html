traceroute

trace route to remote system

Syntax

traceroute [-4] [-6] [-f <TTL>] [-i <source-IP-address>[%<interface>]] [-I] [-m <MaxTTL>] [-p <BaseUDPPort>] <IP-address or hostname>[%<interface>] [packet-size]

Description

'traceroute' displays the route that packets traverse from the current host to reach the specified destination host.

It traces the route an IP packet would follow to the destination host by sending probe packets with an increasing TTL (time to live) and listening for an ICMP TIME EXCEEDED reply from a gateway until the host is reached. Three probes are sent for each TTL value and a line is printed showing the TTL, address of the gateway and the round trip time of each probe.

The probe packets sent to the host are either UDP (default) or ICMP ECHO requests. The optional 'packet-size' argument is the total size of the probe packet (default is 64 bytes), the actual packet size may be increased to include the required packet headers.

'traceroute6' is equivalent to 'traceroute -6'.

If multi-tenancy is enabled the 'source-IP-address' must be supplied with the -i option and the 'interface' must be supplied with either the destination host or the -i option.

To execute a traceroute operation within the context of a specific EVS see the 'traceroute-from-evs' command.

Options:

-4, -6

Explicitly select IPv4 or IPv6 tracerouting. By default, the program will try to resolve the specified host name and use the appropriate protocol; if the host name has both IPv4 and IPv6 addresses, 'traceroute' will use the IPv4 address.

-f <TTL>

Set the initial time-to-live. The default value is 1.

-I

Use ICMP ECHO requests rather than UDP.

-i <source-IP-address> | <interface> | <source-IP-address>%<interface>

Set the source IP address of the request to the supplied interface address. This may be a numeric IP address or the interface name; when specifying an IP address an interface suffix may also be included as '<IP address>%<interface>'. If an interface is specified both here and as part of the destination, then the two interface names must be the same.

-m <MaxTTL>

Set the maximum time-to-live to use. The default value is 30.

-p <BaseUDPPort>

Set the base UDP port to use. The default value is 33434.

Examples

To traceroute to a destination for a range of TTL values:

traceroute -f 5 -m 10 example.com

To traceroute using IPv6 to a destination via a specific source address and interface:

traceroute -6 -i 2001:DB8::1%ag1 example.com

Applies To

Cluster node
