man

Display a manual page

Syntax

man ([--plain-text | -p | --force-ansi | -a] [--width | -w <console-width>] [--apropos | -k] [--narrow | -n] (<man-page-name> | all)) | (--list-pages | -l)

Description

Displays manual ('man') pages. Most, but not all, describe the server's console commands. "man console", by contrast, displays a page containing a general description of the behavior and usage of the command line interface.

Options:

--list-pages, -l

Prints a list of all available man pages. If you use this switch, don't specify a man page name or any other switches.

--plain-text, -p

Suppresses the use of Ansi escape sequences to provide character formatting (bold, underlining and color). This switch is usually unnecessary, because the server's console driver is quite conservative when determining whether your console supports Ansi sequences or not.

--force-ansi, -a

Enables Ansi escape sequences even if the server's console driver believes your console doesn't support them.

--width, -w <console-width>

Tells 'man' the width of your console, in characters. Normally, the command tries to determine the console width for itself, and falls back on a default of 80 characters.

--narrow, -n

Makes less use of indenting. Useful for viewing man pages with sample output or other verbatim text that would otherwise be too wide to display in a narrow window.

--apropos, -k

Tries 'apropos' if no matching man page is found.

'man all' produces a list of all commands available at your current privilege level. You can, however, view man pages for console commands that you don't have the privilege to run.

The "applies to" section exists to help the user to apply the command to the right entity. The common values used in this section are "EVS", "EVS security context", "File system", "Cluster wide", "Cluster node", "Board" and "Console".

To run a command with EVS scope, the "evs-select" command, or an equivalent, must first be used. Such commands will usually be automatically forwarded to the cluster node hosting the EVS in context, as noted in the "needs" section of "man help". "selectfs" is used to set a file system context. File systems are associated with an EVS, so "selectfs" itself requires an EVS context.

A command with "Cluster node" scope will run on the current cluster node unless forwarded explicitly with "cn or, implicitly by "console-context". Note that such a command will not automatically run on the cluster node hosting the EVS in context.

When an EVS with "individual" "evs-security" is in context, an "EVS security context" command will affect just that EVS. When the context specifies an EVS with "global" "evs-security", or the Admin Service, such a command will affect every EVS sharing the cluster-wide security configuration.

Commands with a cluster-wide application can be run without formality, so on any cluster node and from any "console-context", unless the man page warns otherwise. On a multi-board platform, Board-specific commands will run on the current cluster node and on the current board type, which is usually the network interface module (nim), unless explicitly forwarded with "nim", "fsm", "sim" or similar. Commands flagged as Console-specific do not affect other console sessions.

Examples

man autodoc

Applies To

Cluster node

See Also

apropos autodoc cn console console-context evs-security evs-select help selectfs
