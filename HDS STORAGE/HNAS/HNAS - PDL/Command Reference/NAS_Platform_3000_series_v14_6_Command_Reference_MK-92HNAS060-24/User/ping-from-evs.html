ping-from-evs

Ping remote system

Syntax

ping-from-evs [-c <count>][-I <source-IP-address>[%interface]][-s <packetsize>] <IP-address or hostname>[%interface]

Description

ping-from-evs works in a similar way to ping except that it runs on the cluster node that the EVS currently resides on. If routing-by-evs is enabled, then the choice of source address is limited to the source addresses configured on the EVS. If routing-by-evs is disabled, then the choice of source address can be any of the source addresses currently hosted on the cluster node hosting the EVS.

ping-from-evs determines whether a given host is alive by sending a number of ICMP protocol ECHO_REQUEST datagrams using the IPv4 protocol.

Options:

-c <count>

The number of ECHO_REQUESTS to send. The default is 4; the minimum is 1.

-I <source-ip-address>[%interface]

Set the source IP address of the request to the supplied IP address. An optional source interface name may be supplied. If an interface is specified both here and as part of the destination, then the two interface names must be the same.

-s <packetsize>

Set the number of data bytes to be sent. The default is 56 which means that 64 bytes of IPv4 data is sent over the wire.

Examples

To ping an IPv4 address 5 times:

vn 1 ping-from-evs -c 5 172.18.0.170
172.18.0.170...
64 bytes from 172.18.0.170: icmp_seq=0 time=1 ms
64 bytes from 172.18.0.170: icmp_seq=1 time <1 ms
64 bytes from 172.18.0.170: icmp_seq=2 time <1 ms
64 bytes from 172.18.0.170: icmp_seq=3 time <1 ms
64 bytes from 172.18.0.170: icmp_seq=4 time <1 ms
--
5 packets transmitted, 5 packets received, 0% packet loss
round-trip min/avg/max = 0/0/1 ms

Applies To

EVS
