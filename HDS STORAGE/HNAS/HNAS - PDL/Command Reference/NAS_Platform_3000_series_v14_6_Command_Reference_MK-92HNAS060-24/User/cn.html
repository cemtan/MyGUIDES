cn

Run command on specific pnode

Syntax

cn <pnodeid>|(-e <evs-specifier>) <command>
Alias:       pn

Description

Run the specified command on a cluster node other than the local one. The cluster node can be specified by either its cluster node ID or an EVS hosted on the cluster node. This is useful for commands that have specific state per cluster node.

Options:

-e <evs-specifier>

The name or the ID of an EVS hosted on the cluster node on which the command should be run. If there is no token following <evs-specifier>, the pnode id of the cluster node hosting the specified EVS will be displayed.

pnodeid

The ID of the cluster node on which the command should be run

command

The command to run

You may specify "all" as the pnodeid to run the command sequentially on each cluster node

Examples

To run nfs-stats on pnode 1:

   cn 1 nfs-stats

To run uptime on all all cluster nodes:

   cn all uptime

To find out the pnode hosting EVS 1:

   cn -e 1

To run uptime on the pnode hosting EVS 1:

   cn -e 1 uptime
