grep

Print lines matching a pattern

Syntax

... | grep [OPTIONS] PATTERN
	[-A|--after-context <NUM>] [-B|--before-context <NUM>] [-C|--context <NUM>] [-i] [-n|--line-number] [-v] [-w] [-s|--show-all] [-p]

Description

Command output may be piped through grep to select lines containing the given POSIX extended regular expression (equivalent to Unix's "grep -E"). By default, only lines containing a match are printed.

The pattern must not be more than 256 bytes long. If non-ASCII characters are used, this length will correspond to fewer than 256 characters. If the maximum length is exceeded, grep and the command piped through it will refuse to run.

Options are:

-A <NUM>, --after-context <NUM>

Print NUM lines of trailing context after matching lines.

-B <NUM>, --before-context <NUM>

Print NUM lines of leading context before matching lines.

-C <NUM>, --context <NUM>

Print NUM lines of output context.

-i

Ignore case.

-n, --line-number

Prefix each line of output with the 1-based line number within its standard input.

-v

Output non-matching lines.

-w

Match whole words only. Characters that form a word are letters, digits or underscores.

-s, --show-all

Show all lines, color the matching ones and count the number of matching lines.

-p

Paragraph mode; use \n\n as the "line" separator.

Piped output is not colored.

Examples

To look for events pertaining to filesystems Accounts or Engineering:

event-log-show | grep 'Accounts|Engineering'


server:$ echo hello | grep --show-all world
hello
Matching lines: 0
server:$
