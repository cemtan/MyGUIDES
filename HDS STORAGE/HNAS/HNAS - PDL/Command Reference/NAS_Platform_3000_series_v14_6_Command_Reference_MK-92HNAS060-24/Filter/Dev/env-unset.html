env-unset

Unset an environment variable

Syntax

... | env-unset <name>

Description

This command unsets a BALI console environment variable, making it undefined.

A BALI console environment variable name is a string of characters. The first character in the name must be an alphabetic letter or an underscore. Subsequent characters can be alphanumeric or underscores.

Options:

<name>

Specify the name of the BALI environment variable to unset.

When run as a filter, env-unset interprets each line of the output of the preceding command as the name of an environment variable to be unset. If the line contains an equal sign, the name is determined by discarding all characters from the equal sign to the end of the line. This behavior is supported so that the filter can be used to process the output of the 'env' command.

Examples

Define a new BALI environment variable named COUNT and set its value to 4:

   $ env-set COUNT 4

List all BALI environment variables:

   $ env
   COUNT=4

Unset this BALI environment variable:

   $ env-unset COUNT

Verify that COUNT is now undefined:

   $ env --name COUNT
   env: Undefined variable name: COUNT

Unset all BALI environment variables whose names start with "STR":

   $ env | grep "^STR" | env-unset

See Also
