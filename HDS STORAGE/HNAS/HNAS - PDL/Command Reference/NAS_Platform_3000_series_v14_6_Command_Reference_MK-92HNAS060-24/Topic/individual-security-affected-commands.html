individual-security-affected-commands

Description

This manual page details commands that are known to be affected by changing an EVS's configuration from global to individual security, or into a multi-tenant configuration.

If these commands have been used, it is strongly advised that the settings associated with these commands prior to conversion are checked, so that they can be re-added to the converted EVSs.

  • bypass-permissions-check
  • default-owner-policy
  • ignore-unix-security-ops
  • posix-chown-restricted
  • security-mode
  • umask-directory-set
  • umask-file-set
