fs-space-reporting-based-solely-on-vivol-quota

Get/Set whether file system capacity/free space reporting for mount points within the vivol are based solely on the vivol's quota

Syntax

fs-space-reporting-based-solely-on-vivol-quota [--on|--off] <file-system> <vivol-name>

Description

Configures whether file system capacity/free space reporting for mount points within a vivol are based solely on the vivol's quota.

When this option is enabled and a vivol quota is created, capacity/free space counts returned to clients are derived solely from the vivol quota. N.B. When configured, this feature only affects clients that have mounted an export or share within a vivol. A potential use case would be when data migration is configured. In that scenario the primary file system could ingest more data than it has capacity itself for. The administrator can define a quota for a vivol based on available capacity of migration target(s) and enable this feature so that the capacity defined by the quota is reported to protocol clients rather than the primary file system capacity/free space.

Usage:

fs-space-reporting-based-solely-on-vivol-quota [--on|--off] <file-system> <vivol-name>

By default this option is disabled. It may be enabled using --on switch, and disabled again using --off switch. if either --on or --off switch is not specified, the command displays the current setting.

--on

Enable this option.

--off

Disable this option.

<file-system>

The file system name to work on.

<vivol-name>

The name of the vivol.

Examples

To enable this option for the virtual volume 'vivol0' that resides on the file system 'fs0':

  fs-space-reporting-based-solely-on-vivol-quota --on fs0 vivol0

To disable this option for the virtual volume 'vivol0' that resides on the file system 'fs0':

  fs-space-reporting-based-solely-on-vivol-quota --off fs0 vivol0

To get the current setting for the virtual volume 'vivol0' that resides on the file system 'fs0':

  fs-space-reporting-based-solely-on-vivol-quota fs0 vivol0

Applies To

Virtual volume
