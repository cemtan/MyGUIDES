led-identify-node

Use server status LEDs to identify physical node

Syntax

led-identify-node [<pnode-id>|--disable|--cluster] [--dont-wait]

Description

Make the server status LEDs flash in order to physically identify the node. Both status and fault LEDs will flash.

Parameters:

pnode-id

The identifier of the physical node to identify. Node ID 0 is equivalent to --disable. Node ID 255 is equivalent to --cluster.

--disable

Turn off LED node identification.

--cluster

Make all nodes in the cluster flash their pnode number. Each node will flash a number of times equal to their pnode ID. Each flash will be 200ms on then 200ms off. There will be a 1s delay between each count.

Options:

--dont-wait

Normally this command will run and wait for the user to press a key to stop it. This option ignores this and makes the LEDs flash indefinitely, or until this command is run again to disable it.

Examples

led-identify-node

Displays the identifier of the node that is currently flashing.

led-identify-node 1

Make the server status LEDs on node 1 flash.

led-identify-node 2 --dont-wait

Make the server status LEDs on node 2 flash indefinitely.

led-identify-node --disable

Stop the server status LEDs on any node from flashing.

led-identify-node --cluster

Make the server status LEDs on all nodes flash their pnode number.

Privilege Level

Supervisor
