virtualization-path-delete

Deletes the virtualization path for the specified target.

Syntax

virtualization-path-delete -t <fs label> -n <association> [--force] [--confirm]

Description

Deletes a virtualization path association between a remote NAS NFS export source and a local file system or vivol target.

Options:

-t, --target <fs label>

Target file system.

-n, --name <association>

Name of the association. An empty association name matches all associations for the specified target file system, allowing them all to be deleted at once.

--force

Force the source path to be deleted even if that would lead to files becoming inaccessible.

-c, --confirm

Confirm that the deletion is to be carried out without any further prompt.

In migration mode, a check will be made to see if there are still files awaiting migration. If there are, the command will fail unless the --force option has been supplied. In configuration or virtualization mode the command will always fail unless the --force option has been supplied. Note that existing files will become inaccessible after forced deletion. The --force option will cause the virtualization path to be removed even if some files must still be accessed on the legacy NAS and new files may fail to be created, which will lead to IO errors after the virtualization path has been deleted.

This operation is also irreversible. If you delete the association, you cannot recover it, and any virtualization progress will be lost.

Examples

virtualization-path-delete -t fs1 -n assoc1

Delete the virtualization path for assoc1 on file system fs1.

virtualization-path-delete -t Accounts -n HR --force

Delete the virtualization path for the association named HR on file system Accounts, even if some files must still be accessed on the legacy NAS, which will lead to IO errors.
