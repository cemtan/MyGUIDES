migration-cloud-account-modify

Modify an external cloud migration account

Syntax

migration-cloud-account-modify --account-name <account name> --credentials-key <credentials key> --credentials-secret <credentials secret> [--vendor <HCP:HCPS3>] [--server-name <server name>] [--description <description>]

Description

Modifies an account for cloud migration.

--account-name, -a <account name>

The name of the account created with migration-cloud-account-create. This command does not change the name; this option is used to specify the account to modify.

--credentials-key, -u <credentials key>

Specify this option to change the key (user name) for this account.

--credentials-secret, -p <credentials secret>

Specify this option to change the secret (password) for this account.

--vendor, -v <HCP:HCPS3>

Specify this option to change the way files will be uploaded to HCP server. To use S3 multi-part upload feature set vendor to be HCPS3. To use RESTful API set vendor to be HCP. This operation only allows switching between HCP and HCPS3 at this time.

--server-name, -s <server name>

Specify this option to change the server name for this account.

--description, -d <description>

Specify this option to change the description for this account.

Examples

migration-cloud-account-modify --account-name HNAS-HCPTarget --credentials-key newAccessKey

Modifies account HNAS-HCPTarget to use 'newAccessKey' credentials. The user will be prompted for the new secret key upon entering this command.

Applies To

Global security context
