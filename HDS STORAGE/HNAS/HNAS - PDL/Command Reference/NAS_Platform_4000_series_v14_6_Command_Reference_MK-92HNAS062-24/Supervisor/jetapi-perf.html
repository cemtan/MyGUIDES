jetapi-perf

Display JetAPI performance statistics

Syntax

jetapi-perf [[--reset] | [--summary] [--verbose] [--access-type (all|local|localcvl|remotecvl|legacynas)] [--type (elapsed|cpu|nvramreservation|lock|block|wfstime|wfscalls|remotetime|remotecalls|fscontext|security|accessandregionlock|readoplock|writeoplock|handleoplock|gns|renamelock|inq|requestsignatureverification|throughputcontrol|quotacheck|quotarecord|legacynastime|legacynascalls|sharedvirtualdisk|decrypt|encrypt|all)] [--file-system <label> | --all-file-systems]] [<request-pattern>]

Description

The jetapi-perf command displays basic statistics about JetAPI file system activity. The JetAPI is used by external management applications to control certain features of the server.

Usage:

jetapi-perf [OPTION...] [<request-pattern>]

--access-type, -T all | local | localcvl | remotecvl | legacynas

Only displays statistics for accesses to objects of the given type.

--all-file-systems, -a

Show per file system stats for all file systems.

--file-system, -f

Show per file system stats for the specified file system.

--reset, -r

Reset the statistics.

--summary, -s

Only display a summary.

--type, -t elapsed | cpu | nvramreservation | lock | block | wfstime | wfscalls | remotetime | remotecalls | fscontext | security | accessandregionlock | readoplock | writeoplock | handleoplock | gns | renamelock | inq | requestsignatureverification | throughputcontrol | quotacheck | quotarecord | legacynastime | legacynascalls | sharedvirtualdisk | decrypt| encrypt | all

Only display statistics of the given type.

--verbose, -v

Display verbose information.

<request-pattern>

Only show statistics for requests with names that contain the given string.

Applies To

Cluster node
