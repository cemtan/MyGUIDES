ndmp-device-trace

Dump the set of currently enabled device traces or the trace for the specified device

Syntax

ndmp-device-trace [ [--dblog on|off] -i <device number> | -n <device name> ]

Description

Displays the NDMP device subsystems that are currently being traced, or the trace output for a specified device. Not all NDMP device subsystems are traced by default because there is a performance penalty in doing so for certain subsystems. See "ndmp-device-trace-add" for further information.

The data gathered by the NDMP device trace is complementary to that gathered using "ndmp-trace-mask". Device traces consume a fixed amount of processor memory and will therefore wrap after a certain amount of device activity.

Options:

--device-name, -n <device name>

Display for the device specified by its name.

--device-number, -i <device number>

Display for the device specified by its number. The device number for a specific device is displayed in the output of the "ndmp-devices-list" command.

--dblog on|off

A device must be specified to use this option. Enables/disables output of the device's trace to the dblog in real time. By default the trace output is stored in memory and only output to the console when requested using this command specifying a device.

Examples

Display list of NDMP device subsystems currently being traced:

ndmp-device-trace

Display the trace for the specified device:

ndmp-device-trace  --device-name "Tape Drive 2"

Applies To

Cluster node

See Also

ndmp-device-trace-add ndmp-device-trace-remove ndmp-devices-list ndmp-session-trace ndmp-trace-mask

Privilege Level

Supervisor
