event-log-show

Show event log

Syntax

event-log-show [--oldfirst | --from <ISO-8601-start-time>] [--category (s|c|a)] [--info] [--warn] [--severe] [--all] [--explain] [--clusternode <cluster-node>] [--max-events <max-events>]

Description

The event-log-show command displays the event log with various filters

Options:

-o, --oldfirst

Lists event oldest-newest (default: newest-oldest)

-t, --category

Displays event category. 'cat' specifies category:

s

system

c

security

a

all

-i, --info

Displays 'Information' class events

-w, --warn

Displays 'Warning' class events

-s, --severe

Displays 'Severe' class events

-a, --all

Displays all events

-u, --clusternode <cluster-node>

Shows events on a given cluster node

-x, --explain

Prints the cause and resolution for each event

-f, --from <ISO-8601-start-time>

Makes the command terminate as soon as it encounters an event older than the specified timestamp, which should be specified in the same format that appears in event listings and the output of 'settime'. This is more efficient than piping through 'head'. However, the --from switch cannot be used alongside --oldfirst.

-m, --max-events <max-number-of-events>

Prints no more than the specified number of events. (It will print fewer if the event log, after filtering, contains fewer events than specified here.) This is more efficient than piping through 'head'.

Examples

To show severe events from cluster node 2:

event-log-show -s -u 2

To show recent events, adding causes and resolutions:

event-log-show --explain --from '2013-01-16 16:27:02'
event-log-show --explain --from 2013-01-16T16:27:02

Notice that quote marks are required in the first of these commands, because the timestamp contains a space. If the 'T' format is used, quote marks are optional.

Applies To

Cluster node
