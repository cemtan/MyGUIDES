file-filtering

Control the file filtering interface

Syntax

file-filtering [--on|--off] [-b <ip address>] [-d <ip address>] [-r]

Description

The file-filtering command is used to manage the settings of the interface to supported file filtering agents.

If no options are provided, the command displays the list of file filtering agents that are currently registered and reports the current statistics.

Options:

--on

Turn the file filtering agent interface on.

Note: Registration of NTP and NSS Software agents is mutually exclusive. Only one type of file filtering server is permitted to be registered at any one time.

--off

Turn the file filtering agent interface off.

-b, --enable <ip address>

Where a file filtering agent is registered, but for some reason is in a failed state, it can be reset from the failed state using this command.

-d, --deregister <ip address>

Deregister a file filtering agent that's currently registered. Note that the agent may simply re-register itself immediately, so if you don't want the agent to be used again, you should remove the current EVS's IP address(es) from the agent's list of storage devices.

-r, --reset

Reset the statistics.

Examples

To deregister a file filtering agent:

file-filtering -d 10.1.7.50

Applies To

EVS
