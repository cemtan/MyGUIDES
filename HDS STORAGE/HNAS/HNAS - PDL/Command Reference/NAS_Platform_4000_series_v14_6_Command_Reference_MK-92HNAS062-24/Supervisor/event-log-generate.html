event-log-generate

Generate an event

Syntax

event-log-generate [-i|-w|-s|-n <event-id>] <message...>

Description

The event-log-generate command generates an event with the specified text. This can be used to alert all managers of the system (i.e. alert recipients) of something that is happening.

Options:

-i

Generate alert at 'Information' level (the default)

-w

Generate alert at 'Warning' level

-s

Generate alert at 'Severe' level

-n <event-id>

Generate an event with a specified event-id, rather than the default event id of 9995(Information), 9996(Warning), 9997(Severe)

<message>

The message to send. "Test message" is used as a default. A period is appended to the supplied text.

Examples

To log an event to tell alert monitors that we're undergoing maintenance:

event-log-generate -w "Volume data undergoing maintenance; ignore alerts"

(See also the "maintenance" command.)

To log an event 6638 to tell administrators about configuration issues while executing augmentation scripts:

event-log-generate -n 6638 -- "ucoams[0]: /usr/local/bin/create-ldap.conf.rb: Host not set on 127.0.0.1 EVS 0."

Applies To

Cluster node
