apikey-delete

Delete an existing API Key

Syntax

apikey-delete (--description <description> | --prefix <prefix>) [--confirm]

Description

apikey-delete removes an API Key from the system. It is not possible to recreate the same API Key as the prefixes are randomly generated. If a key may need to be used in the future, disabling the key would be a better option than deleting it.

Parameters

Exactly one of description or prefix must be specified:

-d, --description <description>

Specify the key by the description associated with it.

-p, --prefix <prefix>

Specify the key by the randomly generated prefix of the key.

--confirm

Delete the key without prompting for confirmation first.

Examples

Delete the key associated with the description "testkey" and don't prompt to confirm deletion:

server:$ apikey-delete -d testkey --confirm

Delete the key which has the prefix SzKdKXQxur - the user will be prompted for confirmation before the key is deleted:

server:$ apikey-delete -p SzKdKXQxur

Applies To

Cluster
