smb2-client-side-symlink-handling-disable

Disable symlink handling over SMB2.

Syntax

smb2-client-side-symlink-handling-disable 
Alias:       cifs-smb2-client-side-symlink-handling-disable

Description

The smb2-client-side-symlink-handling-* commands govern the behaviour of clients using SMB2 or newer with regard to symlinks.

smb2-client-side-symlink-handling-status

Display the behaviour of symlinks over SMB2.

smb2-client-side-symlink-handling-enable

Enable symlink handling over SMB2.

smb2-client-side-symlink-handling-disable

Disable symlink handling over SMB2.

smb2-client-side-symlink-handling-default

Revert symlink handling behaviour over SMB2 to the default.

When client side symlink handling is enabled, SMB2 or newer clients are able to create and traverse symlinks according to the SMB2 specification. When disabled, the behaviour is determined by the "Follow Symlinks" setting of the share.

The behaviour of SMB1 clients is not affected by this setting.

Windows clients may require a change to their local configuration or Group Policy to allow remote symlink evaluation in order to make use of this feature. See Microsoft documentation on symlink evaluation for further information.

Applies To

EVS
