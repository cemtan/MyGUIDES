span-enable-snapshots

Put a span into copy-on-write snapshot mode

Syntax

span-enable-snapshots <span-label> <new-snapshot-name>
Alias:       snapon

Description

Puts a span into snapshot mode, so that it can be used with storage-based snapshots, as provided by the Hitachi Thin Image (HTI) and Copy-On-Write (CoW) Snapshot features. See 'storage-based-snapshot' and 'storage-based-snapshot-label' for background information.

This command does not create or detect a new copy of the span. It does, however, prepare the span for use with further snapshots. It also assigns a new snapshot name to the existing copy of the span and, in doing so, changes the instance names of the span and all its filesystems.

In order to use storage-based snapshots, it is also necessary to configure the storage and to install the necessary license. Both these operations must carried out in a storage configurator such as Storage Navigator or Hitachi Command Suite.

There is no need to unmount file systems before running 'span-enable-snapshots'.

A span cannot be placed into snapshot mode if it has storage-based mirror relationships or has been assigned to more than one cluster.

Examples

Suppose span Departments contains filesystem Engineering. The administrator wishes to use snapshots with this span. Snapshot name 'Master' is chosen, because the original span contains the master copy of the data.

span-enable-snapshots Departments Master

After this operation, the span will have been renamed as 'Departments-Master' and the filesystem as 'Engineering-Master'. Therefore, to mount and unmount the file system, it will be necessary to type

mount Engineering-Master
unmount Engineering-Master

Applies To

Cluster wide
