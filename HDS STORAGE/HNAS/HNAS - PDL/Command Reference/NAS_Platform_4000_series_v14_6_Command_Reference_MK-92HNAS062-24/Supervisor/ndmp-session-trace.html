ndmp-session-trace

Dump the set of currently enabled traces or the trace for the specified session

Syntax

ndmp-session-trace [ [--dblog on|off] <sess_num> ]

Description

Displays the NDMP subsystems that are currently being traced, or the trace output for a specified session number. Not all NDMP subsystems are traced by default because there is a performance penalty in doing so for certain subsystems. See "ndmp-session-trace-add" for further information.

The data gathered by the NDMP session trace is complementary to that gathered using "ndmp-trace-mask". A certain number of NDMP session traces are always retained at any time (200 by default), meaning that there is always a certain amount of historical NDMP information available.

Options:

--dblog on|off

A session must be specified to use this option. Enables/disables output of the session's trace to the dblog in real time. By default the trace output is stored in memory and only output to the console when requested using this command specifying a session.

Examples

Display list of NDMP subsystems currently being traced:

ndmp-session-trace

Display the trace for the specified session:

ndmp-session-trace 11

Applies To

Cluster node

See Also

ndmp-device-trace ndmp-session-trace-add ndmp-session-trace-remove ndmp-trace-mask

Privilege Level

Supervisor
