span-rename-snapshot

Change the name of a storage-based snapshot

Syntax

span-rename-snapshot <old-instance-name> <new-snapshot-name>
Alias:       mvsnap

Description

This command renames one snapshot in a group of storage-based snapshots. Specify the existing instance name of the span and the new snapshot name; the new instance names of the span and all its filesystems will be determined automatically.

The contents of the file systems will be unchanged, as will the names of any NFS exports and CIFS shares. There is no need to unmount any file systems before renaming a snapshot.

Examples

Suppose span Accounts has two snapshots, Main and Test. Each snapshot has three filesystems: North, East and South. The instance names are as follows:

  • Snapshot Accounts-Main contains filesystems North-Main, East-Main and South-Main.
  • Snapshot Accounts-Test contains filesystems North-Test, East-Test and South-Test.

After the command

span-rename-snapshot Accounts-Test DataMine

the names will be as follows:

  • Snapshot Accounts-Main will contain filesystems North-Main, East-Main and South-Main.
  • Snapshot Accounts-DataMine will contain filesystems North-DataMine, East-DataMine and South-DataMine.

Applies To

Cluster wide
