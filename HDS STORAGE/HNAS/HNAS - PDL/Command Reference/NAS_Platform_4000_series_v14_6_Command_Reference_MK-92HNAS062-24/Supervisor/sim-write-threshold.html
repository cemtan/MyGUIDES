sim-write-threshold

Set dirty page threshold over which user writes are suspended

Syntax

sim-write-threshold [--default | <percentage> | -r]

Description

Disk writes are throttled using a calculation based on sector cache dirtiness. As the number of dirty buffers approaches the threshold, the permitted level of writes rises to 100% of the requests issued to each system drive. If it exceeds the threshold then the file system layer is instructed to suspend write operations in order to allow some of the dirty data to be flushed. sim-write threshold displays the threshold, the amount of dirty data, and statistics on write disabling. It can also be used to change the threshold. Configuration is cluster-wide; statistics are for the local pnode only.

<percentage>

Configure the dirty buffer threshold. NB: Other constraints may cause the threshold in force to be lower than the configured percentage.

--default

Set the configured threshold to the default value, currently 80%.

-r, --reset

Reset the write disable statistics.

Applies To

Cluster wide
