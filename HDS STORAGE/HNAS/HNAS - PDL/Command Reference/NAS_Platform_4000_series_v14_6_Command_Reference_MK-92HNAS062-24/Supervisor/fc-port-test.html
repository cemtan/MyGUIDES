fc-port-test

Loopback FC port test

Syntax

fc-port-test [-n <frames>] [-m <max-fails>] [-l <length>] [-s {off|on}] [-c xx[xx[...]]] <port>

Description

This is a fairly low-level Fibre Channel loopback port test. For it to work the port must be in loop mode, and there should be nothing else participating in the loop. It will transmit a series of frames to itself, and at the end report how many were received and the number of frame errors reported by the hardware.

-n

Number of frames to send. Default: 1000

-m

Number of transmission errors allowed before giving up. (Transmission errors only - receive errors will not make it give up.) Default: 10

-l

Frame payload length. Default: Maximum possible

-s

Force FC 8b/10b scrambler on or off. Default: unchanged (on for 8G, off for lower speeds)

-c

Fill pattern. The argument is one or more pairs of hexadecimal digits (undelimited); each pair is interpreted as an octet value. This pattern is repeated throughout the payload of each frame. Default: An incrementing pattern

<port>

Port to test, numbered from 1.

Examples

Test port 1 with 10000 small frames:

fc-port-test -n 10000 -l 400 1

Test port 4 with ASCII 'J's:

fc-port-test -c 4a 4

A slightly more complicated fill pattern:

fc-port-test -c 4ab5 4

Applies To

Cluster node

Privilege Level

Supervisor
