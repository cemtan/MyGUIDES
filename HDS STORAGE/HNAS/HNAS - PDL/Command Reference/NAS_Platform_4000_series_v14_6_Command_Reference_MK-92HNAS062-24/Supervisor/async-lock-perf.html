async-lock-perf

Display asynchronous file region lock statistics

Syntax

async-lock-perf [-r] [-s]

Description

When a Windows client requests a byte-range lock on a file, it specifies a timeout. If the lock can't be granted immediately, the server queues the request until either the lock can be granted or the timeout expires. Each time a queued lock is granted, the server logs the length of time that it has been queued for. The async-lock-perf command displays the aggregate of these times, therefore giving a guide to how long it generally takes for queued byte-range locks to be granted.

Usage:

async-lock-perf [OPTION...]

--reset, -r

Resets the statistics.

--summary, -s

Only displays a summary.

Applies To

Cluster node
