cifs-spnego-auth

List / change ADS SPNEGO authentication type support.

Syntax

cifs-spnego-auth [  -l  all | log | ntlmssp | ntlmv1  ]

Description

This command controls which authentication mechanisms the server reports support for when negotiating new CIFS connections with ADS-aware clients.

By default, both Kerberos and NTLMSSP mechanisms are supported. This command permits you to restrict the server to only reporting support for NTLMSSP authentication or even only for the NTLMv1 mode of NTLMSSP authentication. This command was added largely to support testing of non-default CIFS authentication modes. This command should not be used in the field, except under guidance from technical support personnel.

This command also permits you to enable logging of each authentication mechanism choice made by an SPNEGO exchange. When enabled, the reports are written to the FSM console. This logging is strongly discouraged, as in a busy CIFS environment it would consume the majority of available FSM console log space.

The cifs-spnego-auth setting is cluster-wide, there are not individual settings for each EVS or EVS security context.

Usage:

--level, -l all | log | ntlmssp | ntlmv1

Sets SPNEGO negotiation behavior.

all

Enables both Kerberos and NTLMSSP authentication negotiation (the default).

log

Enables both Kerberos and NTLMSSP authentication negotiation, and logs the chosen mechanism for each CIFS client authentication (use strongly discouraged).

ntlmssp

Restricts SPNEGO negotiation to the NTLMSSP mechanism (NTLMv2 or NTLMv1) only.

ntlmv1

Restricts SPNEGO negotiation to the NTLMv1 NTLMSSP mechanism only.

Examples

To display the current cluster's cifs-spnego-auth setting:

cifs-spnego-auth

Set the default mode of all authentication types negotiable:

cifs-spnego-auth -l all

Applies To

Cluster wide

Privilege Level

Supervisor
