fs-sdg-utilization

Reports how a file system is utilizing each SD group in the underlying span

Syntax

fs-sdg-utilization <fs-label>

Description

Reports how a file system is utilizing each system drive (SD) group in the underlying span.

The command scans the free space bitmap object in order to calculate where space is used. The time taken is proportional to the number of blocks in the file system. It can take a long time, even several hours for file systems that are several hundreds of TBs in size.

If this is run over SSC and SSC times out, it will continue to run on the server and the output can be found in the dblog.

For each SD group the command reports:

  • which SDs are used by the file system.
  • the number of bytes used by the file system.
  • how that space is utilized (live, snapshot and total allocations).

Examples

server:$ fs-sdg-utilization fs0
             SD Group: 0, 1, 2, 3
Space Allocated to FS: 32.0 GB
    Space Utilization: Live: 92.065% (29.5 GB), Snapshot: 0.000% (0 B), Total: 92.065% (29.5 GB)


             SD Group: 4, 5
Space Allocated to FS: 26.7 GB
    Space Utilization: Live: 0.000% (0 B), Snapshot: 0.000% (0 B), Total: 0.000% (0 B)
