search-directory

Searches the contents of a specified Directory for a particular target and dumps it to the console

Syntax

search-directory [-f <file-system>] [-c <cookie> | -o <object-number> | -n <name>] [-i <entries>] [-x | -w] [--snapshot-name <snapshot name> | --snapshot-checkpoint-number <snapshot checkpoint number> | --snapshot-index <snapshot index>] -d <directory-object-number>

Description

search-directory is used to print out portions of a directory's contents in a variety of different formats. It may be used to search for a particular entry by three criteria: object-number, cookie or entry-name. It may be run with the file system mounted or unmounted. search-directory is typically used as a diagnostic tool when problems with a directory have been encountered, and is not intended as a fancy ls.

Options:

-f <file-system>

The file system on which the directory resides

-c <cookie>

Cookie of entry for which to search

-o <object-number>

Object Number of entry for which to search

-n <name>

Name of entry for which to search

-i <number-of-entries>

Number of entries to display

-x

Display in hexadecimal format

-w

Display in WDIR format

--snapshot-name <snapshot-name>

Search in the given snapshot, rather than the live file system. Can only be used on WFS-2 file systems.

--snapshot-checkpoint-number <checkpoint-number>

Search in the given snapshot, rather than the live file system. Can only be used on WFS-2 file systems.

--snapshot-index <snapshot-index>

Search in the given snapshot, rather than the live file system. Can only be used on WFS-2 file systems.

Examples

To print the entry with cookie 23 for directory 0xa000 on file system 0:

search-directory -f 0 -c 23 -d 0xa000

To print the entry whose name is 'humptydumpty' for directory 0xe8000 on file system 1:

search-directory -f 1 -n humptydumpty -d 0xe8000

To obtain a hexadecimal dump of the contents of the directory at 0xf0000 on file system testfs:

search-directory -x -f testfs -d 0xf0000
