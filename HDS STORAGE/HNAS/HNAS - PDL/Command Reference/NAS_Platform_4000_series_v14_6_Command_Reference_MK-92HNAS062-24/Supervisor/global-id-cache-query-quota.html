global-id-cache-query-quota

List changes in quotas cache since <counter>

Syntax

global-id-cache-query-quota <counter>

Description

There are a number of objects on the NAS server which are tracked and cached based on a globally unique identifier. There are a series of caches of these globally unique identifiers which are used to identify when individual objects have been changed in some way, i.e. added, updated, or deleted. For each of these caches there is a counter which increments each time an entry is made in the cache. This command is used to find out what if any entries have been added to the cache since the user supplied counter, in the context of the current evs.

Note that the counter does not start from zero and although it increments by one each time, each increment may well apply to only one evs. Any changes that do not apply to the current evs will not be shown. Hence we can use global-id-cache-dump to see that there have definitely been changes since the supplied cache counter value but we will not receive any changes on running this command if none of those changes relate to the current evs.

SYNTAX

global-id-cache-query-quota <cache counter value>

where <cache counter value> is a 64 bit number retrieved using global-id-counter-dump.

Examples

Display all cache entries applying to this evs since the supplied cache counter value:

global-id-cache-query-quota 132748621551501317

Applies To

Cluster node
