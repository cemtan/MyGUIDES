shutdown

Shutdown the server or cluster

Syntax

shutdown [-s|--seconds <seconds>] [-f|--force] [-a|--all] [-c|--cluster] [-p|--powerdown]
Alias:       halt

Description

The shutdown command is used to shutdown one or all cluster nodes. When shutting down a cluster, nodes may be shutdown sequentially (--cluster) or simultaneously (--all).

Options:

-s, --seconds

Shutdown after <seconds> seconds.

-f, --force

Force shutdown, otherwise requires user confirmation.

-a, --all

Shutdown all cluster nodes in parallel.

-n, --ship

Disable failover to battery for shipping.

-c, --cluster

Shutdown all cluster nodes one at a time.

-p, --powerdown

Power-down server at end of shutdown sequence.

--now

Request an immediate shutdown without unmounting file systems, migrating EVSs and waiting for services to stop. This is only supported by the reboot command.

Examples

server:$ shutdown
Are you sure you want to shutdown cluster-node-1? (Y/N)[N]:
y
Information: Server is shutting down


server:$ shutdown --all
Are you sure you want to shutdown all cluster nodes? (Y/N)[N]:
y
Information: Server is shutting down

Applies To

Cluster wide
