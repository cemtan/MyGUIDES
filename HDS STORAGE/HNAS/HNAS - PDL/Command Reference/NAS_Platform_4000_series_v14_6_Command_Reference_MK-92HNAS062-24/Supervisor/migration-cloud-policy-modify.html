migration-cloud-policy-modify

Modify an external cloud migration policy

Syntax

migration-cloud-policy-modify <policy-name>|<policy-id> [--source-fs-label <source-fs-label>] [--external-path <external-path>] [--rule:threshold <rule:threshold>]

Description

Modifies a cloud migration policy.

<policy-name>

Name of policy to modify.

<policy-id>

Id of policy to modify.

--source-fs-label, -s <source-fs-label>

Name of policy's file system. Note: the binding to the file system cannot be modified. This parameter is used as a search criteria only.

--external-path, -p <external-path>

Name of file system's external path to modify.

--rule:threshold, -r <rule:threshold>

Name of rule and free-space percentage threshold associated with policy to modify.

Examples

migration-cloud-policy-modify DefaultPolicy --source-fs-label fs0 --rule:threshold gif:30

Modifies policy named 'DefaultPolicy' on filesystem 'fs0' to use rule 'gif' when free space is less than 30%.

Applies To

EVS security context
