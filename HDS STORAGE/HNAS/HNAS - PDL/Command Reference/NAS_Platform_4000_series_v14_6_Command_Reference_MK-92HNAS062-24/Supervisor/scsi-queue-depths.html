scsi-queue-depths

Show SCSI queue depths

Syntax

scsi-queue-depths

Description

This command displays the current SCSI command queue depths and dynamic limits for all system drives in use on the local node. (A SD is considered "in use" if any file system in that SD's storage pool is (locally) open.)

Specifically, it displays the following values for each SD:-

Device

The SD number, as given by sd-list.

Target

The load balancer's determination of the correct queue limit for the SD.

Current

The limit currently in force. It may lag behind Target briefly when limits need to be changed. In particular, a SD newly in use on a given controller and/or cluster node may have to wait for others' queue depth to be lowered before its own can be increased.

Actual

The limit programmed into the FPGA's device descriptor. It should match Current, minus any software allocation.

Software

Capacity temporarily borrowed from the fast path for software-initiated I/O. When the number appears in brackets, it represents a pending allocation.

Examples

server:$ scsi-queue-depths
Current queue depths for all open devices:


Device   Target   Current   Actual   Software
---------------------------------------------
    1      128      128       128
    2      128      128       128
    3      128      128       123        5
    4      128      128       128

Applies To

Cluster node
