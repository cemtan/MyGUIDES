group-mappings-import-from-nis

Imports Unix name <--> id mappings from NIS

Syntax

group-mappings-import-from-nis [--allow-overwrites] [--background] [--commit]

Description

Imports Unix names and group ids from the NIS/LDAP server.

Options:

-o, --allow-overwrites

Update existing entries that match the group name/id

-b, --background

Perform import in the background

-c, --commit

Commit changes to persistent store

If the --allow-overwrites option is not specified, NIS/LDAP entries whose group name/id match existing records will be skipped. If the --allow-overwrites option is specified and the NIS/LDAP group id/name differs from an existing entry, the existing entry will be updated.

If the --background option is specified, the command prompt will return immediately and an event will be raised when the import completes.

If the --background option isn't specified, the command will output a report at the end of the import detailing how many records have been added to the mappings table, how many have been updated, and how many skipped.

Note that this command "fixes" the mappings entries as if they had been manually configured. They will not be refreshed from the NIS/LDAP server, unlike mappings learned via NIS/LDAP during normal operation. Mappings are learned asynchronously, meaning that the first client to trigger a mapping won't immediately see the benefit of the mapping. This will result in unmapped Unix SIDs being initially presented in Windows Explorer and in access being temporarily denied where it would later be granted.

Examples

Import records from NIS/LDAP in the background:

$: group-mappings-import-from-nis --allow-overwrites --background

Applies To

EVS security context
