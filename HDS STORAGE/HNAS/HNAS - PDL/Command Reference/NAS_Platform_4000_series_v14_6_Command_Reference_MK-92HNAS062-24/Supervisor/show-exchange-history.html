show-exchange-history

Display RIS activity log

Syntax

show-exchange-history <key> | -a | [-a] -d <sd> [-o <offset>] [-n <N> | --key-only]
Alias:       showrislog

Description

Show history of errors and retries on individual fast-path SCSI requests that have experienced errors.

-a --active

With -d, list only active records; otherwise, list all active records. A record is active if the request's RIS in SI control memory has not yet been reused.

-d --sd

In combination with -a or --key-only, list records for the specified system drive. Otherwise, display the most recent entries for the device in reverse chronological order, limited to at most 20.

-n --num-records

Limit the number of records to be displayed by -d.

-o --offset

List only records relating to requests at the specified system drive offset.

--key-only

With -d, only list records instead of displaying them.

<key>

Display full details of a specfic record identified by its key, as listed by other forms of this command or logged in certain events.

The information displayed when a key is specified comes in two parts. First is information relating to the request itself and the total time it has spent. Next is a list of log entries relating to each attempt at performing the request, each describing the type of error received and when, and the action taken (resubmit or complete). At most 20 entries are stored for each request, but a request will usually be failed before reaching that many retries, so it should normally be enough to cover the full history of an errored request.

-d (without -a or --key-only) displays not whole request records but individual entries. In order to identify the request associated with each entry it uses a two-line format, where each log entry displayed is preceded by a condensed version of the request record header.

The oldest records are automatically removed if the log grows too large.

Applies To

Cluster node
