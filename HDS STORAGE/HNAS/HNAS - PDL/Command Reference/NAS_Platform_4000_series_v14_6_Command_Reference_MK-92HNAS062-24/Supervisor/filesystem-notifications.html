filesystem-notifications

List the last notification sent by the spanning system for each filesystem

Syntax

filesystem-notifications

Description

The server is composed of several subsystems. One of these is Span Manager, which informs other subsystems when spans and filesystems have been loaded and become healthy or unhealthy, or have been deleted, forgotten, or moved to other clusters.

The 'filesystem-notifications' command lists the most recent notification sent for each filesystem on the local cluster node -- even if the filesystem is no longer loaded. In most cases, each filesystem's permanent ID is also shown. The possible notifications are:

No notification

Span Manager has not sent any notification for this filesystem.

Made available

The filesystem has become healthy, and Span Manager has made it available for use by the rest of the server. The command will show the flags that accompanied the filesystem when it was last made available; these flags vary between releases, but their meaning is understood by your support provider.

Made unavailable

The filesystem or its host span has become unhealthy in some sense (SD failure, an encroachment, or some similar failure), or has been manually hidden by a command such as 'span-hide'. Span Manager has told the rest of the server that the filesystem can no longer be used.

Deletion notification

Span Manager has deleted the filesystem, unlicensed the host span, or reassigned the span to another cluster. It is likely that the filesystem instance name will be omitted.

Examples

server:$ filesystem-notifications
1024 (Alpha, perm ID 14A871223E896BBF): Made available (SANoFlags)
1025 (Bravo, perm ID 14A8712A36FAAEB4): Made unavailable
1026 (perm ID 14A87043FE586989): Deletion notification
1027 (perm ID 14A870400FC5EEA5): Deletion notification
server:$

Applies To

Cluster node
