fc-loopmap-log

dump/clear FC loopmap logs

Syntax

fc-loopmap-log <[-d,--dump] | [-c,--clear] | [-s,--size] | [-h,--help]>
Alias:       fclplog

Description

Dump or clear the contents of the fibre channel loopmap log.

If a loopmap is received by the server it is logged in the loopmap log which may be useful to determine if a target device is no longer presenting itself in the loopmap. Each log entry displays the time that the loopmap was received and its contents.

Note: No loopmap will have been received if the server is running in N-port mode.

Console output may be stopped or aborted with a key press or by CTRL-C.

Options:

-d, --dump

Dump the contents of the FC log for the given host port. The last 20 entries are output if no range is specified.

-h, --hport <1..n>

Specify host port log to be dumped.

-a, --all | -n <[-]no of entries to dump>

Output all entries, last n entries, or first n entries if n is negative. If n = 0, dump entire log; by default, dump last 20 entries.

-c, --clear

Clear the log, default is host port 1 log.

-h, --hport <1..n>

Specify host port log to be cleared.

-C,--Conf

Do not request confirmation.

-s, --size

Show the current and maximum log size.

-h, --help

Show help.

Examples

Dump the entire FC loopmap log for hport 1:

fc-loopmap-log -d -h 1 -a

Dump the last 10 entries from hport 2's FC loopmap log:

fc-loopmap-log -d -h 2 -n 10

Clear the hport 1 loopmap log, no confirmation:

fc-loopmap-log -c -h 1 -C

Applies To

Cluster node

See Also

fc-loopmap fc-ports fc-sns

Privilege Level

Supervisor
