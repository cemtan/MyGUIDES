aggedit

Configure an aggregation interface

Syntax

aggedit lacp agN | static agN | load-normal agN | load-round-robin agN |<add|remove> agN <port> [,<port>]...

Description

The aggedit command is used to manipulate an aggregated interface. An aggregation is a collection of one or more physical interfaces that are bundled together to behave as a single logical interface. All physical interfaces participating in the same aggregation must be of the same speed.

The active members of an aggregation can be configured statically (in which case all interfaces that are physically up will be active members and both ends of the bundle must be identically configured) or the aggregation's members can be determined by use of the 802.3ad link management protocol (LACP).

The default load-balancing algorithm tries to spread conversations equally between all available physical links in order to minimise out-of-order packet arrival at the upstream switch. In round-robin mode, consecutive packets will use the next available physical link in order to maximize the available bandwidth.

Examples

To add 1GbE interfaces to an aggregation:

    aggedit add ag1 ge1,ge2

To add 10GbE interfaces to an aggregation:

    aggedit add ag1 tg1,tg2

To remove 1GbE interfaces from an aggregation:

    aggedit remove ag1 ge1,ge2

To set an aggregation to use LACP:

    aggedit lacp ag1

To set an aggregation to use static aggregation:

    aggedit static ag1

To use the normal load spreading algorithm:

    aggedit load-normal ag1

To use the aggressively balanced load spreading algorithm:

    aggedit load-round-robin ag1

Applies To

Cluster wide
