scsi-ignore-redundancy-check

Suppress loss of redundancy warning alerts for the offline SCSI port

Syntax

scsi-ignore-redundancy-check [<SCSI port id>][--hport | -h <port id>]

Description

If the SCSI port is off-line, stop loss of redundancy warning alerts from being generated for the given SCSI port. This command should only be used if the loss of redundancy is intentional and is only allowed if the SCSI port is off-line; it is intended to be used to suppress alerts for a SCSI port that the server has discovered and which has subsequently been taken off-line. The suppression of the redundancy alert will be automatically removed if the SCSI port comes on line. The request will also be sent to other pnodes of a cluster. If a server host port number is entered, loss of redundancy alert suppression will be applied to all SCSI ports that are off-line because the host port link is down. Note that this command will only suppress warning alerts that are generated by a loss of redundancy. The server will also generate severe alerts if there is no redundancy or if the storage configuration does not consist of 2 host ports and 2 controllers (this does not apply to Nexsan storage). It is not possible to suppress the severe alerts but they will become less frequent until they are generated once every 24 hours.

Options:

<scsi port id>

Specify the SCSI port to suppress loss of redundancy alerts for.

--hport, -h <port_id>

Specify the host port to suppress loss of redundancy alerts for.

Show help.

Examples

scsi-ignore-redundancy-check 3
scsi-ignore-redundancy-check -h 1

Applies To

Cluster wide

Privilege Level

Supervisor
