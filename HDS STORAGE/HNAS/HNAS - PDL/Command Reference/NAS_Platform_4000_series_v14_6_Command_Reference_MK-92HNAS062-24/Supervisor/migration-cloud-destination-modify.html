migration-cloud-destination-modify

Modify an external cloud migration destination

Syntax

migration-cloud-destination-modify <destination name> [--account-name <account name>] [--path-at-destination <path at destination>] [--description <description>] [--encrypt-in-transit <no|yes>] [--force]

Description

Modifies cloud external migration destination.

<destination name>

The name of the destination to modify.

--account-name, -a <account name>

Specify this option to modify the account associated with the named destination.

--path-at-destination, -p <path at destination>

Not recommended. Specify this option to modify the path at the destination associated with the named destination. Files that have already been migrated may become inaccessible.

--description, -d <description>

Specify this option to modify the description of the named destination.

--encrypt-in-transit, -t <no|yes>

Specify this option to modify whether data should be encrypted in transit.

--force

Not recommended. Forces the path at destination change. Warning: files that have been already migrated may become inaccessible.

Examples

migration-cloud-destination-modify CloudDest --path-at-destination newbucket/newdir

Modify destination CloudDest to place files at location newbucket/newdir at the destination on an ongoing basis.

Applies To

Global security context
