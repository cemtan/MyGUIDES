rmdir

Unlink directories

Syntax

rmdir [-f|--force] [-v|--verbose] [--] <dir1> [<dir2>...]

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The rmdir command is used to remove one or more directories. If a directory name begins with a dash, it cannot be specified as the first argument to rmdir. It should be specified after a stand-alone option "--" or after another directory name that does not begin with a dash.

Usage:

--force, -f

Do not fail if the directory does not exist.

--verbose, -v

Display a status message for each non-existent directory when option --force is specified.

--

Optional to signify the end of options and hence all subsequent arguments are directory names to unlink. Useful if the first directory name begins with a dash.

Examples

1. To remove the directories 'dir1', 'dir2' and 'dir3' from the current working directory:

   rmdir dir1 dir2 dir3

2. Specify --force to not fail the command when the directory is non-existent:

   rmdir -f dir1

3. Two ways to remove a directory whose name begins with a dash:

   rmdir -- -testdir  # after stand-alone option --
   rmdir adir -bdir   # after another directory
