group-mappings-refresh

Refreshes group mapping(s)

Syntax

group-mappings-refresh --all | --unix-name <name> | --unix-id <id> | --nt-name <name> | --nt-id <id> |
 --nfsv4-name <name>

Description

Refreshes an entry (or all entries) in the group mappings table.

Options:

--unix-name

Look up the entry to refresh by Unix name

--unix-id

Look up the entry to refresh by Unix group id

--nt-name

Look up the entry to refresh by NT name

--nt-id

Look up the entry to refresh by NT SID

--nfsv4-name

Look up the entry to refresh by NFSv4 name

--all

Refresh all entries

The auto mapper periodically refreshes all entries using a background task. By default it will refresh entries ten times per day where it has failed to auto map fields. Entries where there are no auto mapping failures are refreshed two times per day. You may change these settings if you feel the defaults are inappropriate for your environment - contact your support provider for instructions.

If you choose one entry to refresh, it is refreshed immediately and the result of the refresh entry is displayed. Choosing to refresh all entries starts the background task in a quick refresh mode that refreshes every entry, bypassing the default refresh periods.

Examples

Refresh all mappings now:

$: group-mappings-refresh --all

Applies To

EVS security context
