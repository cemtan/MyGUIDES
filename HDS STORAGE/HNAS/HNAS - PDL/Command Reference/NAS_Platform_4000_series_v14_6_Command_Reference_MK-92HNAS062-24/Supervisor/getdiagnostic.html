getdiagnostic

Copy diagnostic file to local file

Syntax

getdiagnostic [--cnode <id>] [<filename>]

Description

The getdiagnostic command creates a server diagnostic file and copies it locally. The diagnostic file is mostly for use by your support provider for advanced diagnosis of problems. It contains server console logs, configuration information and other long term logs.

--cnode <id>

Specify the cluster node to generate the diagnostics on

filename

A filename to which to save the diagnostics. It is recommended that this argument not be provided, as the default name is unique to the server and current date and time, and will help significantly if multiple diagnostics are generated.

Examples

To get a diagnostic with the default filename:

getdiagnostic

An example of a generated filename

server_diagnostics_2005-08-30_1159+0100.tgz

Applies To

Cluster node
