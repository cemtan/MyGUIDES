variables-show

Display current and default values of SimplePersistentConfig and VnodeLocalVariable items

Syntax

variables-show 
Alias:       twiddles-show
Alias:       simple-persistent-config

Description

Display current and default values of various configuration options.

Use the set or set-for-evs command, as instructed by the output of the variables-show command when given no arguments, to change the value of a configuration option. You are advised to use the unset or unset-for-evs command, rather that set or set-for-evs, to revert to the default value, as the setting will then track any future changes in the default. The "set" configuration items are cluster-wide. The "set-for-evs" configuration items apply to the EVS in context. All the settings are held in the registry, which is persistent across reboots and cluster-node replacements. Most of them take effect immediately but there's no surefire way of telling without inspecting the piece of source code which uses the configuration item. There is no documentation on the individual configuration options. Some of them are set automatically, for example when one-time upgrade checks are performed, and it would be particularly unwise to revert those, at least without consulting the corresponding source code.

Although supervisor level is sufficient to modify any variable, using the set or set-for-evs commands, only variables changed from their default values will be displayed by this command. Values should only be changed under the direction of your support provider.
