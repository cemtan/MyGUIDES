obj-profile

Profiles objm2 state machines

Syntax

obj-profile [-s] [-v] <seconds-to-sample> obj_main|sim_rsp|obj_req_parser|obj_sim_inq|fb_cont|cp_cont|linked_list_cont|vol_info_ram|obj_main_sdram_access_decode|obj_main_vol_info_access_decode

Description

The *-profile commands are used to diagnose performance problems by showing how busy various parts of the hardware are. These commands may either use high resolution profiling implemented in hardware or lower resolution profiling implemented in software. Using the software based profiling may have performance implications on the system.

Only use under instruction from your support provider.

-s, --software

Select use of the software based profiling rather than the hardware based profiling. For profiling commands which support both software and hardware based profiling the default is to use hardware based profiling.

-v, --verbose

This option lists all of the states. By default only those states for which activity has been detected are reported.

Applies To

Cluster node
