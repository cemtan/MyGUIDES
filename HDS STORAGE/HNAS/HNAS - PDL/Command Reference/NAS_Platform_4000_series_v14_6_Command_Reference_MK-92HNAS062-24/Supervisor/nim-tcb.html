nim-tcb

dump active TCBs

Syntax

nim-tcb [ -i tcb-index | -c ] [-s state] [-p port] [-a address] -A -v -V

Description

The nim-tcb command displays information held by the hardware about existing TCP connections. The command options control whether a single line summary or a multi-line detailed information entry is displayed for each connection.

-i, --tcb-index <index>

Displays detailed information about the connection in TCB <index>.

-c, --count

Displays a count of the TCBs.

-s, --state <state>

Display all TCBs that match a given connection state. The state is expressed either by name or number:

  • CLOSED (0)
  • SYN_SENT (1)
  • SYN_RCVD (2)
  • ESTABLISHED (3)
  • FIN_WAIT_1 (4)
  • FIN_WAIT_2 (5)
  • CLOSING (6)
  • CLOSE_WAIT (7)
  • LAST_ACK (8)
-p, --port <port>

Display all TCBs that match a given port (both local and peer ports are considered).

-a, --address <ip-address>

Display all TCBs that match a given IP address (both local and peer addresses are considered).

-A, --list-all

Displays full details for all TCBs. No pagination is provided.

-v, --verbose

Displays additional information when displaying the TCB details.

-V, --very-verbose

Displays explanatory text when displaying the TCB details.

Examples

To display summary information for the first page of TCBs:

nim-tcb

To display all information for TCB 26:

nim-tcb -i 26

To display all information for all TCBs:

nim-tcb -A -V

Applies To

Cluster node

Privilege Level

Supervisor
