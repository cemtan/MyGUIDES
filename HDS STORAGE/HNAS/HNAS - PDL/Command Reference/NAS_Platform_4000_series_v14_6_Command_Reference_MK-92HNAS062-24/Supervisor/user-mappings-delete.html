user-mappings-delete

Deletes an existing user mapping

Syntax

user-mappings-delete --unix-name <name> | --unix-id <id> | --nt-name <name> | --nt-id <id> |
 --nfsv4-name <name> | --kerberos-name <name>

Description

Deletes an entry from the user mappings table.

Options:

--unix-name

Look up the entry to delete by Unix name

--unix-id

Look up the entry to delete by Unix user id

--nt-name

Look up the entry to delete by NT name

--nt-id

Look up the entry to delete by NT SID

--nfsv4-name

Look up the entry to delete by NFSv4 name

--kerberos-name

Look up the entry to delete by Kerberos principal

Only specify one field to lookup the entry. If the entry exists, it will be deleted. An error will be printed if the entry cannot be found.

Examples

$: # delete Unix user joe
$: user-mappings-delete --unix-name joe

Applies To

EVS security context

See Also

user-mappings-add user-mappings-list user-mappings-modify user-mappings-refresh

Privilege Level

Supervisor
