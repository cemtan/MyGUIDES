fs-accessed-time

Get/Set accessed time maintenance

Syntax

fs-accessed-time 
	get [--file-system <label>]
	set [--file-system <label>] [--update-interval <seconds>] <on|off>
	clear <file-system>

Description

Configures file system accessed-time maintenance

Usage:

get [--file-system <label>]

Displays the setting that applies to the cluster, or a specific file system if --file-system is specified.

-f, --file-system

Specifies file system

set [--file-system <label>] [--update-interval <seconds>] <on|off>

Configures the setting that applies to the cluster, or a specific file system if --file-system is specified. File systems 'inherit' the cluster-wide setting, unless explicitly overridden.

-f, --file-system

Specifies file system

-u, --update-interval

This specifies the minimum time in seconds between updates to a file's accessed time. By default, a file's accessed time will not be changed unless it is 4 hours or more since the last update of the accessed time (not the last access to the file).

on|off

The "on" option enables access time maintenance. The "off" option disables access time maintenance - the access time of files will not be updated when clients read from them.

clear <file-system>

Clears a file system's explicit setting, causing it to inherit the cluster-wide setting.

Consideration needs to be taken of the access protocol's caching strategy when enabling accessed time maintenance. A client that caches read requests will not issue a read to the server to update the access time whatever the accessed time maintenance value. This maybe significant when trying to control migrations, replications and changed file notifications using the file access time.

WARNING: Disabling accessed time maintenance, even briefly, will impact data migrations using rules based on accessed time for the file system(s) affected by the change. Subsequent runs of data migration policies that have a migration rule using last accessed time as a criteria will fail if the rule's interval is greater than the time that has elapsed since last accessed time maintenance was enabled.

Examples

Display the cluster-wide setting:

fs-accessed-time get

Enable accessed time maintenance, with an update interval of 1 hour (affects all file systems inheriting the cluster-wide setting):

fs-accessed-time set -u 3600 on

Disable accessed time maintenance on the file system 'fs0' only:

fs-accessed-time set -f fs0 off

Applies To

Cluster wide

Privilege Level

Supervisor
