vlsi-profiler-add

Add a state machine to be included in automated vlsi profiling.

Syntax

vlsi-profiler-add <chip> <stateMachine>

Description

This command is used to mark a state machine as "eligible for profiling" for the automated vlsi-profiler. The vlsi-profiler works by reading the busyness of all given state-machines, and if one has reached the minimum threshold of busyness (vlsi-profiler-threshold-set), it will be profiled.

Note that state-machines added to the profiler are not guaranteed to ever be profiled. This is because their busyness may never surpass the minimum (though if the profile-threshold were set to 0%, that would make them always eligible).

The state machines are read every 10 seconds, but there is a 30 minute cooldown between profiles for individual state-machines to avoid constant profiling.

State machines added to the profiler will only be eligible for profiling on that node, and are not persistent between resets.

To add a state machine, both the chip name and state machine name must be supplied. The command 'vlsi-profiler-supported-list' displays the list of state machines supported by the automated vlsi-profiler. The command uses the same naming scheme for state machines as used by the vlsi-load-* commands, e.g. lower case names and underscores for spaces.

Examples

Add the "seeq" state machine from NIBRX:

vlsi-profiler-add nibrx seeq

Add the "pcx bridge slave" state machine from WLOG:

vlsi-profiler-add wlog pcx_bridge_slave

Applies To

Cluster node
