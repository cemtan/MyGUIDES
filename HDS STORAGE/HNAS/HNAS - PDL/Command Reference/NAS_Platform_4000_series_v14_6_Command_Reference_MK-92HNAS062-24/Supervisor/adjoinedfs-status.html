adjoinedfs-status

While adjoined file systems are deprecated, the command displays the state of a given one.

Syntax

adjoinedfs-status -f <fs label>

Description

Adjoined filesystems are deprecated and administrators are encouraged to transition away from them.

Options:

-f, --fs <file system label>

Main file system through which the adjoined file system is accessed.

Displays information about an adjoined file system. It displays the current status variable settings. Certain actions, such as deleting files, may cause a change in status that is not immediately registered in the status variables. These can be updated using the adjoinedfs-update-status command.

The output includes information about each of the secondary file systems including a current status which is one of the following:

Adjoined but not used

The secondary file system has no files associated with the adjoined file system.

Adjoined file system contains files

There are files at the secondary file system from the adjoined file system.

Adjoined file system deleted/reformatted

The file system is no longer available. It has been removed or has been reformatted.

Unexpected status code
Not adjoined
Active migration in progress

These status values should never be seen.

Examples

adjoinedfs-status  -f fs1

Display details and status of the adjoined file system fs1.

Applies To

EVS
