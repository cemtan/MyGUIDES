snmpv3-user-add

Add an SNMPv3 user

Syntax

snmpv3-user-add <username>

Description

The snmpv3-user-add command adds a new SNMPv3 user to be used for authenticating SNMPv3 requests. An SNMPv3 user consists of a username with a maximum length of 32 characters, an authentication password with a minimum length of 8 characters and maximum of 80, and a privacy password similarly with a minimum length of 8 characters and maximum of 80. The two passwords can either be the same or different. The user's passwords will be securely prompted for after entering the command.

If this command is used when the current SNMP protocol is not v3 (man snmp-protocol) a warning message will be issued but the command will still complete successfully. The warning exists to make the user aware that SNMPv3 users will not affect v1/v2/Bilingual configuration.

Certain characters are not allowed in SNMPv3 usernames or passwords. The characters are: \ / " ' ! £ ¬ ` | < > & ; : * ? % ^

Examples

Add an SNMPv3 user:

snmpv3-user-add newUser
Please enter the authentication password:    ********
Please re-enter the authentication password: ********
Please enter the privacy password:    ********
Please re-enter the privacy password: ********

Applies To

Cluster wide
