replication-source-throughput-limit-set

Limits the aggregate throughput of object replications

Syntax

replication-source-throughput-limit-set <throughput-in-MiB-per-second>

Description

The replication-source-throughput-limit-set command limits the aggregate throughput of object replication per physical node.

For example, if it is set to 50 MiB/s, a physical node will be prevented from sending any more than 50 MiB/s of object replication data (in total across all object replications). In a cluster, each physical node will be able to send up to 50 MiB/s, so the cluster's total throughput will be 50 MiB/s multiplied by the number of nodes.

The setting is persistent over reboots and takes effect immediately for both new and existing replications. The minimum allowed value is 5 MiB/s, and a value of 0 (the default) removes the limit entirely.

To view the current replication throughput, use the replication-show-active command.

Examples

Set the replication throughput limit to 50 MiB/s:

$ replication-source-throughput-limit-set 50
Object replication throughput limit: 50 MiB/s

Remove the replication throughput limit:

$ replication-source-throughput-limit-set 0
Object replication throughput limit: 0 (no limit)

Applies To

Cluster wide
