nfsv3-rsize-set

Set the maximum number of bytes per NFSv3 READ request the client can receive.

Syntax

nfsv3-rsize-set <--tcp | --udp | --all> <value|'default'>

Description

Configures the maximum number of bytes per NFSv3 READ request that a client can receive. These values may be displayed using nfsv3-rsize-get.

Usage:

nfsv3-rsize-set --tcp 32768

Set configuration for TCP transports to 32768 bytes.

nfsv3-rsize-set --udp 28672

Set configuration for UDP transports to 28672 bytes.

nfsv3-rsize-set --all 8192

Set configuration for all settings, i.e. both TCP and UDP transports, to 8192 bytes.

nfsv3-rsize-set --all default

Set configuration for all settings back to their default values.

A note on sizes

The size specified should be a multiple of the block size of any file systems which are exported, i.e. 4k or 32k. nfsv3-rsize-set will enforce the 4k multiple but not the 32k multiple. If you specify a value which is not a multiple of 32k and then export a 32k block size file system, clients will typically ignore the specified value and reduce their read size down to the next multiple.

The maximum specifiable values are as listed below. These are also the default values.

TCP: 65536

UDP: 57344

Note that although the default send and receive values within a transport mechanism are the same the user may configure them to have different values.

Applies To

Cluster wide
