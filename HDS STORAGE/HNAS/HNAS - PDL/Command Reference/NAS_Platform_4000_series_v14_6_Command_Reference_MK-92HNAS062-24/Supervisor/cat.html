cat

Displays a dump of user data of one or more files

Syntax

cat [--hex|--binary] [--offset <offset>] [--length <number bytes to dump>] [--stop-at-zero-byte|-s] <path> [[path1] ...]

Description

The cat command is used to display a dump of the user data of one or more files. The command can be aborted with Ctrl-C.

Usage:

-h, --hex

Produce a hex dump of the file.

-b, --binary

Specify that the file is binary to help the cat command flush the standard output regularly.

-o, --offset <byte offset>

Start dumping from the specified offset.

The <byte-offset> and the <byte-count> below can be specified as a number with unit, or as an algebraic expression as described in the 'offset-and-length' man page.

-l, --length <byte count>

The number of bytes to dump (default is 4096; '0' can be specified to dump to the end of the file).

-s, --stop-at-zero-byte

Stop dumping when encountering the first zero byte starting from the specified offset.

Examples

1. To display the first 512 bytes of file /mydir/myfile:

   cat --length 512 /mydir/myfile

2. To display many files:

   cat file1.txt file2.txt file3.txt

3. To dump the first 1024 bytes, or dump until reaching a zero byte, whichever occurs first:

   cat --length 1024 --stop-at-zero-byte data.txt

4. To unzip a large file

   cat --length 0 --binary zipped.txt.gz | unzip

If the file is very large, --binary option is needed so that cat can flush the standard output frequently.

5. To hex dump:

   cat file.txt --hex

The hex output formatting can be changed by nop or hexdump-set.

Applies To

EVS
