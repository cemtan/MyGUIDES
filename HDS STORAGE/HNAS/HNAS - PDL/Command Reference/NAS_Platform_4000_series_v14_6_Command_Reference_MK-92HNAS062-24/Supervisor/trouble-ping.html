trouble-ping

Contact all sources of trouble reports; display schemata

Syntax

trouble-ping

Description

'trouble-ping' is a tool for troubleshooting the 'trouble' command.

The 'trouble' command interrogates every server in a cluster and prints a prioritised list of faults and suggested solutions. It needs to contact every node in a cluster and, on some servers, it also needs to contact an external process called the 'trouble distributor'.

'trouble-ping' tries to make contact with every source of information used by the 'trouble' command. Where it succeeds, it prints a series of numbers that define the protocols supported by this source. (Any number, including zero, indicates success.) These numbers are intended for use by your support organization.

If 'trouble-ping' tries and fails to contact a trouble distributor, it will recommend that you run a program called 'trouble-distributor-ping', which can be found in the scripts directory. It must be run from a bash prompt in Linux: it cannot be run from the same prompt as 'trouble-ping'.

Examples

This example shows a successful 'trouble-ping' on a single-node cluster that uses trouble distributors (marked here as 'external'):

server:$ trouble-ping
Node 1 internal:
  List schemata  down 0
  List schemata  up   0
  Command header down 0
  Command header up   0
  Report              0
  Run            down 0
  Run            up   0
  List reporters down 0
  List reporters up   0
  List faults    down 0
  List faults    up   0
  Start perf rep down 0
  Start perf rep up   0
  Stop perf rep  down 0
  Stop perf rep  up   0
