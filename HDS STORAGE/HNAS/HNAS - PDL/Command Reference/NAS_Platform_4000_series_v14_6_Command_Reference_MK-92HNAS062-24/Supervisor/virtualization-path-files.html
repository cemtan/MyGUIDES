virtualization-path-files

Reports the list of files and directories that failed to virtualize for the specified association, and allows them to be retried.

Syntax

virtualization-path-files -t <fs label> -n <association name> { --retry | --list-failed }
Extra dev-only option [--convert-file <filename>]

Description

Lists all files and directories that have failed to virtualize correctly. It allows these to be retried as well. Retries for individual files can also be achieved by requesting them as normal, through an 'ls' command

Options:

-t, --target <fs label>

Target file system.

-n, --name <association name>

Name of the association to check.

-l, --list-failed

List the files that the association has failed to correctly process

-r, --retry

Trigger the association to try failed files again.

-c, --convert-file <filename>

Change the file name supplied to a filesystem file, rather than it being a virtualized file. If the file has not been migrated, the contents will be undefined.

This option requires the user to have DEV privilege.

Examples

virtualization-path-files -t Accounts -n assoc0 --list-failed

Lists the files that failed to virtualize on file system Accounts, within assoc0.

Applies To

EVS
