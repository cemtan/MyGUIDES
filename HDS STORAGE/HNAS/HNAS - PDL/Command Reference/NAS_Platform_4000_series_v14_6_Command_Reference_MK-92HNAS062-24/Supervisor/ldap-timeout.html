ldap-timeout

List / change NIS-over-LDAP request timeout value.

Syntax

ldap-timeout [  --timeout | -t  <new timeout in seconds>  ]

Description

This command is used to display or set the timeout value used for all NIS-over-LDAP search operations. A single timeout value is used for all lookup types: user, group and host, across all EVSs in the cluster.

Each NIS-over-LDAP lookup operation is attempted up to three times (original request plus two retries). The timeout value set here applies to each attempt, so the total timeout may be up to three times the value set with this command. Note that this timeout value typically comes into play when an LDAP server is heavily loaded. Simple lookup failures normally happen quite quickly.

If no timeout value is set, the default is 15 seconds. Permissible values range between 5 and 60 seconds.

--timeout, -t <timeout>

set LDAP search timeout to a new value in seconds

Examples

Display the current timeout setting for all NIS-over-LDAP searches:

ldap-timeout

Set the LDAP search timeout to 30 seconds:

ldap-timeout --timeout 30

Applies To

Cluster wide
