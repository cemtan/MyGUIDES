span-set-site-id

Assign a span's system drives to a new site

Syntax

span-set-site-id [--primaries | -p] [--secondaries | -s] <span-instance-name> <site-id>

Description

Assigns the system drives (SDs) in a span to a new site. Sites are described in the 'site' man page.

If you specify the --primaries switch, all the primary SDs in the span are assigned to the new site. If you specify --secondaries, all the secondary SDs are assigned. If you specify neither or both, all the SDs in the span are assigned, regardless of their mirror roles.

Primary SDs are either unmirrored SDs or, usually, mirrored P-Vols; secondary SDs are, usually, S-Vols. The 'storage-based-mirror' man page explains the exceptions.

To work with unspanned SDs, use 'sd-set'.

If the span's SD site IDs were inconsistent before the command was run, 'span-set-site-id' will queue messages that make it possible to mount file systems that could not be mounted beforehand. However, it will return control without waiting for these messages to be processed. A script using this command may need to follow it with 'span-wait-for-loading --mount' to ensure that all the file systems on the span are ready to mount (or have already auto-mounted) before running further commands.

Examples

span-set-site-id --primaries   Accounts 1
span-set-site-id --secondaries Accounts 2

Applies To

Cluster wide

See Also
