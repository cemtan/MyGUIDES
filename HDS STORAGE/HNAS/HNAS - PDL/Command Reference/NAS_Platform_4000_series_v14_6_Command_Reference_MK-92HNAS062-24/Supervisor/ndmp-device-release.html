ndmp-device-release

Release the specified NDMP device

Syntax

ndmp-device-release <device_name>

Description

The command attempts to release an NDMP device that has been left in a reserved state because of a node failure. This command is only relevant if ndmp-option reserve_devices has been set. The release may only be done from the same Fibre Channel port that was used when the device was reserved. So it needs to be used on the same node as the original reservation and backup-device-port must be used to force use of the correct Fibre Channel host port.

Note that this command cannot free devices that are actively in use. The ndmp-session-list command may be used to see if the device is being used. The ndmp-session-kill command might be used to release a device when a session has hung.

Examples

Attempt to free up device /dev/mt_d0l1 which has been left reserved after a previous NDMP session:

ndmp-device-release /dev/mt_d0l1

Applies To

EVS

See Also

backup-device-port ndmp-option ndmp-session-kill ndmp-session-list

Privilege Level

Supervisor
