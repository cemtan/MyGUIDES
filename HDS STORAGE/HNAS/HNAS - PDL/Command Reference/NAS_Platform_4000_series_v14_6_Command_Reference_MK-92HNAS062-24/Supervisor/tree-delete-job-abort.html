tree-delete-job-abort

Abort a tree deleting job specified by its job id

Syntax

tree-delete-job-abort <job-id> [--confirm]

Description

Abort a directory tree deleting job that is currently running or waiting in the queue.

<job-id>

Specifies the id of the job to be aborted. This id was previously returned by the tree-delete-job-submit command if the job was submitted successfully. One can only abort a job that is currently running or is waiting in the queue.

--confirm

If --confirm is not specified, the command will prompt for confirmation. Specifying --confirm suppresses this interactive confirmation.

User-aborted jobs will be moved to a special directory called USER_ABORTED_JOBS in /$__TREEDELETE_TRASH__. They are not automatically rescheduled for background truncation on file system remount. Users can reschedule them by running the tree-delete-job-reschedule command.

Examples

To abort job id 249e0a78-97da-11ce-9acb-13ad0cc2fa41:

$ tree-delete-job-abort 249e0a78-97da-11ce-9acb-13ad0cc2fa41

Applies To

Cluster wide
