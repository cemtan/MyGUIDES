evsipaddr

EVS IP address management

Syntax

evsipaddr --all | --evs <evs-id> [-a|-r|-u] [-i <addr>|<addr/prefix-length>] [-m <mask>] [-p <port>] [-c]

Description

The evsipaddr command lists and manages EVS network configurations.

EVS Types:

-e, --evs <evs-id>

use this option to specify EVS type as Admin Service (-e 0) or File Services EVS (-e 1..32)

-l, --all

use this option to specify both admin and service EVS for display. This cannot be used with any other options. An "(I)" following an IP address indicates an Inactive address.

Actions:

-a, --add

use this option to add an IP address to the admin or service EVS. EVS id, IP address, netmask/prefix-length, and port name are required.

-r, --remove [-c | --confirm]

use this option to remove an IP configuration from the admin or service EVS. EVS and IP address must be specified for the removal. The last IP in any EVS can only be updated but not removed. If an EVS is no longer needed, it can be removed from the cluster with the 'evs' command.

-u, --update [-c | --confirm]

use this option to update the IP configuration for the admin or service EVS. An EVS and at least one of the network settings (-i, -m, or -p) are required.

Network Settings:

-i, --ip

use this option to specify an IP address to be added, removed, or updated. This option can also be used to specify the prefix-length instead of specifying the netmask using -m option. This must be specified together with an EVS type and an action option.

-m, --mask

use this option to specify a netmask to be added or updated. The mask can also be specified as prefix-length using the -i option. This must be specified together with an EVS type and an action option.

-p, --port

use this option to specify a port name to be added or updated. This must be specified together with an EVS type and an action option.

Examples

display network settings for admin and service EVS:

evsipaddr -l

configure an IP address for EVS 1 on ag1 interface:

evsipaddr -e 1 -a -i 192.168.1.1 -m 255.255.255.0 -p ag1

configure an IP address for EVS 1 on ag1 interface using address and prefix-length:

evsipaddr -e 1 -a -i 192.168.1.1/24 -p ag1


evsipaddr -e 1 -a -i fdca:f995:220a:480:1:1:2e:2/64 -p ag1

configure an IP address for the Admin Service on ag2 public interface:

evsipaddr -e 0 -a -i 192.168.1.2 -m 255.255.255.0 -p ag2

update an EVS's last IP address:

evsipaddr -e 1 -u -i 192.168.1.3

update the netmask for an IP address assigned to an EVS:

evsipaddr -e 1 -u -i 192.168.1.4 -m 255.255.240.0

update the netmask using prefix-length

evsipaddr -e 1 -u -i 192.168.1.4/20


evsipaddr -e 1 -u -i fdca:f995:220a:480:1:1:2e:4/64

move an EVS's network settings from one interface to another:

evsipaddr -e 1 -u -p ag2

remove an IP address from an EVS:

evsipaddr -e 1 -r -i 192.168.1.1

Applies To

Cluster wide
