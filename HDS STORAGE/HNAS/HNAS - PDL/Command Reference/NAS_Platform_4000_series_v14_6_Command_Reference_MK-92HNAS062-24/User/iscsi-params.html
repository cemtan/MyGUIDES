iscsi-params

Manages iSCSI configurable parameters

Syntax

iscsi-params [-d] [-s] [<parameter>] [<new value>]

Description

This command manages iSCSI parameters. Parameters are used to control the behavior of the iSCSI service, including data transfer sizes, time outs, etc. The available parameters are:

DataDigest:

A data digest is a CRC that's used to verify the integrity of iSCSI data. Data digests are not currently supported, so this is always set to false.

DataPDUInOrder:

This is always set to true, meaning that iSCSI data PDUs will always be transported in increasing Buffer Offset order and overlays are forbidden.

DataSequenceInOrder:

This is always set to true, meaning that iSCSI data sequences will always be in increasing Buffer Offset order and overlays are forbidden.

DefaultTime2Retain:

This is the maximum time, in seconds, after an unexpected connection reset or termination, and after the Time2Wait period has expired, before the session state will be timed out. A value of 0 means that session state will be discarded immediately.

DefaultTime2Wait:

This is the minimum time, in seconds, after an unexpected connection reset or termination, before which an implicit logout will be attempted. A value of 0 means that an implicit logout will be attempted immediately.

ErrorRecoveryLevel:

Error recovery levels represent a combination of recovery capabilities. Currently, only error recovery level 0 is supported.

FirstBurstLength:

This is the maximum amount in bytes of unsolicited data that an iSCSI initiator may send to a target during the execution of a single SCSI command. This covers the immediate data (if any) and the sequence of unsolicited Data-Out PDUs (if any) that follow the command.

HeaderDigest:

A header digest is a CRC that's used to verify the integrity of iSCSI headers. Header digests are not currently supported, so this is always set to false.

IFMarker:

This is always set to false, meaning that target to initiator markers are not used on iSCSI connections.

ImmediateData:

This is always set to true, meaning that unsolicited data can be sent as part of an iSCSI command PDU, therefore reducing the number of separate PDUs that are required to transport a command and its data.

InitialR2T:

If set to true, the initiator must wait for the target to send it an R2T (ready to transfer) PDU between sending a command and sending its data. If set to false, the initiator can begin sending a command's data as soon as it has sent the command PDU.

MaxBurstLength:

This is the maximum SCSI data payload in bytes in a Data-In or a solicited Data-Out PDU sequence. A sequence consists of one or more consecutive Data-In or Data-Out PDUs that end with a Data-In or Data-Out PDU with the F bit set.

MaxConcurrentCommands:

This controls the size of the CmdSN (command sequence number) window, which governs how many concurrent commands the initiator can have outstanding at the target at any one time.

MaxConnections:

When using MPIO (multi-path I/O), this controls the maximum number of TCP connections that can be used within a single iSCSI session.

MaxOutstandingR2T:

This is always set to 1, meaning that within a given command, there will only ever be 1 outstanding data transfer at any one time. There can, however, be more than one command in progress at any one time.

MaxRecvDataSegmentLength:

This is the maximum amount in bytes of data that can be sent in a single iSCSI PDU.

OFMarker:

This is always set to false, meaning that initiator to target markers are not used on iSCSI connections.

UsePhaseCollapse:

If set to true, the status of a successful iSCSI command will be sent in the final Data-In PDU in the response sequence. If set to false, the status will be sent in an additional Response PDU following the response sequence. So, setting this to true reduces the number of PDUs that are required to send a response.

Usage:

iscsi-params [-d] [-s] [<parameter>] [<new value>]

-d

Resets parameters to defaults.

-s

Show supported parameters.

Examples

iscsi-params


iscsi-params -d


iscsi-params -s


iscsi-params DefaultTime2Wait 60

Applies To

Cluster wide
