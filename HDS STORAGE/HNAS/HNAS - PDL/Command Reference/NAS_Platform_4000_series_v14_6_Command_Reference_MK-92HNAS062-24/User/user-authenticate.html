user-authenticate

Authenticates a network user and reports their credentials

Syntax

user-authenticate [-b|--base64-args] [-i|--interactive] [-u|--report-unix-credentials [-j|--json]] [-s|--no-nis-auth | -t|--no-nt-auth] [-v|--verbose] user password

Description

Authenticates a network user and reports their credentials.

Authentication attempts are made in turn against each supported authentication mechanism, stopping when one succeeds or every mechanism has been tried. Currently supported authentication mechanisms are NT and NIS.

N.B. This command does not authenticate "management" users that are configured with the "user" command.

-b, --base64-args

Specifies that the user and password fields are base64 encoded.

-i, --interactive

Causes the password to be prompted for interactively.

-u, --report-unix-credentials

Map the user's credentials (which might contain NT, NFSv4 or Kerberos principals) to Unix credentials. If the user or primary group principal can't be mapped to a Unix user/group, the anonymous Unix user/group will be reported.

-j, --json

Report a JSON object containing the authentication status, and if successful, the user's Unix credentials.

-s, --no-nis-auth

Don't attempt a NIS authentication.

-t, --no-nt-auth

Don't attempt an NT authentication.

-v, --verbose

Report per authentication mechanism status and timing information.

Examples

$ user-authenticate -u test .test
Successfully authenticated: test
uid=1150(test) gid=504(software) groups=516(development),1010(bk)
