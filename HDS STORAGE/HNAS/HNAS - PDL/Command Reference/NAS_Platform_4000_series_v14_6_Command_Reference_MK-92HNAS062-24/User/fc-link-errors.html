fc-link-errors

Show FC link error stats

Syntax

fc-link-errors [-a] [-r]

Description

This command is used to display or reset statistics on low-level Fibre Channel link errors.

Options

-a, --all

display statistics for all interfaces, not just enabled ones

-r, --reset

reset statistics

Error details

Loss of Signal

As reported by the internal transceiver hardware or a pluggable optics module.

Bad Received Character

8b/10b decoding failures. It is normal for there to be a number of these when the link is first brought up.

Loss of Synchronisation

The FC controller detects an incoming signal but is unable to synchronise with it.

Link Failure

Loss of signal, loss of synchronisation (for longer than R_T_TOV), transmitter fault, N_Port NOS, or loop initialisation time-out.

Received EOFa

Incoming frames terminated with EOFa (End of Frame Abort). This delimiter is used when a transmitter or forwarder detects an error during transmission of a frame.

Discarded Frames

Frames discarded for reasons including: invalid EOF delimiter, CRC error, control character inside a frame, frame interrupted by link failure, frame too long, frame too short. Aborted frames are not included in this count.

Bad CRC

Frames with valid EOF but bad CRC. Invalid and aborted frames are not included.

Protocol Error

Counts N_Port protocol errors and incoming LRR primitive sequences.

Examples

server:$ fc-link-errors
Errors:
                  Lost signal    Bad RX char    Lost sync    Link fail
 FC 1                      0           12              0            0
 FC 3                      0           10              0            0


                     RX EOFa  Discarded frames   Bad CRC  Protocol errors
 FC 1                      0            0              0            0
 FC 3                      0            0              0            0

Applies To

Cluster node
