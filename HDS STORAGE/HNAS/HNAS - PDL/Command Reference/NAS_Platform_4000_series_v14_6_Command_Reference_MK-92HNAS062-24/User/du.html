du

Displays true and apparent disk usage for a specific file

Syntax

du [--human-readable] [--apparent-size] [--exclude-overhead] [--block-size <block size>] [--more] [--file-system <file-system>] <file>

Description

The du command displays disk usage for a selected file. Disk usage differs from file length because it includes metadata overhead. Additionally, if sparse files are in use, disk usage may be reduced significantly, as empty data blocks are not allocated by the file system.

Usage:

-h, --human-readable

(optional) Print sizes in human-readable format as a number of bytes (e.g., 1K 234M 2G). By default usage is reported in blocks.

--apparent-size

(optional) Print apparent size (the "file length") in bytes, rather than disk usage. Although the apparent size is usually smaller than disk usage (when expressed in bytes), it may be larger for sparse files.

-B, --block-size

(optional) Use a specific block size when reporting disk usage in blocks. Use 1 to report disk usage in bytes.

-e, --exclude-overhead

(optional) Prints disk usage excluding metadata overhead. Ignored when used with --apparent-size.

-m, --more

(optional) Prints apparent size, disk usage in bytes, disk usage in blocks, the data/metadata split, and the block size used.

-f, --file-system

(optional) Looks for the requested file on an alternate file system.

<file>

(required) The name of the file to examine.

Examples

du chr2.bed

Displays the disk usage in blocks of the file "chr2.bed".

du -B 1 ubcs.RData

Displays the disk usage in bytes of the file "ubcs.RData".

du -h sequence.wib

Displays in human-readable form, the disk usage in bytes of the file "sequence.wib".

du -h --apparent-size rhesus.scaffold

Displays in human-readable form, the apparent size in bytes of the file "rhesus.scaffold".

du -B 512 --exclude-overhead human-chimp.alignment

Displays the disk usage, excluding metadata overhead, in blocks of size 512 of the file "human-chimp.alignment".

du -m chimp_chr2a.bed

Displays everything for the file "chimp_chr2a.bed".
