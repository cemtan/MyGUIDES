alert-syslog

Report/configure alerts by syslog

Syntax

alert-syslog operation
where 'operation' is one of:
list
set  [-s <i|n>] [-w <i|n>] [-i <i|n>]
add <recipient>
del <recipient>
delall

Description

Reviews or changes the frequency with which alerts are sent to the syslog.

Usage:

list

Displays the current frequency and the configured recipients.

set [-s <i|n>] [-w <i|n>] [-i <i|n>]
-s

Alters the rate at which severe alerts are sent:

i

immediately

n

never

-w

Alters the rate at which warning alerts are sent:

i

immediately

n

never

-i

Alters the rate at which information alerts are sent:

i

immediately

n

never

add <recipient>

Adds a recipient. Recipient can be in the form of an IPv4 address, IPv6 address or host name.

del <recipient>

Deletes a recipient. Recipient can be in the form of an IPv4 address, IPv6 address or host name.

delall

Deletes all the recipients.

The behavior of syslog alerts is affected by the syslog_and_snmptrap_come_from_admin_vnode variable. When set to true (default is false), alerts will be sent from an admin vnode IP address. As its name suggests, this variable also affects the sending of SNMP traps.

Examples

server:$ alert-syslog set -s i -w n -i n

server:$ alert-syslog add acomputer.example.com

server:$ alert-syslog add 10.1.2.3

server:$ alert-syslog add fdca:f995:220a::2:c

server:$ alert-syslog list

Severe alerts:             Immediately
Warning alerts:            Never
Information alerts:        Never
Syslog recipients:  acomputer.example.com
