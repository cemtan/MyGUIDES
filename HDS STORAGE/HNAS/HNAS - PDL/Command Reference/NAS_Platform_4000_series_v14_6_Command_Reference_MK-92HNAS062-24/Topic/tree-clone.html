tree-clone

Description

Tree cloning provides a mechanism for quickly creating a space-efficient copy of a directory tree. The source directory's structure is recreated in the destination directory and all the contained files are cloned. Security information from the source files and directories is applied to the destination. The source directory may be modified while tree cloning is in progress, hence the cloned destination is not a point-in-time replica. Tree cloning uses the same mechanism as file cloning to clone individual files within the tree, so the same limitations apply.

Tree clone jobs are submitted using the tree-clone-job-submit command. Multiple tree cloning jobs may be run on the same or different file systems. Up to 100 jobs may be submitted at a time.

tree-clone-job-list shows a list of running and recently completed jobs. Recently completed jobs are retained for a minimum of 2 minutes after completion. Job status may be queried using the tree-clone-job-status command.

Jobs may be aborted using the tree-clone-job-abort command. All jobs on a file system are aborted when the file system is unmounted.
