filesystem-raw-dump

Hexdump bytes from a filesystem, bypassing server caches

Syntax

filesystem-raw-dump [--terse | --verbose] [--timeout <nr-seconds>] <filesystem-instance-name> (<byte-offset> | ssb|dvcb|dsba|dsbb|zeroes) [<length-in-bytes>]

Description

(This man page contains wide lines. You may wish to widen your console window or use 'man -n'.)

This command is intended for use by your support provider. It translates filesystem offsets into other forms (such as offsets into system drives), and it produces hex dumps of data from filesystems, bypassing server caches and requesting data directly from the storage.

For an explanation of the translated offsets printed by this command, see 'filesystem-translate-offset'.

Arguments:

--terse

Suppresses the usual offset-translation; just dumps data from disk.

--verbose

Prints a system drive device ID and offset before each block of data, and suppresses the usual folding of multiple lines of identical data. Not compatible with --terse.

--timeout

Limits the amount of time that the command spends dumping data. If the specified time is exceeded, the command will abort. The default timeout is 120 seconds.

filesystem

The instance name, device ID or volume number of the filesystem from which you wish to dump data.

byte-offset

The offset into the filesystem at which you wish to start dumping data, either as a number or as a symbolic name. For WFS-2, names are of the form 'dsbX-Y', where X is a DSB number and Y is either 0 (for the original) or 1 (for the copy).

length

The number of bytes you wish to dump. If you specify a length of zero (or omit the length entirely), the command will dump

  • one disk block of data if you specify a numeric offset, or
  • the appropriate size if you specify the name of an on-disk structure.

The command always dumps a whole number of disk blocks of data. It can't dump more than one mebibyte (that is, 2**20 bytes).
