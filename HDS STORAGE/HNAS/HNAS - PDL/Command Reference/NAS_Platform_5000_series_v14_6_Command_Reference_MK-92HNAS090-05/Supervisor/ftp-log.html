ftp-log

Show/modify ftp audit logging configuration

Syntax

ftp-log [-o <on|off>] [-d <directory>] [-f <file system>]
[-s <file size>] [-n <number of files>] [-e <on|off>]

Description

The ftp-log command is used to configure FTP audit logging. When enabled, the FTP server logs details about user logins and the operations they perform.

Options:

-o, --toggle (on|off)

Switches on/off FTP audit logging.

-d, --directory <directory>

Sets the destination directory for the log files (default is the file system's root directory).

-e, --ensure (on|off)

Switches on or off the 'ensure directory exists' option.

-f, --file-system <filesystem-label>

Sets the destination file system for the log files.

-s, --size <number-of-records>

Sets the maximum size of a log file (number of records).

-n, --number <number-of-log-files>

Sets the total number of log files.

Examples

Display FTP log configuration:

ftp-log

Disable FTP audit logging:

ftp-log -o off

Change the logging directory to logdir:

ftp-log -d logdir

Enable ensure-exist option, directories will be created if changed:

ftp-log -e on

Change the logging file system to fs1:

ftp-log -f fs1

Change the log file size to 20000 records:

ftp-log -s 20000

Change the number of log files to 15:

ftp-log -n 15

Supervisor
