dump-disk

Attempts to interpret and dump disk contents to the console

Syntax

dump-disk  [-f <file-system>] [-n <number-of-bytes>] [--dir | --tree | --root | --leaf | --ssb | --dsb | --dvcb | --sd] <sector-number>

Description

dump-disk dumps the contents of specified disk sector addresses in human readable format. Its main use is to display the server metadata structures, but may also be used to display hexadecimal dumps of random data.

If the type of metadata is unknown, dump-disk will attempt to identify the type of metadata by the particular characteristics of the data at the specified location. However, if it is unable to identify the metadata type, dump-disk may be forced to display the data in a variety of different formats using one of several optional command line switches.

This utility can identify and display the following types of metadata: Static Superblock; Dynamic Superblock; Dynamic Volume Config block; Root Onode; Leaf Onode; Directory Chunks; Directory Tree Nodes; Security Descriptor.

Note: dump-disk may be run with the file system mounted or unmounted.

Usage:

-f <file-system> The file system on which the data resides

-n <number-of-bytes> The number of bytes to read

--dir Force decode as a directory's WDIR metadata

--tree Force decode as a directory's WTREE metadata

--root Force decode as a root onode

--leaf Force decode as a leaf onode

--ssb Force decode as a static super block

--dsb Force decode as a dynamic super block

--dvcb Force decode as a dynamic volume config block

--sd Force decode as a security descriptor

Examples

To attempt to interpret and dump 2048 bytes of data resident at disk sector address 0x2c0 on file-system 2:

dump-disk -f 2 -n 2048 0x2c0

To force dump-disk to display the disk contents at sector address 0x1800 as a Leaf Onode on file system testfs:

dump-disk -f testfs --leaf 0x1800

Applies To

EVS

Privilege Level
