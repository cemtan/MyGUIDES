filesystem-leaf-onode-tier

Set/query where leaf onodes of regular files are written to

Syntax

filesystem-leaf-onode-tier [--zero | --one] <filesystem-label>

Description

In a multi-tiered file system, leaf onodes of regular files are by default stored on tier zero storage. The metadata of a file is arranged in a tree like fashion, with each node in the tree called an onode. A leaf onode is the lowest-level node of the metadata tree. In most file systems, leaf onodes will make up the majority of meta data. To reduce the storage requirement of tier zero, this command allows you to specify that leaf onodes of regular files are stored on tier one.

In order to change the tier, the file system must be unmounted. To report the tier leaf onodes are currently stored on, the file system can remain mounted.

If a file system is reformatted, it will revert back to the default of storing leaf onodes on tier zero.

Options

--zero

Change to store leaf onodes on tier zero

--one

Change to store leaf onodes on tier one

<file system>

The file system to query or change.

If neither --zero or --one is specified, the current setting is reported.

Examples

Change file system "fs0" to store leaf onodes on tier one:

filesystem-leaf-onode-tier --one fs0

Change file system "fs0" to store leaf onodes on tier zero:

filesystem-leaf-onode-tier --zero fs0

Report the tier on which leaf onodes will be stored on file system "fs0":

filesystem-leaf-onode-tier fs0
