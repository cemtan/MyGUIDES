span-expedite-compression-space-check

Tell the server to check compressed physical space at once on all spans

Syntax

span-expedite-compression-space-check

Description

This comand relates to compressed HDP pools based on FMD storage.

Every three hours, the server scans for compressed HDP pools whose free physical space (as shown by 'span-list -s') is dangerously low, and which are in danger of failing. Where possible, it recycles enough recently deleted filesystems to create vacated chunks (in the same manner as the 'filesystem-recycle' command) and/or launches an unmapper (in the same manner as 'span-unmap-vacated-chunks'). These measures return unused space to the HDP pool and free up physical space. For more details, see 'span-set-cap-warn-thresh'.

'span-expedite-compression-space-check' instructs the server to perform these checks within the next few seconds, rather than waiting for the next scheduled cycle.

Examples

Applies To

Cluster wide
