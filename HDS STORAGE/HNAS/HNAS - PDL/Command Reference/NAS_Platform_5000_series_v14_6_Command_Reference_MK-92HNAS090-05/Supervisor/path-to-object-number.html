path-to-object-number

Attempt to resolve an object number from a path

Syntax

path-to-object-number  [--dec|--hex|--verbose] [-f <file-system>] <full-path>

Description

Supplied with the path of target file or directory, this utility will attempt to resolve its object number. Because path-to-object-number has no notion of the current working directory when the file system is not mounted and currently selected, it is mandatory to specify an absolute path name starting with "/". If the current working directory is defined, the path can be specified either relative or absolute. Success depends on the integrity of the metadata for the directories included in the path, but not the target object itself.

It is safe to run the path-to-object-number command on a mounted or unmounted file system.

Usage:

-f|--file-system <file-system>

The file system on which the object resides.

-d|--dec

The output consists of only the object number in decimal. Useful for scripting purposes.

-h|--hex

The output consists of only the object number in hexadecimal. Useful for scripting purposes.

-v|--verbose

The object number is displayed in both hex and decimal in a verbose output message.

Note that at most one of the options -d, -h, -v can be specified. If none of them is specified, the object number is displayed, by default, in hex in a verbose output message.

Examples

To obtain the object number for file /home/development/libs/core/core.h on file system fs1:

path-to-object-number -f fs1 /home/development/libs/core/core.h

Alternatively,

selectfs fs1
 cd /home/development/libs
 path-to-object-number core/core.h  # same result
 cd core
 path-to-object-number core.h       # same result
 path-to-object-number ./core.h     # same result
