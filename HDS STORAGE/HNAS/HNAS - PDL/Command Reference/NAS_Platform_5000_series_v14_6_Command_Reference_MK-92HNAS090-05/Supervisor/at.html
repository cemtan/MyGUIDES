at

Manipulate at job table

Syntax

at <operation> [<options>]

Description

Queues a job for later execution by cron. A job is a command or a series of commands.

Operation is one of:

list

Lists jobs.

add [-m <mailList>] <date-time-spec> <command-list>
mail-list

The full email address of the user(s) to mail output to. Multiple users can be specified, in quotes, in the form "user1;user2".

Note: If no email addresses are specified, the output will be displayed in the console.

date-time-spec

The date/time at which to run the command(s). The date/time should be enclosed in double quotes. eg "23:50 12/16/2002".

For further information, see 'DATE AND TIME SPECIFICATION', below.

command-list

The command(s) to run. Multiple commands must be specified, in quotes, in the form:

"command1;command2"

del ID

Deletes a job. ID is that displayed by the list command.

delall

delete all at entries.

Date and time specification

at accepts an argument of the form HH:MM (always enclosed in double quotes) to run a job at a specific time of day. If that time is already past, the next day is assumed. It is also possible to specify "midnight", "noon", or "teatime" (4pm), and time-of-day can be suffixed with AM or PM for running in the morning or the evening. The day the job will be run can be controlled by giving a date in the form "month-name day" with an optional year, or giving a date of the form MMDDYY or MM/DD/YY or DD.MM.YY. If the time of day is also to be specified, it must precede the date.

Times of the form "now + count time-units", where the time-units can be minutes, hours, days, or weeks, can be given, and at can be told to run the job today or tomorrow by suffixing the time with "today" or "tomorrow" respectively. e.g "now + 2 minutes" means 2 minutes from now.

Examples

Adding a job

at add -m support@example.com "08:00" ver

Date and time examples

"4pm + 3 days"

4pm three days from now

"10am Jul 31"

"1am tomorrow"

"now + 4 hours"
