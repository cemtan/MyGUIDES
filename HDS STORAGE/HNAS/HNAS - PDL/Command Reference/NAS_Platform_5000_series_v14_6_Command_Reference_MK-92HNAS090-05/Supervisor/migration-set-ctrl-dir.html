migration-set-ctrl-dir

Sets a control directory for a migration path

Syntax

migration-set-ctrl-dir {-s <source> [-v <vivol>] (-c|-p <dir_path>) [-d]}

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol name>

Name of source Virtual Volume (optional).

-c, --control-dir-default

Use the default control directory location (.control) with this migration path. Note that -c and -p are mutually exclusive.

-p, -control-dir-path <dir path>

The path (relative to the root of the file system/vivol) to the control directory.

-d, --delete

Remove the directory path specified from the list of directories that will be used to contain request files for the specified control path.

Specifies the location of a directory that is to be used to contain request files for a migration path.

The manual page for migration-list-ctrl-dir-requests describes how migration request files should be named.

Examples

migration-set-ctrl-dir -s fs0 -c

Set a control directory for the fs0 migration path to the default location.

migration-set-ctrl-dir -s fs0 -p /path/to/ctrl_dir

Set a control directory for the fs0 migration path to the /path/to/ctrl_dir directory on fs0. On completion the command will display the control directory and migration path affected and whether it was set or deleted.
