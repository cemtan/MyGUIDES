backup-device-port

Control host port used for backup device connection

Syntax

backup-device-port [-h <host port id> | --reset] [<backup devices>]

Description

The backup-device-port command is used to show or change the preferred Fibre Channel host port used to connect to a tape drive or robot device. It is usually unnecessary to set a preferred port. However, if the ndmp-option to reserve devices has been enabled it is recommended that this command be used to select a preferred port for each device.

<backup devices>

Select which backup devices to display or act on. See the sd-spec man page for more details. Only licensed devices will be selected. If this argument is omitted, all licensed backup devices will be selected.

-h <1..N>

Sets the preferred host port for the selected device(s) (-1 for no preference)

--reset

Clears host port preferences for the selected device(s)

With no options, path information is shown for the selected device(s).

Changes to a device's configuration will take effect when it is next opened.

Examples

server:$ backup-device-port
Device 800    Preferred host port: None
Device 801    Preferred host port: None
Device 802    Preferred host port: None

Shows the current preferred port settings. These are the defaults.

server:$ backup-device-port -h 1 801
OK
server:$ backup-device-port -h 2 802
OK
server:$ backup-device-port
Device 800    Preferred host port: None
Device 801    Preferred host port: 1
Device 802    Preferred host port: 2

Sets a preferred host port for all tape drives after running command "ndmp-option reserve_devices tape".

Applies To

Cluster wide

See Also

ndmp-option sd-spec sdpath

Privilege Level

Supervisor
