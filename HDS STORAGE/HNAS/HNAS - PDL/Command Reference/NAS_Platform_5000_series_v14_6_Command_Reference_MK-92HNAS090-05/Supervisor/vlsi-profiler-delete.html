vlsi-profiler-delete

Remove a state machine from automated vlsi profiling.

Syntax

vlsi-profiler-delete <chip> <stateMachine>

Description

This command is used to remove a state machine from those "eligible for profiling" for the automated vlsi-profiler.

Note that once a state machine is removed from the list, the time since it was last profiled will be lost. This means a state machine that is deleted and then re-added is eligible for profiling instantly, even if the 30 minute wait time between profiles has not passed.

To remove a state machine, both the chip name and state machine name must be supplied.

Examples

Remove the "seeq" state machine from NIBRX:

vlsi-profiler-delete nibrx seeq

Applies To

Cluster node
