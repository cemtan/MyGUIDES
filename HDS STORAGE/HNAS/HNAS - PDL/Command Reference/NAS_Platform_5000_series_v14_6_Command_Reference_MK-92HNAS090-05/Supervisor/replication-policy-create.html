replication-policy-create

Create an object replication policy

Syntax

replication-policy-create [-s <source ip address>] [-p <target port>] [--no-target-check] <policy name> <target> <source fs label> <target fs label> [<source-snapshot-option> <source-snapshot-rule> [<target-snapshot-option> <target-snapshot-rule>]]

Description

Create a policy for object replication.

--no-target-check

By default a message is sent to the target server to check that the replication target with the given name is available. If it is not available, the policy creation will fail. Providing this option will enable a policy to be created even if the replication target is not available.

--source-ip-address, -s <source ip address>

Specify source EVS host name or IP address. If this option is not provided, the server will select a source IP address automatically based on the target server's IP address. IP addresses on the same subnet as the target server will be chosen in preference to those on different subnets.

An IP address can be specified as either an IPv4 address in dotted decimal notation (e.g. 203.0.113.0) or as an IPv6 address (e.g. 2001:db8:fe58:3efa:deca:212::c0a9)

--target-ip-port, -p <target port>

Specify object replication listening port of the target host. The default is 59550.

<policy name>

Name of the policy, which must be unique.

<target>

Target EVS host name or IP address.

An IP address can be specified as either an IPv4 address in dotted decimal notation (e.g. 203.0.113.0) or as an IPv6 address (e.g. 2001:db8:fe58:3efa:deca:212::c0a9)

<source fs label>

Source file system label.

<target fs label>

Target file system label.

<source snapshot option>

Source snapshot option, allowed values are:

auto (default)

Take an automatic snapshot and replicate that snapshot. After the object replication completes, the previous snapshot is deleted.

use-named

Replicate the latest snapshot taken with the named snapshot rule.

<source snapshot rule>

Name of the snapshot rule when the source snapshot option is set to use-named.

<target snapshot option>

Target snapshot option, allowed values are:

auto (default)

When the object replication completes, take an automatic snapshot.

use-named

When the object replication completes, take a snapshot using the named snapshot rule.

<target snapshot rule>

Name of the snapshot rule when the target snapshot option is set to use-named.

Examples

replication-policy-create DefaultPolicy FileServer2 fs0 fs0r use-named snap5

Creates an object replication policy named 'DefaultPolicy' that replicates the latest snapshot in the snapshot rule named 'snap5' from file system fs0 to target file system fs0r on EVS hostname FileServer2.

Applies To

EVS
