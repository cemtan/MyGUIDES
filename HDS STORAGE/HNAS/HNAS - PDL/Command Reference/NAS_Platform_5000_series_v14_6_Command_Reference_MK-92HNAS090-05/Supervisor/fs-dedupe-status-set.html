fs-dedupe-status-set

Enable/disable dedupe for a file system

Syntax

fs-dedupe-status-set <file-system> (enable|disable)

Description

Enable/disable deduplication for a file system.

Note that read cache file systems cannot be deduped.

Options:

enable|disable

Enable/disable a file system to be scheduled for deduplication.

Note that dedupe can be enabled for Object replication target file systems.

Dedupe cannot be enabled for WORM file systems.

If option 'enable' is specified and if the dedupe service is in the auto-enable state (i.e., automatically enabled on demand), the command will attempt to enable the dedupe service if there are sufficient free resources. If the attempt fails, the command will fail, leaving the dedupe-enabled state of the file system unchanged.

On disabling dedupe on a file system, note that:

  • Any data that is already deduped continues to remain so even after dedupe is disabled on the file system.
  • Any in-progress dedupe job for the file system will be aborted when dedupe is disabled
<file-system>

File system label.

Examples

Enable file system 'fs0' to be scheduled for deduplication.

fs-dedupe-status-set fs0 enable

Applies To

File system
