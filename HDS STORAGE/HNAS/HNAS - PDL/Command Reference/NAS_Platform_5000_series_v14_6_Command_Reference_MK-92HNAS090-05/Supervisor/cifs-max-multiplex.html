cifs-max-multiplex

Manage the max number of concurrent CIFS requests allowed

Syntax

cifs-max-multiplex [<new value>]

Description

The cifs-max-multiplex command is used to control the maximum number of concurrent SMB1 requests that can be outstanding on a connection at any one time.

When a SMB1 client connects to the server, the server's negotiate protocol (NEGPROT) response includes a MAXMPX, or max-multiplex, value. This value informs the client of the maximum number of concurrent requests that the server wishes to receive.

Note, however, that increasing the max-multiplex value won't necessarily cause the client to send more concurrent requests. It remains up to the application to decide how many concurrent requests it will send, up to the server's max-multiplex value.

The default max-multiplex value is 255.

The maximum max-multiplex value is 65535.

Applies To

Cluster wide

Privilege Level

Supervisor
