test-route-by-evs

tests the router

Syntax

test-route-by-evs -e <evs> dst
test-route-by-evs -e <evs> src dst

Description

The test-route-by-evs command provides similar access to the routing decision making process on the server as the test-route command except that its decisions are influenced by specifying an EVS.

As well as specifying an EVS, the test-route-by-evs command takes one or two IP addresses. If you supply two IP addresses, the first IP address is the source IP address that'll be used in the routing decision and the second IP address is the destination address. If you supply a single IP address, test-route-by-evs works out the route that'll be used to get to that IP address. NB if you supply two IP addresses, then test-route-by-evs behaves like test-route and ignores the EVS setting.

Like the test-route command, the test-route-by-evs command is board specific as different routing decisions are made on the different boards as the available interfaces are different.

Usage:

test-route-by-evs -e <evs> dst


test-route-by-evs -e <evs> src dst

Options:

-e, --evs <evs-id>

Specifies the EVS to use to influence the routing choice.

Examples

To display how a packet would go to 10.1.1.7 from any IP address for evs 1 (ie the source IP address is free to be chosen):

server:$ test-route-by-evs -e 1 10.1.1.7
route from 172.18.0.102->10.1.1.7 via 172.18.254.254 i/f ag1

To display how a packet would go to 10.1.1.7 from any IP address for evs 2 (ie the source IP address is free to be chosen):

server:$ test-route-by-evs -e 2 10.1.1.7
route from 172.18.0.103->10.1.1.7 via 172.18.254.254 i/f ag1

Applies To

Cluster node
