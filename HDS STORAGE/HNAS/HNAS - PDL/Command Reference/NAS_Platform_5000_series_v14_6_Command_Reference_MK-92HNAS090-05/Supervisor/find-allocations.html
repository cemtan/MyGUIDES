find-allocations

Search volume and report on owner of allocations and the FSA bitmap state

Syntax

find-allocations  <-a | <--block <block-number> ... | --byte <byte-number> ... | --sector <sector-number> ...>> -f <file-system>

Description

Supplied with a file-system and one or more address arguments, find-allocations searches the file system for the objects that own the allocations that correspond to them, reporting whether the allocations themselves are root onode allocations, or printing the object number of the object that owns the allocation.

The on-disk address may be supplied as a fs block-number, sector-address, or byte-address. A "block" is a file system unit that is currently either 4K or 32K bytes in size, depending on which size was specified at format time. A block-number is therefore an offset in blocks into a given file system. A sector is 512 bytes in size, and is the basic addressing unit of almost all on-disk metadata.

If searching for addresses found in the on-disk metadata, then the sector-address is likely to be the easiest to use as translation is seldom warranted. If, however, the address has been found by some translation from the underlying storage, then a byte-address might be the obvious way to specify the arguments. It is difficult to imagine why anyone would want to search by block-number as it is not used in on-disk metadata, and the user would have to calculate the block-number from some other unit. Having said that, however, it remains an option although its use is not recommended.

Note: find-allocations is an asynchronous utility and may be run with the file system mounted or unmounted.

Consult the dynamic log to view the results of the search.

Options:

-a

Abort the currently running instance of this utility.

--block

One or more FS block numbers for which to search

--byte

One or more byte addresses for which to search

--sector

One or more sector addresses for which to search

Examples

To determine who owns the blocks at sectors 0x88000 and 0x12800 on file system 0
    find-allocations -f 0 --sector 0x88000 0x12800


To determine who owns the block at byte address 0x45289000 on file system testfs
    find-allocations -f testfs --byte 0x45289000


To determine who owns block numbers 832 1943 on file system 2
    find-allocations -f 2 --block 832 1943


To abort find-allocations on file system myfs
    find-allocations -a -f myfs
