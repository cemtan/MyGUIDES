sd-deny-access

Unlicense (deny access to) one or more system drives

Syntax

sd-deny-access <system-drives-see-'man sd-spec'>

Description

The server will not let you create a new span on a system drive (SD) or load an existing span from an SD until you have allowed access to the SD. Allowing access to an SD is often called licensing the SD.

'sd-deny-access' prevents the server or cluster from using the SD in any way. The SD will, however, continue to appear in the output of commands such as 'sd-list', and can be relicensed by the 'sd-allow-access' command.

Never allow access to an SD that is being used by another server or cluster or a foreign server. If transferring ownership of an SD from one server or cluster to another, always deny access on the old owner before allowing access on the new owner.

You cannot deny access to an SD that's used in a span (unless the SD is in a primary--primary mirror relationship), because it would force the span offline. Use 'span-deny-access' for a way to deny access to all a span's SDs in a single operation, but first read that command's man page to understand its other side effects.

You can deny access to several SDs with a single command; see the 'sd-spec' man page.

Examples

sd-deny-access 0-3

Denies access to SDs 0, 1, 2 and 3.

sd-deny-access !Accounts

Denies access to all SDs that aren't used in span 'Accounts' or, if 'Accounts' is a filesystem, all SDs that aren't used in its host span.

Applies To

Cluster wide

See Also

Privilege Level
