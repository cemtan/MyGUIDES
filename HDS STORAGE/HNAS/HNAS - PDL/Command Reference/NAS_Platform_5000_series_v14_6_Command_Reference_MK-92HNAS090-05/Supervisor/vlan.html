vlan

Display/Remove VLANs (deprecated)

Syntax

vlan [show [vlan#]] | remove vlan# [net-addr[/prefix-length] net-mask] | remove-all

Description

Note - this command is deprecated - use vlan-interface-create and associated commands instead.

The vlan command manipulates the VLAN tables. It is used to view or modify the VLAN membership of individual IP subnets. Each subnet can belong to at most one VLAN. When a VLAN is defined, the server expects tagged traffic on that VLAN and will ignore/discard untagged traffic. One should, therefore, never configure a VLAN on the server for a native VLAN, even if it is a part of the 802.1q trunk.

Without any options, the vlan command displays the contents of the VLAN tables.

To display VLAN information:

vlan

To remove an IP subnet from a VLAN:

vlan remove <tag> <net-addr>[/prefix-length] [<net-mask>]

where:

<tag>

is the VLAN tag that will be used for all transmitted traffic and expected for all received traffic,

<net-addr>[/prefix-length]

is the IP subnet, and

<net-mask>

is the mask corresponding to the IP subnet.

One of (but not both of) prefix-length or net-mask must be supplied.

To remove all subnets from a specific VLAN:

vlan remove <tag>

Note that MTU values can be set for a specific VLAN interface using ipadv.

Examples

To remove the 192.0.2.0/24 network from VLAN 22:

vlan remove 22 192.0.2.0 255.255.255.0

Applies To

Cluster wide
