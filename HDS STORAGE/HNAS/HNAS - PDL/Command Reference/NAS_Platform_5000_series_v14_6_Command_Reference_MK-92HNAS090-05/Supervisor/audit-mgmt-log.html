audit-mgmt-log

View management audit log

Syntax

audit-mgmt-log [--since YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]]

Description

Displays the content of the management audit log on the present cluster node. Each cluster node maintains a log of management change operations performed on EVSes and other resources presently residing on that node. This is the same data which is forwarded to a CEF server when configured, but the local log is written even when no CEF server is configured or reachable.

This command displays only the most recent 1MB of audit log content. If the log contains more data than that then older entries must be accessed by other means.

Options

since

Shows audit log entries added since the specified time. If this would result in more than 1MB of log data being returned, then only the most recent 1MB is displayed. If this option is omitted, the most recent 1MB of entries in the current log are displayed.

Examples

To display all entries in the current cluster node's management audit log:

audit-mgmt-log

To display all entries in cluster node 2's management audit log:

cn 2 audit-mgmt-log

To display audit log entries added since a specified local time:

audit-mgmt-log --since "2016-01-29 17:34:56"

To display audit log entries added since a specified absolute time:

audit-mgmt-log --since "2016-04-14 15:32:45-07:00"

Applies To

Cluster node
