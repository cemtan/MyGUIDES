sd-group-delete

Delete system drive groups

Syntax

sd-group-delete <SD-device-IDs-see-'man sd-spec'>...
Alias:       rmsdg

Description

Deletes one or more system drive (SD) groups. See the 'sd-group' man page for background.

SD groups don't have IDs of their own. To delete a group, specify one or more of the SDs it contains.

A group cannot be deleted if any of its SDs is in an open span. (A span is open if any of its file systems is mounted or is being checked, fixed, etc.)

Removing all the SDs from an SD group (using either 'sd-group-remove' or 'sd-forget') will automatically delete the SD group itself; there is no need for further manual cleanup.

Deleting an SD group does not remove its SDs from any tier to which they have been assigned, even if that tier was assigned by 'sd-group-create' or 'sd-group-add'.

From release 12.3, SD groups are optional. Any SD group containing only one SD has no effect, and can safely be deleted.

Examples

server:$ sd-group-list -m
Serial: 0 1 2 4


Serial: 3 5 6


Serial: 7


Serial: 8


Serial: 9


server:$ sd-group-delete 5-8
Success
server:$ sd-group-list -m
Serial: 0 1 2 4


Serial: 9


server:$

Applies To

Cluster wide
