file-filtering-failures

Report File filtering scan failures

Syntax

file-filtering-failures [--reset|-r]

Description

The file-filtering-failures command is used to report the names of the last 100 files that have encountered file filtering failures. A file filtering scan will fail if the NTP/NSS Software agent encounters an error. Note that the list of failures isn't kept persistently over a reboot.

Usage:

file-filtering-failures [-r]

If no options are provided, the command reports the last 100 files that have encountered scan failures, along with a reason for each failure.

-r, --reset

Clear the failure log.

Applies To

EVS

See Also

file-filtering file-filtering-perf

Supervisor
