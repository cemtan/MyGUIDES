tls-cipher-suite-default

Disable the default insecure cipher suites. Note that using this to change enabled versions will cause the HTTPS server to restart!

Syntax

tls-cipher-suite-default [--confirm]

Description

The tls-cipher-suite-[enable,disable,default] commands are used to enable, disable or reset the enabled cipher suites to the defaults respectively.

The tls-cipher-suite-list command lists the enabled and disabled cipher suites. To list specific TLSv1.3 cipher suites, specify their names in a colon (":") separated list after the command. To list cipher suites for TLSv1.2 and below, specify a cipher list after the command. The format of the cipher list is described at: https://www.openssl.org/docs/man1.1.1/man1/ciphers.html#CIPHER-LIST-FORMAT.

Warning:

Running tls-cipher-suite-[enable,disable,default] will automatically cause the HTTPS server used for server management to restart. Any clients currently connected to it will lose their connection and will need to reconnect.

This forces clients to connect using the desired SSL/TLS version and cipher suite.

SSL/TLS Versions:

Depending on which SSL/TLS versions are enabled using tls-version-set only some of the enabled cipher suites here will be usable.

For example there are new cipher suites that only work in TLSv1.3. Older cipher suites cannot be used for TLSv1.3 connections and the new ones cannot be used for TLSv1.2 (or older) connections.

Note that many cipher suites are upwards compatible. So if you disable TLSv1.1, it does not mean that cipher suites introduced with TLSv1.1 will be unavailable in higher TLS versions.

SSL/TLS Certificate Implications:

The cipher suites you choose to enable or disable will affect the SSL/TLS certificates you will be able to use.

Every certificate will require a certain key agreement algorithm. This algorithm is also part of a cipher suite. So for example if all cipher suites that specify RSA are disabled then certificates that are using RSA as the key agreement algorithm will not be usable.

When no cipher suites that implement a specific key agreement algorithm are enabled a warning will be shown after the command has run.

Enabled cipher suites may not be used:

Because a cipher suite is enabled does not mean that it will (or can) be used. A server and client will use the most preferred suite that both are capable of using.

Options:

--confirm

Specify this argument to confirm that you want to change the settings. When not specified a dry-run will be performed where no settings are changed.

--all

Specify this argument to enable all the possible cipher suites.

Examples

tls-cipher-suite-disable --confirm ECDHE-ECDSA-AES128-CCM

Will disable the cipher suite `ECDHE-ECDSA-AES128-CCM' if it is enabled. If it is disabled nothing will happen.

Note that the HTTPS server will still be restarted.

tls-cipher-suite-enable --confirm ECDHE-ECDSA-AES128-CCM

Will enable the cipher suite `ECDHE-ECDSA-AES128-CCM' if it is disabled. If it is enabled nothing will happen.

Note that the HTTPS server will still be restarted.

tls-cipher-suite-disable --confirm ECDHE-RSA-AES256-SHA ECDHE-RSA-AES128-SHA ECDHE-RSA-AES128-SHA256

Disable multiple cipher suites in one command.

tls-cipher-suite-enable --confirm ECDHE-RSA-AES256-SHA ECDHE-RSA-AES128-SHA

Enable multiple cipher suites in one command.

tls-cipher-suite-enable TLS_AES_128_CCM_8_SHA256

Will do nothing as --confirm was not specified.

tls-cipher-suite-enable --confirm --all

Enables all cipher suites that are available.

tls-cipher-suite-list

List the status all the enabled and disabled cipher suites. Cipher suites without encryption or authentication are not listed and are always disabled.

tls-cipher-suite-list PSK

List that status of all cipher suites that use PSK.

tls-cipher-suite-list DHE ECDHE-PSK-CHACHA20-POLY1305

List the status of all cipher suites that use DHE and also list the status of the cipher suite ECDHE-PSK-CHACHA20-POLY1305

tls-cipher-suite-list TLS_AES_128_GCM_SHA256:TLS_AES_128_CCM_8_SHA256

List the status of the two specified TLSv1.3 cipher suites.

Applies To

Cluster wide
