find-entry

Attempts to report the paths to an object

Syntax

find-entry  <-a | -o <target-object-number>> -f <file-system>

Description

At first glance this command appears the same as the object-number-to-path command, and indeed the goal of both commands is the same. However, object-number-to-path tries to resolve the paths by the reverse links, and is therefore dependent on the integrity of the metadata concerned with such links. find-entry performs a linear search for directories, checking the contents of each one for the target object number, and printing the path when it finds them. Therefore, although somewhat slower to resolve the path, find-entry should succeed even if the target is badly corrupted.

Note: find-entry may only be run with the file system unmounted.

Options:

-o

The object-number of the target entry

-a

Abort the currently running instance of this utility.

Examples

To find what directories contain entries for the object number 0x44ee0 on file system fs3:

find-entry -f fs3 -o 0x44ee0

Applies To

EVS

See Also
