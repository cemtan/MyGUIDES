wfile-profile

Profiles wfilec1 state machines

Syntax

wfile-profile [-s] [-v] <seconds-to-sample> lip|main|rip|cc|autorsp|ie_1|rsp|lmif|ie_2|tan_wfs_transceiver_sl3_tx_arb_cntrl|tan_wfs_transceiver_sl3_rx_demux_cntrl|tan_wfs_transceiver_sl3_rx_link_controller|tan_wfs_transceiver_sl3_inq1_tx_ingress|tan_wfs_transceiver_sl3_inq1_tx_egress|tan_wfs_transceiver_sl3_inq2_tx_ingress|tan_wfs_transceiver_sl3_inq2_tx_egress|tan_wfs_transceiver_sl3_inq3_tx_ingress|tan_wfs_transceiver_sl3_inq3_tx_egress|tan_wfs_transceiver_sl3_inq1_rx_ingress|tan_wfs_transceiver_sl3_inq1_rx_egress|tan_wfs_transceiver_sl3_inq2_rx_ingress|tan_wfs_transceiver_sl3_inq2_rx_egress|hm800_cram_crc_checker_monitor|hm800_cram_crc_emr_unloader|hm800_cram_crc_emr_unloader_crc_err_mon

Description

The *-profile commands are used to diagnose performance problems by showing how busy various parts of the hardware are. These commands may either use high resolution profiling implemented in hardware or lower resolution profiling implemented in software. Using the software based profiling may have performance implications on the system.

Only use under instruction from your support provider.

-s, --software

Select use of the software based profiling rather than the hardware based profiling. For profiling commands which support both software and hardware based profiling the default is to use hardware based profiling.

-v, --verbose

This option lists all of the states. By default only those states for which activity has been detected are reported.

Applies To

Cluster node
