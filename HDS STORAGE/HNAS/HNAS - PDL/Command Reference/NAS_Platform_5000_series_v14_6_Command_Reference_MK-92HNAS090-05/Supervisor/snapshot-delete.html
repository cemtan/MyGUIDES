snapshot-delete

Delete the named snapshot, or all the snapshots associated with a rule.

Syntax

snapshot-delete (--file-system <file-system> <snapshot-name>) | (--rule <rule-name> [--file-system <file-system>]) | (--file-system <file-system> --checkpoint <checkpoint-number>)

Description

This command deletes one or more snapshots of a file system. It affects either:

  • A single snapshot specified by name or checkpoint number, or
  • All the snapshots associated with a rule.

The file system may be identified by specifying a file system label, or a rule name. Specifying a rule name deletes all the snapshots associated with that rule; if the rule name does not unambiguously identify a file system, both a rule name and a file system label may be specified.

--file-system, -f <file-system>

Select file system by label.

--rule, -r <rule-name>

Select rule by name.

--checkpoint, -c <checkpoint-number>

Select snapshot by checkpoint number.

Note: Deleting a snapshot does not immediately free the disk space which it occupies - the space is returned over time by a background task. To recover space quickly, refer to fs-recover-space man page.

Note: Certain types of snapshot, if they are required by the HNAS for correct operation, may be protected from deletion. In this case, a message will be displayed indicating why a snapshot cannot be deleted.

Examples

server:$ snapshot-delete --file-system fs1 testA
Information: Snapshot marked for deletion ("testA" on fs1)


server:$ snapshot-delete --rule hourly
Information: Snapshot marked for deletion ("2007-09-08_1500+0000.hourly" on fs1)
Information: Snapshot marked for deletion ("2007-09-08_1600+0000.hourly" on fs1)


server:$

EVS
