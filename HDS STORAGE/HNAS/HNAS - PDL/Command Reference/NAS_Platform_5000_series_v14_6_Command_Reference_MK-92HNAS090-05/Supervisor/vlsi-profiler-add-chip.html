vlsi-profiler-add-chip

Add all state machines for a chip to automated vlsi profiling.

Syntax

vlsi-profiler-add-chip <chip>

Description

This command is used to mark all the state machines for a chip as "eligible for profiling" for the automated vlsi-profiler. The vlsi-profiler works by reading the busyness of all given state-machines, and if one has reached the minimum threshold of busyness (vlsi-profiler-threshold-set), it will be profiled.

Note that state-machines added to the profiler are not guaranteed to ever be profiled. This is because their busyness may never surpass the minimum (though if the profile-threshold were set to 0%, that would make them always eligible).

The state machines are read every 10 seconds, but there is a 30 minute cooldown between profiles for individual state-machines to avoid constant profiling.

State machines added to the profiler will only be eligible for profiling on that node, and are not persistent between resets.

The chip name must be entered as lower case. The command 'vlsi-profiler-supported-list' displays the list of state machines supported by the automated vlsi-profiler.

Examples

Add the state machines from NIBRX:

vlsi-profiler-add-chip nibrx

Applies To

Cluster node
