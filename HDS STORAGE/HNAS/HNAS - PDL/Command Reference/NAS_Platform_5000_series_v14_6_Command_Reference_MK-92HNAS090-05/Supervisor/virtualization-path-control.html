virtualization-path-control

Controls the background processing of files for the virtualization path for the specified target vivol/volume.

Syntax

virtualization-path-control -t <fs label> -n <association name> { --start | --migrate | --pause | --resume }

Description

Starts, pauses or resumes virtualization or migration background processing of files and directories for a virtualization path association between a remote NAS NFS export source and a local file system or vivol target.

Options:

-t, --target <fs label>

Target file system.

-n, --name <association name>

Name of association

-s, --start

Indicates that virtualization of data from the virtualization source is to be started.

-m, --migrate

Indicates that migration of data from the virtualization source is to be started.

-p, --pause

Indicates that background processing of files from the virtualization source is to be paused.

-r, --resume

Indicates that background processing of files from the virtualization source is to be resumed.

Examples

virtualization-path-control -t fs1 -n assoc1 --pause

Suspends background processing of files for the virtualization path for association assoc1 on file system fs1.

virtualization-path-control -t fs1 -n assoc1 --migrate

Starts migration for the virtualization path for association assoc1 on file system fs1.
