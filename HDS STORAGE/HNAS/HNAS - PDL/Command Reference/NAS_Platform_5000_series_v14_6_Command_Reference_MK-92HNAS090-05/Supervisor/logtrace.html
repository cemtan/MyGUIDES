logtrace

Manage LogTrace objects

Syntax

logtrace [all | none]
logtrace [on | off] {<name> <name> <name>...}
logtrace resize <size-in-K> <name> {<name> <name>...}
logtrace [tson | tsoff] <name> {<name> <name>...}
logtrace dump <name> {<name> <name>...}
logtrace clear <name> {<name> <name>...}
Alias:       lt

Description

A logtrace is a temporary store of diagnostic information that is normally not kept, but which can be displayed at the console during troubleshooting.

A logtrace can be enabled or disabled: enabling it means that information will appear on the console and in the dblog (but not in ssc or ssh) as soon as it is produced. By default, some logtraces are enabled and others not. Manipulating a logtrace on one node has no effect on the same logtrace on other cluster nodes.

In addition, logtracing can be globally enabled or disabled; disabling it means that output that would normally be printed at the console and in the dblog is suppressed. This output can still be displayed manually, however, as long as it has not rotated out of the logtrace. Once again, enabling or disabling logtracing on one cluster node has no effect on other nodes.

Each logtrace has a name, which you can use to manipulate it at the command line. It is also a member of a group, which has a name of its own. You can use group names to manipulate several logtraces at once.

The 'logtrace' command has several subcommands. Their exact syntax is shown below.

Examples

logtrace

Prints a list of all available logtraces on the server where the command is run. A line of output like this --

   span:tr, len 11.1 KB, cap 64 KB, TS

-- means that the logtrace 'tr' is in group 'span'; it currently contains 11.1KiB of text; it can hold 64KiB before the oldest information is lost; and timestamps are enabled for this logtrace.

logtrace dump tr mgspan

Displays the contents of the 'tr' and 'mgspan' logtraces.

logtrace clear tr

Erases all text in the 'tr' logtrace.

logtrace tson macro
logtrace tsoff span

Enables timestamping for the 'macro' logtrace; disables it for all logtraces in the 'span' group.

logtrace all
logtrace none

Enables or disables all logtraces on the server where the command is run.

logtrace off

Disables console and dblog output for all logtraces, even those that are enabled.

logtrace on

Reenables console and dblog output for all enabled logtraces.

logtrace resize 96 macro span

Resizes the 'macro' logtrace, and all logtraces in the 'span' group, to 96KiB. This means that the logtraces can hold 96KiB of text, including timestamps if enabled, before the oldest text is discarded. If the new size is smaller than the amount of text currently stored in the logtrace, the oldest text is discarded. You must be logged in as dev to resize a logtrace.

Applies To

Cluster node
