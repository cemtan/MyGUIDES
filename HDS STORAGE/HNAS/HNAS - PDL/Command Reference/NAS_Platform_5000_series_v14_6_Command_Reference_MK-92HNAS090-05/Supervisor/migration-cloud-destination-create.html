migration-cloud-destination-create

Creates an external cloud migration destination

Syntax

migration-cloud-destination-create --account-name <account-name> --path-at-destination <path-at-destination> [--description <description>] [--destination-uuid <destination-uuid>] [--encrypt-in-transit (no|yes)] <destination-name>

Description

Creates cloud external migration destination.

<destination-name>

Name of the destination for use by the CLI only. It does not have to correspond to any identification names on the cloud storage itself, but will be needed when specifying other cloud objects. May be up to 255 ASCII characters long. The characters ;:/\'"<>*?| and a space character are not allowed.

--account-name, -a <account-name>

Name of the cloud account, which should be created with migration-cloud-account-create.

--path-at-destination, -p <path-at-destination>

Path on the cloud where data will be archived. For Amazon S3 this field is 'bucket/subfolder' where 'bucket' should pre-exist on the target and 'subfolder' is optional. For HCP, HCPS3 and Test providers it is a 'subfolder' that should pre-exist on the target. For Azure provider it is a 'container' that should pre-exist on the target.

--description, -d <description>

Destination description. Any arbitrary string to help clients to manage destinations.

--encrypt-in-transit, -t <no|yes>

The data will be encrypted on the way to the cloud using HTTPS protocol. If not specified the default value for HCP and HCPS3 providers is 'no', for all others it is 'yes'.

--destination-uuid, -u <destination-uuid>

The UUID of the destination. If not specified, will be created by the system.

Examples

migration-cloud-destination-create S3CloudDest --account-name AmazonS3Account --path-at-destination S3bucket/marketingArchive

Creates a cloud migration destination named 'S3CloudDest' that will use cloud credentials specified in 'AmazonS3Account'. The data will be stored in 'S3bucket' under the folder 'marketingArchive'. The data will be encrypted in flight.

migration-cloud-destination-create HCPDest --account-name HCPAccount --path-at-destination marketingArchive

Creates a cloud migration destination named 'HCPDest' that will use cloud credentials specified in 'HCPAccount'. The data will be stored under 'marketingArchive' folder. The data will not be encrypted in flight and will not be encrypted on the destination.

migration-cloud-destination-create TestDest --account-name TestAccount --path-at-destination /test

Creates a test cloud migration destination named 'TestDest' that will use cloud credentials specified in 'TestAccount'.

Applies To

Global security context
