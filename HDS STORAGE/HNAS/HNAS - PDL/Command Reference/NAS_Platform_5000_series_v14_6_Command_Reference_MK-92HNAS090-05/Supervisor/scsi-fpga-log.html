scsi-fpga-log

dump/clear FPGA log

Syntax

scsi-fpga-log [-d, --dump][-c, --clear][-s, --size][-h, --help]

Description

Dumps or clears the FPGA log.

There is a single FPGA log that logs fast path (FPGA) SCSI command responses that have returned with a SCSI check condition status. The log will be empty if no fast path I/O SCSI check conditions have occurred. Console output may be stopped or aborted with a key press or by CTRL-C.

A log entry indicates the SCSI command that generated the SCSI status, as follows:

Command Name
H<n>

Host port that issued the command.

P<n>

The SCSI port number of the target device the command was sent to.

[xxxxxx]

Fibre Channel address of the target port the command was sent to.

DevID <ID>

The device ID of the device that command was sent to.

LUN n

The LUN of the target device.

<: the SCSI status returned by the target device>

Description of the SCSI status returned by the target device.

Options:

-d, --dump

Displays the contents of the SCSI FPGA log. The last 20 entries are output if no range is specified.

-h, --hport <1..n>

Filters by host port.

-p, --port <ID>

Filters by the target SCSI port ID.

-i, --id <ID>

Filters by the target device ID.

-a, --all | -n <[-]no of entries to dump>

Outputs all entries, last n entries or first n entries if n is negative. If n = 0, dumps entire log; by default, dumps last 20 entries.

-c, --clear

Clears the FPGA SCSI log.

-C, --Conf

Clears the log without requesting confirmation.

-s, --size

Shows the current and maximum sizes of the SCSI FPGA log.

-h, --help

Shows help.

Examples

To dump the last 20 entries from the log:

scsi-fpga-log -d

To dump the entire log:

scsi-fpga-log -d -a

To dump last 10 entries filtered by device ID 2:

scsi-fpga-log -d -i 2 -n 10

To dump the entire FPGA log filtered by hport 1:

scsi-fpga-log -d -h 1 -a

To clear the FPGA log with no confirmation:

scsi-fpga-log -c -C

Applies To

Cluster node
