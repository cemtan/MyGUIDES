nsorder

Report/configure name service ordering

Syntax

nsorder [-o [<d|n|w>][<d|n|w>][<d|n|w>]] [-r]

Description

The host name lookup service needs to be configured to work correctly in local environment. nsorder provides a way of using DNS (recommended), NIS or WINS to do that lookup. More than one service can be specified and host name lookup code uses each service in turn until a name is successfully resolved. This functionality is similar to that provided by /etc/nsswitch.conf in other OSs.

Without any options, the nsorder command displays the currently configured list of name services.

To configure the name service order:

nsorder [-o [<d|n|w>][<d|n|w>][<d|n|w>]] [-r]

-o

Specifies the host name order. Use d for DNS, n for NIS (formerly known as yp) or w for WINS.

-r

Removes all name services.

Examples

To specify names should be looked up using DNS followed by NIS:

nsorder -o dn

To specify names should be looked up using DNS followed by WINS:

nsorder -o dw

To display the current name service ordering:

nsorder
