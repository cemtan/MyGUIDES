credentials-list

Lists credentials

Syntax

credentials-list [--fixed] [--unix] [--non-unix] [--match <user name/id>]

Description

List credentials being managed by the EVS

Usage:

credentials-list [--match <name>] [--fixed] [--unix] [--non-unix]

--match

Only shows credentials where the user name/id matches <name>.

--fixed

Only shows fixed credentials.

--unix

Only shows Unix credentials.

--non-unix

Only shows non-Unix credentials.

Every time a user connects to a share or export on the EVS, credentials are created that represent the identity of the user. They include the user's user id and primary group id, a list of additional groups the user belongs to, and a set of local groups the user belongs to. The credentials are used by the file system security layer to determine whether the user has permission to perform file system operations (for example, create a file, write to a file).

The credentials-list command can be used to list the credentials the EVS currently knows about.

Fixed credentials are built-in credentials that aren't supplied by clients. They include anonymous, root, NDMP, console and tree-delete credentials.

A background task periodically reassesses credentials to reflect changes to the local group and primary group tables.

The credentials-reassess command can be run to trigger the background task to reassess credentials immediately, rather than waiting for the next scheduled reassessment.
