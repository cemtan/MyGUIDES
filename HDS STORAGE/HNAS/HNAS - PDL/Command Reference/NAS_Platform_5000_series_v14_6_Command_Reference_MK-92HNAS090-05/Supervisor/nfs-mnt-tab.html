nfs-mnt-tab

Report or remove NFS mount records

Syntax

nfs-mnt-tab operation
	where 'operation' is one of:
		dump [<client> [<path>]]
		delete <client> [<path>]

Description

The server keeps a persistent record of the pre-NFSv4 network clients that have any of its configured NFS exports mounted. Network client crashes, network outages or just bad client behavior can mean that the information on the server becomes stale. This command provides the facility to remove stale network client records held by the server.

Usage:

dump [<client> [<path>]]

Displays the mount records stored by the EVS in context. Additional filters are the client IPv4 address (in dot notation) or IPv6 address (in colon-separated notation) and the mounted path.

delete <client> [<path>]

Deletes the mount records by the EVS in context for the provided client IPv4 address (in dot notation) or IPv6 address (in colon-separated notation). Additionally the path can be provided so that a specific mount record is deleted.

Examples

Display the mount records stored for EVS 1:

vn 1 nfs-mnt-tab dump


Client: 10.1.100.40, has mounted path: /root
Client: fdca:f995:220a:a00:250:56ff:fe06:d3b6, has mounted path: /root
nfs-mnt-tab: *** End of mount entries for EVS 1.

Delete any records stored on EVS 1 for client 10.1.100.40:

vn 1 nfs-mnt-tab delete 10.1.100.40

Delete any records stored on EVS 1 for client fdca:f995:220a:a00:250:56ff:fe06:d3b6:

vn 1 nfs-mnt-tab delete fdca:f995:220a:a00:250:56ff:fe06:d3b6

Applies To

EVS
