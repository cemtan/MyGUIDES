cluster-show

Show cluster parameters and node status

Syntax

cluster-show [-a | --all] [-c | --admin] [-l | --local] [-y | --node] [-q | --quorum] [-v | --EVS]

Description

The cluster-show command displays cluster settings and node status.

Options:

-a, --all

Displays all cluster information.

-c, --admin

Displays cluster Admin Service information.

-l, --local

Displays local EVSs information.

-y, --node

Displays cluster node information.

-q, --quorum

Displays Quorum Device information.

-v, --EVS

Displays EVS information.

The "Cluster Health" status will have one of the following states:

Robust

Cluster heartbeating over the Cluster Interconnect and Management Network is fully functional on all cluster nodes.

Degraded

Cluster heartbeating has failed on one or more cluster links. See "cluster-show -y" for details.

Critical

The cluster is operating with the minimum required quorum. A further cluster link or cluster node failure will cause the whole cluster to reboot.

The "FS Access" status will have one of the following states:

OK

All file systems that have been made available to this node are accessible.

Failure(s)

One or more file systems that have been made available to this node are no longer accessible.

The "Ethernet Aggs" status will have one of the following states:

OK

All Ethernet aggregations are accessible to this node.

Degraded

One or more member interfaces of an Ethernet aggregation are DOWN on this node.

Failure(s)

One or more Ethernet aggregations are not accessible to this node.

The "Mgmnt Netwrk" status will have one of the following states:

OK

The Ethernet management network is accessible to this node.

Failed

The Ethernet management network is not accessible to this node.

The "FC" status will have one of the following states:

OK

All Fibre Channel interfaces are UP on this node.

Degraded

One or more Fibre Channel interfaces are DOWN on this node.

Failed

All Fibre Channel interfaces are DOWN on this node.

NA

FC status is not applicable on this model

The Quorum "Status" will have one of the following states:

Unconfigured

No quorum device has been configured.

Configured

A quorum device has been configured and currently being used(applies to an even-sized cluster).

Granted

A quorum vote is granted to the specified cluster node in an even-sized cluster.

Not Up

A quorum device is configured but is not currently contactable by the cluster nodes.

Misconfigured

The quorum device's internal configuration does not match the UUID of this cluster. The quorum device should be removed and re-added using the "quorumd" command.
