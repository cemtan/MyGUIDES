snmp-hosts

Manipulate authentic SNMP hosts

Syntax

snmp-hosts operation
where 'operation' is one of:
list
add <Host>
del <Host>
delall [--confirm]
mod <NewHost> <CurrentHost>
accept
deny
Alias:       snmphosts

Description

The snmp-hosts command manages the list of hosts allowed to query the server's built-in SNMP agent. Access to the agent can be allowed to all hosts, select hosts, no hosts. When the protocol being used is SNMPv3, hosts will not affect the configuration and snmpv3-user-* should be used instead. Any changes made to hosts while SNMPv3 is enabled will not come into effect until the protocol is changed (man snmp-protocol), though they can still be listed.

Valid operations for snmp-hosts are:

list

List the hosts allowed to query the server's SNMP agent.

add <Host>

Add a host to be allowed to query the agent. Hosts can be added by IP address or hostname. Groups of hosts may be added using CIDR (see EXAMPLES for syntax).

del <Host>

Delete the named host from the list.

delall [--confirm]

Delete all hosts in the list. The --confirm option prevents the command prompting for confirmation.

mod <NewHost> <CurrentHost>

Modify CurrentHost to NewHost.

accept

Allows access to any host.

deny

Allow access only to hosts in the list.

Examples

Add a single host by IP address:

snmp-hosts add 172.16.1.15

Add a single host by hostname:

snmp-hosts add example.com

Add a group of hosts using CIDR notation:

snmp-hosts add fdca:f995:220a:480::/64

Delete a host:

snmp-hosts del 172.16.1.15

Display the access list:

snmp-hosts list
Host: example.com
Only accept SNMP packets from configured hosts

Modify the list:

snmp-hosts mod bar.example.com foo.example.com


snmp-hosts list
Host: bar.example.com
Only accept SNMP packets from configured hosts

Allow any host to query the agent, regardless of the list:

snmp-hosts accept

Allow only hosts in the list to query the agent:

snmp-hosts deny

Applies To

Cluster wide
