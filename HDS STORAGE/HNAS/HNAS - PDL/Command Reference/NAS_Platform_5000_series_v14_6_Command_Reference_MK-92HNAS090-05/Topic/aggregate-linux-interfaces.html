aggregate-linux-interfaces

Description

An aggregate Linux interface is the way a cluster node provides access to the networks available via a file-serving interface from the host Linux operating system. The interface can be used for system management purposes or for host Linux operating system access to network services such as DNS.

An aggregate Linux interface may be used as an alternative to the management network interfaces (eth0/eth1) to achieve the required network connectivity or when higher performance is needed.

You create an aggregate Linux interface for a file-serving aggregate interface using the 'aggregate-linux-interface-create' command. The corresponding aggregate Linux interface created for the file-serving interface named 'ag1' is named 'eth-ag1'. An aggregate Linux interface is removed using the 'aggregate-linux-interface-delete' command.

An Admin Service IP address can then be assigned to and removed from the interface using the 'evsipaddr' command. A cluster node IP address can be assigned to and removed from the interface using the 'ipaddr' command.
