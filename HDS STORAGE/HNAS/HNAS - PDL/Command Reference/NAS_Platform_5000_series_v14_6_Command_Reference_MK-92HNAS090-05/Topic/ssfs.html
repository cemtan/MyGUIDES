ssfs

Description

The term "ssfs" refers to the server's configuration space file system, not a particular command. In general it is unnecessary to manipulate this file system directly, but most traditional file system operations are possible if you do.

WARNING: Incorrect use of ssfs commands can leave the server unbootable and only recoverable by your support provider.

File names

Parts of filenames are separated with a /. You can use // for backwards compatibility. : is used to specify a file within a package.

A full path consists of, in order, and all optional

/cnode

cluster node id

/board

flash name. 'nim', 'fsm' or 'sim' for the 3 module's flash and 'simm' for the flash storing package files.

/file

actual filename.

Example filename

/2/nim/loader.bin

loader.bin on cluster node 2's nim board
