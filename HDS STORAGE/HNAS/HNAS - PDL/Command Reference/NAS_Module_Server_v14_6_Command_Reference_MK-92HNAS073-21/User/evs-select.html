evs-select

Set EVS context

Syntax

evs-select [-q] [<evs-id> [command]]
Alias:       evssel
Alias:       vn

Description

This command sets the EVS that certain following commands apply to. It can also be used as a prefix to set the EVS for just one command.

The first parameter is an EVS id which must be a valid, active EVS in the system.

The id "admin" may be specified to reset the context to its default.

The id "all" may be specified to run the following command in the context of each active EVS sequentially. This does not include the Admin Service.

Commands that are marked as specific to the Admin Service will be forwarded to the cnode hosting the Admin Service. Commands that are marked as specific to an EVS can only be executed when an EVS is "in context", via this command or others. When an EVS is "in context", commands that are marked as being specific to an EVS will be run on the cnode hosting that EVS. Other commands are run locally. A command's specificity to the Admin Service or an EVS is given by the "Needs:" line of its help. Contrast the behavior of "evs-select all" with that of "for-each-evs", which runs its command argument on the cnode hosting the EVS, regardless of the command's requirements.

Options:

-q

Don't echo EVS number with "all" option.

ssrun -c can be useful when trying to run complicated commands.

Examples

The prompt changes to reveal the context:

server-a:$ evs-select 1
server-a[server-s]:$

Some commands perform different functions in the context of the Admin Service and a serving EVS:

server-a:$ cifs-share list
CIFS Share List: No shares are currently configured.
server-a:$ evs-select 1 cifs-share list | wc -l
53
server-a:$

The EVSs enumerated are displayed on standard error:
