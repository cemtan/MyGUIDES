mtu

Display/Configure Local network MTU Table

Syntax

mtu [operation]
where 'operation' is one of:
add <addr> -n <mask> -s <mtu>
del <addr> -n <mask>
delall [--confirm]

Description

The mtu command specifies a maximum IP MTU size to be used when sending data from a specific network. This feature is intended to facilitate the configuration of multiple attached subnets, some of which are jumbo-frame capable and some of which are not.

Each IP MTU setting takes up one available entry in the routing table.

Without any options, the mtu command displays the existing local network IP MTU limitations. To manipulate the set of IP MTU limitations, use the add, del or delall options.

The IP MTU settings are ignored if cluster-node routing is enabled.

Note that the actual IP MTU used will be the lowest of the interface IP MTU, any IP MTU specified by the chosen route and any IP MTU specified by the mtu command.

To configure local network IP MTU limits:

mtu add <addr> -n <mask> -s <mtu>

  ... or with prefix notation ...

mtu add <addr>/<prefix> -s <mtu>

mtu del <addr> -n <mask>

mtu delall [--confirm]

To display local network IP MTU limits:

mtu

Operations

add/del

These specify the type of operation upon the table that you wish to perform. They are mandatory.

delall

Delete all items in the mtu table. Only takes one option which is "--confirm" to avoid the user being prompted for confirmation at the command line.

<addr>[/<prefix>]

Means an IP address format such as 10.1.2.3. There is some checking for invalid addresses. Invalid addresses are: Numbers > 255 and Broadcast Addresses. A prefix can be given to specify how many top bits of the address are significant to the network rather than specifying a netmask.

-n <mask>

Netmask. A <mask> must be supplied. This option is needed if a prefix hasn't been supplied for the <addr>

-s <mtu>

The IP MTU to be used for this route.

-c, --confirm

Confirm that the mass deletion is to be carried out without a further prompt.

Examples

To add an IP MTU limitation for traffic sent from the 192.168.0.0/16 network:

mtu add 192.168.0.0/16 -s 9200

or alternatively the same but with an explicit netmask given instead of an address prefix:

mtu add 192.168.0.0 -n 255.255.0.0 -s 9200

Note that an appropriate ipadv command may be needed to increase the IP MTU of the specified interface. For example, if the interface to be used is ag1, the following command may be required to allow 9200 byte packets to be issued using that subnet on ag1:

ipadv -m 9200 -n 9200 -p ag1

To delete the above IP MTU limit:

mtu del 192.168.0.0/16

or again the alternative form:

mtu del 192.168.0.0 -n 255.255.0.0

To delete all IP MTU limitations without requiring a confirmation:

mtu delall -c

Applies To

Cluster wide
