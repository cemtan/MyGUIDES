evs-security

Change/report per EVS security settings

Syntax

evs-security list | individual -e <evs-id>  | global -e <evs-id>

Description

Used to select whether an EVS has its own individual security settings, or uses the global cluster wide settings.

Usage:

evs-security list | global -e <evs-id>  | individual -e <evs-id>

list

lists the current setting for each EVS

individual

configures an EVS to use its own security settings

global

configures an EVS to share the cluster wide security settings

An EVS's evs-security setting may only be changed when the EVS is disabled (see the evs command).

When changing an EVS's security mode, it is strongly advised to check the configuration defined for affected services. Refer to the 'individual-security-affected-commands' man page for more information.

The following modules will be configured to use per EVS configuration when evs-security is enabled for an EVS:

  • DNS configuration
  • WINS configuration
  • NIS configuration
  • NS order
  • CIFS domain name
  • Kerberos realm
  • User and group mapping tables

When a serving name has been configured on a global EVS using 'cifs-name' and individual security is then enabled, Domain Controllers may not be available due to the change in security context. To resolve this issue, delete then re-add the serving name for the affected EVSs.

Examples

Display the selection for all EVSs:

evs-security list


EVS id     Per EVS security status
------     -----------------------
     1                  individual

Enable an EVS to have its own security settings:

evs-security individual -e 1

Change an EVS back to sharing the cluster wide settings:

evs-security global -e 1

Applies To

Cluster wide

See Also
