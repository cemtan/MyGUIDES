route

Display/Configure Route Table

Syntax

route [operation]
where 'operation' is one of:
[show] [-v]
flush [--confirm]
add gateway -g <addr> [-s <mtu>] 
add host <addr> -g <addr> [-s <mtu>] 
add net <addr>/<prefix-length> -g <addr> [-s <mtu>] 
add net <addr> [-n <mask>] -g <addr> [-s <mtu>] 
del gateway -g <addr> [-s <mtu>] 
del host <addr> -g <addr> [-s <mtu>] 
del net <addr>/<prefix-length> -g <addr> [-s <mtu>] 
del net <addr> [-n <mask>] -g <addr> [-s <mtu>] 
delall [--confirm]

Description

The route command manipulates the IP routing tables. It can be used to set up static routes to specific hosts or networks. It can also be used to set up multiple default gateways to minimize the administration of host and network routes. A gateway route is used to route traffic where no host or network route is given and the destination is not directly reachable via an interface. In this case, the routing code looks at the source IP address to see what interface the packet would go out of and chooses the appropriate gateway on the same subnet. You can minimize the number of host and network routes (or even eliminate them) by specifying one gateway route for each subnet that the server is on.

Without any options, the route command displays the contents of the routing tables. To manipulate the routing tables, you use the add, del or delall options.

The maximum number of routing entries is 127. Note that each MTU entry also takes up a routing entry and counts towards the limit.

To configure routes:

route flush [-c | --confirm]

route add gateway -g <addr> [-s <mtu>]

route del gateway -g <addr> [-s <mtu>]

route add host <addr> -g <addr> [-s <mtu>]

route del host <addr> -g <addr> [-s <mtu>]

route add net <addr> [-n <mask>] -g <addr> [-s <mtu>]

route add net <addr>/<prefix-length> -g <addr> [-s <mtu>]

route del net <addr> [-n <mask>] -g <addr> [-s <mtu>]

route del net <addr>/<prefix-length> -g <addr> [-s <mtu>]

route delall [-c | --confirm]

To display routes:

route [show] [-v]

defaults: dynamic host routes not shown.

-v, --verbose

Displays all routes including dynamic host routes.

Note: Dynamic routes are only displayed for the cluster node on which the command is executed.

Route flags

The following flags indicated various route properties:

H - A host route

G - A gateway route

D - A dynamic route

C - A cluster-node-route

Any route that is not marked H or G is a network route.

Operations

add/del

Add or delete a static route (a single host, a network or a gateway). The type of route and the route details are mandatory.

delall

Delete all static routes in the routing table. Only takes one option, which is "-c" or "--confirm", to avoid the user being prompted for confirmation at the command line.

flush

Remove all dynamic host routes from the routing table. Takes one option, which is "-c" or "--confirm", to avoid the user being prompted for confirmation at the command line.

Note: This operation only applies to the cluster node on which the command is executed.

Note: To flush other dynamic routes from the routing table use the 'irdp flush', 'ndp-flush' and 'rip flush' commands.

show

Display routes. Only takes one option which is "--verbose" which controls the display of dynamic host routes.

host|net|gateway

Specify the type of route, i.e a single host, a network or a gateway. If a network is specified, then you must include either an address prefix length (eg 10.2.3.0/24) or a Subnet mask ( -n <addr> ). For a host, this value will default to 255.255.255.255.

<addr>[/<prefix-length>]

Means an IP address format such as 10.1.2.3. There is some checking for invalid addresses. Invalid addresses are: Numbers > 255, Broadcast Addresses, Unreachable gateways. A prefix length can be given to specify how many top bits of the address are significant to the network rather than specifying a netmask.

-n <addr>

Netmask. Only necessary for "net" type routes if an address prefix length is not supplied. An <addr> must be supplied.

-g <addr>

Gateway. Must be supplied for any route. An <addr> must be supplied.

-s <mtu>

The MTU to be used for this route. This value must lie between 68 and 9600 inclusive for IPv4 routes and between 1280 and 9600 inclusive for IPv6 routes. If the MTU is not specified a default of 1500 will be applied.

Examples

To add a gateway route:

route add gateway -g 10.1.254.50

To delete a gateway route:

route del gateway -g 10.1.254.50

To add a host route to get to 10.1.2.3 via 10.1.3.4:

route add host 10.1.2.3 -g 10.1.3.4

To add a network route for the 10.1.2.0 network via 10.1.3.4:

route add net 10.1.2.0/24 -g 10.1.3.4

or alternatively the same configuration but with an explicit netmask given instead of an address prefix length:

route add net 10.1.2.0 -n 255.255.255.0 -g 10.1.3.4

To delete the host route to get to 10.1.2.3 via 10.1.3.4:

route del host 10.1.2.3 -g 10.1.3.4

Applies To

Cluster wide
