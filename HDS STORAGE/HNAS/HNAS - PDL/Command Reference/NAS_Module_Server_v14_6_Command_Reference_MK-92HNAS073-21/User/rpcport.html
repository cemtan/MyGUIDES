rpcport

List/modify the port numbers associated with RPC services.

Syntax

rpcport list | set <program number> <port number>

Description

Displays the list of RPC-based services and the port numbers that they are available on. This command also allows the port number for a specified service to be changed - the modified port number will take effect after the next server reboot. You are strongly advised against changing the port number for the PortMapper service as this will cause standard network clients to fail to operate with the server. Note changes made using this command are persistent across server reboots. Any changes to the default port numbers must be done with consideration of the ports being used by other services, including those not listed as RPC services. An event will be raised in the event log if an RPC service fails to start because the port number assigned to it is already in use.

Usage:

rpcport list

List the current configuration.

rpcport set <program number> <port number>

Change the port used by an RPC service. The program number for a particular service can be found using the list option above. To revert to the default specify a port number of 0. Note a reboot will be required for modifications to take effect.

Examples

rpcport list
RPC            Program    Current Port       Port After
Service        Number     TCP       UDP      Reboot
----------     -------   -----     -----     -------
PortMapper     100000    111       111       111
NFS            100003    2049      2049      2049
Mountd         100005    4048      4048      4048
Rquota         100011    1028      1024      Any
Lockd          100021    4045      4045      4045
Statd          100024    4050      4050      4050


rpcport set 100005 4049
Information: The Mountd service port number has been updated to 4049
fileprotocol/rpc/rpcport set: Updated successfully.  A reboot will be required for the changes to take effect.

Applies To

Cluster wide
