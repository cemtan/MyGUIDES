replication-recovery-logging-policy-get

Get the logging policy for replication recovery tasks

Syntax

replication-recovery-logging-policy-get

Description

Get the logging policy for replication recovery tasks.

This command displays the following information about each policy:

  1. Policy name
  2. Its current value
  3. Its default value, which is the value that will be set if the --reset option is chosen with replication-recovery-logging-policy-set
  4. The range of acceptable values for this policy

Following logging policies exist:

replication-recovery-max-num-of-logs-per-pnode

Controls the maximum number of logs that will be kept on each physical node.

replication-recovery-max-log-file-size-in-bytes

Controls the maximum size of a log.

replication-recovery-max-entry-size-in-bytes

Controls the maximum size of each entry in a log.

Examples

  replication-recovery-logging-policy-get


  Replication recovery logging policies
  -------------------------------------
	 replication-recovery-max-num-of-logs-per-pnode, value=     20, default=     20, allowed range=[1, 50]
        replication-recovery-max-log-file-size-in-bytes, value=1000000, default=1000000, allowed range=[5000, 1000000]
           replication-recovery-max-entry-size-in-bytes, value=   1000, default=   1000, allowed range=[150, 1000]

Applies To

Cluster wide
