package-combined-install

Install a combined package.

Syntax

package-combined-install  --hm-micro <HM MICRO VERSION> [ --downgrade-allowed ] <PACKAGE FILE>

Description

The package-combined-install command is used to install a combined package.

The full path to the package name must be specified. Other options:

--hm-micro HM_MICRO_VERSION, -h HM_MICRO_VERSION

This is a mandatory option. HM_MICRO_VERSION is the version number of the HM micro code that is to be running with the specified HNAS combined package software versions. This should be in the same format as supplied in the compatibility_check_hm_micro_versions field of the compatibility file or one of two special strings: none - perform no check of the HM micro code or current - use the current HM micro code version of the system.

--downgrade-allowed, -d

If not specified then the check will assume that downgrades from the currently running software are not permissible, no matter what the compatibility file contents.

The package-combined-install command only runs on ssc connections. Once the file transfer over the ssc connection has completed successfully the copy of the file on the client file system will be deleted automatically by the command.

Examples

Install a package:

ssc 192.168.0.1 package-combined-install -h 83-01-00-00/00-01 hnas-package.tar

Install a package when downgrades are allowed:

ssc 192.168.0.1 package-combined-install -d -h 83-01-00-00/00-01 hnas-package.tar

Install a package using the current HM micro code version:

ssc 192.168.0.1 package-combined-install -h current hnas-package.tar

Applies To

Cluster node
