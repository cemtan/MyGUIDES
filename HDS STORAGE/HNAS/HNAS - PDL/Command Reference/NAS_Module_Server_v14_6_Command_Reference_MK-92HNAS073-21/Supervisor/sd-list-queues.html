sd-list-queues

Print SD queues

Syntax

sd-list-queues 
Alias:       sdq

Description

When the server loads spans from Cod at boot time, or when it responds to an I/O error or the recovery of a set of failed system drives (SDs), it breaks down the processing of each SD into several stages and handles several SDs in parallel. This allows the server to respond faster to storage events.

The 'sd-list-queues' command shows the number of SDs at each stage of processing and shows the ID of the thread handling each stage. The output is mainly useful for your support provider.

On a correctly functioning system, SDs don't spend long in any of these queues. It is therefore common to run this command twice, a few seconds apart, and check that the state of each non-empty queue has changed.

To wait until all queues are empty, use 'span-wait-for-loading'.

Examples

sd-list-queues

Applies To

Cluster node
