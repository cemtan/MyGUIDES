snapshot-deletion

Show/set whether a volume is modifiable by the deleter

Syntax

snapshot-deletion [<file system>] [--enable|--disable]

Description

This command shows/sets whether a mounted file system is modifiable by the snapshot deleter. It requires that the Vnode (EVS) where the file system resides be selected and the file system be mounted.

If <file system> is not specified, the current selected file system is assumed. If <file system> is specified but is not mounted, the command will fail. Without --enable and --disable, the command shows the status of snapshot deletion support of the specified file system. If --enable/--disable is specified, the command changes the deletion support as follows:

--enable

Enable snapshot deletion support for the mounted file system. All deletable snapshots associated with the specified file system will be put in the deletion queue for the deleter to delete.

--disable

Disable snapshot deletion support for the mounted file system. All the deletable snapshots associated with the specified file system that are currently in the deletion queue of the deleter will be removed.

This command needs Supervisor privilege.

Examples

server:$ snapshot-deletion fs1 --enable


Snapshot deletion on volume 'fs1' is enabled.


Server:$ snapshot-deletion fs1 --disable


Snapshot deletion on volume 'fs1' is disabled.


Server:$ snapshot-deletion fs1


Snapshot deletion on volume 'fs1' is disabled.


Server:$ snapshot-deletion


Snapshot deletion on volume 'fs1' is disabled.

Applies To

EVS
