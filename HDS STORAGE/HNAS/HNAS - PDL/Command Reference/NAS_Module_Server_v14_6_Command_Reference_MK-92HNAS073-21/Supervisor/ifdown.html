ifdown

makes an interface administratively unavailable

Syntax

ifdown [-v] [-a] [interface]

Description

The 'ifup' and 'ifdown' commands activate and deactivate network interfaces respectively.

Options:

<interface>

If specified, this names the interface that will be acted on.

-a, --all

Causes the command to act on all network interfaces on this node except for the management interfaces and interfaces which are used internally within the server.

-v, --verbose

Logs information as each interface is altered.

WARNING: Deactivating the network interfaces on a node in a cluster may cause the node to be restarted due to a "server lost cluster quorum" condition or trigger an EVS migration.

Examples

To bring up the ag1 interface:

$ ifup ag1
$

To bring down the ag2 interface:

$ ifdown ag2
$

Applies To

Cluster node

See Also

ifconfig ifup

Privilege Level

Supervisor
