vss-account

Manage currently registered VSS provider accounts

Syntax

vss-account list | remove <account id>

Description

The vss-account command is used to display and (if necessary) remove credentials registered with the server on behalf of the server's hardware provider DLLs for the Microsoft Volume Shadow Copy Service (VSS). Credentials may not be created using this command, only the DLL registration process may do that.

VSS credentials exist solely to support VSS hardware provider DLL access to the server via the server's dedicated VSS interface. They may not be used to access any other server management or client interfaces. This is intended as a security measure, since the credential must be stored on the VSS host system.

When listing credentials, they are grouped according to the VSS host system which performed the registration, and then sorted by date. This is meant to simplify the task of identifying obsolete credentials such as those orphaned when a VSS DLL is uninstalled in a disorderly manner.

The remove option is provided for those cases where a VSS DLL was not uninstalled using the normal process, or where the VSS host system could not reach the server during the uninstall operation. When removing a credential, the argument given to remove is the "VSS id" of the credential as found in the first column of the list display.

Removals performed with this command take effect immediately, but will not cause the disconnect of a VSS DLL whose current connection was authenticated using one of the removed credentials.

Examples

Display VSS credentials currently known to the server:

vss-account list

Delete a credential whose id is "3":

vss-account remove 3

Applies To

Cluster wide

Privilege Level

Supervisor
