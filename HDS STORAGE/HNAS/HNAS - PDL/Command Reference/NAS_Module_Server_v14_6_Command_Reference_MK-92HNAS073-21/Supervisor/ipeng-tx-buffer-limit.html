ipeng-tx-buffer-limit

Show or set current tx buffer limits

Syntax

ipeng-tx-buffer-limit [<value>]

Description

This feature is superseded by ipeng-pending-tx-* commands, making these commands deprecated.

The ipeng-tx-buffer-limit commands control a feature that limits the number of buffers used to store transmit data. This was intended to avoid one TCP connection starving all other TCP connections of buffers. It is a global setting, acting on all ports.

This feature is not usually necessary, and hence is turned off by default. Only use under instruction from your support provider.

The commands available are as follows.

ipeng-tx-buffer-limit [<number>]

If a parameter is given, sets the threshold of tx buffers each TCP connection is allowed before new buffers should be discarded.

With no parameter, displays the current and default settings.

ipeng-tx-buffer-limit-enable

Turns on the tx buffer limiting feature.

ipeng-tx-buffer-limit-disable

Turns off the tx buffer limiting feature.

ipeng-tx-buffer-limit-set-defaults

Sets the tx buffer limiting feature back to default values. The default values can be found by typing the command "ipeng-tx-buffer-limit" with no parameters.

Examples

To turn on buffer limit enable, then set the level to 500:

    $ ipeng-tx-buffer-limit-enable
    $ ipeng-tx-buffer-limit 500

To disable buffer limit enable:

    $ ipeng-tx-buffer-limit-set-defaults

Applies To

Cluster wide
