mount-requires-nvram

Queries/sets the mount policy regarding NVRAM.

Syntax

mount-requires-nvram [--on | --off]

Description

Manages the cluster's mount policy. By default, a server will fail to mount a file system for which a record can not be found in NVRAM. Changes made using the command are persistent across reboots.

The "mount policy" affects whether a mount succeeds, rather than whether a mount is automatically attempted. The latter is the "automount" policy, which is managed with the automount command.

Options:

--on

The file system's NVRAM record must exist for the mount to be allowed to proceed.

--off

The mount is allowed to proceed even if the file system's NVRAM record is missing.

With no arguments, the command displays the mount policy.

Examples

Display the current mount policy: mount-requires-nvram Mount will fail if the file system's NVRAM record can not be found.

Change the current mount policy: mount-requires-nvram --off Mount may succeed if the file system's NVRAM record can not be found.

mount-requires-nvram --on
 Mount will fail if the file system's NVRAM record can not be found.

Applies To

Cluster wide
