migration-cloud-account-create

Create an external cloud migration account

Syntax

migration-cloud-account-create --account-name <account name> --credentials-key <credentials key> --credentials-secret <credentials secret> --vendor <S3:HCPS3:HCP:Azure...> [--server-name <server name>] [--description <description>]

Description

Creates an account for cloud migration. The account includes the credentials for uploading and downloading files to a cloud account on a supported vendor and optionally the fully-qualified name of cloud target.

--account-name, -a <account name>

Name of the cloud migration account for use by the CLI only. It does not have to correspond to any identification names on the cloud storage itself, but will be needed when specifying other cloud objects such as destinations. May be up to 255 ASCII characters long. The characters ;:/\'"<>*?| and a space character are not allowed.

--credentials-key, -u <credentials key>

The key (user name) for the cloud account.

--credentials-secret, -p <credentials secret>

The secret (password) for the cloud account. If this option is not specified, the user will be prompted for the password.

--vendor, -v <AmazonS3:IBMCloud:Azure:HCP:HCPS3:S3:Test>

The name of the cloud provider for this account. Currently this value must be one of AmazonS3, IBMCloud, Azure, HCP, HCPS3, S3 or Test.

Use S3 or IBMCloud to configure an IBM Cloud Object Storage account. IBM Cloud Object Storage was previously known as CleverSafe.

HCPS3 cloud provider uses Multi-Part Upload feature of S3 API. It is recommended to use HCPS3 cloud provider in setups 'HNAS <-> HCP <-> HCP S Series' for improved performance.

Use S3 to configure HCP Cloud Scale account.

--server-name, -s <server name>

The fully-qualified name of the cloud storage server if it necessary to be specified. For HCP and HCPS3 providers this should be the address of the namespace that will host data for this account. For S3 and IBMCloud providers this should be an endpoint. AmazonS3, Azure, Test accounts do not require this option to be set.

--description, -d <description>

An optional description for this account.

Examples

migration-cloud-account-create --account-name HNAS-S3target --credentials-key NKRAIBZMZKTDFZ3PM50B --vendor S3 --server s3.endpoint.example.com

Creates a cloud migration account for S3 supported public or private cloud with the endpoint located at s3.endpoint.example.com. The user will be prompted for the secret key upon entering this command.

migration-cloud-account-create --account-name HNAS-HCPtarget --credentials-key user-name --vendor HCP --server-name namespace.tenant.example.com

Creates a cloud migration account 'HNAS-HCPtarget' for a HCP system, where 'user-name' is a HCP tenant user name. The user will be prompted for a tenant password upon entering this command. Specifies the fully-qualified domain name of the HCP system as namespace.tenant.example.com.

migration-cloud-account-create --account-name TestAccount --credentials-key test --credentials-secret test --vendor Test

Creates a cloud migration account named 'TestAccount' for a Test target. This provider doesn't require any target to be setup. Running migration with Test provider will only generate reports without actually moving any data. Note: the only credentials accepted by Test provider are 'test' for both key and secret.

Applies To

Global security context
