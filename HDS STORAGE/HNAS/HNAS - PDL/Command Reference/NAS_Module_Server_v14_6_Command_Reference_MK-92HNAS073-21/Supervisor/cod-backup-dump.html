cod-backup-dump

Show the full contents of a file created by sd-back-up-cod or span-back-up-cod

Syntax

cod-backup-dump [-F | --ssfs] [-t | --terse] <filename> (span | fs | chunk | allcod)
Alias:       cod-back-up-dump

Description

Dumps the complete contents of a Cod backup file produced by the 'sd-back-up-cod' or 'span-back-up-cod' command. Starts by printing the same information as 'cod-backup-list', and then prints one or all of the three types of Cod in a format readable by your support provider.

The second command-line argument specifies which types of Cod you would like to see. Type 'span' for the span Cod (which specifies how system drives are combined into spans), 'fs' for the file system catalog (which lists the base names and IDs of the file systems on the span), 'chunk' for the chunk table (which lists the chunks of space allocated to each file system), or 'allcod' to see all the Cod in the file.

If you're using ssc, the file is read from the ssc file system (i.e. on the computer running the ssc client); you can choose to read it from the server's SSFS file system instead by providing the --ssfs or -F switch. In all other environments the file is always read from the SSFS file system.

If the file is read from SSFS, it is, by default, read from the cluster node where the command is run. To read it from another server's SSFS file system, use the syntax detailed in the 'ssfs' man page.

The --terse switch dumps the chunk table in a more concise format than normal. It has no effect on other types of Cod.

Examples

cod-backup-dump codbackup.scd allcod

Applies To

Cluster wide
