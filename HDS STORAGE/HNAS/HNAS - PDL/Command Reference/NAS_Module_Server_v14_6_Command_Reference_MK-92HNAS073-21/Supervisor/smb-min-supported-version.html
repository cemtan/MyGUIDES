smb-min-supported-version

Set or display the minimum supported version of SMB

Syntax

smb-min-supported-version [ --default | 1 | 2 | 2.1 | 3 ]

Description

The smb-min-supported-version command is used to set or display the minimum supported version of SMB for the current EVS.

Usage:

smb-min-supported-version [ --default | 1 | 2 | 2.1 | 3 ]

Set the minimum supported version of SMB to 1, 2, 2.1, 3 or to its default value of 1.

If no argument is specified, the current minimum supported version is shown.

The minimum version specified must be less than or equal to the maximum supported version.

For example, if smb-maximum-supported-version is SMB2, then it is acceptable to specify an smb-minimum-supported-version of SMB1 or SMB2 but not SMB2.1 or SMB3.

This command affects SMB connections to the HNAS (ie normal file-serving connections) and SMB connections from the HNAS, for example to Domain Controllers or RPC-based virus scanners.

Examples

Display the current minimum supported version of SMB.

  smb-min-supported-version

Set the minimum supported version of SMB to its default of SMB1.

  smb-min-supported-version --default

Set the minimum supported version of SMB to SMB2.

  smb-min-supported-version 2

Applies To

EVS
