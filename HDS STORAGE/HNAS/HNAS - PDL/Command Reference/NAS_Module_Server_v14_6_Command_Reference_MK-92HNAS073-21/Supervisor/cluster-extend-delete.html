cluster-extend-delete

Delete the extended cluster

Syntax

cluster-extend-delete [-c | --confirm]

Description

Removes the configuration changes made by the 'cluster-extend-join' command. It removes the cluster pair consisting of nodes 3 and 4 from the extended unified cluster. If one of the nodes is already removed, it removes the remaining node in the cluster pair. This command needs quorum in Configured state and all the nodes to be in ONLINE state. It cannot be used to remove the last two nodes of the extended cluster. The last two nodes should be removed using the 'cluster-extend-unconfigure' command.

The 'cluster-extend-delete' operation is a two stage process in which nodes removed from the extended cluster reboot to a single node configuration first. In the second stage, they reboot again to form a two-node unified cluster. This command should not be used for a single node re-installation scenario. To remove one of the cluster nodes from an extended configuration, the 'cluster-node-delete' command should be used.

Options:

-c, --confirm

Do not prompt for confirmation.

Applies To

Cluster wide.

See Also

cluster-extend-join cluster-extend-status cluster-extend-unconfigure cluster-node-delete

Privilege Level

Supervisor
