span-stop-unmapping

Stop returning a span's previously-used chunks to the underlying HDP pool, as started by 'span-unmap-vacated-chunks'

Syntax

span-stop-unmapping <span-instance-name>

Description

This man page applies to spans that reside on HDP pools. See the 'hdp' man page.

If you have used 'span-unmap-vacated-chunks' to instruct the server to return previously-used disk space to the HDP pool on which a span resides (in other words, to unmap it), 'span-stop-unmapping' can be used to halt the process. Any space that has already been unmapped will stay unmapped until you create or expand filesystems.

This command does not stop the storage from zero-initializing any disk space that the server has already unmapped. In some configurations, zero-initialization takes substantially longer than it took to unmap disk space, and performance will be lower than normal until it completes.

Examples

span-stop-unmapping Accounts

Applies To

Cluster wide

See Also

hdp span-unmap-vacated-chunks

Privilege Level

Supervisor
