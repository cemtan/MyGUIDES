cluster-extend-join

Join the cluster to form an extended cluster

Syntax

cluster-extend-join [-c | --confirm] <cluster name>

Description

Joins the cluster to an existing unified cluster prepared for cluster extension using the command 'cluster-extend-create'. This configuration is mainly used to provide disaster recovery support for the unified cluster. The command can also be used on a single node unified system to join the existing extended cluster. That is useful if one node of the extended cluster is re-installed.

To join the extended cluster, the nodes have to change their cluster communication links. The internal communication over the backplane does not work in extended configuration. Before running this command, make sure the node IP address on the maintenance network is unique on all four nodes of the extended cluster. The cluster ports must be configured properly to mirror the NVRAM data across the unified cluster. This command will reboot the server automatically to join the extended cluster. The server's configuration (including all EVSs and file systems) will be removed when the node joins the cluster and replaced with the extended cluster's existing configuration.

See the 'cluster-interfaces' man page for guidelines on configuring the cluster network.

Options:

-c, --confirm

Do not prompt for confirmation.

<cluster name>

Specifies the name of the extended unified cluster.

Examples

cluster-extend-join example_server
This command will update the configuration and reboot the cluster.
Do you want to proceed?(Y/N)[N]:
y

Applies To

Cluster wide.
