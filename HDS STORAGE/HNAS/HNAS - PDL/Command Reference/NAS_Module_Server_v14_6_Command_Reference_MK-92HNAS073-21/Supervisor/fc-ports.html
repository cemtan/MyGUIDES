fc-ports

Print remote FC port info

Syntax

fc-ports [-p | --port <address>][-u | --ulp][-v | --verb][-a | --all][hport no]
Alias:       ports

Description

Display summary or verbose information about all the FC remote ports discovered by the server. The ports list may be filtered by host port number, remote port address or ULP sessions. The "state" column shows the current session state. The possible states are:

OFFLINE :

The session does not have a valid FC address for the remote port, the port has either been unplugged or is not responding.

POLLING :

The session has a valid FC address for the remote port but the port is not responding to FC commands. The server sends a LOGO command to the port every 10 seconds for a period of 2 minutes. If the port does not respond in this time it is marked offline. In some circumstances, the remote port will not allow the server to log in to it because LUN security has been enabled. In this case, the server attempts to login to the remote port every 30secs so that a change in security policy will be detected.

INACTIVE :

The remote port does not support port login (The usual state if the port is an initiator).

DISCOVERY :

The session enters this state while it is logging in; it should be transitory

PENDING :

The session enters this state if the session was ACTIVE and the FC link has gone down or in the fabric case if an RSCN is received for a particular target port. It can also occur as the result of SNS polling in the fabric case. If the session status is revalidated successfully, the session will go back to the ACTIVE state, else it will go to DISCOVERY,OFFLINE or POLLING state.

ACTIVE :

The session enters this state when it has successfully logged in to the remote port (PLOGI).

Options:

-p --port <address>

Show remote port(s) with this FC address (hex) only.

-u

Show ULP sessions (SCSI target ports) only.

-v

Show verbose info on each port.

hport no

Show remote ports discovered by this server host port only.

-a

Show all FC remote ports. By default, fabric Domain Controller addresses (0xFFFCxx) are not displayed as the server has no interest in them.

With no parameters the command prints summary info on all remote ports.

Examples

To print summary information on remote ports for host port 1:

fc-ports 1

To print summary information for all remote ports on address 6103EF on host port 1:

fc-ports -p 6103EF 1

Applies To

Cluster node

See Also

fc-hports fc-nodes

Supervisor
