filesystem-release

Allow unlimited auto-expansion of a filesystem if its span is unconfined

Syntax

filesystem-release [--tier <tier>] <filesystems-see-'man fs-spec'>
Alias:       relfs

Description

The 'filesystem-confine' command limits how far the server can automatically expand a set of filesystem when they become full, or forbids auto-expansion altogether. The 'filesystem-release' command removes all restrictions on auto-expansion for one or more filesystems that you have previously confined.

You can simply specify one or more filesystem names after the command name, separated by spaces or slashes, or you can use the full syntax described in the 'fs-spec' man page to select filesystems without having to type out their names explicitly.

If a file system is very full, it may auto-expand, perhaps several times, after the command is run. The command returns control quickly, without waiting for any auto-expansion to take place.

It is also possible to confine a span. Confining a span and confining the filesystems that reside on it are independent actions: if you have also confined the span, you will need to release that before any filesystems on the span can be auto-expanded.

There is no need to release a span or filesystem before expanding the filesystem manually with (for example) the 'filesystem-expand' command: only automatic expansion is affected by confinement.

If a filesystem is tiered, each tier has a separate confining capacity, and each tier can be confined and released independently. When confining or releasing tiered filesystems, use the --tier (or -t) switch to specify which tier you wish to confine or release.

If 'span-hdp-preallocation no' is in force, no filesystem auto-expansion takes place on spans that reside on HDP storage. In addition, no filesystems ever auto-expand on spans that reside on UVM storage.

Examples

To release untiered filesystems Accounts, Marketing and Engineering:

filesystem-release Accounts Marketing Engineering

To release Tier 1 of tiered filesystems ProjectX and ProjectY:

filesystem-release --tier 1 ProjectX ProjectY

Among filesystems whose names start with the word 'Project', the following sequence of commands releases the untiered filesystems and then both tiers of the tiered filesystems:

filesystem-release Project*:untiered
filesystem-release --tier 0 Project*:tiered
filesystem-release --tier 1 Project*:tiered

This sequence satisfies the requirement to use the --tier switch for tiered filesystems and not for others.

Applies To

Cluster wide
