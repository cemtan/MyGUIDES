ndmp-management-ports-set

Set NDMP management relay ports

Syntax

ndmp-management-ports-set <external_port> <internal_port>

Description

SMU connections to control Replication and Data Migration functions on the server usually use the high speed LAN ports associated with the Service EVS. Setting the NDMP management ports with this command causes these connections to be made via the low speed management LAN.

This may be useful if the network layout is such that the SMU cannot connect to the Service EVS addresses. However, it is usually better to use the standard connections to the Service EVSs.

The ports used must be chosen so they do not clash with any other TCP port usage. In particular they must not clash with the port being used to provide the NDMP service on the EVS. (This is usually 10000.)

The values for the ports must be in the range 1 to 65535.

Examples

Set ports so the SMU Replication and Data Migration functions connect to the server via the management LAN:

ndmp-management-ports-set 10001 10002

Applies To

Cluster wide

See Also
