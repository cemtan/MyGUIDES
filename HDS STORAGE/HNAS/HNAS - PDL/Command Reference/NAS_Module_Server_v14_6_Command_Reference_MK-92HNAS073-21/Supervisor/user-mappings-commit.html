user-mappings-commit

Commits changes to the user mappings table

Syntax

user-mappings-commit

Description

Stores modifications to the user mappings table to persistent store.

A background task will periodically commit changes to the user mappings table to flash and synchronize the changes with other nodes in the cluster.

This command can be used to force a commit after completing changes to the user mappings table so changes are propagated cluster wide immediately.

Examples

Commit mapping changes:

$: user-mappings-commit

Applies To

EVS security context

Privilege Level

Supervisor
