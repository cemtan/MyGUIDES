filesystem-decode-si

Dump the config of a filesystem configured into DI's control memory

Syntax

filesystem-decode-si [--hexdump | -h] [--terse | -t] [--no-lock | -L] [--no-abuse-warning | -W] ((--all | -a) | <filesystem-device-ID-or-instance-name-or-FSM-volume-number>)
Alias:       fdi

Description

The server uses hardware acceleration to convert filesystem offsets into system drive (SD) device IDs and offsets. This command dumps the configuration of the chip that does the translation. The output is intended for use by your support organization.

The command has several switches:

--all, -a

Normally, the command dumps the configuration for a single filesystem, which you can specify by its instance name, its device ID (such as 1024), or its volume number (such as 0). If you specify --all instead of a filesystem, the command dumps all filesystems currently configured into the chip.

--terse, -t

Dumps the configuration in a terser format that assumes that the configuration is valid. Certain theoretically-possible misconfigurations won't show up in this format. However, the command detects and identifies them and prints a message, recommending that a full (non-terse) dump be done.

--hexdump, -h:

Produces a raw hex dump of the chip's control memory after the normal dump.

--no-abuse-warning

Suppresses the usual warning against deducing too much from the output of this command.

--no-lock, -L

Bypasses important safety mechanisms inside the server. It is not available to unprivileged users. Use of this switch is not supported.

Examples

filesystem-decode-si Accounts
filesystem-decode-si --terse --all

Applies To

Cluster node
