mclist

Show outstanding management requests on this cluster node

Syntax

mclist [--incoming | -i] [--outgoing | -o] [<minimum-age-in-milliseconds>]

Description

When one of the server's subsystems needs to ask another subsystem to do something, it uses a mechanism called a 'management call'. If you run 'mclist' on a particular server, it lists management calls made by that server that have not yet returned.

The --incoming or -i switch lists management calls being served by this server. The --outgoing or -o switch lists management calls made by this server. If you supply neither switch, the command lists both incoming and outgoing calls. A single call can loop back from the server to itself, and may therefore appear in both lists.

If you supply a minimum age (in milliseconds), calls that are more recent than that age will not be listed. By default, the command lists all calls that are still in progress.

The command prints the following information:

Request

The name of the call.

Age

How long ago the call was made.

Context

Identifies where the call was made from. The two colon-separated values are the originating cluster node ID and a request sequence number that is specific to that cluster node.

Peer

Shows which server is serving the call. A value of 0 indicates a loop-back call that is made and served by the same cluster node.

Thread

Identifies the thread on the local board that either made the call or is busy serving it.

Calls are presented in chronological order, oldest first.

Examples

The following command lists all incoming and outgoing calls on cluster node 1:

server:$ pn 1 mclist
