span-confine

Prevent auto-expansion of all filesystems on a span

Syntax

span-confine <span-instance-name>
Alias:       conspan

Description

In normal use, the server automatically expands filesystems when they become full. If you confine a span, automatic expansion of all the filesystems on that span is forbidden. Manual expansion, however, is still allowed.

One reason to confine a span is to ensure that Cod backups stay valid. When a filesystem is expanded, Cod backups for the host span become out-of-date and hard to use. By confining a span and expanding its filesystems manually, you can avoid this problem.

You can prevent or restrict auto-expansion of a single filesystem on a span using the 'filesystem-confine' command. Confining a span and confining a filesystem are independent actions: if you confine both a filesystem and the span on which it resides, you will need to release them both (in either order) before the filesystem can be auto-expanded.

On a tiered span, all tiers are affected by 'span-confine' and 'span-release'. There is no mechanism for confining or releasing a single tier of a span. For a similar facility, see 'filesystem-confine' and 'filesystem-release', each of which has a --tier switch.

If 'filesystem-thin' has been used to set up filesystem-level thin provisioning, 'span-confine' temporarily disables it by instructing the server to report filesystems' actual capacities to protocol clients. 'span-release' reverses this effect.

If 'span-hdp-preallocation no' is in force, no filesystem auto-expansion takes place on spans that reside on HDP storage.

Examples

span-confine Accounts

Prevents all filesystems on span Accounts from auto-expanding.

Applies To

Cluster wide
