virusscan-icap-engine-add

Add an ICAP virus scanning engine

Syntax

virusscan-icap-engine-add [--service-name <service name>] [--port <port number>] <host>

Description

The virusscan-icap-engine-add command is used to add an ICAP compatible virus scanning engine to the current ICAP engine list. An ICAP engine is automatically enabled when added. If the current virus scanning mode is RPC and virus scanning is enabled then no action will be taken, otherwise ICAP mode will be automatically selected.

Options:

-s, --service-name <service name>

Name of the service to be used. Provided by the ICAP virus scan engine vendor. Defaults to AVSCANRESP.

-p, --port <port number>

The remote port on which the ICAP virus scanning engine is listening. If not specified, the default ICAP port (1344) will be assumed.

<host>

A hostname, or an IP address, that identifies the host running the ICAP virus scanning engine.

Examples

To add the ICAP engine icapengine.example.com to the engine list:

Using a hostname:

virusscan-icap-engine-add --service-name AVSCANRESP --port 1344 icapengine

Alternatively:

virusscan-icap-engine-add --service-name AVSCANRESP --port 1344 icapengine.example.com

Alternatively, using an IP address:

virusscan-icap-engine-add --service-name AVSCANRESP --port 1344 192.168.43.100
