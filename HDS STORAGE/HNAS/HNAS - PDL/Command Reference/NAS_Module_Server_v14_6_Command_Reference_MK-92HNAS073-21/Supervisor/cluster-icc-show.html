cluster-icc-show

Show cluster interconnect status

Syntax

cluster-icc-show [-j|--json]

Description

The cluster-icc-show command displays the intra-cluster connect communication status of configured interfaces.

Values:

OK

The interconnect interface is able receive both normal and jumbo messages.

FAIL

Interconnect interface is failed, so not receiving any messages.

NO-HTBT

Interconnect interface is OK, but unable to receive any heartbeat messages.

NO-JUMBO

Interconnect interface is able to receive normal messages, but failed to receive any jumbo messages.

DEG

Failure of one of the interconnect interfaces, applicable for overall connection status.

C1-C4

Physical ports available on a server, depending on the model. In server models having four physical ports, C1 and C3 are mandatory whereas C2 and C4 are optional.

ICC

Overall connection interfaces, though not a true physical port.

Examples

1. On a 2-node clustered system with only two physical ports, using both:

server:$ cluster-icc-show
Cluster node 1:


Port   To=>     Node 1     Node 2
-----------     ------     ------
         C1       FAIL    NO-HTBT
         C2         OK         OK
        ICC        DEG        DEG

2. On a 2-node clustered system with four physical ports and using all of them:

server:$ cluster-icc-show
Cluster node 1:


Port   To=>     Node 1     Node 2
-----------     ------     ------
         C1         OK         OK
         C3         OK         OK
         C2         OK         OK
         C4         OK         OK
        ICC         OK         OK

3. On a 4-node clustered system with four physical ports and using two of them:

server:$ cluster-icc-show
Cluster node 1:


Port   To=>       Node 1     Node 2     Node 3     Node 4
-----------     --------     ------     ------     ------
         C1     OK             FAIL         OK         OK
         C3     NO-JUMBO         OK         OK         OK
        ICC     NO-JUMBO        DEG         OK         OK

4. On a 4-node clustered system with four physical ports and using all of them:

server:$ cluster-icc-show
Cluster node 1:


Port   To=>       Node 1     Node 2     Node 3     Node 4
-----------     --------     ------     ------     ------
         C1     NO-JUMBO         OK         OK         OK
         C3     NO-JUMBO         OK         OK         OK
         C2     NO-JUMBO         OK         OK         OK
         C4     NO-JUMBO         OK         OK         OK
        ICC     NO-JUMBO         OK         OK         OK


Note: On server models having four physical ports, the mandatory connections
(C1 and C3) are listed before the optional ones as referenced in the examples.

Applies To

Cluster node
