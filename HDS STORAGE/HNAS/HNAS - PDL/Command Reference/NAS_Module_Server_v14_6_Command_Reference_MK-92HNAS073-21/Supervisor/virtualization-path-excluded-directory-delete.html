virtualization-path-excluded-directory-delete

Deletes an entry from the list of directories to be excluded from virtualization.

Syntax

virtualization-path-excluded-directory-delete -t <fs label> -n <association name> -d <directory-to-exclude>

Description

Deletes an entry from the list of directories to be excluded from the virtualization path association between a remote NFS export source and a local file system, vivol or directory target.

This command can only be applied once the association has been created but the virtualization has not yet started.

Options:

-t, --target <fs label>

Target file system.

-n, --name <association name>

Target association name.

-d, --dir <directory>

Target directory relative to the root of the association to exclude from virtualization.

Examples

virtualization-path-excluded-directory-delete -t fs1 -n assoc1 -d .snapshot

Deletes the entry .snapshot from the excluded directory list for the association assoc1 to file system fs1.
