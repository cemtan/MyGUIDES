diag-sources

List, enable or disable sending of certain diagemails

Syntax

diag-sources (source (on|off|default))*

Description

Enables or disables some diagnostic email ('diagemail') messages sent by the Sim. Tells whether these diagemails will be sent.

The diagemail categories that this command can control are:

cod

Messages sent when a problem with Cod (config on disk) prevents a span or a file system from loading

integrity

Messages sent when the server detects an encroachment

Each category is in one of three states: on, off or default. 'On' enables diagemail messages; 'off' disables them. When the server is shipped, all categories are in the default state. On all publicly released builds, 'default' is equivalent to 'on'.

'diag-sources' is persistent: its effects will survive a reboot or a cluster failover.

Some diagemail messages sent by the Sim can't be controlled by 'diag-sources'.

Examples

diag-sources

Shows all categories and their states

diag-sources cod off integrity default

Disables the 'cod' category; returns the 'integrity' category to its default state.

Applies To

Cluster wide
