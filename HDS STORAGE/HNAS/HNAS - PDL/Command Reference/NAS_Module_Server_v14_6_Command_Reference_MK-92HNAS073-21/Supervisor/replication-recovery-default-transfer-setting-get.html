replication-recovery-default-transfer-setting-get

Get the per file system default access point transfer disaster recovery setting.

Syntax

replication-recovery-default-transfer-setting-get <file-system>

Description

This command applies to WFS-2 file systems only.

Gets a file system's default disaster recovery behaviour for shares and exports. This setting is used to decide whether the shares/exports that have their '--disaster-recovery-setting' field set to 'use-fs-default' are recovered or not when an Object Replication target of the file system is promoted to read-write.

Note that when a share/export is created, its '--disaster-recovery-setting' field is set to 'use-fs-default' by default, i.e., its recovery is controlled using the file system level switch. Also note that the file system level switch can be overridden on an individual basis by setting a share/export's '--disaster-recovery-setting' field to 'transfer' or 'do-not-transfer' explicitly (see cifs-share/nfs-export add/mod commands).

<file system>

The file system to query this setting for.

Examples

    replication-recovery-default-transfer-setting-get fs0
    Disaster recovery default access point transfer setting for file system 'fs0' is disabled

Applies To

EVS
