evsfs

EVS file system management

Syntax

evsfs 
	add <filesystem-specifier> <evs-specifier>
	delete <filesystem-specifier> <evs-specifier> [--override-pdm]
	list [<evs-specifier>]

Description

List and manage EVS file systems in a cluster.

Usage:

add <filesystem-specifier> <evs-specifier>

Assigns a file system with no EVS assignment to an EVS. Only the EVS to which it is assigned is allowed to mount a file system.

delete <filesystem-specifier> <evs-specifier> [--override-pdm]

Removes a file system from an EVS. The file system must be unmounted before the removal. From now on, no EVS will be able to mount the file system.

If the file system is currently participating in a data migration path, removing it from an EVS may result in unexpected loss of data. If all file systems involved in the data migration path are deleted from one EVS and added to another, the data migration path will be resurrected. Data migration paths can only be detected on mounted file systems, but an 'evsfs delete' command requires an unmounted file system. An attempt is made to detect any data migration dependencies from currently mounted file systems and, if any are found, a warning will result, stopping the deletion. The '--override-pdm' flag will enable the deletion to proceed, despite the risks.

list [<evs-specifier>]

Displays the EVS file system mapping. Use this command to find out which file systems are bound to each EVS. If you specify an EVS, only that EVS's file systems will be listed.

Examples

To add a file system to an EVS:

evsfs add myfilesystem 1

To list all file systems assigned to an EVS:

evsfs list 1
FS ID         FS Label      FS Permanent ID   EVS ID   EVS Label
-----   --------------   ------------------   ------   ---------
 1024   ny-octopus-fs0   0x6438f46bfff77ba4        1       EVS01

To list all file systems:

evsfs list
FS ID         FS Label      FS Permanent ID   EVS ID   EVS Label
-----   --------------   ------------------   ------   ---------
 1024   ny-octopus-fs0   0x6438f46bfff77ba4        1       EVS01
 None   Unknown          0x686b320f87199b26        2       EVS02

To remove a file system from an EVS:

evsfs delete myfilesystem 1

To remove a file system from an EVS, specifying its permanent ID:

evsfs delete 0x686b320f87199b26 2

To remove a file system from an EVS, specifying its device ID:

evsfs delete 1024 1

Applies To

Cluster wide
