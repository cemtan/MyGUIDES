span-expansion-requirements

Determine how many SDs must be added by 'span-expand', and of what types

Syntax

span-expansion-requirements <span-instance-name> [<tier>]

Description

The 'span-create' man page explains the minimum number of system drives (SDs) that must be added to a span when you expand it in the GUI or via 'span-expand'. There are further restrictions on the type of storage that can be added to a span: for example, HDP DP-Vols can be added to a span that resides on UVM internal LUs but not to a span residing on plain parity groups.

'span-expansion-requirements' tells you the minimum number of SDs needed to expand the span and which storage types are acceptable. For a tiered span, specify both the span instance name and the tier you wish to expand; for an untiered span, specify only the span instance name.

Examples

server:$ span-expansion-requirements AnUntieredSpan
Expansion of this span requires at least 4 SDs of type HDP
server:$ span-expansion-requirements ATieredSpan 0
Expansion of this span requires at least 4 SDs of type HDP
server:$ span-expansion-requirements ATieredSpan 1
Expansion of this span requires at least 6 SDs of type HDP
server:$

Applies To

Cluster wide

See Also

span-expand
