replication-schedule-modify

Modify an object replication schedule

Syntax

replication-schedule-modify [--interval <interval>] [--interval-type <interval-type>] [--interval-unit <interval-unit>] [--next-run <next-run-time>] [--run-forever|--run-until <run-until-time>] [--enable|--disable] <policy name> <source fs label> <schedule id>

Description

Modifies a schedule for object replication.

<policy name>

Name of the policy.

<source fs label>

Source file system label.

<schedule>

Schedule ID.

--next-run <next-run-time>

The next run time in the format YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm].

The scheduler will run the object replication on or within 2 minutes of the next run time.

--run-until <run-until-time>

The run until time in the format YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm].

The scheduler will not run this object replication after the run until time.

--run-forever

The scheduler will run this object replication forever.

--interval-type <interval type>

Interval type, allowed values are:

periodic

Run an object replication at the next run time, and at every <interval> <interval unit> after that.

continuous

Run an object replication at the next run time, then schedule the next run for <interval> <interval unit> after the end of that object replication.

onceonly

Runs an object replication once at the next run time.

detailedtest

Runs an object replication detailed test once at the next run time.

--interval <interval>

The number of interval units.

--interval-unit <interval unit>

The interval unit as minute (valid interval 0-120), hour (valid interval 0-48), day (0-365), week (0-52) or month (0-12).

--enable

Enable the object replication schedule thus allowing it to be scheduled to run.

--disable

Disable the object replication schedule thus disallowing it from being scheduled to run.

Examples

replication-schedule-modify --next-run 2011-02-03T03:30:00-08:00 --interval 3 --interval-type periodic --interval-unit day fs0 a51ddda0-ca9b-11ca-9003-b6a9798eec8b

Modifies the specified object replication schedule on file system fs0 so that the policy is run every 3 days, beginning 3 March 2011 at 3:30am PST.

replication-schedule-modify --interval 1 --interval-type continuous --interval-unit hour fs0 a51ddda0-ca9b-11ca-9003-b6a9798eec8b

Modifies the specified object replication schedule on file system fs0 so that the policy is run continuously, pausing for 1 hour between consecutive runs.

Applies To

EVS
