sd-dump-encroachment-area

Dump one or more SDs' encroachment areas to the console

Syntax

sd-dump-encroachment-area [--denied | -d] <System-drives-see-'man sd-spec'>

Description

'sd-dump-encroachment-area' prints a hexdump of one or more SDs' encroachment areas. (The encroachment area is described in the 'encroachment' man page.) It should consist of several hundred bytes of repeated text, followed by some binary data, and ending with a signature of '0x55, 0xAA'.

Encroachment data is read directly from the storage; cached state is not used.

The --denied or -d switch enables the command to read from unlicensed system drives (those to which you have not enabled access). Without this switch, the command will reject unlicensed SDs. Never use this switch to dump data from SDs that are being used by another cluster.

To see how to specify more than one SD, consult the 'sd-spec' man page.

Examples

sd-dump-encroachment-area 13

Dumps the encroachment area from SD 13.

sd-dump-encroachment-area Accounts:has-bad-encroachment

Dumps the encroachment data from all SDs in span Accounts on which an encroachment has been detected.

Applies To

Cluster wide
