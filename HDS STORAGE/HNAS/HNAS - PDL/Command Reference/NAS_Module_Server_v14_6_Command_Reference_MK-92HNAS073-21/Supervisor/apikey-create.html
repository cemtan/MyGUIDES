apikey-create

Create a new API Key

Syntax

apikey-create <description>

Description

apikey-create generates a new API Key which can be used instead of a username/password combination when authenticating external API requests.

Parameters

<description>

Describe the use for the API Key - it's important to provide a meaningful description when a new key is created, as the keys are made up from a random set of characters. The description will be used for auditing and logging, so should be something that makes it easy to determine what external process was accessing the system via the APIs.

A full API Key is formatted as follows, with the prefix being the characters preceeding the first . character:

SzKdKXQxur.GU0RAlCfxyQ8TJYc7twueT3.x.8VvvvlXBXBP6.bvH3

For security reasons, the full API Key is only displayed upon creation - take a copy, as it is not possible to display the full Key again. The Key prefix or description can be used for deletion/modification, but the full Key is required for authentication by external clients.

New keys are created allowing full access to all supported SOAP and REST APIs, if this is not required, modify the key access level with the apikey-update command.

Examples

To use an API Key to access the external APIs, the key must be supplied as the "X-Api-Key" HTTP header variable. The following example shows how to format the command line for the external "curl" tool to make an API call using an API Key to authenticate:

curl -X POST -H "X-Api-Key: SzKdKXQxur.GU0RAlCfxyQ8TJYc7twueT3.x.8VvvvlXBXBP6.bvH3" https://172.27.5.11:8443/hyas/

Applies To

Cluster
