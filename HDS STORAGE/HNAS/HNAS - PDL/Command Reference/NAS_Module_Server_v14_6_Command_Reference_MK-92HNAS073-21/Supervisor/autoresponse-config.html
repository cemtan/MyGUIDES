autoresponse-config

Enables/disables autoresponse

Syntax

autoresponse-config [--type (all|nfs|smb1|smb2)] [--operation (all:readx:writex:unlink:rmdir:trans2-find-first2:trans2-query-file:trans2-query-path:trans2-set-file:trans2-set-path)] [--confirm] [on|off]

Description

The autoresponse-config command is used to manage which protocols are subject to autoresponse by the server.

Usage:

autoresponse-config [--type <protocol_type>] [--operation <protocol_operation>] [--confirm] [<state>]

Options:

-o, --operation <protocol_operation>

Selects a specific SMB1 protocol operation to configure. Valid choices are:

(all|readx|writex|unlink|rmdir|trans2-find-first2|trans2-query-file|trans2-query-path|trans2-set-file|trans2-set-path)

(Operation level control is not supported for NFSv2/v3 or SMB2)

-t, --type <protocol_type>

Selects a protocol to configure. Valid choices are:

(all|nfs|smb1|smb2)

Notes: The 'nfs' option specifies NFSv2/v3. The 'smb2' option specifies SMB2.x/3.x. Defaults to all.

Turn autoresponse on or off for the given protocol/operation. If no state is given then the current state is shown.

--confirm

Bypass any warning messages which ask the user to confirm their actions on the terminal.

Examples

autoresponse-config --type all

Displays the configuration details for all protocols

autoresponse-config on

Enables autoresponse for all protocols

autoresponse-config --type smb1 on

Enables autoresponse for SMB1

Applies To

Cluster wide
