migration-cloud-cleanup

Purge or un-delete files after FS has been recovered from a snapshot or snapshot was deleted

Syntax

migration-cloud-cleanup <fs-label>

Description

Triggers a background cleanup process on Cloud Gateway. Cleanup process decides whether the file can be purged from the cloud (doesn't belong to any snapshot) or should be undeleted (File system was recovered from a snapshot).

Note: Cleanup cannot be performed on object replication target file systems.

fs-label

File system label, cloud cleanup process will run for.

Examples

migration-cloud-cleanup myFS

Starts cleanup process on the cloud for file system 'myFS'.

Applies To

EVS security context

See Also
