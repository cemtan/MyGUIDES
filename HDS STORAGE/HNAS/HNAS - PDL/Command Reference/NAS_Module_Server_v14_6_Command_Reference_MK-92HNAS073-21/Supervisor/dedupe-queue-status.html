dedupe-queue-status

Reports the pnodes dedupe queue status

Syntax

dedupe-queue-status

Description

Reports the current status of the dedupe queue.

Due to memory constraints not all file systems can be deduped concurrently, so each node in the cluster maintains a queue of file systems waiting to be deduped.

This command shows the current status of the queue - what is queued and what is currently running.

We use a priority queue, with the following levels - highest priority to lowest:

incremental

This level holds file systems that have requested an incremental dedupe using the dedupe-queue-add command

triggered incremental

This level holds file systems that have reached the maximum change threshold that automatically triggers an incremental dedupe request.

daily incremental

This level holds file systems that haven't been deduped in over a day (haven't had enough change to trigger a dedupe)

full

This level holds file systems that require a full dedupe of the entire file system

This command shows the file systems queued at each level, and the file systems that are currently being deduped.

Examples

To report the status of the dedupe queues on all nodes in the cluster:

cn all dedupe-queue-status

Applies To

Cluster Node

See Also

Privilege Level

Supervisor
