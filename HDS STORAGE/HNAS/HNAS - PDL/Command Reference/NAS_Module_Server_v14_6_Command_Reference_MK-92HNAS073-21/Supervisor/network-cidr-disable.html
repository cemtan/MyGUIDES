network-cidr-disable

Disable CIDR routing

Syntax

network-cidr-disable

Description

When routing needs to choose a route, it searches in order through the host route table and then the network route table and finally the gateway table, looking for a suitable route. Once a route is found in one of the tables, subsequent tables are not searched. If more than one route is found within a table, routing has to choose amongst them. If CIDR mode is enabled, which it is by default, then routing will choose based on the length of the prefix match and, if that still produces more than one route, based on the metric of the associated interface, where a numerically lower metric is preferred. For some network configurations, that can produce unworkable routes. To allow for such networks, it is possible to cause routing to ignore the length of the prefix when choosing a route and to choose based solely on the interface metric. However, due to the arbitrary order of routes in the routing table, this may result in the same route being chosen.

CIDR is Classless-Inter Domain Routing, which is an IP address allocation scheme intended to slow the growth of the Internet's routing tables (see RFC 4632).

The network-cidr-enable command causes routing to consider the length of the prefix when choosing a route. This is the default setting.

The network-cidr-disable command causes routing to disregard the length of the prefix when choosing a route.

The network-cidr-show command displays whether routing currently takes prefix length into account when choosing a route.

Examples

To configure routing to choose routes based solely on the interface metric, i.e. CIDR mode disable:

$ network-cidr-disable
$

To configure routing to choose routes based on the longest matching prefix first and the interface metric second, i.e. CIDR mode enabled:

$ network-cidr-enable
$

To display routing's current CIDR mode:

$ network-cidr-show
network CIDR is enabled.
$

Applies To

Cluster wide
