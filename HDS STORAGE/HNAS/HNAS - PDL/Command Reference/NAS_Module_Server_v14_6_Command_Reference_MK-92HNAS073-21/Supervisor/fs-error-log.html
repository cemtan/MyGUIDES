fs-error-log

Displays the file system error log

Syntax

fs-error-log --filesystem <file system label> [--reset]

Description

Utility for dumping or resetting the error log for a given file system.

The error log accumulates errors and stores them on a per file system basis. These errors are due to inconsistencies found on disk, and will be tolerated according to the file system's resiliency level (see fs-resiliency-level). The error log also attempts to record additional information which may be useful in tracking down the root cause (system drive, object number, etc).

If a file system is taken offline from too many errors (see fs-resiliency-level), that file system's error log is dumped to the dblog and automatically cleared. If the server itself is rebooted, all errors logs will be reset. Mounting and unmounting, however, will not affect a file system's error log.

Only unique errors are counted against a file system. For example, if a file has an inconsistent section, multiple attempts to access that inconsistent section will result in one logged error. The count for how many times that particular error has occurred, however, will increase. The number of times an error has occurred is denoted by the "Count" output field (see EXAMPLES).

Options

-f, --filesystem

The label of the file system to operate on.

-r, --reset

Clear the error log for a given file system. This will also reset the recorded errors for a file system to 0 (see fs-resiliency-level).

Examples

Displaying the log:

fs-error-log -f fs1

Example output:

Error                                   Object# Object Type            ObjOffset   FsOffset Sd#    SdOffset Count First seen                Last seen
 --------------------------------------- ------- ---------------------- --------- ---------- --- ----------- ----- ------------------------- -------------------------
 ERR_OBJ_INDIRECT_ONODE_ASCII_IONA_ERROR  0x400c OBJ_TYPE_FSOBJECT_FILE           0x706aa000  15 0x177baa000    34 2010-09-29 12:14:42-07:00 2010-09-29 12:14:42-07:00

Resetting the error count for a particular file system:

fs-error-log --reset -f fs1
