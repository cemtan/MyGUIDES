network-protocol-performance-monitor-report

Report all statistics stored by the network protocol performance monitor.

Syntax

network-protocol-performance-monitor-report [--file-system <file system>] [-a] [-n]

Description

This command will, for each file system with monitoring enabled, display any file system performance statistics which have been gathered. The statistics are intended for resolving performance problems.

Statistics are stored in memory, with each file system having a rolling buffer of statistics storing new samples and discarding old ones.

Options:

-a, --all

Report for both filesystems and non filesystem information.

-n, --nonfs

Report any non filesystem information.

-f, --file-system <filesystem>

Only report for the named filesystem. If omitted, all filesystems are reported.

Report Format

The report output is intended to be easy to parse into a form that can be analysed or fed into graphing software. The output is in sections with each section beginning and ending with a section heading marked with brackets '[]'. The contents of the brackets are a key word to describe which sort of section it is, optionally with an equals sign followed by value after it to give more information about the section such as the label of a file system.

The report is broken into sections with section headings as follows.

[time="2012-11-15 10:19:24.000+00:00"]

Gives the timestamp of the most recent sample received. No data is expected after this header.

[period="10.01905"]

Gives the measured sampling period of the samples. Ideally this is supposed to be 10 seconds exactly, but the data is generated by a part of the system with a different clock to the main clock. Usually this difference in clocks doesn't matter, but here where we can look back for days it is worth adjusting for this difference.

For a while after boot the server will not have enough samples to make an accurate estimate of the period, so it will report exactly 10.00000. By the time there are enough per filesystem samples for the difference to matter, an accurate period should be available.

[headers]

The header is followed by a single line of data which gives a comma separated list of labels. These labels describe the columns of data given in the following file system sections, in the same order as they appear in the lines in that section.

[fs="file-system-name"]

Each line in this section is a sample taken from the named file system. The first line is the most recent sample, the last line the oldest. Each line is comma separated list of values, corresponding to the data given in the headers section before it.

[values]

Each line in this section is a time sample taken from the non filesystem information. The first line is the most recent sample, the last line the oldest. Each line is comma separated list of values, corresponding to the data given in the headers section before it.

[end]

Marks the end of the report section, mainly to show that the file has not been truncated.

Examples

Show current statistics logs for all file systems:

    network-protocol-performance-monitor-report
