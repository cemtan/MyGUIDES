package-check-compatibility

Check whether two package versions can run in the same cluster

Syntax

package-check-compatibility <version1> <version2>

Description

The package-check-compatibility command checks whether two versions of firmware are compatible for running on separate cluster nodes of the same cluster.

The command takes two parameters: text strings representing the versions of firmware to be run on a server.

Note: It is not sufficient to use package names for the parameters, as the command will not inspect the contents of a package to determine its version. The values for the parameters must be obtained either by running the ver command, or by manually inspecting the contents of version.txt in a package.

Examples

package-check-compatibility 5.1.1148.01 5.1.1148.05

Applies To

Cluster wide
