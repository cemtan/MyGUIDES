rm

Unlink files

Syntax

rm [-f|--force] [-v|--verbose] [--] <file1> [<file2>...]

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The rm command is used to unlink one or more files. If a file name begins with a dash, it cannot be specified as the first argument to rm. It should be specified after a stand-alone option "--" or after another file name that does not begin with a dash.

Usage:

--force, -f

Do not fail if the file does not exist.

--verbose, -v

Display a status message for each non-existent file when option --force is specified.

--

Optional to signify the end of options and hence all subsequent arguments are file names to delete. Useful if the first file name begins with a dash.

Examples

1. To unlink the files 'file1', 'file2' and 'file3' from the current working directory:

   rm file1 file2 file3

2. Specify --force to not fail the command when the file is non-existent:

   rmdir -f file1

3. Two ways to unlink a file whose name begins with a dash:

   rmdir -f -- -xfile    # after stand-alone option --
   rmdir -f afile -bfile # after another file
