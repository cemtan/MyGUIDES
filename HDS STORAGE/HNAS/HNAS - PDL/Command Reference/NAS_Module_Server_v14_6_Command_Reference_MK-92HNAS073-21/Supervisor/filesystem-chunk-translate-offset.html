filesystem-chunk-translate-offset

Translate a filesystem chunk number and offset into different formats

Syntax

filesystem-chunk-translate-offset [--dump <nr-bytes> [--verbose] [--timeout <nr-seconds>]] <filesystem> <chunk-number> [<chunk-offset-in-bytes>]

Description

This command is intended for use by your support provider. Given a filesystem instance name, the number of a chunk within the filesystem, and an offset into the chunk, it prints several other representations of the same offset: as an offset into a system drive, as an offset into the filesystem as a whole, and so on.

See 'filesystem-translate-offset' for an explanation of the output.

Specifying '--dump' and a byte-count will read the corresponding data directly from the storage (bypassing the server's caches) and hexdump it on the console. The command always dumps a whole number of disk blocks, and so '--dump 0' will dump the whole of the disk block that contains the offset you specify. If you add '--verbose', the command prints the system drive device ID and offset before each block and dumps the data in full; otherwise, repeated lines are suppressed. '--verbose' implies '--dump'. This command can't dump more than a megabyte or dump any data beyond the end of a chunk; to dump further, issue another '*-translate-offset' command.

The '--timeout' switch limits the amount of time that the command spends dumping data. If the specified time is exceeded, the command will abort. Without '--dump', '--timeout' is ignored. The default timeout is 120 seconds.

There is no syntax for working with an offset into a deleted filesystem. If necessary, undelete a filesystem and then translate offsets.

Examples

See the examples under 'sd-translate-offset'.
