gss-token-cache-info

Report count of entries in GSS token cache (for DDNS)

Syntax

gss-token-cache-info gss-token-cache-info

Description

This command displays information about the GSS token cache. The GSS token cache is used to hold GSS tokens during individual DDNS registration operations. Once a given registration completes, its token is removed from the cache. If a DDNS registration doesn't complete within the defined cache age-out time, then the corresponding GSS token is automatically removed from the cache. This means that the registration, should it eventually proceed, will fail to complete. But the server will retry the registration after a few minutes so it should complete once the DNS server's responsiveness improves.

This command reports cache size and cache entry age-out time. The size is displayed twice, corresponding to two different measurements. Both should be the same. The age-out time cannot be changed, and is shown for reference only.

There is only one GSS token cache per cluster node. When a system is fully configured and has settled in to its environment, it is normal for the GSS token cache to be empty.

Examples

Display current GSS token cache parameters:

gss-token-cache-info

Applies To

Cluster node
