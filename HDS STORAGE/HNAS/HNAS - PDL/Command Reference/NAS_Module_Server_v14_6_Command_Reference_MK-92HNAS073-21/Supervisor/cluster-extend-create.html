cluster-extend-create

Extend the cluster for disaster recovery

Syntax

cluster-extend-create [-c | --confirm] <QD name/ipaddr>

Description

Extends a unified NAS module cluster for disaster recovery. This command prepares the unified cluster to form an extended configuration that can be joined by another unified cluster. The partner cluster can be joined later using the 'cluster-extend-join' command to form 4-node extended unified cluster.

The extended cluster cannot use the internal quorum, so it must be specified with an external Quorum Device.

See the 'cluster-interfaces' man page for guidelines on configuring the cluster network.

Options:

-c, --confirm

Do not prompt for confirmation.

<QD name/ipaddr>

Specifies the quorum device's name or IP address.

Examples

cluster-extend-create qd.example.com
This command will update the configuration and reboot the cluster.
Do you want to proceed?(Y/N)[N]:
y

Applies To

Cluster wide.
