kill-quotas

Description

This dev-level command clears one or both the quota system files of a file system: space constraints file and space trackers file. It requires the file system be unmounted.

Note that the contents of these files can be displayed by the 'space-constraint' and 'space-tracker' commands, respectively.

Options

<file-system>

This is mandatory. It specifies the file system to operate on.

-f, --file trackers|constraints

This is optional. If omitted, the command clears both quota system files. Otherwise, it will clear the specified quota system file only.

Examples

To clear the space trackers of file system named 'fs':

   kill-quotas --file trackers fs

This command is frequently used to fix quota corruption. Once the space trackers file is cleared, it will be rebuilt on file system remount.

Applies To

EVS
