user

Manage management users

Syntax

user add [<username> <password> <level>]|del <username>|delall|list

Description

The user command is used to administer management users. You can add, remove or list users.

If "user add" is used without options, then the command prompts for a username, level and password, in that order. A final interactive password confirmation is required for all "user add" invocations, even if the username, password and level are specified, in that order, on the command line.

The maximum number of users that can be created is 20. Built-in users, e.g. supervisor, do not count towards this limit.

"user list" displays all users of equal or lower level.

Supported levels are "user" and "supervisor". Users of level "user" are only allowed to execute commands of "user" level and lower. These are expected to be read-only commands. Such users may, then, inspect but not alter configuration. Users of "supervisor" level are allowed to execute commands which modify the configuration. A command's minimum level is given by its "help" - see "man help". Some commands have several sub-commands, which may enforce further restrictions. "user list", for example, is available at "user" level, while "user add" requires "supervisor" privilege.

"user del" removes a single user and requires a username. Use "user delall" to remove all users except the current user and supervisor, if extant.

Examples

To add a user called 'viewonly' as a 'user' level user:

user add viewonly password user

To delete all users except the current user and supervisor:

user delall

To show currently configured users:

user list

To show currently logged-in users:

who

To show the user logged in on the current console:

whoami

Applies To

Cluster wide
