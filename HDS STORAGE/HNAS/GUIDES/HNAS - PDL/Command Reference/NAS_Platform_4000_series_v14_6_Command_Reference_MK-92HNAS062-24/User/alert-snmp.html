alert-snmp

Report/configure alerts by SNMP

Syntax

alert-snmp operation
where 'operation' is one of:
list
set  [-s <i|n>] [-w <i|n>] [-i <i|n>]

Description

Reviews or changes the frequency at which SNMP traps are sent by the server.

Usage:

list

Displays the current frequency.

set [-s <i|n>] [-w <i|n>] [-i <i|n>]
-s

Alters the rate at which severe alerts are sent.

-w

Alters the rate at which warning alerts are sent.

-i

Alters the rate at which information alerts are sent:

i

immediately

n

never

The behavior of SNMP traps is affected by the syslog_and_snmptrap_come_from_admin_vnode variable. When set to false (default), traps will be sent from the cluster node where the trap was generated. When set to true, traps will be sent from an admin vnode IP address. As its name suggests, this variable also affects the sending of syslog alerts.

Examples

server:$ alert-snmp set -s i -w n -i n
server:$ alert-snmp list


Severe alerts:             Immediately
Warning alerts:            Never
Information alerts:        Never
SNMP recipients may be displayed using the snmp-traps list command
