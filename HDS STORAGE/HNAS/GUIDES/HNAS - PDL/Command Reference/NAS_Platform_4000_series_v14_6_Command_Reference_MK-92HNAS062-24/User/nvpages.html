nvpages

List/free NVRAM pages reserved for file systems

Syntax

nvpages <list | free -f <file-system-identifier> >

Description

The nvpages command allows you to query the NVRAM usage within the cluster. It can also be used to free the NVRAM pages used by a specified file system on all nodes within the cluster.

Freeing the NVRAM will result in the loss of up to the last 10 seconds of modifying operations for that file system, and will normally only be done under the direction of your support provider.

Usage:

list

Display the current NVRAM usage.

free -f <file-system-identifier>

Frees the NVRAM pages in use by the specified file system. Specifying a file system that is currently mounted or in the process of mounting will cause the command to fail with an appropriate error message. The file system can be identified by either its label or, if that is not available, the hexadecimal "FS Identifier" displayed in the output of the list option.
