NAS Platform Command Line Reference
(Release 15.0)

The Command Line Interface (CLI) can be accessed in any of the following ways:

  • Using Secure Shell (SSH) to connect into the NAS server through the SMU.
  • Using SSH to directly connect into the NAS server.
  • Using the SSC utility, available for Windows and Linux.

These utilities allow you to access the CLI, which then requires that you log in before issuing commands. Contact Customer Support for the login details.

Refer to the System Access Guide for more information about using the utilities listed above.

Filters

The following table lists "filter" commands which can be used along with some of the NAS server commands listed in the previous table. A filter command is a secondary command added to a NAS server command using the command redirection pipe character (|). The filter command reads the NAS server command's output, performs post-processing on it, then writes the resulting output. Filter commands divide, rearrange, or extract portions of the information that passes through them, and are used to sort, view, and select parts of a NAS server command output.

Command Privilege Level Description
devnull User Swallow all output
grep User Print lines matching a pattern
head User Show just the first part of output
hexdump User Hex dump
less User Pause output every page
mail User Mail command output
more User Pause output every page
nop User Copy input unchanged
sort User Sort lines
sstee User Write to console and file
tail User Show just the last part of output
treacle User Slow output down
txt2html User Minimal text-to-HTML filter
unzip User Decompress
wc User Print the number of newlines, words, and bytes
zip User Compress
