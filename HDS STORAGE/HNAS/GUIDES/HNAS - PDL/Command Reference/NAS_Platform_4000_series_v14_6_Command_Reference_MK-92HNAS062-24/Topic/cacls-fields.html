cacls-fields

Description

A group of cacls commands are used to set or modify Security Descriptor fields and to add or remove ACL elements within the Security Descriptor. The commands have several forms, each of which has its own manual page. These commands are listed below.

  • cacls-add
  • cacls-set
  • cacls-mask-in
  • cacls-mask-out
  • cacls-del

The values and associated meanings that can be applied to the various fields are described below. For fields where it is applicable, a list of abbreviations or alternative keywords is also shown.

ACE mask

An Access Control Entry (ACE) contains an access mask describing the permissions that are allowed or denied depending on the type of the ACE. Most bits in the access mask have the same meaning irrespective of the type of object to which they apply. However, a few bits have different meanings for files and directories.

If an ACE pertains to a file, these bits are:

0x00000001

- Read Data

0x00000002

- Write Data

0x00000004

- Append Data

0x00000020

- Execute

0x00000040

- Not used

If an ACE pertains to a directory, these bits are:

0x00000001

- List Contents

0x00000002

- Create Child

0x00000004

- Create Sub-Directory

0x00000020

- Traverse

0x00000040

- Delete Child

To rename an object in a directory a user must have delete child and create child rights. To unlink an object from a directory a user must have either delete right on the child object or delete child right on its parent directory.

The following bits apply to all file system objects:

0x00000008

- Read Extended Attributes

0x00000010

- Write Extended Attributes

0x00000080

- Read Attributes

0x00000100

- Write Attributes

0x00010000

- Delete

0x00020000

- Read Control

0x00040000

- Write DACL

0x00080000

- Write Owner

0x00100000

- Synchronize

0x01000000

- Access SACL

0x02000000

- Maximum Allowed Right

Owner, Group, and ACE SID

The format for numeric SIDS is S-v-i-a1-...-aN-r

The characters 'S-' identify a numeric SID, 'v' is the version, 'i' is the identifier authority, 'a1'...'aN' are the sub authorities and 'r' is the RID. 'v' is an 8 bit unsigned integer as is 'i'; 'a1...aN' are 32 bit unsigned integers as is 'r'. A numeric SID must have at least two digits, e.g S-1-0. SIDS may also be specified using account names, enclosed in quotes if they contain space characters, e.g. "NT AUTHORITY\Anonymous Logon".

ACL explanatory notes

An Access Control List (ACL) is a list that contains Access Control Entries (ACEs). A Security Descriptor has two ACLs, the SACL and the DACL. The SACL or system ACL controls the generation of audit messages for attempts to access a file system object. The DACL or discretionary ACL describes the access permissions that apply to users and groups accessing the file system object.

ACE ordering

The order that ACEs appear in the ACL is important. When checking a user's access rights, the ACEs are examined in turn from the beginning of the ACL. Once a particular right has been granted by an Access Allowed ACE, it cannot be subsequently denied by an Access Denied ACE. Similarly, once a particular right has been denied by an Access Denied ACE, it cannot be subsequently allowed by an Access Allowed ACE. If, on reaching the end of an ACL, a given right has been neither granted nor denied explicitly, it's denied implicitly except where it's conferred by the security model.
