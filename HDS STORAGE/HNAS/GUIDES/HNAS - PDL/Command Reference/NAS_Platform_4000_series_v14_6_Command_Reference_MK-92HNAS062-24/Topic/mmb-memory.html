mmb-memory

Description

This page describes the mmb-memory trouble reporter.

The mmb-memory trouble reporter alerts the administrator if the system is at risk of running out of physical memory.

The trouble reporter sums the maximum memory that could potentially be used by various system components and compares this to the physical memory available. Should the sum exceed the physical memory on the system, the mmb-memory trouble reporter will report how much the physical memory could be exceeded by and the current memory usage values assigned to the system components considered.

Resolving memory over-commitment requires that the maximum memory usage be reduced. Reducing the Bali heap size will reduce the system's file serving capability. Disabling applications will remove certain system functionality. The combination of Bali heap size reduction and disabling applications should be determined by the system's expected use.

The value for "System overhead" represents memory required for Linux, monitoring tasks and other supporting processes. This value cannot be changed by the administrator.

The resource-config command family allows the information presented in the mmb-memory trouble reporter to be viewed and any relevant configuration altered. Run resource-config-show to examine the current system state.

Examples

$ trouble mmb-memory


system:mmb-memory   (on MMB; base priority 20)


  Memory overcommitted:


    Priority 20: MMB:
    MMB Memory is overcommitted by 963 MB.


      Total system memory is 8 GB.


      System overhead is assigned up to 1.598 GB.


      Bali heap is assigned up to 5.694 GB.


      Deduplication is assigned up to 308 MB.


      Data Migrator to Cloud is assigned up to 1.348 GB.


      Embedded SMU is not enabled.


      To fix: man mmb-memory

The example problem above can be solved by any of the following:

1) Disabling Data Migrator to Cloud.

2) Disabling Deduplication and reducing the Bali heap by 655 MB.

3) Reducing the Bali heap by at least 963 MB.
