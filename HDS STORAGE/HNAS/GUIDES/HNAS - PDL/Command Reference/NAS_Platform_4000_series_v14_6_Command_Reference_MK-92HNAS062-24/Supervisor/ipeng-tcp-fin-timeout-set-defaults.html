ipeng-tcp-fin-timeout-set-defaults

Set TCP FIN timeout configuration to defaults.

Syntax

ipeng-tcp-fin-timeout-set-defaults

Description

The ipeng-tcp-fin-timeout commands configure the expiry timeout applied to TCP connections which remain in the FIN-WAIT-2 state. This is to protect the server from peers which do not close the TCP connection which would otherwise leave connection resources allocated forever. On expiry of the timer the connection is removed.

It is a global setting, acting on all file serving interfaces.

Only use under instruction from your support provider.

The commands available are as follows.

ipeng-tcp-fin-timeout [<timeout>]

If a parameter is given, sets the timeout value, in seconds, for TCP connections which remain in the FIN-WAIT-2 state. The valid range for this parameter is 60 to 3600 seconds.

With no parameter, displays the current and default settings.

ipeng-tcp-fin-timeout-disable

Turns off the timeout of TCP connections in the FIN-WAIT-2 state.

ipeng-tcp-fin-timeout-set-defaults

Sets the timeout to the default value.

Examples

To set the timeout to 90 seconds:

    $ ipeng-tcp-fin-timeout 90

To disable the timeout:

    $ ipeng-tcp-fin-timeout-disable

Reset the configuration to the factory defaults.

    $ ipeng-tcp-fin-timeout-set-defaults

Applies To

Cluster wide
