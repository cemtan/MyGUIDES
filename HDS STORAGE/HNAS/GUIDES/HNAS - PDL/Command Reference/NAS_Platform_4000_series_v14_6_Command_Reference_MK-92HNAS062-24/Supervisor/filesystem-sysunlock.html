filesystem-sysunlock

Take a filesystem out of syslocked mode: make it read-write for protocols

Syntax

filesystem-sysunlock <filesystems-see-'man fs-spec'>

Description

Takes file systems out of syslocked mode, in which they read-only to protocols (such as NFS, CIFS) but read-write to NDMP. After this command has run, the filesystems are read-write to protocols as well as to NDMP.

You can specify one or more file systems using the flexible syntax described in the 'fs-spec' man page.

The status of a syslocked, mounted file system is shown in 'filesystem-list' and 'span-list -v' as 'SysLk'.

Examples

filesystem-sysunlock Accounts

Applies To

Cluster wide

See Also
