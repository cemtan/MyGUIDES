replication-start

Perform object replication

Syntax

replication-start [--test|--detailed-test] [--output-stats] {<policy id> | <policy name> <source fs label>}

Description

Start an object replication.

[-t | --test]

Perform a quick object replication test. This tests whether the target server and replication target file system are ready for the object replication to begin and that there's a suitable baseline snapshot from which to perform the object replication.

[-d | --detailed-test]

Perform an object replication detailed test. This performs the same checks as a quick test, but also calculates the amount of data that will be transferred, which is useful for determining how long the object replication is likely to take. A detailed test can itself take many minutes to run, so its progress can be tracked with the 'replication-show-active' and 'replication-history-list' commands.

[--output-stats]

Outputs statistics on replication performance after the replication finishes.

<policy id>

The ID of the policy to run.

<policy name>

The name of the policy to run.

<source fs label>

The source file system to which the policy applies. This is required to identify the policy unambiguously.

Examples

replication-start policy0 span0fs

Starts an object replication as per policy 'policy0' on file system 'span0fs'.

Applies To

EVS
