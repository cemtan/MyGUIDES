devinfo

Display info on a specified device

Syntax

devinfo [-s <SI>] { -a | -e | -r | [-v] <DEVICE> }

Description

devinfo displays information from device descriptor structures in SI control memory.

-a --all

Display summary information on numbers of outstanding and queued requests for all devices that are enabled and/or have outstanding I/O.

-e --enabled

Same as -a but only devices that are enabled.

-r --reset

Reset the maximum counts for all devices.

-s --si

Select which SI to query on systems that have more than one. <SI> is 0-based. By default, the single device form will use the one currently mapped for the device, while -a, -e and -r will be applied to all SIs.

-v --verbose

Dump full descriptor, including the templates for SEST entries, IRBs and FCP frames.

<DEVICE>

Dump detailed information for the specified device number.

The columns of the table shown for -a or -e are as follows:

Device Number, En/Dis

The first column is the device number, as given by sd-list or backup-devices-list. With -a, this is followed by a letter E or D, indicating whether I/O to the device is enabled or disabled. (This is not shown with -e, which displays only the devices that are enabled anyway.) Broadly speaking, a device is usually enabled if it is open, but I/O to a device may need to be disabled temporarily for various reasons.

Disk Requests
Current

The number of reads (R) or writes (W) that have been sent to the storage and for which SI is still waiting for a response. "All" represents the sum of these two. (However, there is a small chance that SI will update these three statistics at the same time as the command is reading them.)

Max

The maximum of the "All" value since the last reset.

Queued Reads & Queued Writes

Queued requests are those queued internally in SI, waiting to be sent out to the storage. These all have both "Cur" (current) and "Max" values. Reads are further divided into "Normal" and "Dedupe"; writes are divided into "Normal" and "High-Priority". High-priority writes are used for certain inquiry tag numbers to help checkpoints complete quickly. Writes being retried are also given high priority.

Reads to Queue

This is an SI-internal bookkeeping value used to maintain an even balance between reads and writes (when both reads and normal-priority writes are queued and could be sent). It represents the number of read requests that SI will issue before the next write.

The detailed output includes some additional fields, most of them of interest only for debugging purposes.

Applies To

Cluster node
