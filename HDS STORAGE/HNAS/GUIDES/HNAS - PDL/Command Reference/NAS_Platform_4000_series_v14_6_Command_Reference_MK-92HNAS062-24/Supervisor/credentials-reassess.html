credentials-reassess

reassess credentials after changes to mappings/local group/primary group tables

Syntax

credentials-reassess

Description

A background task periodically reassesses credentials to reflect changes to the user mappings, group mappings, local group and primary group tables.

The credentials-reassess command can be used to reassess all credentials in the foreground. You may choose to run this command after making changes to one of the mapping tables to force credentials reassessment to happen immediately, rather than wait for the next background reassessment.

Usage:

credentials-reassess

Examples

$: credentials-reassess

Applies To

EVS
