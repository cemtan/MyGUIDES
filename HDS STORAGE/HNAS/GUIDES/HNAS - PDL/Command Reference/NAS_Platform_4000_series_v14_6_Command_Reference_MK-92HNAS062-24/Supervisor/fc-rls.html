fc-rls

Read link status blocks

Syntax

fc-rls [hport id]

Description

The fc-rls command allows you to query the "Link Error Status Block" of remote ports. It uses the fibre channel command "Read Link Error Status Block" (RLS). With a fabric switch topology the command reports the statistics at both the Nx and Fx ports at each end of the link. The use of this command allows you to track down potentially faulty or intermittent connections. Note the counts always increase - there is no way in the fibre channel specification to reset the counts. The exception to this rule is the output of the statistics for the local host ports - these counts may be reset by using the fc-link-errors command. Note not all devices support RLS - in these cases fc-rls will indicate that it failed to recover the information. Command options are:

[hport id]

If specified, only reports devices visible on the specified host port. By default all devices on all (enabled) host ports are queried.

Examples

Query the link statistics on all interfaces:

fc-rls

Query the link statistics on host port 1:

fc-rls 1

Applies To

Cluster node

See Also

fc-link-errors fc-link-status

Supervisor
