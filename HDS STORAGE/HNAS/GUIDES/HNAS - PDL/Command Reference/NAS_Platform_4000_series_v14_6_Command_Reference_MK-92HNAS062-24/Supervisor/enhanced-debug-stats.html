enhanced-debug-stats

Control gathering of enhanced debugging information

Syntax

enhanced-debug-stats [--enable | --disable] [--not-persistent]

Description

Enables/disables gathering of enhanced debugging information.

The maintenance of some debugging information can cause a decrease in system performance. Nevertheless the gathering of this information is enabled by default. The overhead is estimated to be in the order of a few percent. This information is important for problem diagnosis and so should be left enabled during normal operation, unless an extra one or two percent performance is absolutely required.

The setting affects all nodes in a cluster.

Options:

--enable

Enable the gathering of enhanced debugging information.

--disable

Disable the gathering of enhanced debugging information.

--not-persistent

By default, the setting is persistent across reboots. This switch prevents the setting from being saved.

NB. the --not-persistent option applies the changes to the selected pnode only. Use "for-each-cnode " prefix to broadcast non-persistent changes

Examples

  server:$ enhanced-debug-stats

Queries the current enhanced-debug-stats setting

  server:$ enhanced-debug-stats --enable

Enables enhanced-debug-stats gathering

  server:$ enhanced-debug-stats --enable --not-persistent

Enhanced-debug-stats are enabled, but without changing the persistent config. This command only applies to the current pnode

  server:$ for-each-cnode enhanced-debug-stats --enable --not-persistent

Enhanced-debug-stats are enabled, with no persistent config change, as the "for-each-cnode" command is added before the base command this change is broadcast to all pnodes.

  server:$ for-each-cnode enhanced-debug-stats

This will display the enhanced-debug-stats status on all pnodes on the current pnode.

Applies To

Cluster wide Unless --not-persistent is specified, in which case only the current cluster node will be affected
