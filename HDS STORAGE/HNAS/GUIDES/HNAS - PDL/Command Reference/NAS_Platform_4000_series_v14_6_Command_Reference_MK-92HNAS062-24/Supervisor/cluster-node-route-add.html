cluster-node-route-add

Add a cluster-node-route configuration entry

Syntax

cluster-node-route-add host|net|gateway <addr> [-n netmask] -g <gateway-addr> [-s mtu] -p pnode-id

Description

The cluster-node-route-add command adds a new host, network or gateway route entry that is active only on a specific cluster node.

To add routes:

cluster-node-route-add host | net <addr> [ -n <mask> ] -g <addr> [-s mtu] -p pnode-id


cluster-node-route-add gateway -g <addr> [-s mtu] -p pnode-id

host|net|gateway

Specify the type of route, i.e a single host, a network or a gateway. If a network is specified, then you must include a Subnet mask ( -n <addr> ). For a host, this value will default to 255.255.255.255.

<addr>

Means an IP address format such as 10.1.2.3. There is some checking for invalid addresses. Invalid addresses are: Numbers > 255, Broadcast Addresses, Unreachable gateways.

-n <addr>

Netmask. Only necessary for "net" type routes. An <addr> must be supplied.

-g <addr>

Gateway. Must be supplied for any route. An <addr> must be supplied.

-s <mtu>

The MTU to be used for this route.

-p <pnode>

The pnode on which this cluster-node-route will be loaded.

Examples

To add a host cluster-node-route to get to 10.1.2.3 via 10.1.3.4 for pnode 2:

cluster-node-route-add host 10.1.2.3 -g 10.1.3.4 -p 2

To add a network cluster-node-route for the 10.1.2.0 network via 10.1.3.4 for pnode 1:

cluster-node-route-add net 10.1.2.0 -n 255.255.255.0 -g 10.1.3.4 -p 1

To add a gateway cluster-node-route for pnode 7:

cluster-node-route-add gateway -g 10.1.254.50 -p 7

Applies To

Cluster node
