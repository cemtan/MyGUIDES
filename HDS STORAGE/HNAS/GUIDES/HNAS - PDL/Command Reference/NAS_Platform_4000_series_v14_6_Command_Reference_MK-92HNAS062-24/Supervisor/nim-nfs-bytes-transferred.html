nim-nfs-bytes-transferred

Show nim nfs bytes transferred performance statistics

Syntax

nim-nfs-bytes-transferred [-s|--summary] [-f|--file-system <file_system|*|+>] [-o|--operation <wfs_rw_operation|*|+>]

Description

This man page describes a class of commands that show certain per-X (for example, per protocol) statistics. Protocol operations include all requests received by the node whether they are processed in software, hardware or CNS redirected to another node. Incoming CNS redirected operations (received via the ICC link) are not included. The 4 commands listed below show the currently available commands, more are expected to be added in the future.

nim-nfs-perf

Shows response times of individual NFS operations (NFSv2/3 only) and compound NFS operations (NFSv4 only)

nim-nfs-bytes-transferred

Shows bytes transferred per NFS operation

nim-cifs-perf

Shows response times of individual CIFS operations (SMB1/2) and compound CIFS operations (SMB2 only)

nim-cifs-bytes-transferred

Shows bytes transferred per CIFS operation

Supported options:

--summary, -s

Show summary statistics (i.e. don't show histograms)

Supported "per-X" options:

--file-system, -f <file system>

Select file system. Either name or volume number may be used.

--operation, -o <operation>

Select a specific operation. This has to be a numeric opcode.

For the "per-X" options, use * to show statistics for all of those entities. Use + to show a summary for those entities.

The default value for each of these options will be "*" for the most important aspect, and "+" for the others, i.e. roll up most of the "per-X" parameters and detail the most significant one.

Examples

Show NFS performance for all operations

server:$ nim-nfs-perf -o +
