tls-certificate-create-custom

Create a new certificate and private key using the supplied values

Syntax

tls-certificate-create-custom [--key-validity <key-validity>] [--key-size <key-size>] [--organizational-unit <organizational-unit>] [--organization <organization>] [--location <location>] [--state <state>] [--country <country>] [--hostname <hostname>] [--confirm]

Description

The tls-certificate-create-custom command is used to create a custom self-signed certificate and private key pair. It will remove all existing certificates and trust chains installed.

Warning:

Running tls-certificate-create-custom will automatically cause the HTTPS server used for server management to restart. Any clients currently connected to it will lose their connection and will need to reconnect.

This forces clients to connect using the new SSL/TLS certificate.

Options:

--help
--confirm

Specify this to confirm that you want changes to be made. If you do not specify it, no new SSL/TLS certificate will be created or installed.

--key-validity, -v

How many days the certificate will be valid for. Note the validity start date will be 1 day prior to the creation date to allow for date/time discrepancies.

--key-size, -k

The size of the private key. 2048 is the default and recommended value.

--organizational-unit, -u
--organization, -o
--location, -l
--state, -s
--country, -c
--hostname, -r

Allows specifying the hostname you wish to generate a certificate for. It is recommended to not specify this value as the default will be the fully qualified domain name of the machine it is run on.

Examples

tls-certificate-create-custom --confirm

Create a new certificate and private key with the default values.

Applies To

Cluster wide
