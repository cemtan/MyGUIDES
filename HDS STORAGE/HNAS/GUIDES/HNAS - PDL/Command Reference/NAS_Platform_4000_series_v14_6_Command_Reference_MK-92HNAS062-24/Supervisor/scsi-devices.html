scsi-devices

Display information about SCSI devices

Syntax

scsi-devices [[device id | [-d, --device <ID>] | [<system-drives, see 'man sd-spec'>] [options]
  where options are
    -a,  --all
    -c,  --clear
    -e,  --ext
    -i,  --ignore
    -l,  --luns
    -m,  --mirror
    -p,  --port <id>
    -P   --hdp
    -o,  --order
    -r,  --paddr
    -R,  --restricted
    -v,  --verb

Description

Display information about all SCSI devices the server has discovered since boot.

If a device has multiple LUNs, summary information will show the current connected LUN postfixed with '+'. Without any options it will show summary information on all connected devices, sorted in (type, vendor, revision, device id) order.

The summary information "Int LUN" column (Internal LUN/LDEV), if shown, is the LUN that is used internally by the storage array for this device. This value may be displayed in hex or decimal and should match the value displayed in the storage management GUI.

If a device status is shown as secondary, it is the secondary of a mirrored pair.

Notes:

Preferred path refers to the Load Balancer's preferred path. This indicates whether the nexus is the preferred path for the LUN. This will only be displayed in verbose mode and if the storage device supports it.

The sd-spec syntax can only be used to specify system drives that are known by the server.

Options:

[system-drives list]

See the 'sd-spec' man page for an explanation of how to specify a list of system drives using sd-spec syntax.

[device id]

Print verbose information about the system drive with the given device id.

-d, --device <device ID>

Print verbose information about the device with the given device id. This option is not parsed by sd-spec and should be used for devices that are not system drives.

-a, --all

Show all LUNs/devices, including those that are disconnected/removed.

-c, --clear

Clear device error information before displaying.

-e, --ext

Show extended information (verbose format without error information).

-i, --ignore

The verbose option displays all nexus for the device. If LUNs have been re-mapped, nexus' associated with the old LUN will be displayed with "LUN not known/removed" status. Use the ignore flag to avoid displaying these nexus.

-l, --luns

For summary information, show all connected LUNs if device has multiple LUNs.

-m, --mirror

For summary information, show the mirror status of each system drive.

-o, --order

List devices in ascending Device ID order (default is vendor/(system id)/revision/device id order).

-p, --port <id>

Show verbose information about all devices that have a nexus with the given SCSI port id.

-P, --hdp

For summary information, show the HDP pool number if the SD is a member of an HDP pool. If the pool is using compression, the label "(Compr)" will also be shown. The verbose output will show the HDP pool number in the "Info" line.

-r, --paddr

For summary information, show Host port number, port id and online status of the devices' ports, as Hport:Port:Addrs.

-R, --restricted

This option is primarily used to constrain the amount of output that is generated by scsi-devices -v in the showall.txt file when it is called as part of a diagnostics. This option will attempt to display the devices that are of interest, like devices that have failed or have a failed nexus and will display a limited number of devices from each storage array. Usually only the first 4 devices will be listed from each storage array if all devices are online and there are no issues.

-s, --pstat

For summary information, show port id and online status for each of the devices' ports.

-v, --verb

Show verbose information about all devices.

Examples

To show extended device information:

scsi-devices -e

To show all LU LUNs including those that are no longer supported because the LU was mapped to a new LUN:

scsi-devices -al

Show verbose information about all devices:

scsi-devices -v

Show verbose information about all devices on SCSI port 3:

scsi-devices -vp 3

Show verbose information about device 2:

scsi-devices 2

Show system drives in the range 0 to 4 and 7

scsi-devices 0-4,7

Applies To

Cluster node

See Also

scsi-ports scsi-racks

Privilege Level

Supervisor
