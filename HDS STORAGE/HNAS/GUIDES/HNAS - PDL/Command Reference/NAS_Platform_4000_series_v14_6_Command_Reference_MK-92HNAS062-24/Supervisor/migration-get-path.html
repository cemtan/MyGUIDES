migration-get-path

Displays the migration destination for the specified vivol/file system.

Syntax

migration-get-path -s <fs label> [-v <vivol name>]

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol name>

Name of source Virtual Volume (optional).

Displays information about a Data Migration path. This includes its target file system(s) or remote storage path(s) and current status which is one of the following:

Undefined

This return value should never be seen as the path is defined.

No migrated files

The path has no outstanding migrated files. (It may be deleted)

Files have been migrated

There are migrated files at the destination.

Files have been migrated (via file system path)

For a Virtual Volume - there are files that have been migrated via the file system level migration path.

Active migration in progress

A migration process is in progress.

Migration target deleted/reformatted

The target is no longer available. It has been removed or has been reformatted.

Examples

migration-get-path -s vol1

Display details and status of the migration path for file system vol1.

migration-get-path -s Accounts -v HR

Display details and status of the migration path for Virtual Volume HR on file system Accounts.
