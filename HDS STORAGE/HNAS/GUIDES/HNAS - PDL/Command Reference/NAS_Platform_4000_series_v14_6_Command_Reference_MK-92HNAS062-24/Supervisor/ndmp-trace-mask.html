ndmp-trace-mask

Set new NDMP trace mask. (ALL/ON/OFF/NONE/<number>/0x<hex>)

Syntax

ndmp-trace-mask [ all | on | off | none | 0x<hexnum> | <number> ]

Description

This command should only be used under instruction from your Support Organization. The command can be used to cause the NDMP code to write trace information to the diagnostic log on the FSM. The trace output can be extracted by generating a diagnostic E-mail.

Always switch trace off using "ndmp-trace-mask off" after the relevant trace has been gathered.

Switching on tracing may cause a large amount of detail to be written to the log resulting in significant performance degradation. In particular the "all" setting should only be used to diagnose basic configuration and NDMP transfer start problems.

The trace mask is set back to "off" whenever the server is rebooted.

The settings form a bit mask indicating which particular trace points will produce output. The bit settings may be combined. If specified in hexadecimal the parameter must be prefixed by "0x".

Descriptive parameter settings:

off (= 0)          No trace produced.
on  (= 0xfffff)    Standard trace level for all functions.
all (= 0xffffffff) All trace points enabled.

Bit setting that may be used in combination:

0x00000001  NDMP requests.
0x00000002  Tape device.
0x00000004  Media changer.
0x00000008  Transfer of backup data to/from tape.
0x00000010  Production/interpretion of backup image.
0x00000020  Control of NDMP transfers.
0x00000040  NDMP configuration/management.
0x00000080  NDMP process scheduling.
0x00000100  File history (backup catalog).
0x00000200  Tape/media changer device naming/management.
0x00000400  NDMP request splitting.
0x00000800  NDMP notifications.
0x00001000  Recovery name lists.
0x00002000  Direct access recovery.
0x00004000  Checkpoint/restart.
0x00008000  NDMP file system management.
0x00010000  Snapshot restore list.


0x80000000  Recovery name lists - detailed info.
0x40000000  Tape transfer interface - details.
0x20000000  NDMP request splitting - details.
0x10000000  Production/interpretion of backup image - details.
0x08000000  Device interface details.
0x04000000  NDMP process scheduling details.
0x02000000  Intra-cluster relay for CNS access.
0x01000000  Incremental comparison.

Examples

Turn off tracing:

ndmp-trace-mask off

Turn on standard tracing:

ndmp-trace-mask on

Trace NDMP interchanges with the NDMP client:

ndmp-trace-mask 0x801

Detailed trace of all tape commands:

ndmp-trace-mask 0x4800020A

Applies To

Cluster node
