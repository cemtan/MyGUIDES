domain-mappings-replace

Replaces an entry from the domain mapping table

Syntax

domain-mappings-replace < --map-unix-names | --nt <nt domain> | --nfsv4 <NFSv4 domain> | --kerberos <kerberos domain> >[--set-map-unix-names] [--set-nt <nt domain>] [--set-nfsv4 <NFSv4 domain>] [--set-kerberos <kerberos realm>]

Description

Replaces an existing entry in the domain mappings table.

Note that no values in the existing entry persist across the replace operation, they must be explictly specified.

Options:

--map-unix-names

Look up the entry that is set to map to/from Unix names

--nt

Look up the entry that maps to/from the NT domain

--nfsv4

Look up the entry that maps to/from the NFSv4 domain

--kerberos

Look up the entry that maps to/from the Kerberos realm

--set-map-unix-names

Use this mapping entry to map to/from Unix names

--set-nt

Sets the NT domain name to map to/from

--set-nfsv4

Sets the NFSv4 domain name to map to/from

--set-kerberos

Sets the Kerberos domain name to map to/from

To use the command, look up the existing entry to replace by any one of its fields and specify all values for the updated entry.

Examples

$: # Removes the NFSv4 domain as it is not explicitly set and replaces the
$: # kerberos realm for the entry matching NT domain EXAMPLE
$: domain-mappings-list


Unix     NT domain     NFSv4 domain     Kerberos realm
----     ---------     ------------     --------------
   *     EXAMPLE       example.com      example


$: domain-mappings-replace --nt EXAMPLE --set-map-unix-names --set-nt EXAMPLE --set-kerberos newrealm
$: domain-mappings-list


Unix     NT domain     NFSv4 domain     Kerberos realm
----     ---------     ------------     --------------
   *     EXAMPLE                        newrealm


$:

Applies To

EVS security context

Supervisor
