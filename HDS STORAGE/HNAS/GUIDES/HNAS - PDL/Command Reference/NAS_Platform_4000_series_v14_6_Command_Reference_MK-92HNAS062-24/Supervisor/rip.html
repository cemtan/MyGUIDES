rip

Configure/Display RIP listener

Syntax

rip [show] | listen | update | flush | stop | block | unblock

Description

The 'rip' command enables listening for Routing Information Protocol (RIP) route advertisements and, optionally, adding the route entries from those advertisements to the routing table. This reduces the need for manually configured network route entries.

Without any options, the 'rip' command displays details of the RIP configuration.

Operations

show

Display the RIP configuration and the discovered routes.

Note: Discovered routes are only displayed for the cluster node on which the command is executed.

listen

Configure RIP to listen for route advertisements without adding them to the routing table.

update

Configure RIP to listen for route advertisements and update the routing table with the RIP routes.

flush

Flush all previous learned routes and remove them from the routing table.

Note: This operation only applies to the cluster node on which the command is executed.

stop

Configure RIP to stop listening for route advertisements and remove any learned routes from the routing table.

block source <addr>

Block all route advertisements received from a particular source address.

block network <addr> <mask> <gateway>

Block a specific advertised network route via a specific gateway.

unblock source <addr>

Remove a block on receiving route advertisements from a particular source address.

unblock network <addr> <mask> <gateway>

Remove a block for a specific advertised network route via a specific gateway.

Examples

To listen to routes advertised via RIP:

rip listen

To enter all learned RIP routes into the routing table:

rip update

To block all route advertisements from a particular source:

rip block source 10.1.2.3

To block a specific advertised network route via a specific gateway:

rip block network 10.2.0.0 255.255.0.0 10.1.2.3

Applies To

Cluster wide
