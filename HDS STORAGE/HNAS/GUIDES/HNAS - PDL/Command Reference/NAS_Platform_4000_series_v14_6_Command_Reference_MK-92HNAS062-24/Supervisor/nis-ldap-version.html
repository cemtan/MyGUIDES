nis-ldap-version

Displays or changes server's LDAP client protocol version.

Syntax

nis-ldap-version [ -v ldapv3 | ldapv2 ]

Description

The nis-ldap-version command is used to view or change the current EVS security context's LDAP client protocol version. This is the protocol variant which the NAS server uses when contacting LDAP-based NIS servers (RFC-2307 or similar).

Normally the default of LDAP version 3 should be fine, as most modern servers use it. If your environment includes LDAP version 2 servers, this command may be used to configure the NAS server to communicate with them using LDAP version 2.

Note that this setting only applies to non-TLS LDAP connections. When using LDAP-over-TLS, the NAS server always uses LDAP version 3.

Changes made with this command are saved to the EVS context's configuration immediately, but do not affect existing LDAP connections. In the case where the previous LDAP version setting was preventing an LDAP connection, after changing the LDAP version the NAS server should reconnect in a few moments. The ldap-server prod command may be used to accelerate this process.

Examples

Display the current EVS' LDAP protocol version:

nis-ldap-version

Set the server's LDAP protocol version to 3 (the default):

nis-ldap-version -v ldapv3

Set the server's LDAP protocol version to 2:

nis-ldap-version -v ldapv2

Applies To

EVS security context

See Also
