filesystem-reveal

Reveal a filesystem -- undo 'filesystem-hide'

Syntax

filesystem-reveal <span-or-filesystem-instance-name>

Description

The 'filesystem-hide' command prepares a filesystem for span- or storage-level maintenance by making it unavailable for checking, fixing, mounting and formatting. 'filesystem-reveal' reverses this action. Use it after storage- or span-level maintenance is complete.

'filesystem-reveal' sends the Filesystem a message, indicating that the specified file system can now be mounted (or auto-mounted), but it returns control quickly, without waiting for the message to be processed. A script using this command may need follow it with 'span-wait-for-loading --mount' to ensure that the file system is ready for use before the next command is run. If you wish to determine whether the notification has been processed without waiting for it to complete, run 'filesystem-list-queues' and check for the filesystem in the 'Availability' queue. Typically, when you are working interactively, the message is processed fast enough to render a manual check unnecessary.

It is also possible to hide an entire span. If you hide both a filesystem and its host span, you'll need to reveal them both (in either order) before any Filesystem I/O is possible.

Hiding spans and filesystems doesn't prevent them from being shown by 'span-list' and 'filesystem-list'. If either a filesystem or its host span is hidden, 'filesystem-list -a' will display the filesystem as 'Hiddn'; 'span-list', 'filesystem-list' and 'trouble' will print warnings.

Examples

filesystem-reveal Accounts

Applies To

Cluster wide
