ignore-unix-security-ops-show

Configures whether the server ignores NFS operations that attempt to set or change file/directory security attributes

Syntax

ignore-unix-security-ops-show [[-f|--filesystem] <filesystem> [[-i|--vivol] <ViVol name> ]]

Description

The ignore-unix-security-ops-* commands configure under what conditions a server ignores NFS operations that attempt to set or change file/directory security attributes.

Commands

There are 5 commands in this set:

  • ignore-unix-security-ops-show [[-f|--filesystem] <filesystem> [[-i|--vivol] <vivol name> ]]
  • ignore-unix-security-ops-disable [[-f|--filesystem] <filesystem> [[-i|--vivol] <vivol name> ]]
  • ignore-unix-security-ops-enable [[-f|--filesystem] <filesystem> [[-i|--vivol] <vivol name> ]]
  • ignore-unix-security-ops-clear --filesystem <filesystem> [--vivol <vivol>]
  • ignore-unix-security-ops-mode [evs-override-mode|fs-vivol-mode]

Behaviour

This series of commands subsume and extend the older command ignore-unix-security-ops.

The "ignore" behaviour can be configured to work in one of two modes:

  • evs-override-mode (this was the way in which the original ignore-unix-security-ops worked)
  • fs-vivol-mode (this is the new mode)
server: $ ignore-unix-security-ops-mode fs-vivol-mode
 ignore-unix-security-ops-mode: fs-vivol-mode (EVS level ignore setting will now act as a default for all ViVols/filesystems)
 server:$ ignore-unix-security-ops-mode evs-override-mode
 ignore-unix-security-ops-mode: evs-override-mode (EVS level ignore setting will now override all ViVols/filesystems settings)

When "ignore" is enabled, files/directories created over NFS (v2,v3 & v4) will inherit security from the parent directory, or it will be derived from the server's file/directory umask if the parent directory has no inheritable security. If an NFS client attempts to change the security of an existing file/directory (e.g. chmod/chown/chgrp) then the operation succeeds without making the change requested. Changes made by other protocols, such as SMB are unaffected.

This can be used to ensure (remote) security changes are only made via e.g. the SMB protocols.

Setting the mode to fs-vivol-mode will have a small impact on the speed of the performance of security operations.

Examples

server:$ ignore-unix-security-ops-show  -f Span0FS
 ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
 ignore-unix-security-ops: filesystem level [filesystem=Span0FS] is clear
 ignore-unix-security-ops: Effective setting  [filesystem=Span0FS] is disabled (inherited from evs level)


server:$ ignore-unix-security-ops-clear  -f Span0FS -i vv1
 ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
 ignore-unix-security-ops: filesystem level [filesystem=Span0FS] is clear
 ignore-unix-security-ops: vivol level      [filesystem=Span0FS, vivol=vv1] is clear
 ignore-unix-security-ops: Effective setting  [filesystem=Span0FS, vivol=vv1 ] is disabled (inherited from evs level)

Most of the extended behaviour applies to the new mode (fs-vivol-mode) if that is not chosen, then behaviour is largely unchanged. You may choose to use the newer ignore-unix-security-ops-disable and ignore-unix-security-ops-enable in place of ignore-unix-security-ops --disable/--enable but there will be no change in behaviour. You can always revert to the old behaviour by selecting the evs-override-mode.

The "ignore" behaviour can be selected at various levels of granularity:

  • The entire cluster
  • A single EVS
  • A single filesystem
  • A single virtual volume (vivol)

If the mode evs-override-mode is enabled, then ignore setting can be set on an EVS:

  server:$ ignore-unix-security-ops-disable
  ignore-unix-security-ops: EVS level is disabled (evs-override-mode: EVS level setting will override all vivols/filesystems settings)

This will either affect a single EVS or the entire cluster. See evs-security multi-tenancy.

If the fs-vivol-mode is in effect, settings can be made in a more fine grained manner:

  server:$ ignore-unix-security-ops-enable -f Span0FS
  ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
  ignore-unix-security-ops: filesystem level [filesystem=Span0FS] is enabled
  ignore-unix-security-ops: Effective setting  [filesystem=Span0FS] is enabled (explicitly set)

Now files on the filesystem Span0FS will have "ignore" set to enabled, this includes any vivols on that filesystem. Other filesystems on this EVS will have it set to disabled.

This can be further refined to the vivol level:

  server:$ ignore-unix-security-ops-disable  -f Span0FS  -i vv1
  ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
  ignore-unix-security-ops: filesystem level   [filesystem=Span0FS] is enabled
  ignore-unix-security-ops: vivol level        [filesystem=Span0FS, vivol=vv1] is disabled
  ignore-unix-security-ops: Effective setting  [filesystem=Span0FS, vivol=vv1 ] is disabled (explicitly set)

Now the vivol vv1 has a setting of disabled, all other files on Span0FS (including other vivols) have a setting of enabled while all other files on the EVS have a setting of disabled.

In this mode (fs-vivol-mode) there is an inheritance from EVS to filesystem to vivol. Either/both filesystem and vivol level settings can be clear, so they simply inherit from the level above. This allows for simpler configurations:

  server:$ ignore-unix-security-ops-clear  -f Span0FS  -i vv1
  ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
  ignore-unix-security-ops: filesystem level [filesystem=Span0FS] is enabled
  ignore-unix-security-ops: vivol level      [filesystem=Span0FS, vivol=vv1] is clear
  ignore-unix-security-ops: Effective setting  [filesystem=Span0FS, vivol=vv1 ] is enabled (inherited from filesystem level)

and:

  server:$ ignore-unix-security-ops-clear  -f Span0FS
  ignore-unix-security-ops: EVS level is disabled (fs-vivol-mode: EVS level setting acts as a default for all vivols/filesystems)
  ignore-unix-security-ops: filesystem level [filesystem=Span0FS] is clear
  ignore-unix-security-ops: Effective setting  [filesystem=Span0FS] is disabled (inherited from evs level)

The defaults are:

  • ignore-unix-security-op-mode = evs-override-mode
  • ignore-unix-security-op (EVS) = disabled
  • ignore-unix-security-op (filesystem) = clear
  • ignore-unix-security-op (vivol) = clear
