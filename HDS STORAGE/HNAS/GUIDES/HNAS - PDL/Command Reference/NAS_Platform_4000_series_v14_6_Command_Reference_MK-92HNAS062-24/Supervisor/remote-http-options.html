remote-http-options

Configure options for the external migration HTTP client

Syntax

remote-http-options [--persistent-connection-timeout <seconds>]

Description

Configures options for the external migration HTTP client.

Usage:

--persistent-connection-timeout, -p <seconds>

Set the HTTP persistent connection timeout to <seconds>. A value of 0 will reset the timeout to the default.

The remote-http-options command allows the administrator to configure the options for the external migration HTTP client for the current EVS.

When migrating files to a Hitachi Content Platform (HCP) version 3.0 or newer, best performance will achieved if the HTTP persistent connection timeout is set to the same value on the server and the HCP.

Examples

To set the HTTP persistent connection timeout to 2 minutes:

$ remote-http-options --persistent-connection-timeout 120

To return the HTTP persistent connection timeout to the default value:

$ remote-http-options --persistent-connection-timeout 0

To show the current configuration:

$ remote-http-options

Applies To

EVS
