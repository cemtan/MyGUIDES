xvl

Create an external volume link

Syntax

xvl [ --utc-value <utc> ] [ --file-length <length> ] <ipaddr> <cifs:nfs> <export> <path> [<http_target> <target_path> ]

Description

This command checks if a file is an external volume link (XVL) or, if an appropriate remote HTTP target is specified, converts a file to an XVL.

Usage:

<ipaddr> IP address of the EVS containing the file system

<cifs:nfs> Protocol in which the file system is exported to hosts

<export> Name of the NFS export or CIFS share of the file system

<path> Path to the file in the file system

If only the above options are specified, the command returns a message indicating whether the specified file is an XVL or not.

If the following two parameters are also specified, the command converts the file to an XVL, or changes the path of the XVL if it already one:

<http_target> HTTP address of the archive server, which must have been previously specified using the migration-add-external-path command

<target_path> Path to the file on the archive server, which will be postpended to the value of <http_target> in the GET request to the archive server if the XVL is accessed by a client

The following two options may be specified if converting a file to an XVL, which can help protect against stubbing a file that has been changed while in the process of being copied to an archive server:

--utc-value <utc> UTC modification time of the file; this must match the value of the modification time in the file system, otherwise the command will fail

--file-length <length> Length in bytes of the file; this must match the length of the file in the file system, otherwise the command will fail

Examples

If file system testfs hosted on an EVS with IP address 10.3.22.5 and is exported via CIFS as share testshare and contains file dir01/file01.txt to be stubbed, and the file was archived to path archive/dir01/file01.txt on archive server 10.4.5.6, then run

xvl 10.3.22.5 cifs testshare dir01/file01.txt 10.4.5.6 archive/dir01/file01.txt

Applies To

EVS

See Also

migration-add-external-path
