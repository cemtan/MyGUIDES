cluster-gefn-status

Status of GEfN cluster

Syntax

cluster-gefn-status

Description

Reports whether a 4 node cluster has been configured as a GAD Enhanced for NAS (GEfN).

GEfN Cluster

The GEfN Cluster status will have one of the following states:

Configured

If the cluster has been configured as a GEfN

Not Configured

If the cluster is a standard, non-site resilient cluster. This is the default state of a 4 node cluster if the cluster-gefn-configure command has never been run. The following additional contents will not display if the cluster has not been configured as a GEfN.

Mirror Type

The Mirror Type status will have one of the following states:

Remote

A standard GEfN with nvram mirroring that makes it resilient across 2 sites. This assumes that nodes 1 and 2 are in the first site and nodes 3 and 4 are in the second site.

Note that "Remote" is the only mirror type that is available at this time. Other configurations may be introduced in the future.

Port Config

The Port Config status describes the configuration of the InterCluster Connect (ICC) ports and will have one of the following states:

Full

The GEfN uses all the available ICC ports (c1 to c4)

Reduced

The GEfN uses only the minimally required ICC ports (c1 and c3).

For more information about these options, see the cluster-gefn-configure man page.

Examples

1. A 4 node HNAS 5000 cluster GEfN using only c1 and c3 ports (reduced):

server-1:$ cluster-gefn-status
GEfN cluster: Configured
Mirror Type : Remote
Port Config : Reduced

2. A default HNAS 5000 cluster that has not been configured as a GEfN:

server-1:$ cluster-gefn-status
GEfN cluster: Not configured

3. A 4 node HNAS 5000 cluster GEfN using all available ICC ports (full):

server-1:$ cluster-gefn-status
GEfN cluster: Configured
Mirror Type : Remote
Port Config : Full

Applies To

Cluster wide
