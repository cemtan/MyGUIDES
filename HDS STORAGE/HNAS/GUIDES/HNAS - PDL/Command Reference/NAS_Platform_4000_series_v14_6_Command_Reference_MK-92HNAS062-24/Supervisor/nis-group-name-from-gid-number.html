nis-group-name-from-gid-number

test nisclient::api::GroupFromGid

Syntax

nis-group-name-from-gid-number <gid>
Alias:       nis-group-from-gid
Alias:       NisGroupFromGid

Description

The nis-uid-number-from-user-name, nis-user-name-from-uid-number, nis-gid-number-from-group-name and nis-group-name-from-gid-number commands are used to check translations from names and groups to user IDs and group IDs respectively and vice-versa.

The RFC 2307 LDAP schema uses "uid" to mean "user name" and "cn" to mean "group name". Generally, in this product, "uid" and "gid" are used to mean "uid number" and "gid number", as can be seen in the output from these commands.

At most one matching entry is returned. A status of OK_NOT_FOUND means that the directory services server had no matching entries. Failure to contact the directory services server is typically reported as NO_BOUND_SERVER.

Examples

server:$ nis-user-name-from-uid-number 9000
nisclient::api::NameFromUid() failed, status = NO_BOUND_SERVER
server:$ nis-user-name-from-uid-number 90000
nisclient::api::NameFromUid() failed, status = OK_NOT_FOUND
server:$ nis-user-name-from-uid-number 9000
nisclient::api::NameFromUid(9000, name) gives name = Test9000
server:$ nis-uid-number-from-user-name Test9000
nisclient::api::UidFromName(Test9000, uid) gives uid = 9000
server:$ nis-group-name-from-gid-number 1000
nisclient::api::GroupFromGid(1000, name) gives name = group1000
server:$ nis-gid-number-from-group-name group1000
nisclient::api::GidFromGroup(group1000, gid) gives gid = 1000
server:$
