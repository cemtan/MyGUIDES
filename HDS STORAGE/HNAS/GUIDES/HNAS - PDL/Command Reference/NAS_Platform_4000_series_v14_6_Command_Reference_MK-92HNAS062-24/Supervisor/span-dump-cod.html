span-dump-cod

Dump one or all currently loaded spans in the same format as 'sd-dump-cod'

Syntax

span-dump-cod [--terse | --compact-gzip-base64] [<span-instance-name>]

Description

Produces a formatted dump of the Cod for a loaded span, in the same format as 'sd-dump-cod'. Whereas 'sd-dump-cod' reads Cod from disk (and ignores spans loaded into memory), 'span-dump-cod' dumps Cod equivalent to a currently loaded span and its filesystems (and ignores the Cod stored on disk). It can therefore be run even if storage is unhealthy.

If you don't specify a span instance name, 'span-dump-cod' dumps Cod for all currently loaded spans.

The --terse (or -t) switch dumps the chunk table more concisely. It has no effect on other types of Cod.

The --compact-gzip-base64 (or -c) switch takes a compact integrated Cod backup (as if you had run 'span-back-up-cod --compact', compresses it, Base64-encodes it and displays it on the console. Its purpose is to provide a Cod backup in a concise textual form. The server cannot use this Cod backup, but an off-line program called the Cod Converter can convert it into an integrated Cod backup (for use with 'span-restore-cod'), a single-SD Cod backup (for use with 'sd-restore-cod') or a textual listing (as produced by 'span-dump-cod' without the -c switch).

'span-dump-cod' runs in showall.txt, providing a textual Cod backup as part of every set of diagnostics.

Examples

span-dump-cod Accounts


span-dump-cod

Applies To

Cluster wide
