migration-delete-external-path

Deletes the specified target from an external migration path.

Syntax

migration-delete-external-path -s <fs label> [-v <vivol>] -n <logical name> [--force] [--confirm]

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol>

Name of source Virtual Volume (optional).

-n, --name <logical name>

Logical name of the migration target.

--force

Force the target to be deleted (optional, see discussion below).

-c, --confirm

Confirm that the deletion is to be carried out without any further prompt.

Deletes an external migration target from an external Data Migration path. The status of the path will be updated by this command prior to the deletion and if there are cross-file-system links from the source file system to the target still active, the command will fail.

Removing the final external migration target from an external Data Migration path will cause the path to be deleted.

Forcing deletion of a target

In some circumstances it may be necessary to delete an external migration target from an external Data Migration path even if there are still files present on the target system. For instance, if the migration target is a WORM device that is being used to store a long term data archive, it would not be desirable or even possible to delete the target files.

The --force option will cause the migration target to be removed even if files exist on the target. Cross-file-system links referring to the deleted migration target will retain access to the remote file even after the target is removed.

WARNING: The --force option should be used with care: if a target which has active cross-file-system links is removed, it will not be possible to later add a new migration target to the same file-system using the same logical name. Therefore, recreating a migration target that has been deleted in error may not be possible.

Examples

migration-delete-external-path -s fs1 -n Archive1

Delete the external migration target named "Archive1" from migration path for file system fs1.

migration-delete-external-path -s Accounts -v HR -n "HR archive"

Delete the external migration target named "HR archive" from the migration path for Virtual Volume HR on file system Accounts.
