file-access-count-start

Start file access counting

Syntax

file-access-count-start [--file-system <fsname>]

Description

The file-access-count-start command enables the mechanism to count file accesses. Files will continue to be counted (without any performance penalty) in the background.

The file-access-count-stop-and-dump stops the background counting and performs an analysis on the data. During the analysis a performance drop may be noticed, but the analysis normally only takes a few seconds. The output shows the most accessed files along with a count of the number of times they were accessed during the counting period.

The file-access-count-run command starts, delays for the specified number of seconds, and stops and analyses the data.

The file-access-count-status command tells whether the file access counting is already running or not.

Supported options (not all available for each command):

--count

Set the number of files to display. The maximum allowed is 1024 and by default the top 64 most accessed files are displayed.

--max-processing-time

Set the maximum amount of processing time in seconds for the analysis. If the analysis isn't complete after this time, partial results are displayed.

--period

Set the run time in seconds for the file-access-count-run command.

--file-system

Specify the file system to count accesses for. If this option isn't given then accesses to all file systems are counted.

Examples

Show file access counts over a 1 minute period

server:$ file-access-count-run --period 60
Started, running for 60 seconds.
Finished running, analysing data.  This may take minutes.
Total cache pages: 4996000
Active cache pages during counting period: 5
Start time: 2010-03-30 22:19:01+01:00
End time: 2010-03-30 22:19:01+01:00
Duration: 1.00 m
         9 fs1:/a
         6 fs1:/c
         6 fs1:/d
         6 fs1:/b
         5 fs1:/

Start file access counts running on fs1

server:$ file-access-count-start --file-system fs1

Stop a running count, dump the information, but limit processing time to 10 seconds

server:$ file-access-count-stop-and-dump --max-processing-time 10

Applies To

Cluster node
