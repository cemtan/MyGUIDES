filesystem-object-list

List all loaded filesystems

Syntax

filesystem-object-list [--no-lock | -L] [--by-device-id | -d | --by-label | -l | --by-permanent-id | -p | --by-span | -s <filesystem-instance-name>]

Description

This is a diagnostic command, intended for use by your service organization. It lists the filesystems that are currently loaded into memory. The customer-facing equivalent, which is easier to read and should be used in preference, is 'filesystem-list --all'.

By default, the command produces an unsorted list of all loaded filesystems. You can sort the listing (unless you're in the LRC) by giving one of the following switches:

-d, --by-device-id
-l, --by-label
-p, --by-permanent-id
-s, --by-span

You can also list just a single filesystem by specifying its instance name, device ID or FSM volume number.

The '--no-lock' or '-L' switch bypasses important safety mechanisms inside the server. It is not available to unprivileged users. Use of this switch is not supported.

Examples

filesystem-object-list --by-span

Shows brief details of all loaded filesystems, grouping them by span.

filesystem-object-list --no-lock Accounts

Shows brief details of filesystem 'Accounts', bypassing the span-management lock.

Applies To

Cluster wide
