filesystem-space-tracking

Enable or disable file system space tracking

Syntax

filesystem-space-tracking [--file-system <fs-label>] [on|off]

Description

This command enables or disables space tracking on a file system, or reports whether space tracking is enabled or disabled. If the file system is not specified, the currently selected file system will be used (i.e., the file system set by a previous 'selectfs' command). The filesystem needs to be unmounted before enabling or disabling space tracking.

Space tracking is used to maintain quotas and is on by default. If quotas are not being used, space tracking may be disabled to conserve resources.

To disable space tracking for all file systems, set the quota-disable-space-tracking variable to true and reboot. A reboot of all nodes is required to activate the variable correctly on a cluster. When space tracking is disabled using this variable, it overrides the per-file-system settings.

Examples

To turn off space tracking on file system foofs:

    filesystem-space-tracking --file-system foofs off

To show the space tracking status for file system foofs:

    filesystem-space-tracking --file-system foofs

To turn off space tracking on all file systems:

    fsm set quota-disable-space-tracking true
