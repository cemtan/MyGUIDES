shortname

Enable/disable DOS short name generation

Syntax

shortname [-g on | off] [-p on | off]

Description

The shortname command is used to configure the generation of short names.

Legacy applications sometimes can't handle names longer than 8.3 characters (i.e. 8 characters, then an optional dot and an optional 3 extension characters). However, most of today's file names are longer than this. Therefore, in order to allow such applications to operate correctly, short names are created for files whose names are longer than 8.3 characters.

As an example, consider a file named, "New Microsoft Word Document.doc". Its name is longer than 8.3 characters, so a short name will be created for it. The short name will be "NEWM~H27.DOC".

In addition to helping legacy applications access files, short names are also used for file name compatibility purposes. If a file name cannot be returned to a network client for whatever reason, its short name is returned instead.

For example, consider that an NFS client creates a file named, "new\file". The '\' character isn't valid in CIFS file names, as it's used as the path separator. So, if a CIFS client accesses this file, its short name, "NEWF~NOG", will be returned instead of its long name.

Generation of short names for compatibility purposes is always enabled, as it's required for correct operation. However, generation of short names for file names longer than 8.3 characters can be disabled if necessary. This may result in a small performance gain, but should only be done if no applications rely on the presence of short names.

Usage:

shortname [OPTION...]

-g, --global (on|off)

Enable/disable generation of short names for all files whose names are longer than 8.3.

-p, --posix (on|off)

Enable/disable generation of short names for files whose names are longer than 8.3 and that were created by POSIX (taken to mean NFS, FTP, etc.) clients.

Applies To

Cluster wide

Supervisor
