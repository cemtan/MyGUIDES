file-security-descriptor-delete

Delete a file's security descriptor

Syntax

file-security-descriptor-delete [--confirm-ownership-change] <file1> [<file2>...]

Description

WARNING: This command bypasses all security checks so make sure you know what you're doing.

This command is used to delete security descriptors from a set of files and/or directories.

Usage:

-c, --confirm-ownership-change

Allows a file's security descriptor to be deleted, even if that would result in an implicit ownership change (e.g. when the file's UID and/or GID aren't correctly mapped to the security descriptor's owner/group).

Examples

To delete security descriptors from the files 'file1', 'file2' and 'file3' in the current working directory:

file-security-descriptor-delete file1 file2 file3

Applies To

EVS
