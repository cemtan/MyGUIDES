audit-log-save

Save a copy of the current file system audit log for a file system

Syntax

audit-log-save [--confirm] [--empty] <file-system> <backup-file>

Description

Options:

-c, --confirm

Do not prompt for confirmation.

-e, --empty

Empty the current file system audit log after it has been saved.

file-system

File system label.

backup-file

The name of the file into which the audit log records should be saved.

The audit-log-save command saves the file system audit log records from the current audit log for a file system into a file on the same file system.

The backup file name must end with the ".evt" extension.

If the --empty option is specified, the current audit log will be emptied after the records have been saved into the backup file.

If the backup file already exists or the --empty option is specified, the command will prompt for confirmation unless the --confirm option is specified.

Do not save backup files into the .audit directory which contains the audit log and automated backups. The files in this directory are managed automatically and the presence of additional event log files can cause problems.

Examples

To save the audit log from file system fs0 into the file "backups/audit.evt":

audit-log-save fs0 backups/audit.evt

To save the audit log from file system fs0 into the file "backup.evt" and then clear the audit log without further confirmation:

audit-log-save --empty --confirm fs0 backup.evt

Applies To

EVS
