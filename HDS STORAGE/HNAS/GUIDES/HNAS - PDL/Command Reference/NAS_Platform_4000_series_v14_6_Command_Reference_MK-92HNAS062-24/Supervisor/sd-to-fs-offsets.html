sd-to-fs-offsets

Translate SD offsets to fs & offset

Syntax

sd-to-fs-offsets <SD-device-ID> <SD-offset> <SD-offset...>

Description

This command is mainly intended for use by scripts, rather than interactive users. It accepts the device ID of a system drive (an SD), followed by one or more byte-offsets into the SD. It sorts the offsets into ascending numerical order and translates each one into an equivalent offset into a filesystem. For each SD offset, it prints a line containing the offset into the SD (unchanged from the input), the equivalent offset into a filesystem, and the instance name of the filesystem.

Some offsets into the SD may not be used in any filesystem. For those offsets, the command prints 'unmapped'.

All numbers on the command line are taken to be in decimal by default, or hexadecimal if prefixed by '0x'. All numbers printed by this command are in decimal.

Examples

server:$ sd-to-fs-offsets 2 0x4C2EE800 0x15EDFF600 0x8EFF5E00 0xCEA88200 0x11EDD3A00
1278142464: 3827230720, Accounts
2399100416: 1789877760, Engineering
3467149824: 4997022208, Engineering
4812782080: 3637328384, Marketing
5886703104: unmapped
server:$
