virusscan-icap-engine-modify

Modify an existing ICAP virus scanning engine

Syntax

virusscan-icap-engine-modify [--service-name <service name>] [--port <port number>] [--enable | --disable] [--confirm] <host>

Description

The virusscan-icap-engine-modify command is used to modify an existing ICAP virus scanning engine in the current ICAP engine list. Service name, port number and enabled/disabled status can all be modified. At least one of these parameters is required. If the current virus scanning mode is RPC then no action will be taken. Note that if virus scanning is on, but all virus scan engines are disabled, CIFS access to files might be denied if the best effort scanning is disabled.

Options:

-s, --service-name <service name>

New name of the service to be used.

-p, --port <port number>

New port number to be used.

--enable

Is used to enable an ICAP engine for virus scanning.

--disable

Is used to disable an ICAP engine for virus scanning.

--confirm

If the --confirm option is not supplied, and the user is attempting to disable the last ICAP server, the user will be prompted to confirm the request.

<host>

A hostname, or an IP address, that identifies the host running the ICAP virus scanning engine.

Examples

To modify the ICAP engine icapengine.example.com with a new port number:

virusscan-icap-engine-modify --port 1342 icapengine.example.com

To modify the ICAP engine icapengine.example.com with a new port number and service name:

virusscan-icap-engine-modify --service-name SYMCScanResp-AV --port 1342 icapengine.example.com

To enable the ICAP engine using an IP address:

virusscan-icap-engine-modify --enable 192.168.43.100

To disable the ICAP engine icapengine.example.com

virusscan-icap-engine-modify --disable icapengine.example.com

EVS
