filesystem-list-queues

List filesystem notifications waiting to be sent by the spanning system on this server

Syntax

filesystem-list-queues 
Alias:       fsq

Description

Several events cause the Spanning System to queue up notifications for the Filesystem Manager -- for example:

  • Filesystems are loaded at boot time
  • Filesystems become healthy or unhealthy because system drives come up go down or change mirror roles
  • You run 'filesystem-forget-and-delete-nv-data'
  • You run 'span-assign-to-cluster', 'span-add-cluster-uuid' or 'span-remove-cluster-uuid'
  • You run 'filesystem-rename'

These notifications are then spooled asynchronously to the Filesystem Manager while the Spanning System does other work, such as loading further spans and filesystems or responding to further storage events.

'filesystem-list-queues' shows the number of pending notifications of each type, as well as the thread (if any) that is processing them. It is mainly of use to your support provider.

Examples

filesystem-list-queues

Applies To

Cluster node
