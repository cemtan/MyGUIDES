scsi-cap10

Send a SCSI Capacity10 command to the target device

Syntax

scsi-cap10 < [<[sd-spec device specifier] | [-D]>[-p]] | <[-p] [-l]>> [-a] [-r] [-q] [-d]
Alias:       cap10

Description

Sends a SCSI Capacity10 command to the target device on the given SCSI port and LUN.

If the device supports devices greater than 2TB, use the long capacity command, scsi-cap16.

The scsi-devices command can be used to determine which SCSI ports are available for each device.

Options:

[system-drives list]

See the 'sd-spec' man page for an explanation of how to specify a list of system drives using sd-spec syntax. If no SCSI port is specified, the command will be sent to the first on-line port of the device. If the -a option is specified, the command will be sent to all on-line ports.

-D, --Dev <Device id>

device ID of target device. If no SCSI port is specified, the command will be sent to the first on-line port of the device. If the -a option is specified, the command will be sent to all on-line ports.

-p --port <id>

SCSI port id of target device.

-l --lun <LUN>

LUN of the target device.

-a, --all-ports

Attempt to send the SCSI command to all on-line ports of the target device.

-d --dump

Hex dump the returned data.

-r --raw

Attempt to send the command even if the server has not discovered a device on this SCSI port and LUN.

-q --qtag tag

Use this queue tag value which must be one of:

Simple = 0 (default)

Head of q = 1

Ordered q = 2

ACA q = 3

Untagged = 4

Examples

Send a SCSI capacity10 command to SCSI port 0, LUN 1 :

scsi-cap10 -p 0 -l 1

Send a SCSI capacity10 command to device 0 using SCSI port 2 :

scsi-cap10 -D 0 -p 2

Send a SCSI capacity10 command to devices 10 to 15 using the first on-line SCSI port of each device :

scsi-cap10 10-15

Send SCSI capacity10 commands to device 10 using all on-line SCSI ports :

scsi-cap10 10 -a

Applies To

Cluster node

See Also

scsi-cap16 scsi-devices

Privilege Level

Supervisor
