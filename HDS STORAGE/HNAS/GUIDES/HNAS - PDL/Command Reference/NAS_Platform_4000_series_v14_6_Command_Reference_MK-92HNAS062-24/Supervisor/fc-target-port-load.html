fc-target-port-load

Display allocation of SDs to target ports & RCs

Syntax

fc-target-port-load [-v] [rack] [rc] [port] [wwpn <WWPN> | device <N>]

Description

'fc-target-port-load' shows how many system drives are currently assigned to each FC target port/controller/rack, and optionally lists them.

Options:

-v --verbose

List SDs instead of just saying how many there are. Also forces display of individual target ports, whether "port" is given or not.

rack

Show all controllers in the storage rack identified by WWPN/device; ignored otherwise (as redundant).

rc

If WWPN is specified, show the controller; if device is specified, show just the controller through which it is currently accessed (unless "rack" is also given); ignored otherwise.

port

If WWPN or device is specified without "rack" or "rc", show just the corresponding single target port; otherwise break down controller load by target port.

wwpn

Select by WWPN. By default just the load on that target port is displayed.

device

Select by system drive number. By default all controllers in the rack containing that SD are shown.

The "wwpn" and "device" options are mutually exclusive. If neither is given then information will be shown for all controllers known to the system, grouped by storage rack.

The load figure for a controller is simply a count of the SDs open (on at least one cluster node) via that controller. For individual ports, each SD is counted once for each cluster node that has it open via that port; in the verbose output, each additional node is indicated by a dot after the device number (e.g. "23." for device 23 open by two cluster nodes). Normally, a SD should be open on all nodes of a cluster. When SDs are moved from one controller to another, the reported load for a given controller may, temporarily, not exactly match the number of SDs listed against that controller's ports, until all cluster nodes have synchronised.

Examples

Display information for all controllers, grouped by storage rack:

fc-target-port-load

List the SDs being accessed through a specific target port:

fc-target-port-load -v wwpn 200600A0B812DB01

Show how the load is distributed between the ports of a controller:

fc-target-port-load rc port wwpn 2000000402E00556

... or between controllers within a storage rack:

fc-target-port-load rack wwpn 21000080E5126975

Show how the load is distributed between the controllers serving a SD:

fc-target-port-load device 12

List the SDs being accessed through the same target port:

fc-target-port-load -v port device 12

Applies To

Cluster node

See Also
