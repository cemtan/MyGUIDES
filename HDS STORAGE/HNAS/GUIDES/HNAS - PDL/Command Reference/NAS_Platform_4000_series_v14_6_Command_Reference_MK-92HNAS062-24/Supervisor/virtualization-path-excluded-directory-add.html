virtualization-path-excluded-directory-add

Adds an entry to the list of directories to be excluded from virtualization.

Syntax

virtualization-path-excluded-directory-add -t <fs label> -n <association name> -d <directory-to-exclude>

Description

Adds an excluded directory to the virtualization path association between a remote NFS export source and a local file system, vivol or directory target.

This command can be applied while the association has created but the virtualization has not been started.

Options:

-t, --target <fs label>

Target file system.

-n, --name <association name>

Target association name.

-d, --dir <directory>

Target directory relative to the root of the association to exclude from virtualization.

Examples

virtualization-path-excluded-directory-add -t fs1 -n assoc1 -d .snapshot

Add an excluded directory .snapshot to the association assoc1 to file system fs1.
