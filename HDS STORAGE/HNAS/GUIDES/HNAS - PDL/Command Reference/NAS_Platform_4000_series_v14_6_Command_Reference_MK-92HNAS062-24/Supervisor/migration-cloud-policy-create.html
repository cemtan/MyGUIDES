migration-cloud-policy-create

Create an external cloud migration policy

Syntax

migration-cloud-policy-create <policy-name> --source-fs-label <source-fs-label> --external-path <external-path> --rule:threshold <rule:threshold>

Description

Creates a cloud migration policy.

<policy-name>

Name of policy to create. May be up to 255 ASCII characters long. The characters ;:/\'"<>*?| and a space character are not allowed.

--source-fs-label, -s <source-fs-label>

Name of file system on which to apply policy.

--external-path, -p <external-path>

Name of file system's external path on which to apply policy. This external path must have been previously created with the migration-add-external-path command.

--rule:threshold, -r <rule:threshold>

Name of rule to associate with policy. The rule must have been previously created with migration-cloud-rule-create command. The threshold specifies the percentage free space on the file system for the policy to take effect.

Examples

migration-cloud-policy-create DefaultPolicy --source-fs-label fs0 --external-path amazon --rule:threshold jpg:20

Creates a policy called DefaultPolicy in which files will be migrated from filesystem fs0 matching the criteria in rule 'jpg' if free space drops below 20%.

Applies To

EVS security context
