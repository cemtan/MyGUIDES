unified-management-ip-address

Description

Unified management IP addresses are assigned to the non-file-serving interfaces of each controller of a unified system. They are configured using the Maintenance Utility (or HiCommand Suite, or Storage Navigator) and are used for the purposes described below. The examples given assume unified management IP addresses 172.27.12.10 and 172.27.12.11. The address 172.27.12.10 is used, but 172.27.12.11 can be used with exactly the same results.

Uses of the unified management IP addresses
1. Logging into the CLI console of the NAS platform.

The CLI can be accessed using ssh or ssc, in the following way:

ssh manager@172.27.12.10

ssc 172.27.12.10

Either IP address, and any user configured in the NAS platform, can be used. If a command must be handled by a specific cluster node, then prefix the command with "console-context --cnode <id>". There is no need to ensure that the initial ssh or ssc connections are made to a specific controller.

2. Launching the embedded SMU.

The embedded SMU can be launched using the following URL (notice the port 20443):

https://172.27.12.10:20443/mgr/app

Alternatively, the following form can be used, omitting the port. The page navigated to presents the choice between launching the SMU (NAS Manager) or Maintenance Utility.

http[s]://172.27.12.10

The choice may be set as default. To return to the choice, the following form can be used.

http[s]://172.27.12.10/TopPageSelection/

3. Registering the NAS platform with an external SMU.
4. For applications to connect to the SOAP and REST APIs for managing the NAS platform.
5. SNMP queries.
6. For the NAS platform to access network services such as DNS, NTP, SMTP, trap receiver and syslog server, AD.

The two IP addresses can be configured in DNS to resolve individual host names, one for each controller's unified management port. Alternatively a single host name can be resolved by DNS to both IP addresses, in which case the application will simply connect to the IP address it chooses, and get equivalent functionality.

Examples

Commands "ipaddr", "cluster-show" and "evs list" provide information about unified management IP addresses. The example below shows the output of "cluster-show -c".

server:$ cluster-show -c
Node EVS ID    Type     Label Enabled Status    IP Address          Port
---- ------ ------- --------- ------- ------ ---------------------- ----
   1        Cluster    NODE-1     Yes Online       172.27.12.10     CTL1
                                             2001:db8::1:1:4a:2     CTL1
                                                    172.27.12.3     eth0
                                                    172.24.48.1     eth1
                                                2002::ac18:3001     eth1
   1      0   Admin CLUSTER-A     Yes Online        172.24.48.2     eth1
                                                 192.168.38.180      ag1
                                                2002::ac18:3002     eth1
   2        Cluster    NODE-2     Yes Online       172.27.12.11     CTL2
                                             2001:db8::1:1:4a:3     CTL2
                                                    172.27.12.4     eth0
                                                    172.24.52.1     eth1
                                                2002::ac18:3401     eth1

The unified management IP addresses are shown with port "CTL1" and "CTL2", because they are the primary IP addresses for gaining access to controller 1 and controller 2 respectively. These addresses must be changed by using the Maintenance Utility; they cannot be configured using the NAS platform CLI or the SMU. They are, however, the only IP addresses that are contactable from a workstation used to manage the system.

IP addresses configured on eth0 and eth1 are not accessible from outside the system. They are used for two purposes:

1. eth0 IP addresses are for maintenance purposes, and a normally only used by your support team. In normal circumstances these ports would not be connected to a management LAN.

2. eth1 IP addresses are for communication between the internal components of the system. These ports can never be connected to a management LAN.
