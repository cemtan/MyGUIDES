snmp

Description

The Simple Network Management Protocol (SNMP) supports network access to the configuration and status information held on the server.

The SNMP agent in the server supports SNMP versions v1, v2c and v3. The protocol version is configured by the 'snmp-protocol' command. The SNMP agent listens for requests received on port 161; if required a different port can be configured by the 'snmp-port-set' and 'snmp-port-show' commands. SNMP access to management information is restricted to read-only access only.

Using SNMPv1 or SNMPv2c

When SNMPv1 or SNMPv2c is configured, access to the information on the server is restricted by community names which are configured by the 'snmp-communities' command. Access may be further restricted to specific hosts configured by the 'snmp-hosts' command.

SNMPv1 and SNMPv2c may be used at the same time by configuring bilingual support via the 'snmp-protocol' command.

Using SNMPv3

When SNMPv3 is configured, access to the information on the server is restricted to users in the SNMPv3 user list. The user list is configured by the 'snmpv3-user-add', 'snmpv3-user-delete', 'snmpv3-user-delete-all' and 'snmpv3-user-list' commands. The SNMP agent uses HMAC-SHA-96 authentication and AES-128-CFB encryption for data privacy. When SNMPv3 is enabled the SNMP agent will not respond to SNMPv1 or SNMPv2c requests.

Note: The hosts list configured via the 'snmp-hosts' command does not restrict SNMPv3 access to the server.

SNMP Notifications

All notifications are sent using SNMPv1 traps regardless of the configured SNMP protocol version.

The trap generation is configured by the 'snmp-traps' command. The traps are normally sent to port 162; if required a different port can be configured by the 'snmp-trap-port-set' and 'snmp-trap-port-show' commands.
