vlan-interfaces

Description

A VLAN interface is the way an HNAS provides access to a VLAN on a file-serving interface. You should create one of these for each tagged VLAN for each file-serving interface over which the HNAS needs to communicate.

In order to create a VLAN interface the vlan-interface-create command should be used.

IP addresses can then be assigned to and removed from the VLAN interface using the evsipaddr command, and an ipadv configuration can be applied to it using the VLAN interface name.

VLAN interfaces that have been dynamically created by the deprecated vlan command will only appear in the ifconfig display. For these deprecated VLAN interfaces, IP addresses are added using evsipaddr command specifying the aggregation interface name and not the VLAN interface name.

See the ipadv man page for more information about configuration settings for VLAN interfaces. ipadv configuration settings apply identically regardless of the VLAN creation mechanism.
