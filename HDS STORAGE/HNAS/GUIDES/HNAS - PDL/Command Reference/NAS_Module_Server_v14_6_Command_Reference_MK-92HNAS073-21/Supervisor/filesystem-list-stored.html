filesystem-list-stored

List all filesystems known to the system, whether or not they're loaded

Syntax

filesystem-list-stored [-u | --unloaded | <device-id>]
Alias:       fsss

Description

This command lists all filesystems known to the server, regardless of whether they are currently loaded and which EVS they're bound to.

If you specify a device ID, only a single filesystem is displayed.

If you specify the --unloaded switch, only unloaded filesystems are displayed. See the 'cod' man page for an explanation of loading.

Information displayed

Dev

The device ID of the filesystem.

Loaded

'Yes' if the server has loaded the filesystem from Cod; 'No' if the filesystem isn't currently loaded.

Healthy

'Yes' if the filesystem has been loaded and all the system drives on which the host span resides are healthy; 'No' otherwise.

EVS

The EVS to which the filesystem is bound, or a dash if it isn't bound to an EVS.

Cap/GiB

The filesystem's capacity, in gibibytes, where one GiB is 2**30 bytes.

Permanent ID

The 64-bit permanent ID of the filesystem.

Filesystem Instance Name

The instance name of the filesystem.

'filesystem-list' versus 'filesystem-list-stored'

The 'filesystem-list' command displays only filesystems that are currently loaded into memory. 'filesystem-list-stored' reads from the registry, and is therefore able to display even those filesystems that haven't loaded. However, the information it produces is less detailed than the output of 'filesystem-list'.

By default, in the context of a service vnode, 'filesystem-list' shows only filesystems that are bound to the current EVS or to no EVS at all. 'filesystem-list-stored' does not filter by EVS, and displays all filesystems in the registry.

Examples

server:$ filesystem-list-stored
Dev   Loaded  Healthy  EVS  Cap/GiB  Permanent ID      Filesystem instance name
----  ------  -------  ---  -------  ----------------  ------------------------
1024   Yes      Yes     1        70  EB970D68F7C48208  Accounts
1025   Yes      Yes     2        90  EB970D7CDA61DECD  Engineering
1026   Yes      Yes     2        95  EB970D769C482922  Sales
1027   Yes      Yes     1        90  EB970F29F5F5E874  Marketing
1028   No       No      -         4  EB970F7A1080EBD1  fs0
server:$ filesystem-list-stored --unloaded
Dev   Loaded  Healthy  EVS  Cap/GiB  Permanent ID      Filesystem instance name
----  ------  -------  ---  -------  ----------------  ------------------------
1028   No       No      -         4  EB970F7A1080EBD1  fs0
server:$ filesystem-list-stored 1028
Dev   Loaded  Healthy  EVS  Cap/GiB  Permanent ID      Filesystem instance name
----  ------  -------  ---  -------  ----------------  ------------------------
1028   No       No      -         4  EB970F7A1080EBD1  fs0
server:$
