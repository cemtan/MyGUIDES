handle-to-path

Dumps the path of an object given its handle

Syntax

handle-to-path [--file-system <file system name> | --volume-number <volume number>] [--object-number] <handle | object-number>

Description

Given a file handle, handle-to-path attempts to resolve the full pathname for that handle.

This command may only be run when the file system is mounted.

If the handle resolves to a file and that file has multiple hard links, the command will output the path to the first link to the file. If you wish to see all the paths that can reach the file, use the hard-links command.

If corruption is suspected on the file system, consider using object-number-to-path instead of handle-to-path, as object-number-to-path performs more extensive checks on the metadata to protect against file system failure. object-number-to-path can also be run when the file system is unmounted.

Usage:

-f, --file-system <file system>

Look for the handle on the given file system. The default is to use the console's current file system.

-v, --volume-number <volume number>

Look for the handle on the file system with the given volume number. The default is to use the console's current file system.

--object-number

Rather than providing a full handle, only provide the object number. This can be useful in situations where diagnostics haven't output the entire handle.

<handle>

The handle or object number you wish to resolve a path for.

Examples

handle-to-path -v 0 FileHandle[0000000000000138000000006803ffffffffffffffffffff]
 handle-to-path -f my-fs --object-number 1024

Applies To

EVS

See Also

hard-links object-number-to-path selectfs

Privilege Level

Supervisor
