replication-recovery-local-config-update

Updates the local configuration of a file system

Syntax

replication-recovery-local-config-update [--confirm] [--force-create] <filesystem>

Description

This commands brings a file system's local configuration information up-to-date.

A file system's local configuration stores (among other things) information about the CIFS shares and NFS exports that target it. During disaster recovery, this information is used to create shares/exports when promoting a replication target.

Note that a replication policy's "auto" and "use-named" snapshot options automatically ensure that the source snapshot's local configuration is up-to-date.

--confirm

Confirm the update.

--force-create

Force the creation of the configuration object.

<filesystem>

File system label.

Examples

replication-recovery-local-config-update --confirm --force-create sourcefs

Updates the local configuration information of file system "sourcefs".

Applies To

File system
