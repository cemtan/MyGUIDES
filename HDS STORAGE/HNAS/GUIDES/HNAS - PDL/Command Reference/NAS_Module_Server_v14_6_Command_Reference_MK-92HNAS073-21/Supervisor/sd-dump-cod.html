sd-dump-cod

Show the span and filesystem config on a system drive

Syntax

sd-dump-cod [-d | --denied] [-t | --terse] (span | fs | chunk | allcod) <system-drives-see-'man sd-spec'>
Alias:       dcod

Description

'sd-dump-cod' is a low-level command that lists Cod in a form that your support provider can read. Cod is described in the 'cod' man page.

For the first parameter, specify the kind of Cod you wish to see: 'span', 'fs' or 'chunk'. To see all three kinds of cod, specify 'allcod'.

For the second parameter, specify one or more SDs. The syntax is described in the 'sd-spec' man page.

Specifying the --denied switch lets the command run on an SD to which access has been denied. This lets you inspect Cod without allowing access to the SD and potentially loading the Cod into memory.

The --terse switch dumps the chunk table more concisely. It has no effect on other types of Cod.

Examples

sd-dump-cod span 7

Dumps the span Cod on SD 7.

sd-dump-cod --terse allcod 13

Dumps all three Cod structures on SD 13. The chunk table is dumped in a relatively concise format.

sd-dump-cod span 4-7

Dumps the span Cod from SDs 4, 5, 6 and 7.

sd-dump-cod fs Accounts

Dumps the filesystem catalog for each SD in span 'Accounts'.

Applies To

Cluster wide
