default-owner-policy

Set default owner policy

Syntax

default-owner-policy [NT|creator]

Description

The default-owner-policy command allows you to configure strict compatibility with Windows' policy for setting the owner of newly created files.

If the default owner policy is set to "NT", then files created by members of the NAS server's local administrators group (as configured by, for example, the "localgroup add Administrators" command) will be owned, by default, by the local administrators group (BUILTIN\Administrators). The default setting for the default owner policy is "creator". With this setting, newly created files will, by default, be owned by the creating user, even if that user is a member of the local administrators group.

Particularly when inter-operating with NFS and user quotas, we find that the "creator" policy is more useful.

The setting is persistent and takes immediate effect, across the current EVS security context (cluster-wide or per-EVS).

If the applicable security-mode (for the virtual volume, file system, EVS or cluster) is Unix, which is the default, then the server does not store the default owner as a Windows SID. Instead, it has to be translated to a Unix uid. In the default configuration or when there is otherwise no mapping configured, this mapping will fail and the file's owner will become uid 0, conventionally known as "root". (This means that the creator cannot even change the permissions, as they are not root. If the owner defaulted to "nobody" (conventionally uid 65534) instead of "root", then every other unmapped user would have full control over the file.)

Examples

server$ default-owner-policy
default, shared security context: Using "creator" default owner policy.
server$ default-owner-policy NT
default, shared security context: Using "NT" default owner policy.
server$ default-owner-policy
default, shared security context: Using "NT" default owner policy.
server$ default-owner-policy creator
default, shared security context: Using "creator" default owner policy.

Applies To

EVS security context
