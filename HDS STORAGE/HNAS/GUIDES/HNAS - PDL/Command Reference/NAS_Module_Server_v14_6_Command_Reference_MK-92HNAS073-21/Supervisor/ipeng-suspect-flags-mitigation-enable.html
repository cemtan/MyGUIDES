ipeng-suspect-flags-mitigation-enable

Enable countermeasures for segments with suspect header flags (Obsolete).

Syntax

ipeng-suspect-flags-mitigation-enable [-g <interface>]

Description

The ipeng-suspect-flags-mitigation-enable and ipeng-suspect-flags-mitigation-disable commands are obsolete and will be removed in a future release.

The ipeng-suspect-flags-mitigation-* commands previously controlled if the system applied the countermeasures detailed in RFC 5961 for the possible attacks described in CVE-2004-0230. The countermeasures are now permanently enabled; this was previously the default setting for this configuration item.

The commands available are as follows.

ipeng-suspect-flags-mitigation-enable [-g <interface>]

Deal with segments with suspect header flags as described in RFC 5961. This behaviour is permanently enabled.

ipeng-suspect-flags-mitigation-disable [-g <interface>]

This operation is no longer supported. Previously this command reverted to the pre-RFC 5961 TCP behaviour.

ipeng-suspect-flags-mitigation-show [-g <interface>]

Display the configuration setting.

Options:

-g, --aggportname <interface>

Name of the interface on which to act.

Examples

To display the configuration setting:

    $ ipeng-suspect-flags-mitigation-show

Applies To

Cluster wide
