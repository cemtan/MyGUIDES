sd-restore-cod-signature

Restore a Cod signature previously wiped by sd-wipe-cod-signature

Syntax

sd-restore-cod-signature [--danger-ignore-sd-licensing] <System-drives-see-'man sd-spec'>

Description

Restores one or more SDs' Cod signatures, reversing the effect of the 'sd-wipe-cod-signature' or 'span-delete' command. Cod signatures are described in the 'cod' man page.

Restoring Cod signatures on all SDs in a deleted span will allow the span to load again.

There is no equivalent way to undelete the filesystems on a span. However, the 'cod' man page explains where Cod backups can be found.

The --danger-ignore-sd-licensing switch enables the command to write to unlicensed system drives (those to which you have not allowed access). Without this switch, the command will reject unlicensed SDs. Never use this switch to do I/O to SDs that are being used by another cluster.

You can specify more than one SD in a single command. See the 'sd-spec' man page.

After wiping or restoring Cod signatures, run 'sd-rescan-cod' to make the server load the updated Cod into memory.

When a span is deleted, the server destroys its SDs' encroachment data. If you undelete the span, you will need to rewrite this encroachment data before the span can be used.

Examples

If a span consisting of SDs 0 to 7 has been deleted, and if the SDs are healthy and licensed and have not been reused in a new span, the following commands will undelete the span:

sd-restore-cod-signature 0-7
sd-rescan-cod

Applies To

Cluster wide
