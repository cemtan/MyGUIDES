sd-group-create

Create a new system drive group

Syntax

sd-group-create [(--tier | -t) <tier>] <SD-device-IDs-see-'man sd-spec'>...
Alias:       mksdg

Description

Creates a new system drive (SD) group, containing one or more SDs. See the 'sd-group' man page for background.

The SDs you specify must exist, must not already be members of other groups, and must not be used in open spans. (A span is open if any of its file systems is mounted or is being checked, fixed, etc.) Within these restrictions, there is no limit to the number of SDs in a group.

From release 12.3 onwards, SD groups are optional unless (in contravention of best practice) a single parity group containing magnetic disks has been divided into multiple SDs. Therefore, it will rarely be necessary to run 'sd-group-create'.

The --tier parameter assigns the SDs to a tier before creating the SD group. To remove any pre-existing tier assignments, specify Tier -1. If the tier assignments cannot be made or removed, no SD group is created. If --tier is not supplied, all the SDs specified must be in the same tier (or in no tier at all).

Examples

server:$ sd-group-list
No groups found
server:$ sd-group-create 0-2
Success
server:$ sd-group-create --tier 1 6-8
Success
server:$ sd-group-list
SDs 0-2


(In tier 1) SDs 6-8


server:$

Applies To

Cluster wide
