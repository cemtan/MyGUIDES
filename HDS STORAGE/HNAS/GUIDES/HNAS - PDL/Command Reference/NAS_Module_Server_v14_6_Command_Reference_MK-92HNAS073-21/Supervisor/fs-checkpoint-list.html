fs-checkpoint-list

Lists checkpoints available to fs-checkpoint-select

Syntax

fs-checkpoint-list <file system>

Description

On WFS-2 file systems, the server stores a fixed number of historic checkpoints of the file system (see the 'dsb-count' option in the format command). These checkpoints may be selected using the fs-checkpoint-select command to rollback the file system to a previous checkpoint.

fs-checkpoint-list can be used to list these checkpoints. It will output three fields for each checkpoint:

Checkpoint number

This number uniquely identifies the checkpoint. If you wish to rollback the file system to this checkpoint, provide this checkpoint number to fs-checkpoint-select.

Time stamp

The time the checkpoint was created.

Age

The age of this checkpoint relative to the latest checkpoint.

The command will display the table from the newest selectable checkpoint to the oldest. If fs-checkpoint-select has been used to rollback the file system to an older checkpoint, some checkpoints will be flagged as being newer than the latest checkpoint, with an indication that the file system has been rolled back from these checkpoints.

The command will fail if we can't read any of the DSBs (which store the checkpoint information) off disk. In this case run fs-checkpoint-health, which performs more extensive checks to determine which checkpoints can safely be selected.

Examples

To list available checkpoints that may be selected on fs3:

fs-checkpoint-list fs3
