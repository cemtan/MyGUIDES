group-mappings-add

Creates a group mapping

Syntax

group-mappings-add     [--unix-name <name>] [--unix-id <id>] [--nt-name <name>]
    [--nt-id <id>] [--nfsv4-name <name>]
    [--no-unix-mapping] [--no-nt-mapping] [--no-nfsv4-mapping]

Description

Adds a new entry to the group mappings table.

Options:

--unix-name

Sets the Unix name to be used for the new entry

--unix-id

Sets the Unix group id for the new entry

--nt-name

Sets the NT name (of the form DOMAIN\group) for the new entry

--nt-id

Sets the NT SID for the new entry

--nfsv4-name

Sets the NFSv4 name for the new entry

--no-unix-mapping

The entry will never have a valid Unix mapping

--no-nt-mapping

The entry will never have a valid NT mapping

--no-nfsv4-mapping

The entry will never have a valid NFSv4 mapping

Any fields set from the command line will be marked as fixed and won't be changed by the auto mapper.

Any fields marked as having no mapping will be ignored by the auto mapper.

The auto mapper will attempt to fill in any fields that haven't been either explicitly set or marked as having no mapping.

If the server finds an existing entry that matches some of the fields set from the command line, the command will not create a new entry or modify the existing entry (use group-mappings-modify to modify an existing entry).

Examples

Add the Unix group badgers with a group id of 100 who maps to the NT group EXAMPLE\Badgers:

$: group-mappings-add --unix-name badgers --unix-id 100 --nt-name "EXAMPLE\Badgers"
Created entry
Unix name     : badgers*
Unix group id : 100*
NT name       : EXAMPLE\Badgers*
NT id         : *unknown*
NFSv4 name    : *unknown*

Applies To

EVS security context
