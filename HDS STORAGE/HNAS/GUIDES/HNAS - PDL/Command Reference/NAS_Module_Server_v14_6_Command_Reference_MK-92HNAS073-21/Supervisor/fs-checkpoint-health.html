fs-checkpoint-health

Perform a health check on critical structures for recent checkpoints of the file system, or abort an existing check

Syntax

fs-checkpoint-health [-a] -f <file-system>

Description

On WFS-2 file systems, the server stores a fixed number of historic checkpoints of the file system (see the 'dsb-count' option in the format command).

fs-checkpoint-health validates the essential health of these checkpoints.

Starting with the most recent checkpoint, several file system checkpoints are validated concurrently. Validation stops when a checkpoint version is found that passes all of its checks and all more recent versions have failed one or more checks, or when there are no more checkpoints to validate.

Validation consists of a number of checks that are performed on critical file system objects. A report of the results of these checks is generated for the most recent checkpoint. Thereafter, reports will only be generated for checkpoints which have at least one failed check during validation, and one final report for the checkpoint which passed validation.

fs-checkpoint-health is a synchronous utility.

Users should consult the FSM dblog or the dynamic-log to view the reports.

Note: fs-checkpoint-health may only be run on an unmounted WFS-2 file system.

Options:

-a

Abort an existing health check.

Examples

To validate checkpoints of file system fs3:

fs-checkpoint-health -f fs3

To abort the health check on file system fs3:

fs-checkpoint-health -a -f fs3

Applies To

EVS
