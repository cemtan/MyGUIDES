replication-listening-port

Query/change the object replication port number

Syntax

replication-listening-port [--default | <new port number>]

Description

This command is used to configure the port on which a server receives object replication traffic. The default is 59550.

When the port is changed, the new port number is propagated to every node in the cluster. Each node then attempts to switch to using the new port number. If a node isn't able to start listening on the new port, perhaps because it's already in use, then that node will need to be rebooted in order for the change to take effect. This command will highlight any nodes that are affected in this way.

Usage:

--default, -d

Return the port to its default value.

Examples

replication-listening-port 42262
The configured object replication listening port is 42262
  Cluster node 1: Listening on port 42262
  Cluster node 2: Listening on port 59550 *

* Some nodes are still listening on the old port number. It may be that the port change hasn't yet propagated around the cluster, so wait a few seconds and run this command again. If a node hasn't applied the port change after a couple of minutes, check its event log for errors. If an error has occurred, the affected node will need to be rebooted in order for the change to take effect.

Applies To

Cluster wide

Privilege Level

Supervisor
