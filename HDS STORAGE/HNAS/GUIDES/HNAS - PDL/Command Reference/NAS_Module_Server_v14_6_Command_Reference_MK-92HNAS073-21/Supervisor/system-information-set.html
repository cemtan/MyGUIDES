system-information-set

Set system information

Syntax

system-information-set [--name <name>] [--location <location>] [--contact <contact>]

Description

System information consists of a name, contact and location. The name is the same value as the cluster's name, which can also be seen in the output of 'cluster-show', and modified using the command 'cluster-rename'. When the name value is changed using this command it will:

1. Change the cluster name

2. Change each cluster node name, using the cluster name with the simple ID of the node appended e.g. clustername-1.

"name" may be up to 13 alpha-numeric or hyphen ASCII characters long.

"contact" may be up to 180 ASCII characters long. The characters \/:,;*?"<>|%&^ are not allowed. Example use is the name, phone number, and email of a person/group responsible for the server.

"location" may be up to 180 ASCII characters long. The characters \/:,;*?"<>|%&^ are not allowed. Example use is address or location of the rack hosting the server.

Other purposes of the fields are to provide information reported in the SNMP MIB-II System Group objects "sysName", "sysContact" and "sysLocation", and to identify a system in a notification email.

Note:

Changing the cluster name will trigger creation of a new self-signed SSL/TLS certificate for the SOAP HTTPS management server and (if present) for the embedded SMU. However, if you had installed a custom certificate it will not be affected (but you may want to consider updating it to match the new domain name).

Note: The system information is applied to the block component. This can take up to several minutes to complete.

Examples

Setting values

system-information-set --name "NEWCLUSTER" --contact "Fred Smith +44 121 496 0123 fred@example.com" --location "123 Any Street Birmingham UK"

The settings will be confirmed with the following output:

Successfully saved settings

      Name: NEWCLUSTER
  Location: 123 Any Street Birmingham UK
   Contact: Fred Smith +44 121 496 0123 fred@example.com

Clearing values

To clear the settings, use:


system-information-set --contact "" --location ""


(Note: The name field may not be cleared.)

Applies To

Cluster wide

See Also

Supervisor
