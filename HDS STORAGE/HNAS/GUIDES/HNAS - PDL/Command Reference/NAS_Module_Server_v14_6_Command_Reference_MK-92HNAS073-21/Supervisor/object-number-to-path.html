object-number-to-path

Dumps the path(s) of a given object

Syntax

object-number-to-path  [-f <file system>] [--snapshot-name <snapshot name> | --snapshot-checkpoint-number <snapshot checkpoint number> | --snapshot-index <snapshot index>] <object-number>

Description

Given an object number, object-number-to-path attempts to resolve the full pathname for that object. If the command runs successfully the full path is displayed on the current console. object-number-to-path may be run with the file system mounted or unmounted, and may be safely invoked to resolve the path of objects suspected of being corrupt in some way. However, should the nature of the corruption interfere with the path resolution, then object-number-to-path will fail.

If you wish to find the path name with the file system mounted and do not suspect corruption, consider using handle-to-path, which performs fewer checks and is much less resource hungry.

Options:

-f <file-system>

The file system on which the onode resides

--snapshot-name <snapshot-name>

Find the path as it was in the given snapshot, rather than as it is now in the live file system. Can only be used on WFS-2 file systems.

--snapshot-checkpoint-number <checkpoint-number>

Find the path as it was in the given snapshot, rather than as it is now in the live file system. Can only be used on WFS-2 file systems.

--snapshot-index <snapshot-index>

Find the path as it was in the given snapshot, rather than as it is now in the live file system. Can only be used on WFS-2 file systems.

Examples

To print the path of the object located at 0x3860 on file system testfs:

object-number-to-path -f testfs 0x3860

Applies To

EVS
