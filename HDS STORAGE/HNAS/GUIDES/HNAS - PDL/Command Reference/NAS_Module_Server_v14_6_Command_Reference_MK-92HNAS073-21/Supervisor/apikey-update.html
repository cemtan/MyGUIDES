apikey-update

Update an existing API Key

Syntax

apikey-update (--description <description> | --prefix <prefix>) [--enable] [--disable] [--new-description <new-description>] [--add-access (full-access|native-rest-all|native-rest-read|other-all)] [--remove-access (full-access|native-rest-all|other-all)]

Description

apikey-update can be used to enable or disable an API Key, and also update the description and access level associated with a key.

The description associated with an API Key should be changed if the key were being put to a new use, or had been transferred to another end user.

The access level of a key describes which external interfaces access has been granted to. Newly created keys have full-access, but the access level can be a combination of the following values:

full-access - allows full read/write access to all APIs

native-rest-all - allows full read/write access to the native RestAPI

native-rest-read - allows read only access to the native RestAPI

other-all - allows full read/write access to the SOAP APIs

Parameters

Exactly one of description or prefix must be specified:

-d, --description <description>

Specify the key by the description associated with it.

-p, --prefix <prefix>

Specify the key by the randomly generated prefix of the key.

--enable

Enables the specified API Key.

--disable

Disables the specified API Key.

-n, --new-description <new-description>

A new description to associate with the specified API Key.

-a, --add-access <access-level>

Adds an access level to the specified API Key. Valid choices are: (full-access|native-rest-all|native-rest-read|other-all)

-r, --remove-access <access-level>

Remove an access level from the specified API Key. Valid choices are: (full-access|native-rest-all|other-all)

Examples

Disable the key associated with the description "testkey":

server:$ apikey-update -d testkey --disable

Update the description associated with the key which has the prefix SzKdKXQxur:

server:$ apikey-update -p SzKdKXQxur --new-description "permanent key"

Disable full access from a key which has the prefix SzKdKXQxur:

server:$ apikey-update -p SzKdKXQxur --remove-access full-access

Enables read only access to the native RestAPI for a key which has the prefix SzKdKXQxur:

server:$ apikey-update -p SzKdKXQxur --add-access native-rest-read

Applies To

Cluster
