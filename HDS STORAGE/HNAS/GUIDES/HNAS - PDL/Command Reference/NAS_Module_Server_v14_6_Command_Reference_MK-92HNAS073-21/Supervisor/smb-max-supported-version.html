smb-max-supported-version

Set or display the maximum supported version of SMB

Syntax

smb-max-supported-version [ --default | 1 | 2 | 2.1 | 3 ]

Description

The smb-max-supported-version command is used to set or display the maximum supported version of SMB for the current EVS.

Usage:

smb-max-supported-version [ --default | 1 | 2 | 2.1 | 3 ]

Set the maximum supported version of SMB to 1, 2, 2.1, 3 or to its default value of 2.

If no argument is specified, the current maximum supported version is shown.

The maximum version specified must be greater than or equal to the minimum supported version.

For example, if smb-minimum-supported-version is SMB2, then it is acceptable to specify an smb-maximum-supported-version of SMB2, SMB2.1 or SMB3, but not SMB1.

This command affects SMB connections to the HNAS (ie normal file-serving connections) and SMB connections from the HNAS, for example to Domain Controllers or RPC-based virus scanners.

Examples

Display the current maximum supported version of SMB.

  smb-max-supported-version

Set the maximum supported version of SMB to its default of SMB2.

  smb-max-supported-version --default

Set the maximum supported version of SMB to SMB3.

  smb-max-supported-version 3

Applies To

EVS
