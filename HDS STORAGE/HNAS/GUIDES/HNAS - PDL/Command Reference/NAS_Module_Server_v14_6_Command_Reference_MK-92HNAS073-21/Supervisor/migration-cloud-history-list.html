migration-cloud-history-list

Lists cloud migration history for a specified policy.

Syntax

migration-cloud-history-list [--verbose] [[--running] | [--all] | [--recent]] [--source-fs-label <source-fs-label>] [--policy-name-or-id <policy-name-or-id> [--history-record-id <history-record-id>]]

Description

Lists the cloud migration job history.

--source-fs-label, -s <source-fs-label>

Name or id of the file system for which to list the history jobs.

--policy-name-or-id, -n <policy-name-or-id>

Name or id of the policy for which to list the history jobs.

--history-record-id, -i <history-record-id>

Id of the history job to display. Requires policy-name-or-id to be also specified.

--verbose, -v

List history jobs in a verbose mode

--all, -a

List all history jobs.

--recent, -r

List only recent history jobs.

--running, -R

List only active history jobs.

Examples

migration-cloud-history-list --policy-name-or-id DefaultPolicy --recent

Lists recent history for policy 'DefaultPolicy'.

migration-cloud-history-list --source-fs-label FileSystemName --running --verbose

Lists running history jobs in a verbose mode for the file system 'FileSystemName'.

migration-cloud-history-list

Lists 2 recent history jobs for each policy on this EVS.

set ext-migration-policy-history-records 40

A set variable can be applied to extend the default limit from 20 to 40.

Applies To

EVS security context
