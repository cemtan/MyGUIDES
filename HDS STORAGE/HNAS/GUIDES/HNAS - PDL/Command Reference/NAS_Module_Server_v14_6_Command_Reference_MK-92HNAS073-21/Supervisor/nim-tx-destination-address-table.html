nim-tx-destination-address-table

Dump Destination Discovery Table

Syntax

nim-tx-destination-address-table [--interface <interface> | --summary | --statistics | --reset]

Description

The nim-tx-destination-address-table command displays a record of the software Discovery table.

Without any options, the command displays the contents of the table.

-i, --interface <name>

Limits a table display operation to the entries associated with a specific interface.

-s, --summary

Outputs the number of entries in the Discovery table and the remaining free-space.

-s, --statistics

Outputs counters for the number of table entries created.

-r, --reset

Reset the statistics.

The table columns record the following information:

Local IP

The server IP address to which this entry relates.

Peer IP

The IP address of a remote machine to which this entry relates.

Next hop MAC

The MAC address to which all traffic for this entry will be sent. If the remote machine is within the same subnet, this will be a MAC address of that machine; otherwise it will be a MAC address of a local gateway. If the address is not yet resolved it will be 00-00-00-00-00-00.

TCP MTU

The MTU to be used for TCP traffic.

Oth MTU

The MTU to be used for non-TCP traffic.

Flags

Various flags that record the state of the entry in the VLSI table.

Examples

$ nim-tx-destination-address-table
         Local IP           Peer IP       Next hop MAC  TCP MTU Oth MTU Flags
     172.18.0.112         10.1.1.14  00-12-f2-e1-00-00     1500    1500 HW Completed
     172.18.0.112         10.1.16.4  00-00-00-00-00-00     1500    1500 HW
     172.18.0.112        172.18.0.1  00-e0-81-02-7a-99     1500    1500 HW Completed
     172.18.0.112        172.18.1.1  00-0c-29-48-22-f7     1500    1500 HW Completed
     172.18.0.112    172.18.255.255  ff-ff-ff-ff-ff-ff     1500    1500 HW Completed
     172.18.0.112       243.0.1.255  00-12-f2-e1-00-00     1500    1500 HW Completed

Applies To

Cluster node

Privilege Level

Supervisor
