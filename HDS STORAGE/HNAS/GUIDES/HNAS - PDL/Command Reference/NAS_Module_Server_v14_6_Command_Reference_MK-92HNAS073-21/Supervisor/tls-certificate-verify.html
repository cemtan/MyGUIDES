tls-certificate-verify

Check that the installed certificate is valid.

Syntax

tls-certificate-verify

Description

tls-certificate-verify is used to check that the installed SSL/TLS certificate is valid. It will only inform you of the area where something is wrong with the installed certificate. To find out what is wrong you should compare the output of this command with the output of tls-certificate-show --all which will show all information about the installed certificate and trust chain.

Examples

Valid

An example of output from a valid certificate:

$ tls-certificate-verify
Check that the installed certificate's identity is correct...
The identity of the cluster's Admin EVS is 'example.com'.
The identity of the cluster's Admin EVS matches that in the certificate.


Check that the current date is within the installed certificate's valid date range...
The installed certificate is valid between Feb 28 14:38:48 2013 GMT and Feb 26 14:38:48 2024 GMT.
The installed certificate has not expired.


All checks passed.

Invalid identity

An example of output when the identity of the certificate is not as expected. Note that the certificate's identity not matching the Admin EVS's FQDN is not necessarily an error.

$ tls-certificate-verify
Check that the installed certificate's identity is correct...
The identity of the cluster's Admin EVS is 'example.com'.
The identity of the cluster's Admin EVS does not match the identity of the certificate.
This is not necessarily a failure.


Check that the current date is within the installed certificate's valid date range...
The installed certificate is valid between Feb 28 14:38:48 2013 GMT and Feb 26 14:38:48 2024 GMT.
The installed certificate has not expired.


All checks passed.

Invalid date range

An example of output when the identity is correct, but the time range the certificate is valid for is not:

$ tls-certificate-verify
Check that the installed certificate's identity is correct...
The identity of the cluster's Admin EVS is 'example.com'.
The identity of the cluster's Admin EVS matches that in the certificate.


Check that the current date is within the installed certificate's valid date range...
The installed certificate is valid between Mar  3 09:19:26 2013 GMT and Mar  4 09:19:26 2013 GMT.
Verify failed: The installed certificate expires on Mar  4 09:19:26 2013 GMT. Install a new certificate before this date.


There were one or more failures.
Use the output of <tls-certificate-show --all> to help pinpoint the issue.

Applies To

Cluster wide
