migration-cloud-rule-create

Create an external migration rule

Syntax

migration-cloud-rule-create <rule name> --rule <rule> [--description <description>] [--case-sensitive <no|yes>]

Description

Creates cloud external migration rule.

<rule name>

Name of the rule, which must be unique. May be up to 30 ASCII characters long. The characters ;:/\'"<>*?| and a space character are not allowed.

--rule, -r <rule>

Syntax of the rule to be created. For details on rule syntax see the Administrator's Guide.

--description, -d <description>

Rule description. Any arbitrary string to help clients to manage rules. May be up to 50 ASCII characters long.

--case-sensitive, -c <no|yes>

Whether the rule should be treated as case sensitive or not. Default value is 'no'.

Examples

migration-cloud-rule-create JpgFilesRule --rule "INCLUDE (<FILENAME *.jpg> AND <INACTIVE_OVER 30 DAY> AND <UNCHANGED_OVER 30 DAY>)"

Creates a cloud migration rule named 'JpgFilesRule' that will match all files with .jpg extension, not case sensitive, that have not been accessed or modified for over 30 days.

Applies To

Global security context
