cacls-del

Delete ACLs and ACEs from a Security Descriptor

Syntax

cacls-del <acl id> | [<acl id> ace <ace id>] <file name>

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The cacls-del command is used to modify the Windows file access permissions for a set of files and/or directories by directly deleting parts of the Security Descriptors for those items.

The following options are available:

cacls-del

<acl id> <file name>

delete the ACL <acl id> for the file <file name>.

cacls-del

<acl id> ace <ace id> <file name>

delete the ACE in position <ace id> in ACL <acl id> for the file <file name>.

Meaning of terms

<acl id>

'sacl' or 'dacl'; used to select the appropriate ACL.

<ace id>

<uint16>

A <uint16> that selects, by position, the ACE to be deleted. Note that ACEs are numbered from 1, not 0.

<uint16>

an unsigned 16 bit integer expressed as a decimal number.

Examples

cacls-del dacl myfile

Delete the DACL for file myfile.

cacls-del dacl ace 2 myfile

For the file myfile, delete from within the DACL, the ACE at position 2

Applies To

EVS
