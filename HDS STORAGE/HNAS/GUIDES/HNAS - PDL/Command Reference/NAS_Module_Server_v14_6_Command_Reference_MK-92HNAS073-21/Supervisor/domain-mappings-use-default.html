domain-mappings-use-default

Sets whether the default domain mapping entry should be used if no other entries exist

Syntax

domain-mappings-use-default [--enable | --disable]

Description

Controls whether the default domain mapping should be used by the auto mapper.

Options:

--enable

Use the default domain mapping if no explicit mappings exist.

--disable

Never use the default domain mapping.

If neither the enable or disable option is specified, the current setting is displayed.

Examples

$: domain-mappings-use-default --enable

Applies To

EVS security context

See Also

domain-mappings-add domain-mappings-delete domain-mappings-list domain-mappings-replace
