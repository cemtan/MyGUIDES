mse-unset

Unset a minishell environment variable

Syntax

... | mse-unset <name>

Description

This command unsets a variable in the global minishell environment, making it undefined. It can also be run as a filter.

For a brief overview of the minishell environment, see the mse-env man page. For detailed information, see the mse man page.

Options:

<name>

Specify the name of the global minishell environment variable. But if the <name> is "--all", unset all variables in the global minishell environment.

Run as a filter

When run as a filter, mse-unset interprets each line of the output of the preceding command as the name of a global minishell environment variable to be unset. If the line contains an equal sign, the name is determined by discarding all characters from the equal sign to the end of the line. This behavior is supported so that the filter can be used to process the output of the 'mse-env' command.

Examples

1. Define a new global minishell environment variable named COUNT and set its value to 4:

   $ mse-set COUNT 4


   List all global minishell environment variables:


   $ mse-env
   COUNT=4


   Unset this environment variable:


   $ mse-unset COUNT


   Verify:


   $ mse-env --name COUNT
   mse-env: Undefined variable name: COUNT

2. Unset all environment variables whose names start with "STR":

   $ mse-env | grep "^STR" | mse-unset

3. Unset all variables in the global minishell environment

   $ mse-unset --all

4. Unset a variable in an mse script

The mse interpreter supports a built-in command called 'myenv-unset', which has the same syntax and functionality as mse-unset. The 'myenv-unset' built-in command unsets variables in the minishell environment currently in effect in the running instance of mse. For more information, see section D8.6 of the mse man page.

   $ mse 'x=5; myenv-unset x; myenv --name x'
   myenv: Undefined variable name: x
