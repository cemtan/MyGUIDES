replication-recovery-task-log-events

Description

This man page describes the DR log events for the recovery of NFS export and CIFS share mount points, and CNS and EVS links.

Error events

Share failed to recover.

Failed to recover the share. See DR log for further details.

Export failed to recover.

Failed to recover the export. See DR log for further details.

Link failed to recover.

Failed to recover the link. See DR log for further details.

Share deletion failure.

Failed to delete the share. See DR log for further details.

Export deletion failure.

Failed to delete the export. See DR log for further details.

Warning events

Failed to Log to DR log.

Failed to log an event to the recovery log.

This can happen for example when the log size exceeds the limit set by the DR logging policy. A paced event containing the details is logged to the console and the system log.

Share failed to recover due to name clash.

Failed to resolve the name clash with a new name. See DR log for further details.

Share name clash fixed.

The share was successfully recovered with a new name.

Export failed to recover due to name clash.

Failed to resolve the name clash with a new name. See DR log for further details.

Export recovered with name clash fixed.

The export was successfully recovered with a new name.

Export Id clash.

Failed to resolve the name clash due to a failure to generate a unique export Id. See DR log for further details.

Info events

Share recovered.

Successfully recovered the share.

Share skipped (name clash).

The share was not recovered due to a name clash. Use the --fix-name-clash option to recover it.

Share skipped (do-not-transfer).

The share was not recovered as its --disaster-recovery-setting field is set to 'do-not-transfer' setting.

Share skipped (use-fs-default).

The share was not recovered as its --disaster-recovery-setting field is set to 'use-fs-default', and either the file system's default DR transfer setting is disabled or the --override-fs-dr-default option is used with 'do-not-transfer' setting.

Share skipped (name clash with identical).

The share was not recovered due to a name clash with an identical share. Use the --do-not-skip-identical option to recover it.

Export recovered.

Successfully recovered the export.

Export skipped (name clash).

The export was not recovered due to a name clash. Use the --fix-name-clash option to recover it.

Export skipped (do-not-transfer).

The export was not recovered as its --disaster-recovery-setting field is set to 'do-not-transfer' setting.

Export skipped (use-fs-default).

The export was not recovered as its --disaster-recovery-setting field is set to 'use-fs-default', and either the file system's default DR transfer setting is disabled or the --override-fs-dr-default option is used with 'do-not-transfer' setting.

Export skipped (name clash with identical).

The export was not recovered due to a name clash with an identical share. Use the --do-not-skip-identical option to recover it.

Share deleted.

Successfully deleted the share.

Export deleted.

Successfully deleted the export.

Link recovered.

Successfully recovered the link. The --rebind-existing-export option was used and the link was rebound to the promoted file system.
