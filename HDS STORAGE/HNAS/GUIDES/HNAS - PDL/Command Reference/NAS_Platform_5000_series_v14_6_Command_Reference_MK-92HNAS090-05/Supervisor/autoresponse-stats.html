autoresponse-stats

Display statistics for autoresponse

Syntax

autoresponse-stats [--reset] [--type (all|nfs|smb1|smb2)]

Description

The autoresponse-stats command is used to report or reset the current counts of protocol requests which have been autoresponded by the server.

Usage:

autoresponse-stats [--reset] [--type <protocol_type>]

Options:

-r, reset

Reset the selected counters back to 0

-t, --type <protocol_type>

Selects a set of protocol stats. Valid choices are:

(all|nfs|smb1|smb2).

Notes: The 'nfs' option specifies NFSv2/v3. The 'smb2' option specifies SMB2.x/3.x. Defaults to all.

The statistics are grouped into three sections:

Autoresponse:

Total requests - Count of all requests seen, regardless of suitability for autoresponse.

AR requested - Count of requests identified by the protocol as potentially suitable for autoresponse.

AR sent - Count of requests for which autoresponse was generated and sent.

AR cancelled by SW - Count of requests lower-level (filesystem) software determined were not suitable for autoresponse.

Autoresponse not requested reasons:

Protocol-specific reasons that autoresponse was not requested by software.

Autoresponse cancelled reasons:

Reasons autoresponse was cancelled by lower-level (filesystem) software despite being requested.

Examples

autoresponse-stats --type all

Displays the number of autoresponded operations for every protocol.

autoresponse-stats

Also displays the number of autoresponded operations for every protocol.

autoresponse-stats --type smb1 --reset

Resets the count of SMB1 operations that have been autoresponded.

Applies To

Cluster node
