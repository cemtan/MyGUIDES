ignore-unix-security-ops

Configures whether the server ignores NFS operations that attempt to set or change file/directory security attributes

Syntax

ignore-unix-security-ops [--enable | --disable]

Description

The ignore-unix-security-ops command configures whether the server ignores NFS operations that attempt to set or change file/directory security attributes.

By default this setting is disabled.

When enabled, security can only be managed via CIFS. Files/directories created over NFS will inherit security from the parent directory, or it will be derived from the server's file/directory umask if the parent directory has no inheritable security. If an NFS client attempts to change the security of an existing file/directory (e.g. chmod/chown/chgrp) then the operation is succeeded without making the change requested.

Examples

server:$ ignore-unix-security-ops
ignore-unix-security-ops: disabled


server:$ ignore-unix-security-ops --enable
ignore-unix-security-ops: enabled
