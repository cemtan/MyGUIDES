maintenance

Suppresses alerts for the specified number of hours

Syntax

maintenance start <hours>|stop|status

Description

The maintenance command allows the user to throttle SMTP alerts for a set period of time, usually during a maintenance window when a large number of events might be expected. Options:

start <n>

Start throttling alerts for the specified number of hours

stop

Stop throttling alerts

status

Show if maintenance window active and when it finishes

The throttling stops when the command is run with "stop", the cluster node is rebooted, or the number of hours passes. An event is logged when the command is run, or when the time expires.

Examples

To throttle events for maintenance expected to last 2 hours

maintenance start 2

Applies To

Cluster node
