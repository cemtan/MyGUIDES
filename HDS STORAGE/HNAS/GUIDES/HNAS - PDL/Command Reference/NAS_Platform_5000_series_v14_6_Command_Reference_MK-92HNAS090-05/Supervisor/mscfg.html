mscfg

Set management server configuration

Syntax

mscfg <server> (listhosts | delall [confirm] | port <port> [confirm] | maxconns <conns> | enable | disable [confirm] | restart | restrict (on [confirm] | off) | evsipaddr (on|off) | addhost <host> | removehost <host>)

Description

The mscfg command configures various parameters for the various management server interfaces to the server.

Valid servers are:

HTTP

Remote management (API access)

HTTPS

Remote management (secure API access)

rest

Native RestAPI management

ssc

Command line (CLI)

vss

VSS hardware provider DLL connection

Options for setting up a restricted list of clients allowed to access each server are

listhosts

List the authorized hosts (default option)

addhost <host>

Add a host to the list of authorized hosts. The host can be in any of the following formats:

  • An IPv4 address in dot notation (e.g. 203.0.113.0)
  • An IPv4 address range in CIDR notation (e.g. 203.0.133.0/24)
  • An IPv4 address range in wildcard notation (e.g. 203.0.133.*)
  • An IPv6 address in colon notation (e.g. 2001:db8:fe58:3efa:deca:212::c0a9)
  • An IPv6 address range in prefix-length notation (e.g. 2001:db8::/32)
  • A hostname
removehost <host>

Remove a host from the list of authorized hosts

delall [confirm]

Delete all authorized hosts. If the "confirm" argument is not given then the user is prompted to confirm the action before any changes are made.

restrict (on [confirm]|off)

Enable or disable restriction of the named server to authorized hosts. If restriction is being turned on for the HTTPS or SSC server and the "confirm" argument is not given then the user is prompted to confirm the action before any changes are made.

evsipaddr (on|off)

Allow / disallow use of EVS IP addresses by server. This will be disabled and have no effect if multi-tenancy is enabled.

Other options:

port <portno> [confirm]

Configure the service to run on the specified port. If the port is being changed for the HTTPS or SSC server then and "confirm" argument is not given then the user is prompted to confirm the action before any changes are made.

maxconns <number>

Specify the maximum number of connections to the named server

enable

Enable the named server

disable [confirm]

Disable the named server. If the HTTPS or SSC server is being disabled and the "confirm" argument is not given then the user is prompted to confirm the action before any changes are made.

restart

Stops the server and then starts it again.

Examples

Disable the SSC management server (the "confirm" argument is not given so the user will have to press "y" when prompted to confirm the action):

mscfg ssc disable

List the hosts allowed to talk to the SSC server:

mscfg ssc listhosts

Applies To

Cluster wide
