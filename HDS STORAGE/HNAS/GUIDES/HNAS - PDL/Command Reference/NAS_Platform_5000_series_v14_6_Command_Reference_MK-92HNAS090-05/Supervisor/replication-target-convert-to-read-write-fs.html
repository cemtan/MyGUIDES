replication-target-convert-to-read-write-fs

Converts a replication target file system to be generally modifiable, and not modifiable by object replication

Syntax

replication-target-convert-to-read-write-fs <file-system>

Description

This command converts the specified file system so that it will be read-write on next mount, and non-modifiable by the object replication engine. The specified file system must be unmounted when this command is run.

This command would generally be used when it becomes necessary to allow network clients (e.g. NFS or CIFS) read-write access to the file system, perhaps because the object replication source file system is unavailable.

Note that once a file system is mounted read-write, it can not successfully act again as a replication target until the live version of the file system has been rolled back to a valid replica of object replication source file system.

Examples

To make a replication target file system accessible as a normal read-write file system:

  replication-target-convert-to-read-write-fs fs1

Applies To

EVS

See Also
