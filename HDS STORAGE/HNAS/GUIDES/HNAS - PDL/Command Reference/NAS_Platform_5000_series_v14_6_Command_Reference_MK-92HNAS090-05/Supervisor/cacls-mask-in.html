cacls-mask-in

Mask (logically OR) in Security Descriptor element values

Syntax

cacls-mask-in [<acl id> ace <ace id> <mask>+]+ | control <uint16>  <file name>

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The cacls-mask-in command is used to set the Windows file access permissions for a file or directory by logically ORing a value into a given field.

The following option is available:

cacls-mask-in

<acl id> ace <ace id> <mask>+ <file name>

mask in <mask> in the ACE in position <ace id> in the ACL <acl id> for file <file name>. Multiple <mask> statements can be given to mask-in multiple fields in the same ACE.

The following option is not currently supported but the syntax is listed for completeness:

cacls-mask-in

control <uint16> <file name>

mask in Security Descriptor control mask with <uint16> for file <file name>.

For a description of the meaning of values that can be applied to fields within a Security Descriptor see the cacls-fields manual page.

Meaning of terms

<mask>

Set the nominated maskable field from the following:

<expression shortcut>

flags <hex8>

flags <f | d | p | i | I | S | F>

mask <hex32>

mask <r | w | a | x | d | D | t | T | n | N | c | C | o | y>

<acl id>

'sacl' or 'dacl'; used to select the appropriate ACL.

<expression shortcut>

The following shortcuts effectively provide a convenient mask for a set of values but without the need to use the mask keyword. See the cacls-fields manual page for details of the values associated with each shortcut.

read

write

readexecute

modify

fullcontrol

<uintN>

an unsigned 'N' bit integer expressed as a decimal number, e.g. uint8, uint16, uint32.

<hexN>

an unsigned 'N' bit integer expressed as a hexadecimal number, e.g. hex8, hex16, hex32.

<ace id>

<uint16>

A <uint16> that selects, by position, the ACE to be modified. Note that ACEs are numbered from 1, not 0.

Examples

cacls-mask-in dacl ace 1 mask rwT myfile

For the file myfile, update the ACE that is at position 1 in the DACL. In that ACE, OR into the mask field the bits for Read Data / List Contents, Write Data / Create Child and Write Attributes.

cacls-mask-in dacl ace 1 flags 0x1 myfile

For the file myfile, update the ACE that is at position 1 in the DACL. In that ACE, OR into the flags field the value 0x1

cacls-mask-in dacl ace 1 flags i mask r dacl ace 2 mask w myfile

For the file myfile, update the ACE that is at position 1 in the DACL. In that ACE, OR into the flags field the bit for Inherit Only ACE and then OR into the mask field the bit for Read. Then update the ACE that is at position 2 in the DACL. In that ACE, OR into the mask field the bit for Write.

Applies To

EVS
