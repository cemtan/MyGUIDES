global-id-cache-dump

Dump current state of Global Id change cache

Syntax

global-id-cache-dump

Description

There are a number of objects on the NAS server which are tracked and cached based on a globally unique identifier. There are a series of caches of these globally unique identifiers which are used to identify when individual objects have been changed in some way, i.e. added, updated, or deleted. This command is used to print out all of these caches and hence the output can be quite large.

SCOPING

There is one cache per tracked class on each cluster node regardless on the natural scope of the objects identified by the cached globally unique identifiers. For a description of all the tracked classes and their scoping see the global-id-map-dump manual page.

Examples

Display all caches:

global-id-cache-dump

Applies To

Cluster node
