virtualization-path-modify

Modifies a virtualization path for the specified target vivol/volume.

Syntax

virtualization-path-modify [--characterset {latin1|utf8} | --port <port> | --mountport <mountport> | -s <source>] -t <fs label> -n <association name>

Description

Changes the source URI associated with a pre-existing virtualization path that is mapped to a local file system, vivol or directory target.

This command will change the location that data is received from when scanning for entries to virtualize. After virtualization process has started changes by virtualization-path-modify command are no longer allowed. To change the source in such case the virtualization path needs to be deleted and created again.

Options:

--characterset <latin1|utf8>

Character set of the legacy NAS source.

--port <port>

NFS port of the legacy NAS source.

--mountport <mountport>

Mount port of the legacy NAS source.

-t, --target <fs name>

The name of the filesystem containing the association

-n, --name <association name>

The name of the association to be modified

-s, --source <legacy NAS path URI>

Legacy NAS path URI. The URI can contain a hostname, an IPv4 literal address, or an IPv6 literal address.

Examples

virtualization-path-modify -t fs1 -n assoc1 -s nfs://legacy.example.com/example_fs

Sets nfs://legacy.example.com/example_fs as the new path associated with the association 'assoc1' on file system fs1.

Applies To

EVS
