migration-cloud-schedule-create

Create an external cloud migration schedule

Syntax

migration-cloud-schedule-create <policy name> <run-type> [ ( --next-run | -n ) <run-time> ] [ ( --max-duration | -d ) <max-duration>]
where
  <run-type> = simulateOnly | onceOnly | daily | weekly
  <run-time> = YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]
  <max-duration> = [nD]hh:mm

Description

Create a schedule for cloud migration.

<policy name>

Name of the cloud migration policy which this schedule applies.

<run type>

Schedule run type, allowed values are:

onceOnly

Run a cloud migration once at the next run time.

daily

Run a cloud migration at the next run time, and at the same time every day after that.

weekly

Run a cloud migration at the next run time, and then at the same time once every week after that.

simulateOnly

Run a cloud migration detailed test once at the next run time. This exercises the selected policy, but no files are actually migrated.

--next-run <run-time>

The next run time in the format YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]. Defaults to the current time.

--max-duration <limit>

The maximum amount of time a given migration run started by this schedule will be allowed to run. If this limit is hit then the migration will be halted until the next scheduled run-time, if any.

The limit is in the format [nD]HH:MM. For example, 1D12:30 is 1 day, 12 and one half hours (or 36.5 hours).

If the --max-duration option is not given, the default is no limit.

Examples

migration-cloud-schedule-create DefaultPolicy daily --next-run 2011-02-03T03:30:00-08:00

Creates a cloud migration schedule that runs the policy 'DefaultPolicy' on its associated file system every day, beginning 3 February 2011 at 3:30am PST.

migration-cloud-schedule-create DefaultPolicy weekly -d 1d18:00

Creates a cloud migration schedule that runs the policy 'DefaultPolicy' on its associated file system once each week, starting now, with each weekly run lasting at most 1 day and 18 hours.
