ndmp-user

Set or get NDMP user information

Syntax

ndmp-user [<user-name> <password>]
Alias:       configuser

Description

ndmp-user sets or displays the NDMP username and password. The NDMP username and password are used to authenticate NDMP clients. These must be used when configuring a backup application to access the server over NDMP. They must also be supplied in adc copies and in SMU replications when the destination machine is unmanaged. Where replication or data migration controlled by the SMU runs on managed servers, the NDMP user and password are still used and must be set using ndmp-user or the GUI equivalent. However, it will not be necessary to supply them as part of the configuration of the replication or data migration.

Note that the NDMP username and password allow access to all data on disks in the scope of that EVS security context. It effectively gives "root" permissions. Therefore, it is very important to ensure that the password is secure.

Typing the command without parameters will display only the current NDMP username. The password is not shown.

If for some reason NDMP access is needed by non-administrative staff for limited use then the NDMP restricted user facility can be used instead. See ndmp-ruser for details.

Examples

ndmp-user ndmproot xwX49L

Sets the NDMP username to "ndmproot" and the password to "xwX49L".

ndmp-user

Displays in plain text the current NDMP username.

Applies To

EVS security context

See Also

ndmp-option ndmp-ruser ndmp-status

Privilege Level

Supervisor
