sd-group-add

Add system drives to an existing group

Syntax

sd-group-add [(--set-tier | -t) <tier>] <device-ID-of-SD-in-group>... <device-ID-of-SD-to-add-to-group-see-'man sd-spec'>...
Alias:       adsdg

Description

Adds one or more system drives (SDs) to an existing SD group. See the 'sd-group' man page for background.

Specify at least one SD from the group you want to extend and at least one SD that is not yet in a group. The SDs can be specified in any order: the order of SDs in a group is not significant.

The SDs you add to the group must not be in an open span. (A span is open if any of its file systems is mounted or is being checked, fixed, etc.) The SDs that are already in the group when you run 'sd-group-add' are not subject to this restriction.

The --set-tier switch assigns the new SDs to the same tier as those already in the SD group. If the tier assignments cannot be made, no SD groups are created. If --set-tier is not provided, the new SDs must be in the same tier as those already in the group before the command is run.

Examples

server:$ sd-group-list -m
Serial: 0 1


server:$ sd-group-add 0 2 3
Success
server:$ sd-group-list -m
Serial: 0 1 2 3


server:$

You can specify any SD from the existing group; it need not be the first. You can also specify more than one SD from the existing group if it is more convenient. So the command demonstrated here could have been written in any of the following ways:

sd-group-add 0 2 3
sd-group-add 1 2 3
sd-group-add 1-3
sd-group-add 0-3

Applies To

Cluster wide
