smb3-encryption-enable

Enable SMB3 encryption on the current EVS

Syntax

smb3-encryption-enable

Description

The smb3-encryption-* commands determine whether SMB encryption is enabled on an EVS. These commands are only available on an EVS if it has the maximum supported version set to SMB3.

Effect on clients using SMB3:

When enabled, the use of encryption is enforced on the current EVS. Any unencrypted clients will be rejected.

Effect on clients using SMB2:

Works in combination with the smb3-reject-unencrypted-access-* commands. If SMB3 'reject unencrypted access' is disabled on this EVS then enabling or disabling EVS encryption has no effect. If SMB3 'reject unencrypted access' is enabled on this EVS then EVS encryption when enabled causes SMB2 clients to be rejected from this EVS.

Effect on clients using SMB1:

Has no effect. Access by SMB1 clients can only be prevented by turning off the SMB1 server. The SMB1 server can be turned off by using the smb-min-supported-version command to set the minimum supported version to SMB2

smb3-encryption-status

Display the encryption status of the current EVS.

smb3-encryption-enable

Enable encryption on the current EVS.

smb3-encryption-disable

Disable encryption on the current EVS.

smb3-encryption-default

Set the encryption status of the current EVS to the default state of disabled.

Applies To

EVS
