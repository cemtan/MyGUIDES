directory-services-config-dump

Dumps directory services configuration for an EVS in context

Syntax

directory-services-config-dump

Description

The directory-services-config-dump command displays the directory services information for current EVS security context. The main purpose of this command is to allow augmentation scripts to automatically retrieve this information. Hence, it is displayed in a machine-readable format using Ruby syntax and resembling the syntax of the ldap.conf.rb file. Currently, it has been implemented for NIS and LDAP only.

This command accepts no arguments. The directory services have to be configured using other commands: domainname, ldap-schema, ldap-search-config, ldap-security, ldap-server, nis-ldap-mode and nis-server.

Examples

Display the current NIS and LDAP settings:

vn 1 directory-services-config-dump


NISDomain = "organization.example.com"
NISServer = nil
Schema = LdapSchema::RFC2307
Context = "CN=Directory Manager"
Transport = :simple_tls
Scope = Net::LDAP::SearchScope_WholeSubtree
SearchUsers = nil
SearchGroups = "ou=Groups,DC=organization,DC=example,DC=com"
SearchNetgroup = nil
SearchHost = nil
Search = "DC=organization,DC=example,DC=com"
Host = "10.0.0.10"

Applies To

EVS security context
