migration-recreate-links-mode

Configures the recreation mode for migrated files

Syntax

migration-recreate-links-mode [always-recreate-links | test-before-recreate]

Description

The migration-recreate-links-mode command is used to configure how to recreate external migration links during a replication or restore operation which is using the "re-create links" option.

The setting affects the current EVS.

The following options are available:

always-recreate-links

Always recreate an external link, even if the target file on the external migration target is not currently accessible.

This mode is useful when parallel migrations are being performed (a) for a server file system containing externally migrated files and (b) for the third party external migration target storage system. In this situation it is not always possible to determine whether the third party storage system will have replicated a particular file before the server attempts to recreate the external migration link to it.

In particular, this mode will provide benefits when combined with replications using the "re-create links" option for externally migrated files.

test-before-recreate

Test whether the target file on the external migration target is accessible, and only recreate the external migration link to it if the file can be accessed.

This is the default.

Examples

To display the current setting:

$ migration-recreate-links-mode

To change the current setting to "always-recreate-links":

$ migration-recreate-links-mode always-recreate-links

Applies To

EVS
