namespace-delete

Removes a cluster or EVS namespace

Syntax

namespace-delete  [--confirm] <cluster or EVS namespace>
Alias:       cns-delete

Description

Removes a cluster or EVS namespace including any directories or links created within it.

The namespace that is accessible using this command depends on the type of node in context.

- If an admin node or an EVS node which is using global EVS security is in context, then only a cluster-wide namespace may be deleted using this command.

- If an EVS node which is using individual EVS security is in context then only a namespace local to that EVS may be deleted using this command.

Use console-context to set the proper context.

Note this command will not have any effect on file systems referenced by the links. This command will ask for confirmation unless the --confirm option is specified.

Usage:

namespace-delete [--confirm] <cluster or EVS namespace>

Examples

namespace-delete cns

This will delete a namespace named 'cns' if it is available under the current node in context.

Applies To

Cluster wide

See Also
