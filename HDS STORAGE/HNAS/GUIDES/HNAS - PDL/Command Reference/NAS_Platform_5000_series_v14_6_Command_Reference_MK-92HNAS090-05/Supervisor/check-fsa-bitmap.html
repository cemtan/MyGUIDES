check-fsa-bitmap

Schedules a check of the FSA bitmap on the specified, unmounted file system

Syntax

check-fsa-bitmap [-a] -f <file-system>

Description

Performs a check of the FSA bitmap for a given file system. If diagnostics indicate that the FSA bitmap is corrupt, then check-fsa-bitmap can identify corruption quicker than would checkfs. However, it is possible that check-fsa-bitmap will erroneously flag over-allocations. Such warnings are best ignored.

If more widespread corruption is suspected, then checkfs or check-directory is the appropriate tool to use.

Users should consult the event log to see the results of the check or the dynamic-log for more detailed analysis.

check-fsa-bitmap will only run on a file system that is unmounted. check-fsa-bitmap is an asynchronous utility.

Options:

-a

Abort the instance of this utility.

Allocation discrepancy warnings due to file system conversion to support dedupe

fs-convert-to-support-dedupe may leave behind some discrepancies in the interest of speed. checkfs invoked internally by check-fsa-bitmap reports these as "allocation type discrepancies possibly due to conversion to support dedupe". Refer to checkfs man page for details.

Examples

To check the FSA bitmap on the file system fs1:

check-fsa-bitmap -f fs1

Applies To

EVS
