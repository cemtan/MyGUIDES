filesystem-remake-tiers

Recalculates the MTFS state for a file system

Syntax

filesystem-remake-tiers [--remake-on-next-mount] <filesystem-label>

Description

When you create a file system on a tiered span, the server creates structures on the file system to help it write data efficiently to the correct tier. If you later discover that SD tiers were set up incorrectly before the file system was created, these structures will need to be recreated after SD tiers have been corrected. This command will recreate those structures. The file system must be unmounted when the command is executed.

Options

--remake-on-next-mount

Instead of recreating the structures immediately, the command puts a flag on the file system so that structures are recreated the next time the file system is mounted read/write. Consider using this option if several file systems are affected, because the server can mount several file systems at the same time, parallelizing the recalculations. Note this option is not supported for WFS-2 file systems that support dedupe.

Examples

Rebuild the tiered structures on file system "fs0":

filesystem-remake-tiers fs0

Rebuild the tiered structures on file system "fs1" the next time it is mounted read/write (not supported on WFS-2 file systems that support dedupe):

filesystem-remake-tiers --remake-on-next-mount fs1
