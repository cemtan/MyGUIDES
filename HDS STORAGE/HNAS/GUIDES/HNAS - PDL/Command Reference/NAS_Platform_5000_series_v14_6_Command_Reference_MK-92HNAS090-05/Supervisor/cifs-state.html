cifs-state

Show count of CIFS objects in use

Syntax

cifs-state

Description

The cifs-state command shows the resources used by CIFS clients. If the resource is limited by the size of a heap pool or a fixed limit, such as max-network-open-files, then the lowest limiting factor is shown. The number of persistent opens can sometimes slightly exceed its limit.

Usage:

cifs-state

Examples

server(FSA):$ cifs-state
Current SMB1 state:
 Session handles: 0 out of 32583 (limited by Session heap pool size)
  (there is space for up to 1832659 more SIDGroupSet elements of size 136 B in LongTermSizeBasedPool-136,
   there are 0 SMB1 Sessions and 0 FTP Sessions giving an average of 0 SIDGroupSet elements per session)
    Tree handles: 0 out of 32583 (limited by Tree heap pool size)
    File handles: 0 out of 108435 (limited by max-network-open-files)
  Search handles: 0 out of 10843 (limited by Search heap pool size)


Current SMB2.x and SMB3.x state:
       Session handles: 0 out of 32583 (limited by Session heap pool size)
  Tree Connect handles: 0 out of 32583 (limited by Connection heap pool size)
 Volatile file handles: 0 out of 108435 (limited by OpenFile heap pool size)
  Durable file handles: 0 out of 108435 (limited by OpenFile heap pool size)
      Persistent opens: 0 out of 10843 (limited by one tenth of the OpenFile heap pool size)

Applies To

Cluster node
