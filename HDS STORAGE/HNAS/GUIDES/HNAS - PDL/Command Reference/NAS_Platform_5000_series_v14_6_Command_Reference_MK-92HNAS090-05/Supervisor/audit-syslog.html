audit-syslog

Configure file system audit logging by syslog

Syntax

audit-syslog <operation>
where <operation> is one of:
    list
    add <server> [facility]
    del
    mod <server> [facility]
    facilities

Description

Configures file system audit log events to be sent to a syslog server.

Usage:

list

Displays the current configuration.

add <server> [facility]

Adds a new syslog server.

del

Deletes the syslog server.

mod <server> [facility]

Modifies the configured syslog server.

facilities

Displays the list of known syslog facility names and their corresponding numeric values.

The audit-syslog command configures whether file system audit log events should be sent to a syslog server using the BSD Syslog Protocol (RFC 3164).

It is possible to configure both which syslog server audit events should be sent to, and the syslog facility that should be used.

The syslog server can be specified as a resolvable hostname, or an IP address (IPv4 or IPv6 addresses can be used). The facility can be specified by name or numeric value, and if not specified the "auth" facility (4) will be used.

Syslog messages generated from a Success audit event will be sent with a severity of Information (6), and those from a Failure audit event will be sent with a severity of Notice (5). These values cannot be changed.

Examples

To send file system audit events to a syslog server "logserver" using the "auth" facility:

audit-syslog add logserver auth

To show the current configuration:

$ audit-syslog list
Server:   logserver
Facility: auth (4)
