span-set-default-cap-warn-thresh

Set the default span capacity warning threshold

Syntax

span-set-default-cap-warn-thresh <percentage> | --default

Description

If filesystem auto-expansion leaves a span full, or nearly so, then the server logs an event to notify the administrator that filesystems cannot auto-expand much further. Warnings are also issued by 'trouble' when a span is nearly or actually full. The percentage fullness at which these warnings start is formally known as the capacity warning threshold.

This threshold can be set for each span individually by 'span-set-cap-warn-thresh'. However, a newly created span defaults to the percentage set by 'span-set-default-cap-warn-thresh', the command described in the present man page. If that command also has not been used, a newly created span uses the default default threshold set by the software release that creates it, which is currently 90%.

Setting a good default can avoid the need to change the threshold for newly created spans. For example, setting it to zero will disable warnings for all new spans; this can be desirable if spans are deliberately kept almost full.

When a span is created, a capacity warning threshold is written to span Cod. The span's threshold does not change unless 'span-set-cap-warn-thresh' is used, even if a new release changes the default or 'span-set-default-cap-warn-thresh' is used to set a new default.

Examples

span-set-default-cap-warn-thresh 95

Increases the default threshold to 95% for spans created from now on.

span-set-default-cap-warn-thresh 0

Disables capacity warnings by default for spans created from now on.

span-set-default-cap-warn-thresh --default

Returns to the default set by the software release, which is currently 90%. Newly created spans will use whichever default percentage applies in the software release under which the spans are created.

Applies To

Cluster wide
