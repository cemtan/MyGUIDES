sd-tier-import

Import SD tiers from a text file

Syntax

sd-tier-import <filename>

Description

Reads a file produced by 'sd-tier-export'; imports system drive tiers into the registry. Overwrites any existing SD tiers for SDs specified in the import file, but leaves other SD tiers unchanged.

Inside an SSC session, the command will look for the import file on your local client. In other environments, the file will be read from the server's local file system.

This command is rarely needed. There is no need to make or restore regular backups of a server's SD tiers: tiers are stored in the registry, and a registry backup will back them up. Exporting and importing tiers is part of the process of migrating a span between clusters, but the SMU can carry out that task more simply and efficiently.

Files of 128KiB or larger can't be imported. To work around the limit, either split the import file manually or (more conveniently) export fewer SD tiers at once.

Examples

sd-tier-import some-sds.txt

Applies To

Cluster wide

See Also
