global-id-counter-dump

Dump current state of Global Id change counters

Syntax

global-id-counter-dump

Description

There are a number of objects on the NAS server which are tracked and cached based on a globally unique identifier. There are a series of caches of these globally unique identifiers which are used to identify when individual objects have been changed in some way, i.e. added, updated, or deleted. For each of these caches there is a counter which increments each time an entry is made in the cache. This command is used to print out all of these counters.

The counter value is needed when running any of the cache querying commands, e.g. global-id-cache-query-export. Note that the counter does not start from zero and although it increments by one each time, each increment may well apply to only one evs.

Examples

Display all caches:

global-id-counter-dump

Applies To

Cluster node
