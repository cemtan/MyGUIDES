ipeng

Report/set engineering IP configuration

Syntax

ipeng 	 [-g <interface>] [-w <bytes>] [-y on|off] [-s on|off]
	 [-r on|off] [-n on|off] [-a on|off] [-i on|off] [-f on|off]
	 [-l on|off] [-q on|off] [-o <min_rto>] [-O <max_rto>] [-k on|off]
	 [-T on|off] [-S on|off] [-P on|off] [-W on|off] [-F <factor>]
	 [-R on|off] [-d] [-x init|create|delete|list|show-all]

Description

The ipeng command permits manipulation of low level engineering IP parameters. Only use under instruction from your support provider.

Some of these parameters can be set on a per-interface basis, the rest are global settings.

To create a specific configuration for an interface use "-x create -g <interface>" Note that an interface-specific configuration must be created before specific values can be set for that interface, the interface-specific configuration can be created prior to creating the interface.

To delete an interface-specific configuration use "-x delete -g <interface>"

The following options are available:

-w, --def-wnd <bytes>

Sets default window size. The valid range is 250 to 65534 bytes.

-y, --delayedacks on|off

Enable/disable delayed acks.

-s, --slowstartca on|off

Enable/disable slow start congestion avoidance. This may be configured on a specific interface.

-r, --ssrestartdoublemss on|off

Enable/disable SS restart double mss.

-n, --nagle on|off

Enable/disable nagle.

-a, --sillywindowavoid on|off

Enable/disable silly window avoid.

-i, --slowstartonidle on|off

Enable/disable slow start on idle.

-f, --fastretxfastrecovery on|off

Enable/disable fast retx fast recovery.

-l, --intelliseg on|off

Enable/disable intelliseg.

-q, --quickr1teardown on|off

Enable/disable quick R1 teardown.

-o, --minimumrto <minrto>

Sets the TCP minimum RTO in milliseconds. The valid range is 300 to 1000 milliseconds.

-O, --maximumrto <maxrto>

Sets the TCP maximum RTO in milliseconds. The valid range is 10000 to 240000 milliseconds.

-k, --loosekarn on|off

Enable/disable use of loose Karn.

-T, --stricttimewait on|off

Enable/disable use of strict TIME_WAIT.

-S, --sack on|off

Enable/disable use of TCP SACK. This may be configured on a specific interface.

-P, --paws on|off

Enable/disable use of TCP PAWS.

-W, --windowscaling on|off

Enable/disable use of TCP window scaling. This may be configured on a specific interface.

-F, --windowscalefactor <factor>

Sets the TCP window scale factor. The valid range is 0 to 14.

-R, --tcp-rate-control on|off

Enables the TCP Rate Control feature. This may be configured on a specific interface.

-p, --path-mtu-timeout

Sets the Path MTU timeout. The valid range is 300 to 3600 seconds.

-d, --default

Restores the default IP Engineering configuration. Must be the only option used.

-g, --aggportname

Name of interface on which to act.

-x, --xtraaction

Specifies an extra action to take (init, create, delete, list, show-all).

Note that settings are not applied to eth0, eth1 or aggregate Linux interfaces (e.g. eth-ag1).

Examples

To set a value globally:
    Change the default window to 32000 bytes:
    ipeng -w 32000


To create a specific configuration for port ag1:
    ipeng -x create -g ag1


To set a specific value for a specific port:
    Disable Selective Acknowledge just on port ag1:
    ipeng -g ag1 -S off


To delete a specific configuration for port ag1:
    ipeng -x delete -g ag1


To show the current global configuration settings and the default values:
    ipeng


To show the current configuration settings for port ag1:
    ipeng -g ag1

Applies To

Cluster wide
