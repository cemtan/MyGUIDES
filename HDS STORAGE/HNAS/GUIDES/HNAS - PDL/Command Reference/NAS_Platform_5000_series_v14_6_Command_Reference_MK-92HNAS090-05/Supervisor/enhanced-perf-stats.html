enhanced-perf-stats

Control enhanced performance statistics gathering

Syntax

enhanced-perf-stats [--enable | --disable] [--not-persistent]

Description

Enables/disables enhanced statistics gathering.

Options:

--enable

Enable the gathering of enhanced statistics information.

--disable

Disable the gathering of enhanced statistics information.

--not-persistent

By default, the setting is persistent across reboots. This switch prevents the setting from being saved.

NB. the --not-persistent option applies the changes to the selected pnode only. Use "for-each-cnode " prefix to broadcast non-persistent changes

The maintenance of some statistics can cause a decrease in system performance, and by default these statistics are not gathered. We estimate the overhead to be in the order of 10%.

These statistics will be automatically enabled when a performance info report is running.

The setting affects all nodes in a cluster.

Examples

  server:$ enhanced-perf-stats

Queries the current enhanced-perf-stats setting

  server:$ enhanced-perf-stats --enable

Enables enhanced-perf-stats gathering

  server:$ enhanced-perf-stats --enable --not-persistent

Enhanced-perf-stats are enabled, but without changing the persistent config. This command only applies to the current pnode

  server:$ for-each-cnode enhanced-perf-stats --enable --not-persistent

Enhanced-perf-stats are enabled, with no persistent config change, as the "for-each-cnode" command is added before the base command this change is broadcast to all pnodes.

  server:$ for-each-cnode enhanced-perf-stats

This will display the enhanced-perf-stats status on all pnodes on the current pnode.

Applies To

Cluster wide Unless --not-persistent is specified, in which case only the current cluster node will be affected
