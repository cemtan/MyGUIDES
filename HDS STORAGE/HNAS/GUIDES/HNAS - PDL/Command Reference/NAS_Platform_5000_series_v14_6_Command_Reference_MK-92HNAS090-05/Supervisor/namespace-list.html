namespace-list

Displays the name of the configured namespace and, optionally, its links

Syntax

namespace-list [-l]
Alias:       cns

Description

Displays the name of the cluster or EVS namespace and, optionally, its links.

The namespace that is listed using this command depends on the type of node in context.

- If an admin node or an EVS node which is using global EVS security is in context, then only a cluster-wide namespace will be listed.

- If an EVS node which is using individual EVS security is in context then only a namespace local to that EVS will be listed.

Use console-context to set the proper context.

Usage:

namespace-list [OPTIONS]

--links, -l

Display the links within the cluster or EVS namespace.

Examples

	namespace-list
	Cluster namespace
	Name: cns

If an admin node or an EVS node with global EVS security is in context, then this command will list any cluster namespace that has been configured for the node. However, if an EVS node with local EVS security is in context, then it will list any EVS namespace local to that EVS.

The first line of the output refers to the type of namespace listed and also, optionally, the type of the node in context.

For example, in the context of an admin node with a cluster namespace 'cns', the namespace-list command will produce:

	Cluster namespace
	Name: cns

In the context of an EVS node with global EVS security enabled and a cluster namespace 'cns' configured, the namespace-list command will produce:

	Cluster namespace - EVS is using global EVS security
	Name: cns

And in the context of an EVS node with individual EVS security and an EVS namespace 'evsns' configured, the namespace-list command will produce:

	EVS namespace - EVS is using individual EVS security
	Name: evsns

Applies To

Cluster wide
