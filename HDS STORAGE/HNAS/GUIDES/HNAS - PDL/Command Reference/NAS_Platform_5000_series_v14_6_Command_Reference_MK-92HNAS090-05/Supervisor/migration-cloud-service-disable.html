migration-cloud-service-disable

Disable the Data Migrator to Cloud service

Syntax

migration-cloud-service-disable

Description

Disable the Data Migrator to Cloud service.

The service can be disabled only when all cloud configuration is deleted and all cloud migrated files are recalled back to HNAS.

To disable the service follow the next steps:

    1. delete cloud policies and schedules in order for new migrations not to take place (migration-cloud-policy-delete, migration-cloud-schedule-delete).
    2. reverse-migrate all cloud migrated data back (reverse-migrate)
    3. delete HNAS snapshots if needed on file systems that have external cloud path configured (snapshot-delete)
    4. delete external cloud migration path (migration-delete-external-path)
    5. delete cloud destinations (migration-cloud-destination-delete)
    6. delete cloud accounts (migration-cloud-account-delete)
    7. run migration-cloud-service-disable

Examples

migration-cloud-service-disable
Attempting to apply modifications...
Cloud Gateway is not running.
Data Migrator to Cloud service has been disabled
Successfully applied configuration modifications.

Disables the Data Migrator to Cloud service if no cloud configuration exists. Returns an error if cloud accounts or destinations are configured.

Applies To

Cluster wide
