multi-tenancy-enable

enable multi-tenant environment

Syntax

multi-tenancy-enable

Description

Enables the multi-tenant environment.

It is likely that this command will only be run when first configuring a new system. See the multi-tenancy manual page for further details.

Enabling the multi-tenant environment will cause a temporary loss of service to all EVSs while they are enabled for use.

It is important to note that all connections to the server will be disconnected during this enabling process. This includes the connection that may be being used to execute the enabling command.

Once converted, if an EVS was enabled and was ready to serve files, it will serve files as before, but be configurable as required for a multi-tenant environment.

Restrictions

Multi-tenancy can only be enabled if all cluster nodes are multi-tenant capable, and are all online. It also requires an "EVS Security" license.

Multi-tenancy cannot be enabled:

  • if any EVS is configured with global security settings. This can be resolved by setting the EVSs to individual security by use of the 'evs-security' command and copying the required configuration.
  • if any deprecated VLAN subnet is still configured in the system. VLAN interfaces must be used instead. VLAN subnets can be removed with the 'vlan remove-all' command.
  • if there is a cluster namespace configured, you will need to use the 'namespace-delete' command prior to enabling.

Examples

Enabling the multi-tenant environment with EVSs configured with individual security.

server:$ multi-tenancy-enable
Warning: Enabling multi-tenancy significantly affects the configuration of the HNAS.

y
