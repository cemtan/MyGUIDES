fs-recover-space

Quickly recovers space scheduled to be freed

Syntax

fs-recover-space [--file-system <file system>]

Description

Deleting a file or a snapshot does not immediately free the disk space that it is occupying. The space is freed over time by a background process. This command speeds up the freeing process.

One of the factors affecting how long it takes to reclaim space occupied by deleted files or snapshots is the number of DSBs the file system is configured with. To change this number, refer to the fs-set-dsb-count man page.

Note: This command creates a number of checkpoints in rapid succession and may temporarily reduce the utility of fs-checkpoint-select.

Usage:

fs-recover-space [--file-system <fs-label>]

-f, --file-system

Selects a file system. If no file system is specified, the currently selected file system is used.

Examples

Recover space on file system 'fs1':

fs-recover-space --file-system fs1

Or

selectfs fs1
 fs-recover-space

Applies To

EVS

See Also

fs-checkpoint-select fs-set-dsb-count selectfs snapshot-delete snapshot-delete-all

Privilege Level

Supervisor
