migration-change-external-path

Modifies a target of the external migration path for the specified vivol/volume.

Syntax

migration-change-external-path -s <fs label> [-v <vivol>] -n <logical name> ([-A] [-H] [-R] [-W] [--type <type>] [-u <user> [-p <password>]] -t <target> | -O) [--force] [--confirm] [--use-ignore-file]

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol>

Name of source Virtual Volume (optional).

-W, --WORM

Flag this target as being a system with WORM behavior (optional).

-R, --RAIN

Flag this target as being a system with RAIN behavior (optional).

-A, --HCAP-archive

Flags this target as being a Hitachi Content Archive Platform, HCAP (optional).

-T, --type { HCAP2 | HCP-default-namespace | HCP-namespace | external-control | cloud }

Flags this target as being of the type specified. Note that --type HCAP2 is equivalent to --HCAP-archive.

-H, --hash

Turn on hash generation for files as they are migrated (optional).

-n, --name <logical name>

Logical name of the migration target.

-t, --target <external path URI>

Protocol specific external path.

-O, --offline

Flag this target as being offline.

-c, --confirm

Confirm that the modification is to be carried out without any further prompt.

-F, --force

Force the change of the external target even when it is not accessible.

-I, --use-ignore-file

Attempt to find and use an ignorefile on the external target. See migration-add-external-path for more information on ignore files.

-u, --user <username>

Specify the data access account username for an HCP namespace target.

-p, --password <password>

Specify the data access account password for an HCP namespace target. If this option is not supplied on the command line, the command will prompt for the password interactively, in which case the password will not be echoed to the terminal.

Modifies the target or options of an existing external migration target.

The first form of this command allows the protocol specific URI and target options to be modified. For instance this command could be used to update an external migration target when an external NFS server is renamed.

The second form of this command causes the specific external migration target to be marked as offline. An offline target will no longer be used for new migrations, but existing cross-file-system links may refer to files via this path. An offline target may not be made online again; however, it is possible to change the protocol specific URI and other target specific flags of an offline target by using the first form of this command, for instance if an NFS server is renamed.

If the new URI is not currently accessible, then the command will fail unless the -F (--force) flag is specified.

With the exception of -O (--offline), the options are the same as for the migration-add-external-path command.

Examples

migration-change-external-path -s fs1 -O -n Archive1

Mark the "Archive1" external target as offline.

migration-change-external-path -s fs1 -W -R -H -n Archive1 -t nfs://new-archive.example.com/fcfs_data/example_archive

Change the target of the "Archive1" external target to be the /fcfs_data/example_archive mount point on new-archive.example.com. If this migration path had previously been marked offline, the offline status would be retained but the target details would still be changed.

For cloud migration use migration-cloud-destination-modify or migration-cloud-accout-modify to update cloud related information.
