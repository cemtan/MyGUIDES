dedupe-queue-add

Queue a file system to be deduped

Syntax

dedupe-queue-add <--full | --incremental> -f <fs> [--synchronous]

Description

Adds a file system to the queue of file systems to be deduped.

You must specify if you want an incremental dedupe (changes written since the last incremental are deduped) or a full dedupe (the entire file system is deduped).

The requested dedupe is added to the queue and will be processed once it reaches the head of the queue as follows:

- If the dedupe service state is enabled, the requested dedupe will be started.

- If the dedupe service state is disabled, the requested dedupe will not be started.

- If the dedupe service state is auto-enable (i.e., is automatically enabled on demand), the server will attempt to change the dedupe service state to enabled if there are sufficient free resources. If the attempt fails, it will raise an event to advise users to check the resource-config commands for details of the system state and how to ensure the dedupe service can be started.

The 'dedupe-queue-status' command can be used to see the current state of the dedupe queue.

This command is only supported on file systems that have been formatted/converted to support dedupe.

Options:

--full

Requests a full dedupe.

--incremental

Requests an incremental dedupe.

--file-system, -f

Specifies the file system to be deduped.

--synchronous

By default, this command will exit once the file system has been placed on the queue. The --synchronous option changes the behaviour to wait until the requested dedupe has completed before exiting. This could cause the command to take a very long time to complete, depending on where the requested dedupe is placed in the queue, and how long the dedupe job takes to run.

Examples

To queue a full dedupe on file system 'fs0':

dedupe-queue-add --full -f fs0

Applies To

File system
