for-each-evs

Run a command in the context of each configured EVS

Syntax

for-each-evs [--quiet] [--admin] [--evs-security] <command> [args]

Description

The for-each-evs command runs the specified command once for each active EVS, setting the console context appropriately first, and running the command on the cnode hosting the EVS.

Options:

--admin, -a

Include the Admin Service context. Useful for a handful of commands (e.g. DNS configuration) that have cluster-wide configuration with per-EVS overrides.

--quiet, -q

Don't display each EVS id/name as commands are run.

--evs-security, -s

With this option, the specified command is run for each active EVS configured with an individual evs-security context and, if there is such an EVS, once in the context of the first active EVS that shares the global evs-security context. When used in conjunction with the --admin option, the command is first run in the context of the Admin Service followed by once in the context of each active EVS configured with an individual evs-security context.

ssrun -c can be useful when trying to run complicated commands.

Examples

Show NFS exports for all EVSs:

for-each-evs nfs-export list

Do a name lookup for each EVS and the Admin Service:

for-each-evs --admin host server.example.com

Show ldap schema configured in the context of the Admin Service and in each active individual evs-security context:

for-each-evs --admin --evs-security ldap-schema

Applies To

Cluster node
