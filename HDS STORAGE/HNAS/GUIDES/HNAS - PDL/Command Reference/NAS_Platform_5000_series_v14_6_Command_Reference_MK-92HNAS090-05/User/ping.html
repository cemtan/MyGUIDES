ping

Ping remote system

Syntax

ping [-c <count>][-I <source-IP-address>[%interface]][-s <packetsize>] <IP-address or hostname>[%interface]

Description

ping determines whether a given host is alive by sending a number of ICMP protocol ECHO_REQUEST datagrams using the IPv4 protocol.

If multi-tenancy is enabled the 'source-ip-address' must be supplied with the -I option and the 'interface' must be supplied with either the destination host or source ip address.

Options:

-c <count>

The number of ECHO_REQUESTS to send. The default is 4; the minimum is 1.

-I <source-ip-address>[%interface]

Set the source IP address of the request to the supplied IP address. An optional source interface name may be supplied. If an interface is specified both here and as part of the destination, then the two interface names must be the same.

-s <packetsize>

Set the number of data bytes to be sent. The default is 56 which means that 64 bytes of IPv4 data is sent over the wire.

Examples

To ping an IPv4 address 5 times:

ping -c 5 10.1.1.82
10.1.1.82...
64 bytes from 10.1.1.82: icmp_seq=0 time=2 ms
64 bytes from 10.1.1.82: icmp_seq=1 time <1 ms
64 bytes from 10.1.1.82: icmp_seq=2 time <1 ms
64 bytes from 10.1.1.82: icmp_seq=3 time <1 ms
64 bytes from 10.1.1.82: icmp_seq=4 time <1 ms
--
5 packets transmitted, 5 packets received, 0% packet loss
round-trip min/avg/max = 0/0/2 ms

Applies To

Cluster node
