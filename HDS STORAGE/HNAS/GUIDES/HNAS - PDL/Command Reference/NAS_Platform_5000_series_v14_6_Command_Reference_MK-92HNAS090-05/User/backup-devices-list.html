backup-devices-list

List NDMP devices, such as tape drives and media changers

Syntax

backup-devices-list [-v | --verbose] [<Device-ID>]
Alias:       tapes

Description

Lists NDMP backup devices, such as tape drives and media changers.

If you specify a device ID, the command shows details of just that device. Otherwise, it shows details of all backup devices in its database. This may include devices that haven't been present since the server was booted.

By default, the command uses a concise format, printing one line per device. If you provide the -v or --verbose switch, it uses a more verbose format showing more information.

Fields printed in concise mode

Nodenames

This field is present only in verbose mode. It shows the world wide node name(s) of the FC controller(s) through which the device is accessed.

Device ID

The persistent ID that you can use to refer to the device. This ID persists across reboots and failovers.

OK

'Yes' if the backup device is online; 'No' otherwise.

Allow

'Yes' if the server is allowed to access the backup device; 'No' otherwise.

EVS

The ID of the EVS that's allowed to use the backup device. Special values are 'None', meaning that the device hasn't been bound to an EVS, and 'Any', meaning that all EVSs are permitted to use the device.

Model

Shows the vendor, model, revision number and vendor-unique information that the backup device reports to the server.

Fields printed in verbose mode

Device ID, Model, EVS

As for concise mode.

Status

The status of the backup device. Provides more detail than simply 'OK' or 'Not OK'.

Access

'Allowed' if the server is allowed to access the backup device; 'Denied' otherwise.

Examples

backup-devices-list
backup-devices-list --verbose
backup-devices-list 800

Applies To

Cluster wide
