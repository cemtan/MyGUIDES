sd-group-list

List some or all system drive groups

Syntax

sd-group-list [--show-count | -c] [--machine-readable | -m] [<device-IDs-see-'man sd-spec'>...]
Alias:       lsdg

Description

Lists some or all of the system drive (SD) groups in the server's registry. It is not necessary for the SDs to be visible to the server or to have any particular status or mirror role when the command is run. See the 'sd-group' man page for background.

If you specify one or more SDs, only groups that use those SDs will be listed. (Those groups, however, may include SDs that aren't specified on the command line, and those SDs will appear in the listing.)

Otherwise, all groups will be listed.

The --show-count or -c switch displays the number of groups listed.

The --machine-readable or -m switch produces a listing designed for easy machine parsing. By default, a more concise, human-readable listing is produced.

In a machine-readable listing, the sort order is simple: the SD group containing the lowest SD device ID is shown first. A human-readable listing shows single-SD groups separately for conciseness, leading to a more elaborate order. Multi-SD groups are shown first, sorted in the same order as in a machine-readable listing. After that come single-SD SD groups. Tiered SD groups come before untiered groups; SD groups in the lowest-numbered tier are shown first.

Examples

The first example shows the difference between a human-readable listing and a machine-readable listing:

server:$ sd-group-list
SDs 8-15


(In tier 0) SDs 16-19


(In tier 1) SDs 20 and 21


SDs 22 and 23


(In tier 0) Single-SD groups on SDs 0 and 1


(In tier 1) Single-SD groups on SDs 2-5


(In tier 1) Single-SD groups on SDs 24 and 25


Single-SD groups on SDs 6 and 7


server:$ sd-group-list -m
(In tier 0) Serial: 0


(In tier 0) Serial: 1


(In tier 1) Serial: 2


(In tier 1) Serial: 3


(In tier 1) Serial: 4


(In tier 1) Serial: 5


Serial: 6


Serial: 7


Serial: 8 9 10 11 12 13 14 15


(In tier 0) Serial: 16 17 18 19


(In tier 1) Serial: 20 21


Serial: 22 23


(In tier 1) Serial: 24


(In tier 1) Serial: 25


server:$

The remaining examples demonstrate how a listing can be flexibly restricted to certain system drives:

server:$ sd-group-list 8-18
SDs 8-15


(In tier 0) SDs 16-19


server:$ sd-group-list all:in-tier=1
(In tier 1) SDs 20 and 21


(In tier 1) Single-SD groups on SDs 2-5


(In tier 1) Single-SD groups on SDs 24 and 25


server:$

Applies To

Cluster wide
