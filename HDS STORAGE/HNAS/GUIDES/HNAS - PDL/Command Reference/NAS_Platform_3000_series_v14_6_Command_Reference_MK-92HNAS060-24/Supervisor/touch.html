touch

Change file time/owner information

Syntax

touch [-a] [-m] [-e] [-t STAMP] [--uid <uid> --gid <gid>] <file1> [<file2>...]

Description

WARNING: This command bypasses all security checks and doesn't ask for confirmation, so make sure you know what you're doing.

The touch command is used to 'touch' files or directories. The command takes one or more paths. If a file system object already exists at a given path, its last-accessed and userdata-modification times will be set to the current time, or the time specified with -t. The metadata-modification time will always be updated to the current time.

If a file system object doesn't already exist at a given path, an empty file will be created.

Usage:

-a

Change only the last-accessed time (doubles as the retention time for WORM file systems)

-m

Change only the userdata-modification time

-e

Change only the creation time

-t STAMP

Use YYYYMMDDhhmm[.ss] instead of current time

-u, --uid <uid>

Set the file or directory's Unix UID to the given value (a poor man's chown)

-g, --gid <gid>

Set the file or directory's Unix GID to the given value (a poor man's chgrp)

Examples

To create the files 'file1', 'file2' and 'file3' in the current working directory, with UID 550 and GID 504:

touch -u 550 -g 504 file1 file2 file3

To set the creation time and last-accessed time to the current time:

touch -a -e file

Applies To

EVS
