namespace-create

Creates a cluster or EVS namespace

Syntax

namespace-create <cluster or EVS namespace>
Alias:       cns-create

Description

Creates a cluster or EVS namespace with the specified name.

The type of namespace that is created depends on the type of node in context.

- If an admin node or an EVS node which is using global EVS security is in context, then a cluster-wide namespace is created.

- If an EVS node which is using individual EVS security is in context then a namespace local to that EVS is created.

Use console-context to set the proper context.

Only one cluster namespace can exist at any time.

An EVS can only have one EVS namespace at any time. An EVS namespace can not be created if the EVS has file systems that are already participating in the cluster namespace.

Similarly, once an EVS namespace has been created, the file systems owned by the EVS can not participate in the cluster namespace.

File systems that are participating in an EVS namespace can not be migrated away from the EVS.

If multi-tenancy is enabled, cluster namespaces cannot be created.

A cluster namespace will need to be deleted if multi-tenancy is to be enabled.

Usage:

namespace-create <cluster or EVS namespace>

Examples

namespace-create cns

This will create a cluster-namespace named 'cns' if an admin node or an EVS node configured with global EVS security is in context. However, if an EVS node with individual EVS security is in context, then this command will create a namespace local to that EVS.

Applies To

Cluster wide

See Also
