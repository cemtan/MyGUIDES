group-mappings-commit

Commits changes to the group mappings table

Syntax

group-mappings-commit

Description

Stores modifications to the group mappings table to persistent store.

A background task will periodically commit changes to the group mappings table to flash and synchronize the changes with other nodes in the cluster.

This command can be used to force a commit after completing changes to the group mappings table so changes are propagated cluster wide immediately.

Examples

Commit mapping changes:

$: group-mappings-commit

Applies To

EVS security context

Privilege Level

Supervisor
