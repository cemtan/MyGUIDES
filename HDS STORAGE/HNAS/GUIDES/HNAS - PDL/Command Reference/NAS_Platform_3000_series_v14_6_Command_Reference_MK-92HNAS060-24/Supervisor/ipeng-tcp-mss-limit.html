ipeng-tcp-mss-limit

Show or set current tcp MSS upper limit value

Syntax

ipeng-tcp-mss-limit [<value>]

Description

The ipeng-tcp-mss-limit commands allow an upper limit to be set on the MSS value that the system advertises as part of the initial TCP handshake. The MSS is usually derived from the MTU, but for performance reasons on some servers it can be useful to cap the value.

It is a global setting, acting on all file serving interfaces.

This feature should already be optimally set for the server performance. Only use under instruction from your support provider.

The commands available are as follows.

ipeng-tcp-mss-limit [<number>]

If a parameter is given, sets the upper limit for the MSS value and enables the cap.

With no parameter, displays the current and default settings.

ipeng-tcp-mss-limit-disable

Turns off the tcp mss limiting feature.

ipeng-tcp-mss-limit-set-defaults

Sets the MSS limit back to the default value. The default value can be found by typing the command "ipeng-tcp-mss-limit" with no parameters. This may enable or disable the limit depending on platform.

Examples

To turn on tcp MSS limit with a cap at 8000:

    $ ipeng-tcp-mss-limit 8000

To disable MSS limit:

    $ ipeng-tcp-mss-limit-disable

Reset the feature to factory default value.

    $ ipeng-tcp-mss-limit-set-defaults

Applies To

Cluster wide
