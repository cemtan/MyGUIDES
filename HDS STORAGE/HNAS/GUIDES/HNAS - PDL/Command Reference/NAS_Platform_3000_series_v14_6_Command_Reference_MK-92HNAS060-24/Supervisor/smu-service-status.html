smu-service-status

Report the SMU service status.

Syntax

smu-service-status

Description

Display the status of the SMU service.

SMU service status is reported as 'running' when both webserver and database are running. If webserver is not running but database is, SMU service status will be reported as 'database running'. If webserver is running but database is not, SMU service status will be reported as 'enabled'.

Note: smu-service-restart will restart the entire SMU service (webserver and database).

The SMU might be disabled if the command smu-service-disable has been run, or if a registry, backed up when the SMU was disabled, is restored to the server.

When the server is being managed by an external SMU, it is expected that the embedded SMU will have been disabled, but this is an optional step.

The SMU will always be disabled when the server is configured as a cluster, and it will not be possible to enable it.

Examples

smu-service-status


     The SMU status is: running


          or


     The SMU status is: database running


          or


     The SMU status is: enabled

Applies To

Cluster wide
