ipadv

Report/set advanced IP configuration

Syntax

ipadv 	 [-p <interface>] [-a <timeout>] [-r <time>] [-i on|off]
	 [-s on|off] [-o <mtu>] [-k on|off] [-t <timeout>]
	 [-m <tcpmtu>] [-n <othermtu>] [-x init|create|delete|list|show-all] [-d]

Description

The ipadv command permits manipulation of IP parameters. Some of these parameters can be set on a per-interface basis, the rest are global settings.

To create a specific configuration for an interface use "-x create -p <interface>" Note that an interface-specific configuration must be created before specific values can be set for that interface, the interface-specific configuration can be created prior to creating the interface.

To delete an interface-specific configuration use "-x delete -p <interface>"

A global setting applies to all interfaces, including VLAN interfaces, except those for which a specific configuration exists. Settings for the parent aggregation interface do not apply to the associated VLAN interfaces. To apply an MTU setting only to a specific VLAN interface it is necessary to create a specific configuration for the full interface name of the VLAN as shown by ifconfig. So, for example, to affect the MTU for VLAN 17 on ag2, create a specific configuration for ag2-vlan0017 and set the desired values for the VLAN interface; a configuration for ag2 will not affect traffic over ag2-vlan0017 and will not have the desired effect.

The following options are available:

-a, --arpcachetimeout <timeout>

Changes ARP cache timeout to the specified number of seconds. The valid range is 50 to 600 seconds.

-r, --ipreassemtim <time>

Changes the IP reassembly timer to the specified number of seconds. The valid range is 1 to 300 seconds.

-i, --ignoreicmpecho on|off

Ignores or acts on ICMP echo requests. This may be configured on a specific interface.

-s, --ignoreicmpredirect on|off

Ignores or acts on ICMP redirects.

-o, --offsubnetmtu <mtu>

Sets IP MTU off-subnet transmits to the specified number of bytes. The valid range is 68 to 9600 bytes, however some interfaces may further limit the effective MTU value. For IPv6 traffic the effective MTU will be 1280 bytes when this option is configured to be less than 1280. This may be configured on a specific interface.

-k, --keepTCPalive on|off

Enables or disables TCP keep alive.

-t, --timeTCPkeepalive

Sets the TCP timeout timer to specified number of seconds which must be atleast 900 seconds.

-m, --tcpmtu <tcpmtu>

Sets the IP MTU used for transmission of TCP packets. The valid range is 68 to 9600 bytes, however some interfaces may further limit the effective MTU value. For IPv6 traffic the effective MTU will be 1280 bytes when this option is configured to be less than 1280. This may be configured on a specific interface.

-n, --othermtu <othermtu>

Sets the IP MTU used for transmission of non-TCP packets. The valid range is 68 to 9600 bytes, however some interfaces may further limit the effective MTU value. For IPv6 traffic the effective MTU will be 1280 bytes when this option is configured to be less than 1280. This may be configured on a specific interface.

-d, --default

Restores the default IP Advanced configuration. Must be the only option used.

-p, --aggportname <interface-name>

interface-name on which to act.

-x, --xtraaction

Specifies an extra action to take (init, create, delete, list, show-all).

Note that settings are not applied to eth0, eth1 or aggregate Linux interfaces (e.g. eth-ag1).

Examples

To set a value globally:
    Change the ARP cache timeout to 65 seconds:
    ipadv -a 65


To create a specific configuration for interface ag1:
    ipadv -x create -p ag1


To set a specific value for a specific interface:
    Ignore ICMP Echo requests on ag1:
    ipadv -p ag1 -i off


To delete a specific configuration for interface ag1:
    ipadv -x delete -p ag1


To show the current global configuration settings and the default values:
    ipadv


To show the current configuration settings for interface ag1:
    ipadv -p ag1


To set the MTU to 9000 for TCP on VLAN 17 on interface ag2:
    ipadv -x create -p ag2-vlan0017
    ipadv -m 9000 -p ag2-vlan0017

Cluster wide
