migration-expand-path

Adds new target file systems to a multi-target migration path.

Syntax

migration-expand-path -s <fs label> [-v <vivol name>] -t <fs label list>

Description

Options:

-s, --source <fs label>

Source file system.

-v, --vivol <vivol name>

Name of source Virtual Volume (optional).

-t, --target <fs label list>

Additional target file systems.

Adds additional target file systems to a data migration path. The source and target file systems must be assigned to the same EVS.

Examples

migration-expand-path -s fs1 -t fs5:fs6

Adds file systems fs5 and fs6 to the existing migration path from fs1.

Applies To

EVS
