migration-cloud-rule-modify

Modify an external migration rule

Syntax

migration-cloud-rule-modify <rule name> --rule <rule> [--description <description>] [--case-sensitive <no|yes>]

Description

Modifies cloud external migration rule.

<rule name>

Name of the rule, which must be unique.

--rule, -r <rule>

Syntax of the rule to be created. For details on rule syntax see the Administrator's Guide.

--description, -d <description>

Rule description. Any arbitrary string to help clients to manage rules.

--case-sensitive, -c <no|yes>

Whether the rule should be treated as case sensitive or not.

Examples

migration-cloud-rule-modify JpgFilesRule --rule  "INCLUDE (<FILENAME *.conf> AND <INACTIVE_OVER 30> AND <UNCHANGED_OVER 30>)" --case-sensitive yes

Modifies cloud external migration rule 'JpgFilesRule' to match the file with .JPG extension, where the match will be case sensitive, and where those files have not be accessed or modified in over 30 days.

Applies To

Global security context
