scsi-io-errors

Display error stats for SCSI ports & devices

Syntax

scsi-io-errors [-p [<port-id>] | [-p] -d [<device-id>] | [-p] [-d] {-s <span name> | -f <fs name/num>} | -r [-p <port-id> | -d <device-id>]]

Description

Displays error statistics by SCSI port or device. (The ports are those reported by the scsi-ports command, i.e. FC channels.) If no options are given, it will display counts for all ports with errors and all devices with errors, as if -d -p had been given. If the counts displayed include FCP or SCSI errors, further details will automatically be displayed.

Options:

-r

Reset counts. Use with -p to reset counts for a specific port, -d for a specific device, or by itself to reset all counts. Resetting error counts also resets any fault status associated with a port or device. Description of other options assumes no -r.

-p

Display by port. If <port-id> is given, display error counts for that port only.

-d

Display by system drive. If <device-id> is given, display error counts for that device only; with -p, error counts for each port serving that device (including errors relating to other devices on the same ports).

-s

Display by span - show error counts for the SDs in the specified span and/or the ports serving those devices. (-p -d implied if neither set.)

-f

Display by file system - show error counts for the SDs in the span containing the specified file system and/or the ports serving those devices. (-p -d implied if neither set.)

Examples

Display counts for all ports with errors and all devices with errors: scsi-io-errors

scsi-io-errors -p -d

Display counts for all ports with errors:

scsi-io-errors -p

Display counts for port 5:

scsi-io-errors -p 5

Display counts for all devices with errors:

scsi-io-errors -d

Display counts for system drive 8:

scsi-io-errors -d 8

Display counts for all ports serving system drive 8:

scsi-io-errors -p -d 8

Display counts for all SDs in span "BigSpan" and their ports:

scsi-io-errors -s BigSpan

Display counts for all devices in the filesystem "Home":

scsi-io-errors -d -f Home

Reset all counts and fault statuses:

scsi-io-errors -r

Reset counts (and fault status if any) for port 5:

scsi-io-errors -r -p 5

Reset counts (and fault status) for system drive 8:

scsi-io-errors -r -d 8

Applies To

Cluster node
