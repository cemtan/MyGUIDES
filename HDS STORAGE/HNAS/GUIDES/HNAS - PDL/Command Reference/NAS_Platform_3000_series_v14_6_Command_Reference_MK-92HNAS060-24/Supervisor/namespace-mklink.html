namespace-mklink

Creates a link to a file system within a cluster or EVS namespace

Syntax

namespace-mklink [--remote-read-caching (disable|enable-all|enable-cvl)] [--local-read-caching (disable|enable-all|enable-cvl)] <file-system> <path-on-file-system> <cluster or EVS namespace> <namespace-path>
Alias:       cns-mklink

Description

Creates a link within a cluster or EVS namespace to the directory in the specified file system. N.B. Files cannot be linked to.

Usage:

namespace-mklink [--remote-read-caching (disable|enable-all)] [--local-read-caching (disable|enable-all|enable-cvl)] <file-system> <path-on-file-system> <cluster or EVS namespace> <namespace-path>

-r, --remote-read-caching

Applicable to cluster namespaces only. Controls which files below the link are candidates for read caching, when accessed remotely:

disable

Disables read caching.

enable-all

All files are candidates for read caching.

enable-cvl

Only cross volume links are candidates for read caching.

-l, --local-read-caching

Controls which files below the link are candidates for read caching, when accessed locally. Values are the same as for --remote-read-caching.

If the directory specified by path-on-file-system does not exist, it will be created.

Examples

Create a link 'filesystems/users' in the namespace 'cns0' to the root directory of the file system 'users':

namespace-mklink users / cns0 filesystems/users

Applies To

Cluster wide

See Also

Privilege Level

Supervisor
