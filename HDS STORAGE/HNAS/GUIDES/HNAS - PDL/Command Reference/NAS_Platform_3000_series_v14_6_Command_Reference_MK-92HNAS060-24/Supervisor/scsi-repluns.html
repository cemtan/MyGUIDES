scsi-repluns

Send a SCSI Report LUNs command to the target device

Syntax

scsi-repluns <-p, --port> | <-P, --port-list>[-l, --lun][-d, --dump][-q, --qtag]
Alias:       repluns

Description

Sends a SCSI report LUNs command to the target device on the given SCSI port. This command is used to request that a SCSI RAID controller return the LUN inventory. The command is normally sent to LUN 0.

Options:

-p, --port <port>

SCSI port id of target device.

-P <comma-separated list of SCSI ports>

Sends the command to all the SCSI ports listed.

-l, --lun <LUN>

LUN of the target device. The command is handled by the controller, which will probably ignore any LUN value specified; the default value is 0.

-d, --dump

Hex-dumps the returned data.

-q, --qtag <tag>

Uses this queue tag value, which must be one of:

Simple = 0 (default)

Head of q = 1

Ordered q = 2

ACA q = 3

Untagged = 4

Examples

Send a SCSI report LUNs command to SCSI port 3 and hex dump the result:

scsi-repluns -p 3 -d

Send a SCSI report LUNs command to SCSI ports 3,4 and 5:

scsi-repluns -P3,4,5

Applies To

Cluster node

See Also

scsi-ports

Privilege Level

Supervisor
