set-for-evs

Sets a variable (or lists them)

Syntax

set-for-evs [[-d | -h | -b | -s | -r] <variable name> [(--current-board | --all-boards) <value>]]

Description

The set-for-evs command configures various advanced server parameters for the evs currently in context. It should only be used under guidance from your support provider.

The related command "set" performs the same role as set-for-evs but in a global context. It has a comprehensive manual page to which the reader may refer to more fully understand the set and registry mechanisms. This page will therefore only describe the key differences and usage of set-for-evs.

The values of configured "set-for-evs" variables are not listed in showall.txt but, nonetheless, can easily be divined from server diagnostics. To do this, decompress registry5.tgz and use a text viewer to inspect registry5.db0 for the Admin evs or registry5.<evs id>.db0 for a serving evs. The registry is a collection of key-value pairs. The key for "set-for-evs" variables is \S\vnode\VnodeLocalVariable\<board type>\<variable name>, where "board type" is "mmb" on single-board platforms or "allBoards" for non board specific variables. Values are encoded in a human-readable format, except for values of type "raw", where the registry encodes non-printable characters. This encoding is grokked by the regdump and regundump scripts.

-d

Force value to be interpreted as decimal

-h

Force value to be interpreted as hexadecimal

-b

Force value to be interpreted as a boolean

-s

Force value to be interpreted as a string

-r

Force value to be interpreted as raw

<variable name>

The name of the variable being set. If the variable name is unspecified a list of current variables are shown.

<value>

The value to set the variable to. An attempt to automatically derive the variable type is made based on this value, but can be overridden by the various switches. If value is unspecified then the current value (if any) of the specified variable is shown.

--current-board | -c

Define the variable in the current board scope. Typically this will be the "mmb" board on single-board platforms.

--all-boards | -a

Define the variable in the "allBoards" board scope. Note that "allBoards" is a specific board designation just like "mmb" and using this switch will delete the variable in that scope if it exists. This will not cause the command to define the variable in all board scopes.

Examples

List all variables currently customized in evs 1:

server:$ evs-select 1
server:$ set-for-evs
G  allBoards       CifsAuthConfig = '0x1' (1 dec)
G  allBoards per-evs-security-config = '0x0' (0 dec)
G        mmb             NSMState = '0x178b' (6027 dec)
G        mmb    VnodeLastExportId = '0xe6' (230 dec)
G        mmb VnodePublicFileHandle = '\/'
G        mmb localgroup-upgrade-check-performed = '0x1' (1 dec)
G        mmb         smb2_enabled = '0x1' (1 dec)

Set the variable "myvar1" on the current board and for "allBoards" for the evs currently in context:

server:$ set-for-evs -s myvar1 --current-board "cval1"
server:$ set-for-evs -s myvar1 --all-boards "aval1"

List all variables matching "myvar1" for the evs currently in context:

server:$ set-for-evs  myvar1
G  allBoards               myvar1 = 'aval1'
G        mmb               myvar1 = 'cval1'

See Also

evs-select set unset-for-evs
