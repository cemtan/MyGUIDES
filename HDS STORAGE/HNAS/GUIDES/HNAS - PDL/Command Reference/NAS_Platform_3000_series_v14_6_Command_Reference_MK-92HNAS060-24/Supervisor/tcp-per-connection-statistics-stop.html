tcp-per-connection-statistics-stop

Stop per-connection TCP statistics gathering and display results

Syntax

tcp-per-connection-statistics-stop [-v] [-d] [-s absolute|percentage|per-second] [-p]
Alias:       tpcs-stop

Description

Halts any per-connection TCP statistics collection that may be in progress and displays the latest results, by default restricting each table of results to no more than 50 entries.

-d, --display N

Restricts each table of results to no more than N entries.

-p, --show-peer

Display the client IP and port number for each reported connection.

-s, --style STYLE

Selects a display style. Permissible values are:

per-second

Values displayed are normalised to a per-second basis.

absolute

Values displayed are totals over the collection interval.

percentage

RX Duplicate Ack packets and RX Out of Order packets are displayed as a percentage of RX Data packets. TX RTO packets, TX Fast RETX packets and TX SACK RETX packets are displayed as a percentage of TX data packets. TX RTO ms is displayed as a percentage of the collection interval.

The default style is per-second.

-v, --verbose

Displays information about how long it took to read statistics from the hardware.

Examples

tcp-per-connection-statistics-stop

Applies To

Cluster Node
