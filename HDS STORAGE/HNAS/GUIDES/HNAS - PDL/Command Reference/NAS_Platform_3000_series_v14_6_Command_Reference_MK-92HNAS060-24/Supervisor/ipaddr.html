ipaddr

Cluster node IP address management

Syntax

ipaddr [-s | --show-internal-addresses] [-r] [-p <port>] [-i <addr> -m <mask>] [-c | --confirm]

Description

The ipaddr command lists and manages a cluster node's physical IP addresses.

Options:

-i, --ip <addr>

Use this option to add or update the cluster node IP address.

This address is applied to the eth1 10/100 port by default and is used for inter-node communication within a cluster.

-m, --mask <mask>

Use this option to specify a netmask to be added or updated.

-p, --port <port>

When configuring an additional cluster node IP address, use this option to explicitly specify a different port name. This IP address will not be used for inter-node cluster communication. It will be node specific and will not migrate to a different node in case of node failure.

The IP address is applied to eth1 by default, which is used for inter-node cluster communication.

-r, --remove

Use this option to remove the cluster node IP configuration.

This can only be done if the server is configured for non-clustered operation. By default this option is applied to eth1. To remove the IP address on eth0, use -p option to specify the port name.

-c, --confirm

Use this option to confirm an update to, or removal of, the existing cluster node IP address.

-s, --show-internal-addresses

Has no effect on this platform.

Examples

To display current cluster node IP configuration, run the command without any arguments.

server:$ ipaddr
Cluster node ID = 1
##  IP Address       Mask             Port
--  ---------------  ---------------  ----------
1   192.168.43.70    255.255.240.0    eth1
Cluster node ID = 2
##  IP Address       Mask             Port
--  ---------------  ---------------  ----------
1   192.168.43.75    255.255.240.0    eth1

To configure the cluster node IP address:

server:$ ipaddr -i 192.168.1.1 -m 255.255.255.0

To configure an additional cluster node IP address on eth0:

server:$ ipaddr -p eth0 -i 192.168.1.2 -m 255.255.255.0

Applies To

Cluster node
