nat-mapping-list

Display the NAT mappings

Syntax

nat-mapping-list

Description

Display the list NAT mappings.

NAT mappings allow UIs for devices on a private network to be launched from the embedded SMU's System Monitor page. This command provides a summary of the NAT mappings applied to the server's firewall.

Applying NAT mappings to the firewall requires that the private management network be set up according to best practice:

eth1 is connected to just one subnet

eth1's subnet has a netmask of 255.255.255.0

eth0 and eth1 must not be connected to the same subnet

nat-mapping-list reports why it may not be possible to apply those mappings to the firewall.

Examples

nat-mapping-list

Lists the NAT mappings to be configured in the firewall.

An example of its output follows:

eth0 Port      Device IP     Device Port
---------     ----------     -----------
    28010     192.0.2.10             100
    28011     192.0.2.11              80

If the private management network has not been configured according to best practice, the command will be followed by an explanation, such as:

eth0:1 and eth1 are on the same network (192.0.2.0), so NAT cannot be configured.

Applies To

Cluster wide
