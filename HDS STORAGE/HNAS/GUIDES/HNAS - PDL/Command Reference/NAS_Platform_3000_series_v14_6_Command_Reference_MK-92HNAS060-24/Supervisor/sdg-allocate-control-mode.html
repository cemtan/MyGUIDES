sdg-allocate-control-mode

Control System Drive Group allocation mode

Syntax

sdg-allocate-control-mode --mode <NO_READ_AHEAD|READ_AHEAD> -f <file-system>

Description

The command requests a change to the free space allocation mode for the specified file system.

The NO_READ_AHEAD sdg allocate mode ensures that pages of the free space bitmap are only read as they are required by the allocation process.

The READ_AHEAD sdg allocate mode allows pages of the free space bitmap to be read ahead of the time they are required by the allocation process. This can reduce the latency of the allocation process in and should result in a significant reduction in the FSA_CREATE_CHECKPOINT component of checkpoint dead time. The level of read-ahead is tuned automatically to match the amount of memory fitted, which is dependent on the model of server being used, and is also tuned automatically to match the activity of the file system. Further manual tuning is possible, but not recommended and should only be considered under supervision of the development team.

The command may be applied to a mounted or unmounted file system. However, if applied to an unmounted file system it will only take effect when the file system is mounted. It is persistent across an unmount/mount or reboot.

Options:

--mode <NO_READ_AHEAD|READ_AHEAD>

Selects the free space allocation mode - READ_AHEAD or NO_READ_AHEAD

Examples

Set the free space allocation mode of the fs0 file system to READ_AHEAD

sdg-allocate-control-mode --mode READ_AHEAD -f fs0
