tls-version-list

Lists the disabled and enabled SSL/TLS versions.

Syntax

tls-version-list

Description

The tls-version-set command is used to control which SSL/TLS versions are enabled on the HTTPS server that is externally contactable.

The tls-version-list command lists the enabled and disabled SSL/TLS versions.

Warning:

Running tls-version-set will automatically cause the HTTPS server used for server management to restart. Any clients currently connected to it will lose their connection and will need to reconnect.

This forces clients to connect using the desired SSL/TLS version and cipher suite.

Options:

Specify the following arguments to enable the desired SSL/TLS version. Any versions not specified will be disabled.

When --default or --enable-all are specified it is not possible to specify any versions to enable.

--confirm

Specify this argument to confirm that you want to change the settings. When not specified a dry-run will be performed where no settings are changed.

--default

Sets the enabled SSL/TLS version to the default. Any other SSL/TLS versions specified will be ignored.

The default is to enable all SSL/TLS versions except for SSLv2 and SSLv3.

--enable-all

All SSL/TLS versions will become enabled.

--ssl2

Enables SSLv2

--ssl3

Enables SSLv3

--tls1.0

Enables TLSv1.0

--tls1.1

Enables TLSv1.1

--tls1.2

Enables TLSv1.2

--tls1.3

Enables TLSv1.3

Examples

Here are some examples:

tls-version-set --confirm --default --ssl2

Will set the enabled SSL/TLS version to the defaults. In this case SSLv2 will not be enabled.

tls-version-set --confirm --ssl2 --ssl3 --tls1.0 --tls1.1 --tls1.2 --tls1.3

Enable all SSL/TLS versions.

tls-version-set --confirm --enable-all

Enable all SSL/TLS versions.

tls-version-set --confirm

Disable all SSL/TLS versions.

tls-version-set --tls1.1 --tls1.2

Will not change the enabled SSL/TLS versions, as --confirm was not specified.

Applies To

Cluster wide
