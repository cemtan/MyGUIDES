cifs-server-comment-set

Syntax

cifs-server-comment-set <comment>

Description

Sets the CIFS comment of the server (also sometimes referred to as the description). The CIFS comment describes the server in a NetrServerGetInfo response and can be retrieved from an SMB server using the DCERPC call. Customers should assess whether their environment requires this value to be set. This command replaces the "comment" field of the deprecated "id" command. Its default value will be whatever was set in the comment field when the HNAS was running older firmware. The comment is limited to 48 UTF-8 characters and can be removed by providing an empty string. The setting is applied at once and stored in the registry so that it is available after reboot and upgrade.

cifs-server-comment-set <comment>
    Sets the CIFS comment.

Options

    None.

Examples

server:$ cifs-server-comment-set "Archive server"

Applies To

Cluster wide.
