timestamp-decode

Display WFS file system timestamp in human readable format

Syntax

timestamp-decode [--nanosecond-resolution] [--terse] <raw-time-value>

Description

This command converts a 64-bit raw value of a file system timestamp to a human-readable form.

The 64-bit raw value is defined as follows:

    - the high-order 32-bit value is the number of seconds since 1970/01/01.
    - the low-order 32-bit value is the fractional second expressed in nanoseconds.

Note that the human-readble format is subject to 100-nanosecond resolution. Therefore, converting the human-readable format back to a 64-bit time value causes the result to have the same 100-nanosecond resolution.

Usage:

--terse, -t

Display only the human-readable text, omitting the raw value. By default, if --terse is not specified, the command will display the human-readable timestamp followed by the raw value in parentheses.

--nanosecond-resolution, -n

Display the fractional part of second with 100-nanosecond resolution.

<raw-time-value>

The 64-bit raw value of the file system timestamp to convert.

Examples

The onode command in previous releases may not decode an on-disk onode's format timestamp, displaying it as a raw value only. For example:

   $ onode -f fs2 -o 16400
   onode/fs2:   Format TimeStamp : 0x5387c44f11e71346

This value encodes the time when the file system "fs2" was formatted. To convert such a raw value to human-readable form:

   $ timestamp-decode 0x5387c44f11e71346
   2014-05-29 16:35:43-07:00 (0x5387c44f11e71346)


   $ timestamp-decode --terse 0x5387c44f11e71346
   2014-05-29 16:35:43-07:00


   $ timestamp-decode --terse --nano 0x5387c44f11e71346
   2014-05-29 16:35:43.300356400-07:00
