domain-mappings-add

Adds a domain mapping to the mapping table

Syntax

domain-mappings-add [--map-unix-names] [--nt <nt domain>] [--nfsv4 <NFSv4 domain>] [--kerberos <kerberos realm>]

Description

Adds a new entry to the domain mappings table.

Options:

--map-unix-names

Use this mapping entry to map to/from Unix names

--nt

Sets the NT domain name to map to/from

--nfsv4

Sets the NFSv4 domain name to map to/from

--kerberos

Sets the Kerberos domain name to map to/from

The auto mapper uses the domain mappings table to determine how to map name fields.

By default it uses an entry that maps Unix names to/from names from the NT domain the server is configured to connect to. If you need to map to an alternate domain or map NFSv4 or Kerberos ids, then you will need to add an explicit mapping.

The domain-mappings-add command can be used to add multiple mappings. However, each field in the mapping table must be unique (for example, you cannot have two domain mapping entries that map to the NT domain, nor can you have two mapping entries that map Unix names). This is to ensure the auto mapper can only ever match one entry, preventing conflicts.

The server will return an error if you try and add a conflicting entry.

Examples

$: domain-mappings-add --map-unix --nt EXAMPLE --nfsv4 example.com --kerberos example
$: # Now if we create a user mapping with nt name EXAMPLE\joe the auto mapper
$: # will set the Unix name to "joe", the NFSv4 name to "joe@example.com", and
$: # the Kerberos principal to "joe@example"

Applies To

EVS security context
