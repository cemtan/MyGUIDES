span-encode-tiers

Produce a command line for 'span-decode-tiers'

Syntax

span-encode-tiers <span-instance-name>

Description

'span-encode-tiers' and 'span-decode-tiers' are used in the following circumstances:

  • A replication product such as TrueCopy or HUR is in use; and
  • Each side of the mirror is at a different site; and
  • Each site has a cluster (or stand-alone server) of its own -- the two sites' servers are not in the same cluster; and
  • One or more of the replicated spans are tiered.

These two commands are used for copying tier numbers from the production cluster to the backup cluster.

HOW TO USE THESE COMMANDS

Follow these steps:

  1. At the production site, set up a tiered span and tiered file systems in the usual way. It is likely that you will need 'sd-mirror-remotely'.
  2. At the production site, run 'span-encode-tiers' against the span. It will display a 'span-decode-tiers' command which can, if necessary, be transmitted by email.
  3. At the backup site, run the 'span-decode-tiers' command produced in the previous step.

'span-list -s' can then be used to confirm that the SDs at the backup site have been assigned to tiers in the same way as at the production site.

Examples

production-cluster:$ span-encode-tiers Accounts
Run this command on the backup cluster:


span-decode-tiers Accounts 13968353963550894874
production-cluster:$


backup-cluster:$ span-decode-tiers Accounts 13968353963550894874
Success
backup-cluster:$

Applies To

Cluster wide
