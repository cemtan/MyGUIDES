sd-object-dump

List one or all SDs

Syntax

sd-object-dump [--no-lock | -L] [<sd-device-id>]

Description

This is a diagnostic command, intended for use by your service organization. It lists the system drives (SDs) known to the server. The nearest customer-facing equivalent is 'sd-list --vverbose'.

Unlike 'sd-list', 'sd-object-dump' can list SDs that are hidden because (for example) they are used for monitoring and managing the storage, rather than for storing data. Such SDs will have device IDs at the top of the available range -- starting at 1023 and working down. A hidden SD's device ID is the same all over a single cluster, but (unusually) doesn't persist across server reboots.

By default, 'sd-object-dump' will dump details of all SDs. You can limit the listing to a single SD by specifying its device ID on the command line.

The '--no-lock' or '-L' switch bypasses important safety mechanisms inside the server. It is not available to unprivileged users. Use of this switch is not supported.

Examples

sd-object-dump -L 42

Shows details of SD 42, bypassing the span-management lock.

Applies To

Cluster node
