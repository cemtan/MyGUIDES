root-of-mount

Get/set the current root of mount

Syntax

root-of-mount [<path>]

Description

The root-of-mount command allows the console to 'mount' a sub-directory of a file system. 'Mount' in this sense is akin to when an NFS client mounts an export, and should not be confused with a physical mount/unmount of the underlying file system.

The selectfs command is used to provide the console with access to a given file system. By default, the console 'mounts' the root of this file system. However, using the root-of-mount command, the console can 'mount' a sub-directory of the file system instead.

Note that this only affects the CLI's access to its current file system (i.e. ls, cd, etc.). Access to CIFS shares and NFS exports, for example, is unaffected.

Examples

Select file system 'fs0' for use by the CLI. Effectively, the CLI 'mounts' the root of 'fs0':

selectfs fs0

Instead of mounting the root of fs0, configure the console to 'mount' the path '/home/user' on 'fs0':

selectfs fs0
root-of-mount /home/user

EVS
