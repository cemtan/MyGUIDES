autoinquiry-stats

Display statistics for autoinquiry

Syntax

autoinquiry-stats [--boot] [--reset] [--type (all|nfs|smb1|smb2|smb2.1|smb3)]

Description

The autoinquiry-stats command reports the number of requests that have attempted to be, have been or have failed to be autoinquired by the server.

Usage:

autoinquiry-stats [--boot] [--reset] [--type <protocol_type>]

-t, --type <protocol_type>

Selects a set of protocol stats. Valid choices are (all|nfs|smb1|smb2|smb2.1|smb3). Defaults to all.

-r, --reset

Resets the statistic counters back to 0

-b, --boot

Displays the statistics since the server boot rather than since the previous use of --reset

Examples

autoinquiry-stats

Displays the number of autoinquired operations for every protocol since the last use of the --reset option.

autoinquiry-stats --type smb1 --reset

Resets the count of SMB1 operations that have been autoinquired.

autoinquiry-stats --type smb1 --boot

Displays the count of SMB1 operations that have been autoinquired since the system was booted.

Applies To

Cluster node

See Also
