posix-chown-restricted

Turns _POSIX_CHOWN_RESTRICTED on and off

Syntax

posix-chown-restricted [on|off]

Description

The posix-chown-restricted command manipulates the _POSIX_CHOWN_RESTRICTED mode.

If this option is in effect (which is the default and is mandated by POSIX 1003.1-2001) then only a user with TakeOwnership right may change the owner of a file. The group of a file may only be changed to one of the groups to which the invoker of chgrp belongs, unless the invoker is root-equivalent or this setting is disabled.

TakeOwnership right is granted to root-equivalent users, that is:

  • unsquashed NFS root
  • mapped equivalents of root
  • members of certain local groups, like Root Users

TakeOwnership right is also granted to:

  • any SID explicitly granted this right by the file's security descriptor
  • the owner of a file

This last point doesn't have a great deal of value but it does allow a file's owner to chown it to themselves - a null operation - without error.

The setting is persistent and takes immediate effect across the current EVS security context (cluster-wide or per-EVS). All operations which modify the owner or group are affected, regardless of protocol and security mode, unless the user is root-equivalent.

The setting is provided to emulate systems which don't have this restriction. It allows users to circumvent quotas by giving away ownership. Relaxing this setting doesn't open the system to the kind of accident that can happen if unsquashed root access is allowed. The identity-conferring mode bits (setuid and setgid) are cleared when a non-root user gives away a file.

Examples

server$ posix-chown-restricted
_POSIX_CHOWN_RESTRICTED is on.
server$ posix-chown-restricted off
server$ posix-chown-restricted
_POSIX_CHOWN_RESTRICTED is off.
server$ posix-chown-restricted on
server$
