scsi-refresh

Force a refresh of SCSI state

Syntax

scsi-refresh [-a, --all] [-d, --disks] [-f, --force] [-t, --tapes][-l, --local]
Alias:       ref

Description

Forces a refresh of server SCSI state. With no parameters, refreshes disk devices only.

This command requests the server to re-validate its SCSI device inventory and refresh its internal state.

Requesting the server to re-validate the status of tape devices will cause the server to send SCSI commands to the tape library which may interfere with a tape backup if one is running at the time. For this reason, by default, tape devices that are in use are not refreshed. Use the --force parameter to override this. By default, scsi-refresh will run on all nodes of the cluster.

Options:

-a, --all

Refresh all devices.

-d, --disks

Refresh disk devices (default).

-t, --tapes

Refresh tape devices but not tape devices that are in use.

-f, --force

Refresh all tape devices, including those that are in use.

Note: This option has no effect unless either the -t or -a option is specified and there are tape drives on the system.

-l, --local

Specify the local flag to run on the cluster node only.

Examples

scsi-refresh

Refresh state of all disk devices across all cluster node.

scsi-refresh -a

Refresh state of all SCSI devices across all cluster node.

Applies To

Cluster wide

Privilege Level

Supervisor
