user-mappings-modify

Modifies an existing user mapping

Syntax

user-mappings-modify <--unix-name <name> | --unix-id <id> | --nt-name <name> | --nt-id <id> |
 --nfsv4-name <name> | --kerberos-name <name> >
    [--set-unix-name <name>] [--set-unix-id <id>] [--set-nt-name <name>]
    [--set-nt-id <id>] [--set-nfsv4-name <name>] [--set-kerberos-name <name>]
    [--set-no-unix-mapping] [--set-no-nt-mapping] [--set-no-nfsv4-mapping]
    [--set-no-kerberos-mapping]
    [--clear-unix-name] [--clear-unix-id] [--clear-nt-name] [--clear-nt-id]
    [--clear-nfsv4-name] [--clear-kerberos-name]

Description

Modifies an existing entry in the user mappings table.

Options:

--unix-name

Look up the entry to modify by Unix name

--unix-id

Look up the entry to modify by Unix user id

--nt-name

Look up the entry to modify by NT name

--nt-id

Look up the entry to modify by NT SID

--nfsv4-name

Look up the entry to modify by NFSv4 name

--kerberos-name

Look up the entry to modify by Kerberos principal

--set-unix-name

Sets the Unix name to be used for the modified entry

--set-unix-id

Sets the Unix user id for the modified entry

--set-nt-name

Sets the NT name (of the form DOMAIN\user) for the modified entry

--set-nt-id

Sets the NT SID for the modified entry

--set-nfsv4-name

Sets the NFSv4 name for the modified entry

--set-kerberos-name

Sets the Kerberos principal (of the form user@realm) for the modified entry

--set-no-unix-mapping

The entry will never have a valid Unix mapping

--set-no-nt-mapping

The entry will never have a valid NT mapping

--set-no-nfsv4-mapping

The entry will never have a valid NFSv4 mapping

--set-no-kerberos-mapping

The entry will never have a valid Kerberos mapping

--clear-unix-name

Clears the existing Unix name in the modified entry

--clear-unix-id

Clears the existing Unix user id in the modified entry

--clear-nt-name

Clears the existing NT name in the modified entry

--clear-nt-id

Clears the existing NT SID in the modified entry

--clear-nfsv4-name

Clears the existing NFSv4 name in the modified entry

--clear-kerberos-name

Clears the existing Kerberos principal in the modified entry

To use the command, look up the existing entry to modify by any of its fields and specify the changes you wish to apply to the entry.

Use the --set-* options to specify new/changed values for fields.

Use the --set-no-*-mapping options to specify fields that will never contain a valid entry.

Use the --clear-* options to remove the "fixed" flag from a field, making it subject to auto mapping.

Examples

Modify the Unix user joe to have a Unix user id of 200 and flag that there is no NT mapping:

$: user-mappings-modify --unix-name joe --set-unix-id 200 --set-no-nt-mapping
Modified entry
Unix name     : joe*
Unix user id  : 200*
NT name       :
NT id         :
NFSv4 name    : *unknown*
Kerberos id   : *unknown*
$:

To properly map root to the local administrators group, which is used (as the owner) when a Windows administrator performs a "Take Ownership" operation, the "fixed" flag for the NT id field needs to be cleared:

$: user-mappings-modify --unix-name root --set-nt-name BUILTIN\Administrators --clear-nt-id
Modified entry
Unix name         : root*
Unix user id      : 0*
NT name           : BUILTIN\Administrators*
NT id             : S-1-5-32-544
NFSv4 name        :
Kerberos id       :
$: user-mappings-commit
$:

If this is not done, then the NT id appears empty.

Applies To

EVS security context
