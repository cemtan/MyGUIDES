nis-netgroups-nobyhost-config

Show/set configuration for netgroup lookups without netgroup.byhost map

Syntax

nis-netgroups-nobyhost-config [--on | --off] [--depth n] [--verbose]

Description

Controls whether the netgroup.byhost map is queried.

This command only applies to classic NIS - its configuration is ignored while in NIS/LDAP mode.

This command allows a user to enable or disable the NIS netgroups 'nobyhost' feature. When this feature is enabled, the server will not use the NIS netgroup.byhost map in order to determine whether a host is a member of a netgroup. Instead, it will make repeated recursive lookups in the netgroup map. This may be appropriate if, for example, the netgroup.byhost map is not available.

Usage:

--on

Enable the 'nobyhost' feature. This means the NIS netgroup.byhost map will not be used.

--off

Disable the 'nobyhost' feature. This means the NIS netgroup.byhost map will be used - and the server will expect it to be available.

-d n, --depth n

This option is deprecated in favor of nis-netgroups-lookup-depth. Set the maximum netgroup lookup depth to n.

-v, --verbose

This option is deprecated in favor of nis-netgroups-lookup-depth. Display the maximum netgroup lookup depth if the 'nobyhost' feature is enabled. Implied by -d.

Examples

Display the current settings:

        $ nis-netgroups-nobyhost-config
        The NIS netgroups 'nobyhost' feature is switched OFF
        This means the NIS netgroup.byhost map will be used
        $

Switch the 'nobyhost' feature on:

        $ nis-netgroups-nobyhost-config --on
        The NIS netgroups 'nobyhost' feature is switched ON
        This means the NIS netgroup.byhost map will not be used
        $
