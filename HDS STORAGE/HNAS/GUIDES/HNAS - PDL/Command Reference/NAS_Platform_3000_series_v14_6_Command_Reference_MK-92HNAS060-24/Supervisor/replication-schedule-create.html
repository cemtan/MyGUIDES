replication-schedule-create

Create an object replication schedule

Syntax

replication-schedule-create [--next-run <next-run-time>] [--run-until <run-until-time>] <policy name> <source fs label> <interval-type> [<interval> <interval-unit>]

Description

Create a schedule for object replication.

<policy name>

Name of the policy.

<source fs label>

Source file system label.

--next-run <next-run-time>

The next run time in the format YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]. Defaults to the current time.

The scheduler will run the object replication on or within 2 minutes of the next run time.

--run-until <run-until-time>

The run until time in the format YYYY-MM-DD{T| }HH:MM[:SS][{+|-}hh:mm]. Defaults to run forever.

The scheduler will not run this object replication after the run until time.

<interval type>

Interval type, allowed values are:

periodic

Run an object replication at the next run time, and at every <interval> <interval unit> after that.

continuous

Run an object replication at the next run time, then schedule the next run for <interval> <interval unit> after the end of that object replication.

onceonly

Runs an object replication once at the next run time.

detailedtest

Runs an object replication detailed test once at the next run time.

<interval>

The number of interval units.

<interval unit>

The interval unit as minute (valid interval 0-120), hour (valid interval 0-48), day (0-365), week (0-52) or month (0-12).

Examples

replication-schedule-create --next-run 2011-02-03T03:30:00-08:00 DefaultPolicy fs0 periodic 3 day

Creates an object replication schedule that runs the policy 'DefaultPolicy' on file system fs0 every 3 days, beginning 3 March 2011 at 3:30am PST.

replication-schedule-create --run-until "2010-12-31 23:00" DefaultPolicy fs0 continuous 1 hour

Creates an object replication schedule that runs the policy 'DefaultPolicy' on file system fs0 continuously, pausing for 1 hour between consecutive runs, until 31 December 2010 at 11:00pm local time.
