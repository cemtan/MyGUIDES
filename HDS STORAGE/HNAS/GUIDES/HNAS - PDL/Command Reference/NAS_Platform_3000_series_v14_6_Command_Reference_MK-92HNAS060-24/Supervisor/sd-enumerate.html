sd-enumerate

List system drives specified on the command line

Syntax

sd-enumerate [--tolerate-empty-terms] [--tolerate-empty-list] <system-drives-see-'man sd-spec'>

Description

Lists system drives (SDs) on a single line, without reporting any information about them. Uses 'sd-spec' to enable flexible specification of the required SDs.

There are two switches:

--tolerate-empty-terms, -t

Omits the usual error message if a term matches no SDs. For example, 'sd-enumerate all:unhealthy all:denied' has two terms, which are 'all:unhealthy' and 'all:denied'.

--tolerate-empty-list, -l

Omits the usual syntax summary if the command as a whole displays no SD device IDs.

These switches are useful when there is a reasonable expectation that the command may display no devices.

To see details of SDs, rather than just summarising their device IDs, use 'sd-list'.

Examples

To see which SDs are licensed and have come up on this server:

server:$ sd-enumerate all:allowed:healthy
SDs 0-3 and 9-24
server:$

To see all unspanned SDs that this server has recognized as GAD LUs:

server:$ sd-enumerate all:is-gad:not-in-a-span
SDs 11-24
server:$

To see all TrueCopy S-Vols without displaying an error message if no S-Vols have been detected:

server:$ sd-enumerate -lt all:is-secondary
No SDs found
server:$

If you give no command-line arguments, the command lists the device IDs of all SDs:

server:$ sd-enumerate
SDs 0-3 and 9-24
server:$

Depending on timing and on the health of cables and other connection infrastructure, it is possible for an SD to be healthy (or recognized as a GAD device, a mirrored secondary, etc.) on one server but not on another server in the same cluster.

Applies To

Cluster node

See Also

sd-list sd-spec

Privilege Level

Supervisor
