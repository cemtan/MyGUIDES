console-prompt-style-default-set

Description

This man page describes a family of commands used to set the appearance of the BOS command prompt.

The console-context and evs-select commands (and their aliases) can be used to change the context of the current console session, altering the EVS and / or cnode any commands are going to be executed on. This context is reflected in the prompt presented to the user, which is referred to as 'context' style.

With the admin vnode in context, the prompt takes the form of:

cnode-in-context:$

With any other evs in context, the prompt takes the form of:

cnode-in-context[evs-in-context]:$

Prior to software version 12.1 this context information would not be presented in the prompt. To revert to this behaviour, a 'legacy' style is provided. In this style, the prompt takes the form of:

local-cnode:$

The local-cnode above refers to the node that has been logged into. This does not change as the console context changes.

console-prompt-style-show

Displays both the current session's style and the default style for new sessions.

console-prompt-style-set (legacy|context)

Set the current console to use either 'legacy' or 'context' style.

console-prompt-style-default-set

Use the current session's style as the default for all new console sessions on the cluster. Current console sessions are not affected by changes to the default style. Supervisor privilege level is required to execute this command.

Examples

The initial login prompt is the same for both styles:

cluster-node-one:$ console-prompt-style-show
 Current console session: context
Default for new sessions: context
cluster-node-one:$ console-context
Admin Service in context
Current cnode 1


cluster-node-one:$ console-prompt-style-show
 Current console session: legacy
Default for new sessions: legacy
cluster-node-one:$ console-context
Admin Service in context
Current cnode 1

With the 'context' style, changes to the console context are reflected in the prompt:

cluster-node-one:$ console-prompt-style-show
 Current console session: context
Default for new sessions: context
cluster-node-one:$ evs-select 1
cluster-node-one[EVS01]:$ console-context --cnode 2
Current evs: 1
Current cnode 1
Commands will be run on cnode 2
cluster-node-two[EVS01]:$ console-prompt-style-show
 Current console session: context
Default for new sessions: context

Setting the session's prompt style is reflected immediately in the prompt:

cluster-node-two[EVS01]:$ console-context
Current evs: 1
Current cnode 1
Commands will be run on cnode 2
cluster-node-two[EVS01]:$ console-prompt-style-show
 Current console session: context
Default for new sessions: context
cluster-node-two[EVS01]:$ console-prompt-style-set legacy
 Current console session: legacy
cluster-node-one:$ console-prompt-style-show
 Current console session: legacy
Default for new sessions: context
cluster-node-one:$ console-context
Current evs: 1
Current cnode 1
Commands will be run on cnode 2
cluster-node-one:$ console-prompt-style-set context
 Current console session: context
cluster-node-two[EVS01]:$ console-prompt-style-show
 Current console session: context
Default for new sessions: context

Supervisor users may set the default style for new connections to match their session's current style:

cluster-node-one:$ console-prompt-style-show
 Current console session: legacy
Default for new sessions: context
cluster-node-one:$ console-prompt-style-default-set
Default for new sessions: legacy
cluster-node-one:$ console-prompt-style-show
 Current console session: legacy
Default for new sessions: legacy

Console
