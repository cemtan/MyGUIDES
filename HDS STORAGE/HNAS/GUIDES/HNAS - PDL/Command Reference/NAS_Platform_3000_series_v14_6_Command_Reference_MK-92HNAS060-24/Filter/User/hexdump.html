hexdump

Hex dump

Syntax

... | hexdump [-v] [-t] [--no-symbols] <start address> [<byte count>|<line count>l]

Description

The hexdump command (with start and count arguments) is used for trouble shooting and should not be used without direction.

Note that hexdump can also be run as a filter. Command output may also be piped through the hexdump filter to display the output as hex bytes and ASCII, in the style of Unix's "hexdump -C". In this case the hexdump filter supports the following options:

-v

Dump all input data even though this may result in groups of identical output lines. This is similar to Unix's "hexdump -v".

-t

Terse output. Dump only the data in hex; do not display the address field, the ASCII field, or any extra messages.

--no-symbols

Don't display recognized addresses as pointers with a symbolic decode below.

If the options above are unable to produce a desired output, one can change the formatting with the nop filter or the hexdump-set command/filter.

The output can be piped through wrdata or sswrdata to convert the hex dump back to the original data to store in a file.

Examples

The "cat --hex" command can hex dump the contents of any file in the current selected file system. But for illustration purposes, let's use the hexdump filter:

   cat --length 0 file | hexdump

To display only the data column:

   cat --length 0 file | hexdump -t

To display all data:

   cat --length 0 file | hexdump -t -v

To disable coloring:

   cat --length 0 file | hexdump | nop

To display only the address and data columns:

   cat --length 0 file | hexdump-set --show-byte-address | hexdump

Console
