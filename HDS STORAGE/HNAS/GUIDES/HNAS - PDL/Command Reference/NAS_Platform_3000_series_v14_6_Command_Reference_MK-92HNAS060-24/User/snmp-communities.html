snmp-communities

Manipulate authentic SNMP communities

Syntax

snmp-communities operation
where 'operation' is one of:
list
add <CommunityName>
del <CommunityName>
delall [--confirm]
modcom <NewCommunityName> <CurrentCommunityName>
authtrap
noauthtrap
Alias:       snmpauth

Description

The snmp-communities command manages the list of authentic SNMP communities. The server's built-in SNMP agent may only be configured for read-only access by community. snmp-communities also controls whether authentication traps are sent. When the protocol being used is SNMPv3, communities will not affect the configuration and snmpv3-user-* should be used instead. Any changes made to communities while SNMPv3 is enabled will not come into effect until the protocol is changed (man snmp-protocol), though they can still be listed.

Valid operations for snmp-communities are:

list

Display the list of configured communities.

add <CommunityName>

Add configured communities, where 'Access' is always read-only.

del <CommunityName>

Delete the configured community.

delall [--confirm]

Delete all configured communities. The --confirm option prevents the command prompting for confirmation.

modcom <NewCommunityName> <CurrentCommunityName>

Modify the community name.

authtrap

Send authentication failure trap.

noauthtrap

Do not send authentication trap.

modcom is used to modify the community name. authtrap and noauthtrap turn authentication traps on or off. Authentication traps will be sent when an SNMP packet is not authenticated.

Examples

Show the list on an unconfigured agent:

snmp-communities list
No authentic SNMP communities are currently configured
Authentication failure trap will NOT be sent on failure to
authenticate an SNMP packet

Add the community 'public' with read-only access:

snmp-communities add public
snmp-communities list


Community name: public
        Access: read-only
Authentication failure trap will NOT be sent on failure to
authenticate an SNMP packet

Modify the community name; note that the new name comes first:

snmp-communities modcom notsopublic public
snmp-communities list


Community name: notsopublic
        Access: read-only
Authentication failure trap will NOT be sent on failure to
authenticate an SNMP packet

Turn on authentication traps:

snmp-communities authtrap
snmp-communities list


Community name: notsopublic
        Access: read-only
Authentication failure trap will be sent on failure to
authenticate an SNMP packet

Applies To

Cluster wide
