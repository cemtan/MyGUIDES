ops

Show average number of OPS per second

Syntax

ops [-n | --no-reset] [-p | --protocols] [-c | --show-counts] [< [-a | --all] cluster-node-id >]

Description

Gives an indication of the average of client operations per second being processed by the specified cluster node/s.

After system boot, protocol operations are counted. These counts are time averaged to get operations per second for the serviced protocols. In normal operation ops displays the current rates (and optionally counts) and then resets the counts (see also stats-logger below). This reset can be suppressed by using the -n option.

NFS operations that are handled solely by the hardware are counted against the 'Auto-Inq' stat. All other operations are broken down by protocol.

Usage:

ops [-n | --no-reset] [-p | --protocols] [-c | --show-counts] [< [-a | --all] cluster-node-id >]

Options:

-n, --no-reset

Suppress reset of counts used to calculate time averages.

-p, --protocols

Show the stats for each serviced protocol.

-c, --show-counts

Show the counts as well as the rates.

-a, --all

Show rates/counts for all cluster nodes.

Applies To

Cluster node

See Also

fsload stats-logger
